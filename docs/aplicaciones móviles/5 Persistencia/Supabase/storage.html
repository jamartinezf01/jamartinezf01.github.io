5.5.2 Supabase Sorage

El servicio de Storage en Supabase ofrece una forma sencilla de gestionar archivos (imágenes, documentos, videos, etc.) en la nube, con una estructura de buckets y permisos similar a Amazon S3. En aplicaciones Android escritas en Java, podemos interactuar con esta funcionalidad usando las APIs REST de Supabase y una librería HTTP como OkHttp. A continuación, se detalla la configuración y los ejemplos necesarios para su implementación.

1. Conceptos Clave​

  • Bucket: Contenedor raíz para tus archivos (similar a una carpeta en la nube).
  • Objeto / Archivo: Cada archivo subido se identifica por un path único dentro del bucket.
  • Bucket Público: Se puede leer sin necesidad de autenticación. La subida y eliminación requieren claves o permisos específicos.
  • Bucket Privado: Se requiere autenticación y claves válidas para leer y escribir.
  • Signed URL: Enlaces con firma criptográfica que dan acceso temporal a un archivo, aun cuando el bucket sea privado.

Estos conceptos se trasladan de forma directa al desarrollo en Android, utilizando peticiones HTTP a la API de Supabase.

2. Creación y administración de buckets​

Creación de un bucket​

Puedes crear un bucket desde la consola de Supabase o utilizando un SDK/HTTP en otro entorno (no existe un SDK oficial para Java Android, pero podrías hacerlo con peticiones HTTP en cualquier backend).

  1. En la consola, ve a Storage → New Bucket.
  2. Especifica el nombre (ej. imagenes-producto) y la visibilidad (Public o Private).

Administración de buckets​

  • Listar buckets: En la consola verás todos los buckets creados.
  • Modificar visibilidad: Puedes cambiar un bucket de public a private y viceversa.
  • Eliminar buckets: Elimina el bucket y todos los archivos que contiene (irreversible).
  • Además puedes crear carpetas o subir archivos de forma manual

3. Permisos y visibilidad de archivos​

Cuando creas un bucket, puedes marcarlo como:

  1. Público:

    • Cualquier persona con la URL puede ver los archivos.
    • Ideal para imágenes u otros recursos que no requieren privacidad.
    • Para subir o borrar, sí se requiere autenticación (con la clave o token JWT apropiado).

  2. Privado:

    • Solo usuarios autenticados o con tokens válidos pueden ver, subir o eliminar archivos.
    • Para compartir un archivo concreto, se generan Signed URLs con validez limitada.

En ambos casos, la configuración de acceso puede complementarse con Policies y Row Level Security (RLS) para un control aún más granular.

políticas de acceso

Para que tus contenedores sean accesibles mediante la API de supabase es necesario asignar alguna políca de acceso.

Para ello puedes usar alguna de las plantillas que se ofrecen o crearlas de manera manual.

4. Uso del servicio Storage en Android (OkHttp)​

Para interactuar desde Android con el servicio de Storage, usaremos:

  • OkHttp: una librería HTTP potente y muy popular en Android.
  • API de Supabase Storage: Endpoints REST que nos permiten subir, descargar y eliminar archivos.

Configuración de dependencias​

En tu archivo app/build.gradle (módulo de la aplicación), asegúrate de incluir OkHttp:

dependencies {
    // ...
    implementation 'com.squareup.okhttp3:okhttp:4.10.0'
    // ...
}

(La versión puede variar; revisa la página oficial de OkHttp.)

Subir archivos​

Para subir un archivo a Supabase Storage, necesitas:

  • URL Base: https://<PROJECT_ID>.supabase.co/storage/v1/object
  • Nombre del bucket: Ej. imagenes-producto
  • Cabeceras: apikey y Authorization: Bearer {TU_API_KEY}

Estos datos los encontrarás en la parte de settings de tu proyecto, en la parte de API

Ejemplo: Subir imagen desde la galería​

Supongamos que quieres subir una imagen seleccionada de la galería. El usuario elige una imagen, obtienes su Uri y luego lees el archivo para enviarlo a Supabase.

private static final String SUPABASE_URL = "https://TU_PROJECT_ID.supabase.co/storage/v1/object";
private static final String BUCKET_NAME = "imagenes-producto";
private static final String SUPABASE_API_KEY = "TU_ANON_O_SERVICE_KEY"; // O la que corresponda

// Método para subir un archivo a Supabase
public void subirArchivoASupabase(Uri uri, String nombreArchivo) {
    // 1. Crear instancia de OkHttpClient
    OkHttpClient client = new OkHttpClient();

    // 2. Obtener el path en Supabase (ej. "uploads/nombreArchivo.jpg")
    String rutaEnSupabase = "uploads/" + nombreArchivo;

    // 3. Construir la URL final
    String endpoint = SUPABASE_URL + "/" + BUCKET_NAME + "/" + rutaEnSupabase;

    // 4. Crear RequestBody con contenido del archivo
    //    Primero convertimos el uri de la galería en un InputStream
    try {
        InputStream inputStream = getContentResolver().openInputStream(uri);
        byte[] fileBytes = toByteArray(inputStream); // Método auxiliar para leer el stream

        RequestBody fileBody = RequestBody.create(
                fileBytes,
                MediaType.parse("image/jpeg")
        );

        // 5. Construir la petición PUT
        Request request = new Request.Builder()
                .url(endpoint)
                .header("apikey", SUPABASE_API_KEY)
                .header("Authorization", "Bearer " + SUPABASE_API_KEY)
                // "x-upsert" define si se sobrescribe el archivo si ya existe
                .header("x-upsert", "false")
                .put(fileBody)
                .build();

        // 6. Enviar la petición en un hilo secundario (importante para no bloquear el main thread)
        new Thread(() -> {
            try (Response response = client.newCall(request).execute()) {
                if (response.isSuccessful()) {
                    Log.d("Supabase", "Archivo subido con éxito: " + response.code());
                } else {
                    Log.e("Supabase", "Error al subir archivo: " + response.code()
                            + " - " + response.message());
                    Log.e("Supabase", "Detalles: " + response.body().string());
                }
            } catch (Exception e) {
                e.printStackTrace();
            }
        }).start();
    } catch (Exception e) {
        e.printStackTrace();
    }
}

// Método auxiliar para convertir un InputStream a byte[]
private byte[] toByteArray(InputStream inputStream) throws IOException {
    ByteArrayOutputStream buffer = new ByteArrayOutputStream();
    int nRead;
    byte[] data = new byte[4096];
    while ((nRead = inputStream.read(data, 0, data.length)) != -1) {
        buffer.write(data, 0, nRead);
    }
    return buffer.toByteArray();
}
  • Nota: MediaType.parse("image/jpeg") asume que subes un JPEG. Ajusta según el tipo real (ej. image/png, application/pdf, etc.).
  • Para no bloquear la UI, la petición se hace en un Thread separado. Puedes usar RxJava, Coroutine (si fuera Kotlin) o cualquier otra forma de asincronía que prefieras.

Descargar archivos​

Para descargar un archivo, haremos una petición GET al mismo endpoint (en realidad es una petición sin verbo explícito: se obtiene con GET la URL completa):

public void descargarArchivoASupabase(String nombreArchivo) {
    OkHttpClient client = new OkHttpClient();

    // Asumimos que el archivo está en "uploads/nombreArchivo"
    String objectPath = "uploads/" + nombreArchivo;
    String endpoint = SUPABASE_URL + "/" + BUCKET_NAME + "/" + objectPath;

    Request request = new Request.Builder()
            .url(endpoint)
            .header("apikey", SUPABASE_API_KEY)
            .header("Authorization", "Bearer " + SUPABASE_API_KEY)
            .build();

    new Thread(() -> {
        try (Response response = client.newCall(request).execute()) {
            if (response.isSuccessful()) {
                // Obtener los bytes del archivo
                byte[] fileBytes = response.body().bytes();

                // Guardar en disco local (directorio interno, por ejemplo)
                File outputFile = new File(getFilesDir(), nombreArchivo);
                FileOutputStream fos = new FileOutputStream(outputFile);
                fos.write(fileBytes);
                fos.close();

                Log.d("Supabase", "Archivo descargado y guardado en: " + outputFile.getAbsolutePath());
            } else {
                Log.e("Supabase", "Error en la descarga: " + response.code() + " - " + response.message());
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }).start();
}
  • Si el bucket es público, la descarga no requeriría Authorization. Sin embargo, en caso de ser privado, necesitas la cabecera con el token.

Eliminar archivos​

Para eliminar un archivo, se envía una petición DELETE al endpoint:

public void eliminarArchivoSupabase(String nombreArchivo) {
    OkHttpClient client = new OkHttpClient();
    
    String objectPath = "uploads/" + nombreArchivo;
    String endpoint = SUPABASE_URL + "/" + BUCKET_NAME + "/" + objectPath;

    Request request = new Request.Builder()
            .url(endpoint)
            .header("apikey", SUPABASE_API_KEY)
            .header("Authorization", "Bearer " + SUPABASE_API_KEY)
            .delete()
            .build();

    new Thread(() -> {
        try (Response response = client.newCall(request).execute()) {
            if (response.isSuccessful()) {
                Log.d("Supabase", "Archivo eliminado correctamente: " + nombreArchivo);
            } else {
                Log.e("Supabase", "Error al eliminar: " + response.code() + " - " + response.message());
                Log.e("Supabase", "Detalles: " + response.body().string());
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }).start();
}
