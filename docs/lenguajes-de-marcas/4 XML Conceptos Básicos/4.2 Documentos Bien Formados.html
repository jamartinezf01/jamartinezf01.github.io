4.2 Documentos Bien Formados

La sintaxis de XML sigue reglas simples que garantizan que un documento sea bien formado. Un documento bien formado es aquel que cumple con todas las reglas estructurales definidas por XML.

Siguiendo estas reglas, los documentos XML serán legibles y procesables en múltiples plataformas sin errores. 🚀

1. Un único elemento raíz​

Todo documento XML debe tener un único elemento raíz que contenga todos los demás elementos.

  • Ejemplo Correcto
<pokedex>
    <pokemon>
        <nombre>Pikachu</nombre>
    </pokemon>
</pokedex>
  • Ejemplo Incorrecto (Múltiples Elementos Raíz)
<pokedex>
    <pokemon>
        <nombre>Pikachu</nombre>
    </pokemon>
</pokedex>
<entrenadores>
    <entrenador>
        <nombre>Ash</nombre>
    </entrenador>
</entrenadores>
Error:

Hay dos elementos raíz (<pokedex> y <entrenadores>). Debe haber solo uno.

2. Prólogo opcional, pero siempre al inicio​

El prólogo XML indica la versión y codificación, pero es opcional. Si se usa, debe ser la primera línea del documento.

  • Ejemplo Correcto
<?xml version="1.0" encoding="UTF-8"?>
<pokedex>
    <pokemon>
        <nombre>Charmander</nombre>
    </pokemon>
</pokedex>
  • Ejemplo Incorrecto (Prólogo Mal Ubicado)
<pokedex>
    <?xml version="1.0" encoding="UTF-8"?>
    <pokemon>
        <nombre>Charmander</nombre>
    </pokemon>
</pokedex>
Error:

El prólogo debe ir antes del primer elemento, no dentro del XML.

3. Etiquetas correctamente cerradas​

Cada elemento debe abrirse y cerrarse correctamente.

  • Ejemplo Correcto
<pokemon>
    <nombre>Pikachu</nombre>
</pokemon>
  • Ejemplo Incorrecto (Etiqueta Sin Cierre)
<pokemon>
    <nombre>Pikachu
</pokemon>
Error:

Falta la etiqueta de cierre </nombre>.

tip

Para elementos vacíos, se puede usar la forma auto-contenida:

<imagen src="pikachu.png" />

4. Case Sensitive​

XML diferencia entre mayúsculas y minúsculas, por lo que <Pokemon> y <pokemon> son distintos.

  • Ejemplo Correcto
<pokemon>
    <nombre>Pikachu</nombre>
</pokemon>
  • Ejemplo Incorrecto (Inconsistencia de Mayúsculas)
<Pokemon>
    <nombre>Pikachu</nombre>
</pokemon>
Error:

Se abre como <Pokemon> pero se cierra como </pokemon> (deberían ser idénticos).

5. Los elementos XML deberán estar debidamente anidados​

Los elementos deben cerrarse en el orden correcto.

  • Ejemplo Correcto
<pokemon>
    <nombre>Pikachu</nombre>
    <tipo>
        <principal>Eléctrico</principal>
    </tipo>
</pokemon>
  • Ejemplo Incorrecto (Anidación Incorrecta)
<pokemon>
    <nombre>Pikachu
    <tipo>
        <principal>Eléctrico</tipo>
    </principal>
</pokemon>
Error:

<tipo> se cerró antes de que <principal> estuviera cerrado.

6. Atributos entre comillas​

Los atributos deben siempre estar entre comillas dobles (") o simples (').

  • Ejemplo Correcto
<pokemon id="025">
    <nombre>Pikachu</nombre>
</pokemon>
  • Ejemplo Incorrecto (Atributo sin Comillas)
<pokemon id=025>
    <nombre>Pikachu</nombre>
</pokemon>
Error:

El valor del atributo id no está entre comillas.

7. Sustitución de caracteres especiales​

Algunos caracteres tienen significado especial en XML y deben sustituirse por entidades.

CarácterEntidad
< (menor que)&lt;
> (mayor que)&gt;
& (ampersand)&amp;
" (comillas dobles)&quot;
' (comillas simples)&apos;
  • Ejemplo Correcto
<descripcion>Pikachu &lt; Raichu en evolución</descripcion>
  • Ejemplo Incorrecto (Carácter No Escapado)
<descripcion>Pikachu < Raichu en evolución</descripcion>
Error:

El < en "Pikachu < Raichu" será interpretado como el inicio de una etiqueta.

8. Comentarios bien estructurados​

Los comentarios en XML usan <!-- --> y pueden contener texto explicativo.

  • Ejemplo Correcto
<!-- Este XML almacena información sobre Pokémon -->
<pokemon>
    <nombre>Pikachu</nombre>
</pokemon>
  • Ejemplo Incorrecto (Comentarios Anidados)
<!-- Este es un <!-- comentario anidado --> -->
<pokemon>
    <nombre>Pikachu</nombre>
</pokemon>
Error:

XML no permite comentarios anidados.

9. Preservación de espacios en blanco​

XML no elimina espacios en blanco en el contenido de los elementos.

  • Ejemplo Correcto
<pokemon>
    <nombre> Pikachu </nombre>
</pokemon>

El espacio en Pikachu se conservará.

  • Uso de Espacios en Blanco en Texto Multilínea
<descripcion>
    Pikachu es un Pokémon eléctrico.
    Puede liberar descargas eléctricas por sus mejillas.
</descripcion>
tip

Los saltos de línea y espacios se preservan.
