Actividad 2 — Errores XML

Ejercicio 1​

Indica por qué los siguientes ejemplos XML son incorrectos y corrige los errores.

Ejemplo 1​

<?xml version="1.0" encoding="UTF-8"?>
<persona>
<nombre>Pedro</nombre>
<DNI>123456789</DNI>
</persona>
<persona>
<nombre>María</nombre>
<DNI>987654321</DNI>
</persona>
Solución
  • Un documento XML debe tener un único elemento raíz. Aquí hay dos elementos <persona> al mismo nivel sin ningún nodo raíz que los envuelva.

Corrección:

<?xml version="1.0" encoding="UTF-8"?>
<personas>
  <persona>
    <nombre>Pedro</nombre>
    <DNI>123456789</DNI>
  </persona>
  <persona>
    <nombre>María</nombre>
    <DNI>987654321</DNI>
  </persona>
</personas>

Ejemplo 2​

<elemento>Coche</ELEMENTO>
Solución
  • Las etiquetas XML son case-sensitive. La apertura <elemento> no coincide con el cierre </ELEMENTO>.

Corrección:

<elemento>Coche</elemento>

Ejemplo 3​

<negrita><italica>Texto XML</negrita></italica>
Solución
  • Los elementos anidados deben cerrarse en orden inverso al de apertura. <italica> se abrió dentro de <negrita>, por lo que debe cerrarse antes que <negrita>.

Corrección:

<negrita><italica>Texto XML</italica></negrita>

Ejemplo 4​

<rango>1 < 5 & 11 > 7</rango>
Solución
  • Los caracteres <, > y & son caracteres reservados en XML y no pueden usarse directamente en el contenido de texto. Deben sustituirse por sus entidades predefinidas.
CarácterEntidad
<&lt;
>&gt;
&&amp;

Corrección:

<rango>1 &lt; 5 &amp; 11 &gt; 7</rango>

Ejemplo 5​

<XMLfichero>Marcas.xml</XMLfichero>
Solución
  • Los nombres de etiquetas no pueden empezar por las letras XML (en cualquier combinación de mayúsculas/minúsculas), ya que ese prefijo está reservado por la especificación XML.

Corrección:

<ficheroXML>Marcas.xml</ficheroXML>

Ejemplo 6​

<![CDATA[ <[[aa]]>]]>
Solución
  • La secuencia ]]> dentro de un bloque CDATA finaliza la sección prematuramente. El contenido <[[aa]]> contiene ]]> y cierra el CDATA antes de tiempo, dejando el > final fuera.

Corrección: Separar o escapar la secuencia problemática:

<![CDATA[ <[[aa]]]]><![CDATA[>]]>

Ejemplo 7​

<user@uo>Pedro@Empleados</user@guo>
Solución
  • Los nombres de etiquetas no pueden contener el carácter @. Además, la apertura <user@uo> no coincide con el cierre </user@guo>.

Corrección:

<userUO>Pedro@Empleados</userUO>

Ejemplo 8​

<texto>El titular de hoy se basa en esta
<subrayado>noticia<subrayado></texto>
Solución
  • La etiqueta de cierre <subrayado> debería ser </subrayado> (falta la barra /).

Corrección:

<texto>El titular de hoy se basa en esta
<subrayado>noticia</subrayado></texto>

Ejercicio 2​

Dado el siguiente documento XML, corregir los posibles errores de sintaxis.

<!-- Documento XML con errores de sintaxis. --!>
<? xml versión="1.0" encodin = "UTF-8" >
<terrestres>
    <vehiculo>bicicleta<vehiculo>
    <vehiculo>coche<vehiculo>
    <vehiculo>tractor<vehiculo>
<acuaticos>
    <vehiculo>canoa<vehiculo>
<aereos>
    <vehiculo>avioneta<vehiculo>
    <vehiculo>helicóptero<vehiculo>
Solución
  1. La declaración XML tiene un espacio: <? xml → <?xml.
  2. El atributo es version (sin tilde), no versión.
  3. El atributo es encoding, no encodin.
  4. La declaración debe cerrar con ?>, no con >.
  5. Todas las etiquetas <vehiculo> de cierre son de apertura; deben ser </vehiculo>.
  6. Faltan las etiquetas de cierre </terrestres>, </acuaticos> y </aereos>.
  7. No hay un elemento raíz único que contenga todo el documento.

Corrección:


<?xml version="1.0" encoding="UTF-8"?>
<vehiculos>
  <terrestres>
    <vehiculo>bicicleta</vehiculo>
    <vehiculo>coche</vehiculo>
    <vehiculo>tractor</vehiculo>
  </terrestres>
  <acuaticos>
    <vehiculo>canoa</vehiculo>
  </acuaticos>
  <aereos>
    <vehiculo>avioneta</vehiculo>
    <vehiculo>helicóptero</vehiculo>
  </aereos>
</vehiculos>

Ejercicio 3​

Corregir los errores que hay en el siguiente documento XML para que esté bien formado.

<?xml version="1.0" encoding="UTF-8">
< frutas >
  < fruta >
    < nombre >cereza< nombre \>
  < fruta \>
  < fruta >
    < nombre >naranja< nombre \>
  < fruta \>
< frutas \>
Solución
  1. La declaración XML debe cerrar con ?>, no con >.
  2. Los nombres de etiqueta no pueden contener espacios: < frutas > → <frutas>.
  3. Las etiquetas de cierre usan </etiqueta>, no < etiqueta \>.

Corrección:

<?xml version="1.0" encoding="UTF-8"?>
<frutas>
  <fruta>
    <nombre>cereza</nombre>
  </fruta>
  <fruta>
    <nombre>naranja</nombre>
  </fruta>
</frutas>

Ejercicio 4​

Corregir los errores que hay en el siguiente documento XML para que esté bien formado.

<?xml version="1.0" encoding="UTF-8"?>
<figuras>
  <figura plana>
      <nombre>cuadrado</nombre>
      </lados 4>
  </figura>
  <figura plana>
      <nombre>triángulo</nombre>
      </lados 3>
  </figura>
  <figura tridimensional>
      <nombre>cubo</nombre>
      </aristas 12>
      </caras 6>
  </figura>
</figuras>
Solución
  1. <figura plana> no es válido: los espacios en nombres de etiqueta no están permitidos. Si plana es un atributo, debe escribirse como <figura tipo="plana">.
  2. Las etiquetas </lados 4>, </aristas 12>, </caras 6> son inválidas: las etiquetas de cierre no pueden tener contenido ni atributos, y además deberían ser etiquetas de apertura seguidas de contenido y cierre.

Corrección:

<?xml version="1.0" encoding="UTF-8"?>
<figuras>
  <figura tipo="plana">
    <nombre>cuadrado</nombre>
    <lados>4</lados>
  </figura>
  <figura tipo="plana">
    <nombre>triángulo</nombre>
    <lados>3</lados>
  </figura>
  <figura tipo="tridimensional">
    <nombre>cubo</nombre>
    <aristas>12</aristas>
    <caras>6</caras>
  </figura>
</figuras>

Ejercicio 5​

Corregir los errores que hay en el siguiente documento XML para que esté bien formado.

<?Xml version="1,0" encoding="UTF8"?>
<triangulo base="7"altura="5">
<triangulo base="2"altura="6">
<triangulo base="3"altura="3">
Solución
  1. La declaración XML debe ser <?xml en minúsculas.
  2. El separador decimal en version es un punto: "1,0" → "1.0".
  3. El valor de encoding estándar es "UTF-8" (con guion), no "UTF8".
  4. Entre atributos debe haber un espacio separador: base="7"altura="5" → base="7" altura="5".
  5. Las etiquetas <triangulo ...> no se cierran. Si son elementos vacíos, deben usar />.
  6. No hay un elemento raíz único.

Corrección:

<?xml version="1.0" encoding="UTF-8"?>
<triangulos>
  <triangulo base="7" altura="5"/>
  <triangulo base="2" altura="6"/>
  <triangulo base="3" altura="3"/>
</triangulos>

Ejercicio 6​

Corregir los errores que hay en el siguiente documento XML para que esté bien formado.

<?xml version="1.0" encoding="UTF-8"?>
<numeros>
  <1 letra="u" letra="n" letra="o">1</>
  <2 letra="d" letra="o" letra="s">22</>
  <6 letra="s" letra="e" letra="i" letra="s">666666</>
</numeros>
Solución
  1. Los nombres de etiqueta no pueden empezar por un número: <1>, <2>, <6> son inválidos.
  2. No se puede repetir el mismo nombre de atributo en un mismo elemento: cada letra="..." es un atributo duplicado.
  3. La etiqueta de cierre </> no es válida en XML estándar; debe incluir el nombre: </numero>.

Corrección (usando un atributo valor y eliminando duplicados):

<?xml version="1.0" encoding="UTF-8"?>
<numeros>
  <numero valor="1" letras="uno">1</numero>
  <numero valor="2" letras="dos">22</numero>
  <numero valor="6" letras="seis">666666</numero>
</numeros>

Ejercicio 7​

Corregir los errores que hay en el siguiente documento XML para que esté bien formado.

<?xml version="1.0" encoding="UTF-8"?>
<biblioteca>
  <libro ISBN=978-3-16-148410-0>
    <titulo>El Quijote</titulo>
    <autor>Miguel de Cervantes</autor>
    <año>1605</año>
  </libro>
  <libro ISBN="978-0-14-028329-7">
    <titulo>Don Juan Tenorio</titulo>
    <autor>José Zorrilla<autor>
    <año>1844</año>
  </libro>
</biblioteca>
Solución
  1. El valor del atributo ISBN del primer libro no está entre comillas: ISBN=978-3-16-148410-0 → ISBN="978-3-16-148410-0".
  2. La etiqueta de cierre del segundo <autor> es de apertura: <autor> → </autor>.

Corrección:

<?xml version="1.0" encoding="UTF-8"?>
<biblioteca>
  <libro ISBN="978-3-16-148410-0">
    <titulo>El Quijote</titulo>
    <autor>Miguel de Cervantes</autor>
    <año>1605</año>
  </libro>
  <libro ISBN="978-0-14-028329-7">
    <titulo>Don Juan Tenorio</titulo>
    <autor>José Zorrilla</autor>
    <año>1844</año>
  </libro>
</biblioteca>

Ejercicio 8​

Corregir los errores que hay en el siguiente documento XML para que esté bien formado.

<?xml version="1.0" encoding="UTF-8"?>
<agenda>
  <contacto>
    <nombre>Laura García</nombre>
    <telefono>+34 600 123 456</telefono>
    <email>laura@empresa.com</email>
    <nota>Llamar antes de las 18:00 & preguntar por el presupuesto</nota>
  <contacto>
    <nombre>Marcos López</nombre>
    <telefono>+34 611 789 012</telefono>
    <email>marcos<empresa.com</email>
  </contacto>
</agenda>
Solución
  1. El carácter & en el contenido de <nota> debe escaparse como &amp;.
  2. La etiqueta de cierre del primer contacto es <contacto> (apertura) en lugar de </contacto>.
  3. El carácter < dentro de <email> debe escaparse como &lt; (o usar @ si es un error tipográfico).

Corrección:

<?xml version="1.0" encoding="UTF-8"?>
<agenda>
  <contacto>
    <nombre>Laura García</nombre>
    <telefono>+34 600 123 456</telefono>
    <email>laura@empresa.com</email>
    <nota>Llamar antes de las 18:00 &amp; preguntar por el presupuesto</nota>
  </contacto>
  <contacto>
    <nombre>Marcos López</nombre>
    <telefono>+34 611 789 012</telefono>
    <email>marcos@empresa.com</email>
  </contacto>
</agenda>

Ejercicio 9​

Corregir los errores que hay en el siguiente documento XML para que esté bien formado.

<?xml version="1.0" encoding="UTF-8"?>
<instituto>
  <alumno id="001" curso="1DAW">
    <nombre>Ana Martínez</nombre>
    <modulos>
      <modulo>Bases de Datos</modulo>
      <modulo>Programación</modulo>
    </modulos>
  </alumno>
  <Alumno id="002" curso="2DAW">
    <nombre>Carlos Ruiz</nombre>
    <modulos>
      <modulo>Acceso a Datos</modulo>
      <modulo>Desarrollo Web</modulo>
    </modulos>
  </alumno>
  <alumno id="003" id="004" curso="1DAW">
    <nombre>Elena Torres</nombre>
  </alumno>
</instituto>
Solución
  1. La etiqueta de apertura <Alumno> no coincide con el cierre </alumno> — XML es case-sensitive.
  2. El tercer alumno tiene el atributo id duplicado: id="003" id="004". Un elemento no puede tener dos atributos con el mismo nombre.

Corrección:

<?xml version="1.0" encoding="UTF-8"?>
<instituto>
  <alumno id="001" curso="1DAW">
    <nombre>Ana Martínez</nombre>
    <modulos>
      <modulo>Bases de Datos</modulo>
      <modulo>Programación</modulo>
    </modulos>
  </alumno>
  <alumno id="002" curso="2DAW">
    <nombre>Carlos Ruiz</nombre>
    <modulos>
      <modulo>Acceso a Datos</modulo>
      <modulo>Desarrollo Web</modulo>
    </modulos>
  </alumno>
  <alumno id="003" curso="1DAW">
    <nombre>Elena Torres</nombre>
  </alumno>
</instituto>

Ejercicio 10​

Corregir los errores que hay en el siguiente documento XML para que esté bien formado.

<?xml version="1.0" encoding="UTF-8"?>
<tienda>
  <producto ref="P01">
    <nombre>Teclado mecánico</nombre>
    <precio>79.99€</precio>
    <descripcion>Teclado con switches Cherry MX.
      Compatible con Windows & Mac.
      Incluye reposamuñecas.</descripcion>
    <stock disponible=true/>
  </producto>
  <producto ref="P02">
    <nombre>Monitor 27"</nombre>
    <precio>349.00€</precio>
    <stock disponible=false/>
  </producto>
</tienda>
Solución
  1. El carácter & en <descripcion> debe escaparse como &amp;.
  2. Los valores de los atributos siempre deben ir entre comillas: disponible=true → disponible="true" y disponible=false → disponible="false".

Corrección:

<?xml version="1.0" encoding="UTF-8"?>
<tienda>
  <producto ref="P01">
    <nombre>Teclado mecánico</nombre>
    <precio>79.99€</precio>
    <descripcion>Teclado con switches Cherry MX.
      Compatible con Windows &amp; Mac.
      Incluye reposamuñecas.</descripcion>
    <stock disponible="true"/>
  </producto>
  <producto ref="P02">
    <nombre>Monitor 27"</nombre>
    <precio>349.00€</precio>
    <stock disponible="false"/>
  </producto>
</tienda>
