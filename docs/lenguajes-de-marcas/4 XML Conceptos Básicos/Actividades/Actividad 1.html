Actividad 1 — Errores XML

Dado los siguientes documentos XML, indica los posibles errores de sintaxis.

Ejercicio 1​

<!-- Documento XML -->
<? xml version="1.0" encoding="UTF-8" ?>
<pokedex>
    <pokemon id="001">
        <nombre>Bulbasaur</nombre>
        <tipo>Planta
        <tipo>Veneno</tipo>
    <pokemon id="004">
        <nombre>Charmander</nombre>
        <tipo>Fuego</tipo>
    </pokemon>
</pokedex>
Solución
  • La declaración XML empieza con <? xml (hay un espacio), debe ser <?xml.
  • La etiqueta <tipo>Planta no está cerrada (falta </tipo>).
  • Falta la etiqueta de cierre </pokemon> del primer pokémon.

Ejercicio 2​

<?xml version="1.0" encoding="UTF-8"?>
<pokedex>
    <pokemon id=007 tipo=Agua>
        <nombre>Squirtle</nombre>
        <descripcion>Squirtle < Wartortle < Blastoise en evolución.</descripcion>
    </pokemon>
    <pokemon id="025" tipo=Eléctrico>
        <nombre>Pikachu</nombre>
    <pokemon>
</pokedex>
Solución
  • Los valores de los atributos deben ir entre comillas: id=007 → id="007" y tipo=Agua → tipo="Agua".
  • El carácter < dentro del texto de <descripcion> no está permitido; debe usarse la entidad &lt;.
  • El atributo tipo=Eléctrico también carece de comillas.
  • La etiqueta de cierre <pokemon> debería ser </pokemon>.

Ejercicio 3​

<?xml version="1.0" encoding="UTF-8"?>
<pokedex>
    <pokemon id="004">
        <nombre>Charmander</nombre>
        <tipo>Fuego</tipo>
    <descripcion>
        <![CDATA[Charmander puede evolucionar a Charmeleon y luego a Charizard.]]>
</pokedex>
Solución
  • Falta la etiqueta de cierre </pokemon>.
  • La etiqueta <descripcion> no está cerrada (falta </descripcion>).

Ejercicio 4​

<!-- Archivo XML Pokedex -->
<? xml version= "1.0" encoding = "UTF-8" ?>
<pokedex>
    <pokemon id= "001">
        <nombre>Bulbasaur</nombre>
        <tipo>Planta</tipo>
    <pokemon id="002">
        <nombre>Ivysaur</nombre>
        <tipo>Planta</tipo>
    </pokemon>
</pokedex>
Solución
  • La declaración XML tiene un espacio: <? xml → <?xml.
  • Los espacios alrededor de = en los atributos de la declaración (version= "1.0", encoding = "UTF-8") no están permitidos.
  • El atributo id= "001" tiene un espacio antes del valor.
  • Falta la etiqueta de cierre </pokemon> del primer pokémon.

Ejercicio 5​

<?xml version="1.0" encoding="UTF-8"?>
<pokedex>
    <pokemon id="025" id="001">
        <nombre>Pikachu</nombre>
        <fecha_captura>2023/03/10</fecha_captura>
    </pokemon>
</pokedex>
Solución
  • Un elemento no puede tener dos atributos con el mismo nombre: id="025" id="001" es inválido.

Ejercicio 6​

<?xml version="1.0" encoding="UTF-8"?>
<pokemon>
    <nombre>Pikachu & Raichu</nombre>
    <descripcion>Pikachu < Raichu en evolución.</descripcion>
</pokemon>
Solución
  • El carácter & en el texto debe escaparse como &amp;.
  • El carácter < en el texto debe escaparse como &lt;.

Ejercicio 7​

<?xml version="1.0" encoding="UTF-8"?>
<pokemon id="001">
    <nombre>Bulbasaur</nombre>
</pokemon>
<pokemon id="004">
    <nombre>Charmander</nombre>
</pokemon>
Solución
  • Un documento XML debe tener un único elemento raíz. Aquí hay dos elementos <pokemon> al mismo nivel sin un nodo raíz que los contenga.

Ejercicio 8​

<?xml version="1.0" encoding="UTF-8"?>
<123pokemon>
    <nombre>Pikachu</nombre>
</123pokemon>
Solución
  • Los nombres de etiquetas no pueden empezar por un número. <123pokemon> es inválido; debería ser, por ejemplo, <pokemon123> o <pokemon>.

Ejercicio 9​

<?xml version="1.0" encoding="UTF-8"?>
<pokedex>
    <Pokemon id="001">
        <nombre>Bulbasaur</nombre>
    </pokemon>
</pokedex>
Solución
  • Las etiquetas XML distinguen mayúsculas de minúsculas (case-sensitive). La apertura <Pokemon> no coincide con el cierre </pokemon>; deben ser idénticas.

Ejercicio 10​

<?xml version="1.0" encoding="UTF-8"?>
<pokedex>
    <pokemon id="007">
        <nombre>Squirtle<nombre>
        <tipo>Agua</tipo>
    </pokemon>
</pokedex>
Solución
  • La etiqueta de cierre <nombre> debería ser </nombre> (falta la barra /).

Ejercicio 11​

<?xml version="1.0" encoding="UTF-8"?>
<pokedex>
    <pokemon id="006">
        <nombre>Charizard</nombre>
        <tipo>Fuego</tipo>
        <altura>1.7m</altura>
    </tipo>
</pokedex>
Solución
  • La etiqueta de cierre </tipo> está fuera de lugar y no tiene apertura correspondiente en ese nivel. Lo correcto sería cerrar </pokemon> antes de cerrar </pokedex>.

Ejercicio 12​

<?xml version="1.0" encoding="UTF-8"?>
<pokedex>
    <pokemon id="094">
        <nombre>Gengar</nombre>
        <descripcion>Gengar usa "Bola Sombra" como ataque principal.</descripcion>
    </pokemon>
</pokedex>
Solución
  • Las comillas dobles " dentro del contenido de texto son válidas en XML (solo están prohibidas sin escapar dentro de los valores de atributos). Sin embargo, si el texto se encontrase dentro de un valor de atributo, habría que usar &quot;. Este ejercicio puede usarse para debatir cuándo escapar las comillas.

Nota: Este documento es técnicamente válido. Se propone para reflexionar sobre el contexto (texto vs. atributo) en el uso de caracteres especiales.
