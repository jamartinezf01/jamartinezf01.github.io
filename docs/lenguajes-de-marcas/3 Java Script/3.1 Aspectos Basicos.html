3.1 Aspectos Básicos

En este punto comenzaremos a trabajar con JavaScript en su entorno natural, el navegador. Antes de avanzar a tareas más complejas, conviene dejar bien asentadas las bases de trabajo y el flujo habitual de prueba y comprobación.

1️⃣ Métodos de Integración​

JavaScript necesita integrarse con HTML para poder ejecutarse en un navegador. 🌐 Esto se logra a través de diversos métodos que permiten incluir código JavaScript en un archivo HTML. 🧩 Es fundamental entender estas opciones, ya que afectan la estructura, legibilidad y mantenimiento del código. ✅

🟩 Inline (En línea)​

En este método, el código JavaScript se escribe directamente dentro de una etiqueta <script> en el archivo HTML. 📜 Es la forma más simple de incluir JavaScript en una página web.

Ejemplo básico de integración inline:

<!DOCTYPE html>
<html lang="es">
<head>
    <title>JavaScript Inline</title>
</head>
<body>
    <h1>Ejemplo de JavaScript Inline</h1>
    <script>
        console.log("Hola desde un script en línea");
    </script>
</body>
</html>
Ventajas 🌟:
  • Simplicidad: Ideal para scripts pequeños o pruebas rápidas. 💡
  • No requiere configuraciones adicionales. 🔧
Desventajas ⚠️:
  • Dificulta el mantenimiento: Si el script es largo, puede desorganizar el HTML. 📂
  • Problemas de reutilización: El código no puede reutilizarse fácilmente en otros archivos. ♻️
  • Mezcla de lenguajes: HTML y JavaScript quedan entremezclados, lo que dificulta la lectura. 📖

🟨 Archivo Externo​

Este método separa el código JavaScript en un archivo independiente con extensión .js, que se enlaza al archivo HTML mediante la etiqueta <script> con el atributo src. Este enfoque es el más recomendado para proyectos profesionales y páginas más complejas. 🏗️

Estructura de archivos:

  • HTML:
<!DOCTYPE html>
<html lang="es">
<head>
    <title>JavaScript Externo</title>
</head>
<body>
    <h1>Ejemplo de JavaScript Externo</h1>
    <script src="script.js"></script>
</body>
</html>
  • Archivo externo (script.js):
console.log("Hola desde un archivo externo");
Ventajas 🌟:
  • Separación de responsabilidades: HTML y JavaScript se mantienen separados, mejorando la organización. 🧹
  • Reutilización: El mismo archivo JavaScript puede ser usado en múltiples páginas. 🔁
  • Caché del navegador: Los navegadores almacenan el archivo en caché, mejorando la velocidad de carga. ⚡
Desventajas ⚠️:
  • Más pasos iniciales: Es necesario gestionar varios archivos. 🗂️
  • Dependencia de conexión: Si el archivo JavaScript está en un servidor externo y no se carga, el script no funcionará. 🌐

🟧 Archivo Inline con Atributos Especiales​

El código JavaScript puede escribirse dentro de la etiqueta <script>, pero utilizando atributos específicos para determinar cómo y cuándo se ejecuta.

Atributos comunes de <script>​

AtributoDescripción
deferRetrasa la ejecución del script hasta que el DOM esté completamente cargado. ⏳
asyncCarga el script de forma asíncrona, en paralelo al resto de la página, pero no garantiza el orden. ⚡

Ejemplo con defer:

<!DOCTYPE html>
<html lang="es">
<head>
    <title>JavaScript con defer</title>
    <script src="script.js" defer></script>
</head>
<body>
    <h1>Ejemplo de uso de defer</h1>
</body>
</html>

Ejemplo con async:

<!DOCTYPE html>
<html lang="es">
<head>
    <title>JavaScript con async</title>
    <script src="script.js" async></script>
</head>
<body>
    <h1>Ejemplo de uso de async</h1>
</body>
</html>

Diferencia entre defer y async​

AspectoDeferAsync
Orden de ejecuciónGarantiza el orden si hay varios scripts. ✅No garantiza el orden; se ejecutan tan pronto como se cargan. ❌
Bloqueo de la páginaNo bloquea la carga de la página. 🚀Tampoco bloquea la carga, pero puede afectar el orden. 📉
Uso recomendadoIdeal para scripts que dependen del DOM. 🌐Útil para scripts independientes del DOM. 💡

🟥 Ejecución Condicionada: DOMContentLoaded​

Cuando el código JavaScript interactúa con el DOM (Modelo de Objetos del Documento), es importante asegurarse de que todos los elementos HTML estén completamente cargados antes de ejecutar el script. 🔄

Ejemplo con DOMContentLoaded:

<!DOCTYPE html>
<html lang="es">
<head>
    <title>DOM Listo</title>
</head>
<body>
    <h1 id="titulo">Ejemplo de DOM Listo</h1>
    <script>
        document.addEventListener("DOMContentLoaded", () => {
            const titulo = document.getElementById("titulo");
            titulo.textContent = "¡El DOM está completamente cargado!";
            console.log("DOM Listo para ser manipulado");
        });
    </script>
</body>
</html>
¿Por qué usar DOMContentLoaded?
  • Asegura que el script no intente acceder a elementos HTML que aún no se han cargado. 🛡️
  • Evita errores comunes como Cannot read properties of null. 🚨

2️⃣ Funciones de consola​

En JavaScript, el objeto console ofrece un conjunto de funciones que permiten interactuar con la consola del navegador. Estas herramientas son muy útiles para depuración, registro de mensajes, advertencias y errores. Aquí exploraremos las funciones más comunes:

🟩 console.log()​

La función más utilizada para mostrar mensajes en la consola. Es ideal para imprimir información general, como valores de variables, mensajes personalizados o resultados de cálculos.

Ejemplo de uso de console.log()
let nombre = "Juan";
console.log("Hola, " + nombre); // Muestra: Hola, Juan
console.log(5 + 3); // Muestra: 8
Aplicación práctica

Úsalo para verificar el estado de variables o el flujo del programa durante su ejecución.

🟨 console.info()​

Es similar a console.log(), pero está diseñada para mensajes informativos. Algunos navegadores pueden mostrar estos mensajes con un estilo visual ligeramente diferente.

Ejemplo de uso de console.info()
console.info("La carga de datos ha sido exitosa.");
Diferencia con console.log()

En la mayoría de los casos, ambas funcionan de manera similar, pero console.info() puede ser útil para diferenciar visualmente mensajes informativos en proyectos grandes.

🟧 console.warn()​

Se utiliza para mostrar advertencias. Estos mensajes aparecen en la consola con un color amarillo (o un ícono de advertencia, dependiendo del navegador).

Ejemplo de uso de console.warn()
let edad = 17;
if (edad < 18) {
    console.warn("Advertencia: El usuario es menor de edad.");
}
nota

Perfecto para alertar sobre situaciones que no detendrán el programa, pero que podrían requerir atención, como valores inesperados.

🟥 console.error()​

Sirve para mostrar errores críticos. Los mensajes aparecen en la consola con un color rojo, lo que indica problemas que podrían afectar el funcionamiento del programa.

Ejemplo de uso de console.error()
try {
    throw new Error("Ha ocurrido un error grave.");
} catch (e) {
    console.error(e.message);
}
nota

Úsalo para registrar errores al manejar excepciones o identificar fallos graves en el código.

🟪 console.group() y console.groupEnd()​

Sirven para agrupar mensajes en la consola bajo un mismo bloque plegable. Es muy útil cuando quieres mostrar varios console.log/info/warn/error relacionados con un mismo proceso (por ejemplo, datos de un usuario, pasos de una función o resultados de una búsqueda).

  • console.group("Título"): abre el grupo (se puede plegar y desplegar)
  • console.groupEnd(): cierra el grupo
  • Extra: console.groupCollapsed("Título") abre el grupo plegado por defecto
Ejemplo de uso de console.group()
let usuario = "Ana";
let edad = 20;

console.group("Datos del usuario");
console.log("Nombre:", usuario);
console.info("Edad:", edad);

if (edad < 18) {
  console.warn("Advertencia: Usuario menor de edad.");
} else {
  console.log("El usuario es mayor de edad.");
}

console.groupEnd();
Aplicación práctica

Úsalo para organizar la consola cuando imprimes muchos mensajes. Por ejemplo, puedes crear un grupo por cada “paso” del programa o por cada usuario procesado.

🟦 console.clear()​

Limpia el contenido actual de la consola, dejándola vacía. Es útil para evitar confusiones en sesiones de depuración largas.

Ejemplo de uso de console.clear()
console.clear(); // Limpia toda la consola

🟫 Ejemplo práctico combinando​

console.log("Iniciando la aplicación...");

let usuario = "Ana";
let edad = 20;

// Agrupar información relacionada del usuario
console.group("Datos del usuario");
console.info("Usuario: " + usuario);
console.info("Edad: " + edad);

if (edad < 18) {
    console.warn("Advertencia: Usuario menor de edad.");
} else {
    console.log("El usuario es mayor de edad.");
}
console.groupEnd();

try {
    throw new Error("No se pudo cargar la configuración.");
} catch (error) {
    console.error("Error detectado: " + error.message);
}

// Limpiar la consola
console.clear();

en este ejemplo
  1. console.log: Mensaje inicial y datos del usuario.
  2. console.group y console.groupEnd: Agrupan los mensajes relacionados con el usuario.
  3. console.info: Información adicional sobre el usuario.
  4. console.warn: Advertencia si es menor de edad.
  5. console.error: Mensaje de error.
  6. console.clear: Limpia todo lo anterior.

3️⃣ Interacción con el usuario​

La API del DOM (Document Object Model) en JavaScript nos permite interactuar con los usuarios directamente a través de la ventana del navegador. Las funciones más comunes para este propósito son alert, prompt y confirm. A continuación veremos cómo funcionan y cuándo utilizarlas:

🟩 alert()​

Esta función muestra un cuadro de diálogo modal con un mensaje y un botón "Aceptar". Es útil para notificar al usuario sobre algo importante o para proporcionar información rápida.

✍️ Sintaxis de alert()
alert("Este es un mensaje de alerta.");
Ejemplo de uso de alert()
let mensaje = "Bienvenido a nuestra página web.";
alert(mensaje);
nota

Aunque es sencilla de usar, el uso excesivo de alert puede resultar molesto para los usuarios, ya que bloquea la interacción con la página hasta que se cierre el cuadro de diálogo.

🟨prompt()​

Permite solicitar información al usuario mediante un cuadro de diálogo con un campo de entrada. Devuelve el texto ingresado como un string, o null si el usuario cancela.

✍️ Sintaxis de prompt()
let valor = prompt("¿Cuál es tu nombre?");
Ejemplo de uso de prompt()
let nombre = prompt("¿Cómo te llamas?");
if (nombre) {
    alert("Hola, " + nombre + "! Encantado de conocerte.");
} else {
    alert("No proporcionaste tu nombre.");
}
Aplicaciones:
  • Obtener datos básicos del usuario.
  • Crear experiencias interactivas, como personalización de contenido.

🟧 confirm()​

Muestra un cuadro de diálogo con un mensaje y dos opciones: "Aceptar" y "Cancelar". Devuelve true si el usuario hace clic en "Aceptar" y false si hace clic en "Cancelar".

✍️ Sintaxis de confirm()
let decision = confirm("¿Estás seguro de que deseas continuar?");
Ejemplo de uso de confirm()
let confirmar = confirm("¿Quieres eliminar este archivo?");
if (confirmar) {
    alert("El archivo ha sido eliminado.");
} else {
    alert("Acción cancelada.");
}
Uso típico:
  • Solicitar confirmación para acciones importantes, como eliminar datos o enviar formularios.

🟥 document.write()​

Escribe directamente contenido HTML o texto en la página web. Reemplaza todo el contenido actual del documento si se usa después de que la página haya terminado de cargarse.

✍️ Sintaxis de confdocument.write()
document.write("¡Hola, mundo!");
Ejemplo de uso de document.write()
document.write("<h1>Bienvenidos a mi sitio web</h1>");
document.write("<p>Este texto fue generado dinámicamente con JavaScript.</p>");
Precaución:
  • document.write() no es recomendable en aplicaciones modernas porque puede sobrescribir el contenido de la página si no se usa con cuidado. En su lugar, se prefieren métodos como innerHTML o appendChild para manipular el DOM.

🟪 Ejemplo combinando​

// Mostrar un mensaje de alerta
alert("¡Bienvenido al sitio web!");

// Pedir el nombre del usuario
let nombre = prompt("Por favor, dinos tu nombre:");

// Confirmar si quiere continuar
if (nombre) {
    let continuar = confirm("Hola " + nombre + ", ¿quieres explorar nuestra página?");
    if (continuar) {
        document.write("<h1>¡Hola, " + nombre + "!</h1>");
        document.write("<p>¡Gracias por visitarnos! Estamos encantados de tenerte aquí.</p>");
    } else {
        document.write("<p>¡Esperamos verte pronto, " + nombre + "!</p>");
    }
} else {
    alert("Parece que no proporcionaste tu nombre. ¡Inténtalo de nuevo más tarde!");
}

🟦 Notas​

nota

  1. Modalidad: alert, prompt y confirm son cuadros de diálogo modales, lo que significa que bloquean la interacción con la página web hasta que el usuario responda.

  2. Usabilidad: Estas funciones pueden ser útiles en prototipos o demostraciones rápidas, pero no se recomiendan en interfaces de usuario modernas debido a la interrupción de la experiencia.

  3. Alternativas modernas: Para una mejor experiencia de usuario, puedes utilizar librerías como SweetAlert o modales personalizados creados con HTML, CSS y JavaScript.
