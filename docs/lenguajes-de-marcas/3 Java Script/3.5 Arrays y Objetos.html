5Ô∏è‚É£ Objetos (estructuras con propiedades)‚Äã

Un objeto es una forma de agrupar datos relacionados en una sola variable usando pares clave: valor. En web se utiliza constantemente para representar cosas del mundo real, por ejemplo un usuario con su nombre, su correo y su rol, o un libro con su titulo, autor y numero de paginas.

  • Un objeto responde bien a la pregunta: "tengo una cosa con varias caracteristicas".

üü© Crear objetos‚Äã

  nombre: "Ana",
  edad: 20,
  activo: true
};

console.log(usuario);
Como leerlo

Se puede leer como: "usuario tiene nombre Ana, edad 20 y activo true".

1. Tipos de valores dentro de un objeto‚Äã

Una propiedad puede guardar cualquier tipo: strings, numbers, booleans, arrays, otros objetos e incluso funciones.

  titulo: "Dune",
  paginas: 412,
  generos: ["ciencia ficcion", "aventura"],
  autor: { nombre: "Frank Herbert", pais: "EEUU" }

üü® Acceso a propiedades: punto y corchetes‚Äã

Se usa cuando se conoce la clave y es un nombre valido.

2. Notacion de corchetes (cuando la clave es dinamica)‚Äã

Se usa cuando:

  • la clave viene en una variable
  • la clave tiene caracteres raros o espacios
  • la clave no se conoce de antemano

Por ejemplo, se tiene un formulario y se quiere actualizar una propiedad segun el nombre del input.

Regla rapida
  • Si la clave es fija y sencilla, usar punto.
  • Si la clave es variable o rara, usar corchetes.

Borrar propiedades no siempre es necesario. En web suele ser mas practico:

6Ô∏è‚É£ Objetos dentro de arrays (caso web mas comun)‚Äã

8Ô∏è‚É£ Referencia vs copia‚Äã

Los objetos y arrays se guardan por referencia, no por copia. Esto significa que cuando se asigna un objeto a otra variable, no se duplica, solo se crea otra referencia al mismo objeto.

Referencia: dos variables apuntan al mismo objeto
let a = { nombre: "Ana" };
let b = a;

b.nombre = "Luis";
console.log(a.nombre); // "Luis"
Problema tipico

Se modifica b pensando que es una copia, pero realmente se esta modificando tambien a. En frontend esto causa bugs muy tipicos cuando varias funciones comparten el mismo objeto.

üü© Copia superficial (shallow copy) con spread‚Äã

El spread (...) crea una copia nueva del primer nivel.

let copia = { ...original };

copia.edad = 21;

console.log(original.edad); // 20
console.log(copia.edad);    // 21
Copiar array con spread
let base = [1, 2, 3];
let copia = [...base];

copia.push(4);

console.log(base);  // [1, 2, 3]
console.log(copia); // [1, 2, 3, 4]
Importante

El spread copia solo el primer nivel. Si dentro hay objetos, esos objetos se siguen compartiendo por referencia.

Shallow copy: objetos internos siguen compartidos
let lista1 = [{ id: 1 }, { id: 2 }];
let lista2 = [...lista1];

lista2[0].id = 999;

console.log(lista1[0].id); // 999 (se modifico el mismo objeto interno)

üü® Copia profunda (deep copy)‚Äã

Cuando se quiere duplicar todo el contenido (incluyendo objetos anidados), en navegadores modernos se puede usar structuredClone.

let origen = [{ id: 1 }, { id: 2 }];
let clon = structuredClone(origen);

clon[0].id = 999;

console.log(origen[0].id); // 1
console.log(clon[0].id);   // 999
Alternativa clasica

JSON.parse(JSON.stringify(obj)) clona en muchos casos, pero pierde cosas como:

  • Date
  • undefined
  • funciones
  • algunos tipos avanzados

9Ô∏è‚É£ JSON y objetos‚Äã

JSON es el formato mas habitual para:

  • guardar datos en el navegador
  • enviar datos a una API
  • recibir datos desde una API

En JavaScript se convierte un objeto a JSON (texto) y se puede volver a convertir a objeto.

‚úÖ Buenas practicas con arrays y objetos‚Äã

Sigue estos consejos...

  4. Tener cuidado con mutaciones, sobre todo si varios sitios usan la misma referencia.
  5. Usar spread para copias rapidas, pero recordar que es copia superficial.
