3.5 Arrays y Objetos

En desarrollo web, la mayoria de datos que se manejan en JavaScript se almacenan como listas o como estructuras con propiedades. Por eso, dominar arrays y objetos es clave para trabajar con datos reales, pintar informacion en pantalla y responder a interacciones del usuario.

1️⃣ Arrays (listas de elementos)​

Los arrays en JavaScript son estructuras de datos que permiten almacenar múltiples valores en una sola variable. Son dinámicos, lo que significa que pueden crecer o decrecer según sea necesario, y pueden contener cualquier tipo de dato (números 🔢, cadenas 📝, objetos 🛠️, funciones ➡️, otros arrays 🔄, etc.).

Los elementos del array están ordenados, cada elemento ocupa una posicion llamada indice, que empieza en 0.

Ejemplo básico
let numeros = [1, 2, 3, 4, 5];
console.log(numeros[0]); // 1
console.log(numeros[3]); // 4

🟩 Crear arrays​

Existen 4 métodos diferentes para crear un array:

1. Declaración usando corchetes 🖇️​

Es la forma más común de declarar un array.

Crear arrays
let numeros = [10, 20, 30];
let nombres = ["Ana", "Luis", "Marta"];
let mezcla = [1, "texto", true]; // posible, pero no recomendable en proyectos reales

console.log(numeros);
console.log(nombres);

2. Declaración usando Array 🏗️​

let numeros = new Array(1, 2, 3, 4, 5);

3. Array vacío 🕳️​

let vacio = [];

4. Array con longitud específica 📏​

Crea un array con un tamaño definido (pero vacío).

let longitudEspecifica = new Array(5);
console.log(longitudEspecifica.length); // 5
Buena practica

En proyectos web, suele ser mejor que un array contenga elementos del mismo tipo. Por ejemplo, todos numeros o todos objetos con el mismo formato.

Arrays Multidimensionales

Un array multidimensional es un array que contiene otros arrays como elementos. Podemos crearlos de la siguiente manera...

let matriz = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9]
];

console.log(matriz[1][2]); // 6

🟨 Acceder y modificar por indice​

Indices y modificacion
let colores = ["rojo", "verde", "azul"];

console.log(colores[0]); // "rojo"
console.log(colores[2]); // "azul"

colores[1] = "amarillo";
console.log(colores); // ["rojo", "amarillo", "azul"]

🟧 Propiedad length​

length indica el numero de elementos del array.

length
let items = ["a", "b", "c"];
console.log(items.length); // 3
Ojo

El ultimo indice es length - 1.

Ultimo elemento
let items = ["a", "b", "c"];
console.log(items[items.length - 1]); // "c"

2️⃣ Metodos basicos de arrays​

🟩 Añadir y quitar al final: push y pop​

Son los métodos más utilizados para añadir o eliminar datos. Funcionan como una pila (LIFO: Last In, First Out).

  • push(): Añade uno o más elementos al final del array y devuelve la nueva longitud.
    • Sintaxis: array.push(elemento1, elemento2, ...)
  • pop(): Elimina el último elemento del array y lo devuelve.
    • Sintaxis: array.pop()
push y pop
let tareas = ["estudiar", "entrenar"];

tareas.push("descansar");
console.log(tareas); // ["estudiar", "entrenar", "descansar"]

let ultima = tareas.pop();
console.log(ultima); // "descansar"
console.log(tareas); // ["estudiar", "entrenar"]

🟨 Añadir y quitar al inicio: unshift y shift​

Son similares a push y pop pero actúan sobre el principio del array.

  • unshift(): Añade uno o más elementos al inicio del array y devuelve la nueva longitud.
    • Sintaxis: array.unshift(elemento1, elemento2, ...)
  • shift(): Elimina el primer elemento del array y lo devuelve (desplaza al resto).
    • Sintaxis: array.shift()
unshift y shift
let cola = ["persona2", "persona3"];

cola.unshift("persona1");
console.log(cola); // ["persona1", "persona2", "persona3"]

let primero = cola.shift();
console.log(primero); // "persona1"
console.log(cola); // ["persona2", "persona3"]
Rendimiento

shift y unshift mueven muchos elementos de posicion. En listas grandes es mejor evitarlos si se puede.

🟧 Cortar, insertar o borrar en medio: splice​

splice es una navaja suiza: permite borrar, insertar o reemplazar elementos en cualquier posición.

  • Sintaxis: array.splice(inicio, cantidadBorrar, item1, item2...)
    • inicio: Índice donde empezar.
    • cantidadBorrar: Número de elementos a eliminar.
    • items: (Opcional) Elementos nuevos para insertar en esa posición.
Mutabilidad

splice modifica el array original.

splice: borrar 1 elemento desde indice 1
let letras = ["a", "b", "c", "d"];
letras.splice(1, 1);
console.log(letras); // ["a", "c", "d"]
splice: insertar sin borrar
let letras = ["a", "c", "d"];
letras.splice(1, 0, "b");
console.log(letras); // ["a", "b", "c", "d"]
splice: reemplazar
let letras = ["a", "b", "c", "d"];
letras.splice(2, 1, "X");
console.log(letras); // ["a", "b", "X", "d"]

🟥 Copiar una parte sin modificar: slice​

Devuelve una copia superficial de una parte del array en un nuevo array. No modifica el original.

  • Sintaxis: array.slice(inicio, fin)
    • inicio: Índice donde empieza el corte (incluido).
    • fin: (Opcional) Índice donde termina (no incluido).
slice: copia desde 1 hasta 3 sin incluir 3
let base = ["a", "b", "c", "d"];
let parte = base.slice(1, 3);

console.log(parte); // ["b", "c"]
console.log(base);  // ["a", "b", "c", "d"]

🟪 Buscar en arrays: includes e indexOf​

Permiten saber si un elemento existe en el array y dónde está.

  • includes(): Devuelve true si el elemento está en el array, o false si no.
    • Sintaxis: array.includes(elemento)
  • indexOf(): Devuelve el índice de la primera aparición del elemento. Si no está, devuelve -1.
    • Sintaxis: array.indexOf(elemento)
includes e indexOf
let frutas = ["manzana", "pera", "uva"];

console.log(frutas.includes("pera")); // true
console.log(frutas.includes("kiwi")); // false

console.log(frutas.indexOf("uva")); // 2
console.log(frutas.indexOf("kiwi")); // -1

🟦 Ordenar y revertir​

📜 sort​

Ordena los elementos del array in situ (modifica el array original). Por defecto, convierte los elementos a strings y los ordena por código ASCII (alfabéticamente).

  • Sintaxis: array.sort([funcionDeComparacion])
let numeros = [10, 2, 5, 1];
numeros.sort();
console.log(numeros); // [1, 10, 2, 5]

↩️ reverse​

Invierte el orden de los elementos del array in situ (el primero pasa a ser el último y viceversa).

  • Sintaxis: array.reverse()
let numeros = [1, 2, 3];
numeros.reverse();
console.log(numeros); // [3, 2, 1]

3️⃣ Recorrer arrays​

🟩 for clasico​

for clasico
let nums = [10, 20, 30];

for (let i = 0; i < nums.length; i++) {
  console.log("Indice:", i, "Valor:", nums[i]);
}

🟨 for...of (muy comodo)​

for...of
let nums = [10, 20, 30];

for (let n of nums) {
  console.log(n);
}

🟧 forEach (funcion por cada elemento)​

forEach
let nums = [10, 20, 30];

nums.forEach((n, i) => {
  console.log("Indice:", i, "Valor:", n);
});
Regla rapida
  • for o for...of cuando se quiere poder usar break o continue.
  • forEach cuando solo se quiere ejecutar algo para cada elemento.

4️⃣ Metodos modernos de arrays​

Estos metodos ayudan a trabajar con datos sin escribir bucles largos.

🟩 map (transformar)​

Crea un nuevo array con los resultados de llamar a una función para cada elemento del array original. Es fundamental en React y frameworks modernos para pintar listas.

  • Sintaxis: let nuevoArray = array.map(function(elemento, indice) { ... })
map
let precios = [10, 20, 30];
let preciosConIva = precios.map((p) => p * 1.21);

console.log(preciosConIva); // [12.1, 24.2, 36.3]

🟨 filter (filtrar)​

Crea un nuevo array con todos los elementos que cumplan la condición implementada por la función dada. Si ninguno cumple la condición, devuelve un array vacío.

  • Sintaxis: let nuevoArray = array.filter(function(elemento) { ... })
filter
let edades = [12, 18, 25, 16];
let mayores = edades.filter((e) => e >= 18);

console.log(mayores); // [18, 25]

🟧 find (encontrar el primero)​

Devuelve el valor del primer elemento del array que cumple la función de comprobación proporcionada. En caso contrario devuelve undefined.

  • Sintaxis: let elemento = array.find(function(elemento) { ... })
find
let usuarios = ["ana", "luis", "marta"];
let encontrado = usuarios.find((u) => u.startsWith("m"));

console.log(encontrado); // "marta"

🟥 some y every (comprobaciones booleanas)​

Sirven para verificar condiciones en el array sin necesidad de recorrerlo manualmente. Devuelven true o false.

  • some(): Comprueba si al menos un elemento del array cumple la condición.
    • Sintaxis: array.some(callback)
  • every(): Comprueba si todos los elementos del array cumplen la condición.
    • Sintaxis: array.every(callback)
some y every
let notas = [4, 6, 8];

console.log(notas.some((n) => n >= 5));  // true
console.log(notas.every((n) => n >= 5)); // false

🟪 reduce (acumular)​

Ejecuta una función reductora sobre cada elemento del array, devolviendo como resultado un único valor. Es útil para sumar totales, agrupar datos o transformar estructuras.

  • Sintaxis: let resultado = array.reduce(function(acumulador, valorActual) { ... }, valorInicial)
reduce: suma total
let cesta = [5, 10, 20];
let total = cesta.reduce((acum, precio) => acum + precio, 0);

console.log(total); // 35
Consejo

reduce es potente pero puede ser dificil de leer si se abusa. En nivel inicial, usarlo sobre todo para sumas y acumulaciones simples.

5️⃣ Objetos (estructuras con propiedades)​

Un objeto es una forma de agrupar datos relacionados en una sola variable usando pares clave: valor. En web se utiliza constantemente para representar cosas del mundo real, por ejemplo un usuario con su nombre, su correo y su rol, o un libro con su titulo, autor y numero de paginas.

La idea importante es esta:

  • Un array responde bien a la pregunta: "tengo una lista de cosas".
  • Un objeto responde bien a la pregunta: "tengo una cosa con varias caracteristicas".

🟩 Crear objetos​

La forma mas habitual es el objeto literal con llaves {}. Cada propiedad se escribe como clave: valor y se separa con comas.

Objeto literal
let usuario = {
  nombre: "Ana",
  edad: 20,
  activo: true
};

console.log(usuario);
Como leerlo

Se puede leer como: "usuario tiene nombre Ana, edad 20 y activo true".

1. Tipos de valores dentro de un objeto​

Una propiedad puede guardar cualquier tipo: strings, numbers, booleans, arrays, otros objetos e incluso funciones.

Objeto con distintos tipos
let libro = {
  titulo: "Dune",
  paginas: 412,
  generos: ["ciencia ficcion", "aventura"],
  autor: { nombre: "Frank Herbert", pais: "EEUU" }
};

console.log(libro.generos[0]);      // "ciencia ficcion"
console.log(libro.autor.nombre);    // "Frank Herbert"

🟨 Acceso a propiedades: punto y corchetes​

Hay dos maneras principales de acceder a una propiedad.

1. Notacion de punto (la mas usada)​

Se usa cuando se conoce la clave y es un nombre valido.

Acceso por punto
console.log(usuario.nombre); // "Ana"
console.log(usuario.activo); // true

2. Notacion de corchetes (cuando la clave es dinamica)​

Se usa cuando:

  • la clave viene en una variable
  • la clave tiene caracteres raros o espacios
  • la clave no se conoce de antemano
Acceso por corchetes
console.log(usuario["edad"]); // 20

Por ejemplo, se tiene un formulario y se quiere actualizar una propiedad segun el nombre del input.

Clave dinamica
let clave = "nombre";
console.log(usuario[clave]); // "Ana"
Claves con espacios (solo corchetes)
let config = {
  "color tema": "oscuro"
};

console.log(config["color tema"]); // "oscuro"
Regla rapida
  • Si la clave es fija y sencilla, usar punto.
  • Si la clave es variable o rara, usar corchetes.

🟧 Modificar, Añadir y borrar​

1. Modificar una propiedad existente​

Modificar propiedad
usuario.edad = 21;
console.log(usuario.edad); // 21

2. Añadir una propiedad nueva​

JavaScript permite Añadir propiedades aunque no existieran antes.

Añadir propiedad
usuario.ciudad = "Madrid";
console.log(usuario.ciudad); // "Madrid"
Idea clave

Los objetos en JavaScript son estructuras flexibles. Eso facilita prototipos rapidos, pero en proyectos grandes conviene ser consistente con las claves.

3. Borrar una propiedad​

Borrar propiedad
delete usuario.activo;
console.log(usuario);
Buena practica

Borrar propiedades no siempre es necesario. En web suele ser mas practico:

  • asignar null si se quiere indicar ausencia intencional
  • asignar false si es un estado
  • o crear un nuevo objeto actualizado (cuando se quiere evitar mutaciones)

6️⃣ Objetos dentro de arrays (caso web mas comun)​

En frontend, lo tipico es manejar colecciones de datos como listas. Por eso es muy habitual trabajar con un array donde cada elemento es un objeto con el mismo formato.

Esto encaja con un patron muy comun:

  • el array es "la base de datos en memoria"
  • el DOM se pinta a partir de ese array
  • cuando el usuario interactua, se modifica el array y se vuelve a pintar
Array de objetos
let libros = [
  { id: "A1", titulo: "Dune", autor: "Frank Herbert", paginas: 412 },
  { id: "A2", titulo: "1984", autor: "George Orwell", paginas: 328 },
  { id: "A3", titulo: "El Hobbit", autor: "J. R. R. Tolkien", paginas: 310 }
];

🟩 Buscar un elemento: find​

find devuelve el primer objeto que cumpla la condicion. Si no lo encuentra, devuelve undefined.

find: buscar por id
let libroA2 = libros.find((l) => l.id === "A2");

console.log(libroA2);         // objeto del libro A2
console.log(libroA2.titulo);  // "1984"
Ojo

Si find no encuentra nada, el resultado es undefined. Conviene comprobarlo antes de acceder a propiedades para evitar errores.

find con comprobacion
let libro = libros.find((l) => l.id === "ZZ");

if (!libro) {
  console.log("No existe ese libro");
} else {
  console.log(libro.titulo);
}

🟨 Filtrar elementos: filter​

filter devuelve un array nuevo solo con los elementos que cumplan la condicion.

filter: libros largos
let largos = libros.filter((l) => l.paginas >= 350);
console.log(largos); // array con Dune
Diferencia clave
  • find devuelve un solo elemento o undefined.
  • filter devuelve siempre un array, aunque sea vacio.

🟦 Ejemplo tipico: buscador por titulo​

En web, un buscador suele:

  • leer texto del usuario
  • filtrar
  • pintar resultados
Filtro por texto (busqueda simple)
let query = "el";

let resultados = libros.filter((l) =>
  l.titulo.toLowerCase().includes(query.toLowerCase())
);

console.log(resultados);
Conexion con el DOM

Este filtro se conecta facilmente con un input. Cada vez que el usuario escribe, se recalculan resultados y se actualiza la lista en pantalla.

7️⃣ Recorrer objetos​

A veces se necesita listar todas las propiedades de un objeto, por ejemplo para depuracion o para construir una tabla.

🟩 for...in​

Recorre las claves (propiedades) del objeto.

for...in
let perfil = { nombre: "Ana", correo: "ana@mail.com", rol: "user" };

for (let clave in perfil) {
  console.log(clave, "=>", perfil[clave]);
}
Nota

for...in recorre claves. En arrays no se recomienda como bucle principal, porque puede incluir claves no numericas.

🟨 Object.keys, Object.values, Object.entries​

Son mas claros cuando se quiere trabajar de forma controlada.

keys, values, entries
let perfil = { nombre: "Ana", correo: "ana@mail.com", rol: "user" };

console.log(Object.keys(perfil));   // ["nombre", "correo", "rol"]
console.log(Object.values(perfil)); // ["Ana", "ana@mail.com", "user"]

Object.entries(perfil).forEach(([k, v]) => {
  console.log(k, v);
});
Cuando usar cada uno
  • keys si solo interesan las claves.
  • values si solo interesan los valores.
  • entries si se necesitan clave y valor a la vez.

8️⃣ Referencia vs copia​

Los objetos y arrays se guardan por referencia, no por copia. Esto significa que cuando se asigna un objeto a otra variable, no se duplica, solo se crea otra referencia al mismo objeto.

Referencia: dos variables apuntan al mismo objeto
let a = { nombre: "Ana" };
let b = a;

b.nombre = "Luis";
console.log(a.nombre); // "Luis"
Problema tipico

Se modifica b pensando que es una copia, pero realmente se esta modificando tambien a. En frontend esto causa bugs muy tipicos cuando varias funciones comparten el mismo objeto.

🟩 Copia superficial (shallow copy) con spread​

El spread (...) crea una copia nueva del primer nivel.

Copiar objeto con spread
let original = { nombre: "Ana", edad: 20 };
let copia = { ...original };

copia.edad = 21;

console.log(original.edad); // 20
console.log(copia.edad);    // 21
Copiar array con spread
let base = [1, 2, 3];
let copia = [...base];

copia.push(4);

console.log(base);  // [1, 2, 3]
console.log(copia); // [1, 2, 3, 4]
Importante

El spread copia solo el primer nivel. Si dentro hay objetos, esos objetos se siguen compartiendo por referencia.

Shallow copy: objetos internos siguen compartidos
let lista1 = [{ id: 1 }, { id: 2 }];
let lista2 = [...lista1];

lista2[0].id = 999;

console.log(lista1[0].id); // 999 (se modifico el mismo objeto interno)

🟨 Copia profunda (deep copy)​

Cuando se quiere duplicar todo el contenido (incluyendo objetos anidados), en navegadores modernos se puede usar structuredClone.

Deep copy con structuredClone
let origen = [{ id: 1 }, { id: 2 }];
let clon = structuredClone(origen);

clon[0].id = 999;

console.log(origen[0].id); // 1
console.log(clon[0].id);   // 999
Alternativa clasica

JSON.parse(JSON.stringify(obj)) clona en muchos casos, pero pierde cosas como:

  • Date
  • undefined
  • funciones
  • algunos tipos avanzados

9️⃣ JSON y objetos​

JSON es el formato mas habitual para:

  • guardar datos en el navegador
  • enviar datos a una API
  • recibir datos desde una API

En JavaScript se convierte un objeto a JSON (texto) y se puede volver a convertir a objeto.

🟩 JSON.stringify y JSON.parse​

Convertir a JSON y volver a objeto
let perfil = { nombre: "Ana", edad: 20 };

let textoJson = JSON.stringify(perfil);
console.log(textoJson); // {"nombre":"Ana","edad":20}

let vuelta = JSON.parse(textoJson);
console.log(vuelta.nombre); // "Ana"
Conexion con almacenamiento

localStorage solo guarda texto. Por eso se guarda con JSON.stringify y se recupera con JSON.parse.

✅ Buenas practicas con arrays y objetos​

Sigue estos consejos...
  1. Evitar arrays con tipos mezclados en proyectos reales.
  2. Mantener un formato consistente en arrays de objetos, mismas claves para todos.
  3. Preferir map, filter y find para trabajar con datos de UI de forma clara.
  4. Tener cuidado con mutaciones, sobre todo si varios sitios usan la misma referencia.
  5. Usar spread para copias rapidas, pero recordar que es copia superficial.

📚 Ejercicios​

Misión: El Gremio de Aventureros

¡Bienvenido al Reino de Scriptonia! Has sido reclutado por el Gremio de Aventureros para gestionar los datos de los héroes, su inventario y las amenazas que acechan en las sombras.

🎒 Misión 1: El Inventario Inicial​

Todo aventurero comienza con lo básico.

  • Objetivo: Crea un array llamado inventario que contenga: "Espada", "Escudo", "Poción".
  • Requisito: Muestra en consola el primer elemento y el número total de objetos.
  • Pista: Usa corchetes [] e índices.

⚔️ Misión 2: Pertrechando al Héroe​

Has encontrado un botín y gastado un objeto.

  • Objetivo:
    1. Añade "Mapa" al final del inventario.
    2. Añade "Cuerda" al principio del inventario.
    3. Elimina el último objeto del array.
  • Pista: Usa push, unshift y pop.

🔍 Misión 3: ¿Llevo la Poción?​

Antes de entrar en la mazmorra, revisas tu mochila.

  • Objetivo: Crea una lógica que verifique si "Poción" está en tu inventario.
  • Requisito: Si está, imprime "¡Listo para el combate! 🧪"; si no, imprime "Necesito ir a la tienda 🏪".
  • Pista: Usa includes() y un condicional if.

👤 Misión 4: Ficha de Personaje​

Define a tu protagonista con todo detalle.

  • Objetivo: Crea un objeto llamado heroe con: nombre, clase (Guerrero, Mago, etc.), nivel (número) y salud (número).
  • Requisito: Añade también una propiedad habilidades que sea un array con dos nombres de ataques.
  • Pista: Usa la estructura de objeto literal {}.

📈 Misión 5: Subida de Nivel​

¡Has derrotado a un trasgo! Tus estadísticas mejoran.

  • Objetivo:
    1. Incrementa el nivel del héroe en 1.
    2. Cambia su salud a 150 usando notación de punto.
    3. Añade una nueva propiedad oro con valor 50 usando notación de corchetes.
  • Pista: Los objetos son mutables.

👿 Misión 6: El Bestiario​

La enciclopedia de monstruos del gremio debe estar organizada.

  • Objetivo: Crea un array de objetos llamado monstruos. Cada objeto debe tener nombre, tipo y nivel.
  • Requisito: Añade al menos 3 monstruos (ej: Orco, Esqueleto, Dragón).
  • Pista: Es un array donde cada posición tiene un {}.

🛡️ Misión 7: Filtrando Peligros​

El Rey solo quiere que te encargues de las amenazas reales.

  • Objetivo: Crea un nuevo array llamado amenazas que contenga solo los monstruos de nivel 10 o superior.
  • Pista: Usa el método filter().

🎯 Misión 8: Caza de Recompensa​

Buscas a un fugitivo específico por su nombre.

  • Objetivo: Encuentra el objeto completo del "Dragón" (o el nombre que hayas puesto) dentro del array de monstruos.
  • Requisito: Imprime su nivel en consola.
  • Pista: Usa el método find().

📜 Misión 9: Listado de Objetos Mágicos​

Quieres que todo tu inventario aparezca en el Registro Real en mayúsculas.

  • Objetivo: Crea un nuevo array basado en inventario donde todos los nombres de objetos estén en MAYÚSCULAS.
  • Pista: Usa map() y toUpperCase().

💎 Misión 10: El Tesoro Total​

Al final de la jornada, el gremio suma el oro de todos los miembros del equipo.

  • Objetivo: Tienes un array de números de oro: [50, 120, 30, 200]. Calcula el total de oro usando un método de array.
  • Pista: Usa reduce().
