3.6 String y Números

En desarrollo web se trabaja constantemente con texto y números. Los textos llegan desde inputs, URLs, atributos HTML y datos de APIs. Los números se usan para precios, contadores, puntuaciones, posiciones o cálculos. En este apartado se reúnen las operaciones y métodos más útiles para manejar ambos con seguridad, evitando errores típicos como concatenaciones inesperadas o decimales mal redondeados.

1️⃣ Strings (texto)​

Un string es una cadena de caracteres. Se puede escribir con comillas simples, dobles o backticks.

🟩 Crear strings​

Tres formas de escribir strings
let a = "Hola";
let b = 'Mundo';
let nombre = "Ana";
let c = `Hola, ${nombre}`;

console.log(a);
console.log(b);
console.log(c);
Recomendación

En web se usa mucho el backtick porque permite interpolar variables con ${...} sin concatenar.

2️⃣ Propiedades y métodos básicos de string​

Los strings incluyen propiedades (datos que puedes consultar, como su longitud) y métodos (funciones que puedes ejecutar sobre el texto). En desarrollo web se usan constantemente para validar inputs, normalizar texto, comparar valores y preparar contenido para mostrarlo en pantalla.

🟩 Longitud: length​

Antes de manipular un texto, muchas veces interesa saber cuántos caracteres tiene. Esto es útil para validar formularios (mínimo de caracteres), limitar mensajes o comprobar si un input está vacío (aunque ojo: si tiene espacios, length también los cuenta).

length
let texto = "JavaScript";
console.log(texto.length); // 10

🟨 Acceder a caracteres​

Un string se puede tratar como una “lista” de caracteres. Esto permite acceder a posiciones concretas, por ejemplo para obtener la primera letra, comprobar un símbolo o extraer el último carácter. El índice empieza en 0, igual que en los arrays.

Acceso por índice
let texto = "Hola";
console.log(texto[0]); // "H"
console.log(texto[texto.length - 1]); // "a"

🟧 Pasar a mayúsculas o minúsculas​

Para comparar textos en web, lo normal es normalizar el contenido. Por ejemplo, si el usuario escribe “ANA”, “Ana” o “ana”, se quiere tratar como lo mismo. Para eso se usan toUpperCase() y toLowerCase(), que devuelven una copia del string transformada.

toUpperCase y toLowerCase
let nombre = "AnA";

console.log(nombre.toUpperCase()); // "ANA"
console.log(nombre.toLowerCase()); // "ana"
Caso típico

Para comparar textos en buscadores, se suele convertir todo a minúsculas.

🟦 Quitar espacios: trim​

Cuando un texto viene de un input, es muy común que el usuario escriba espacios delante o detrás sin querer. trim() elimina esos espacios “sobrantes” al principio y al final, lo cual ayuda a validar mejor campos como nombre, correo o usuario.

trim
let input = "   hola   ";
console.log(input.trim()); // "hola"
Comprobar si un string está vacío (tras trim)

En formularios, un campo puede parecer “relleno” si solo tiene espacios. Por eso, antes de validar, conviene hacer trim() y comparar con "".

Validar string
let texto = "   ";

if (texto.trim() === "") {
  console.log("Campo vacío");
}

3️⃣ Buscar dentro de un texto​

En desarrollo web se necesita buscar texto continuamente: comprobar si un usuario ha escrito una palabra, validar formatos simples, filtrar listas, detectar extensiones de archivo o analizar URLs. Estos métodos permiten responder rápidamente a preguntas del tipo: “¿este texto contiene esto”, “¿empieza por”, “¿en qué posición aparece”.

🟩 includes​

includes sirve para comprobar si un string contiene otro string. Devuelve true o false. Es muy útil para buscadores sencillos, filtros rápidos y validaciones básicas.

includes
let frase = "Aprender JavaScript es útil";
console.log(frase.includes("JavaScript")); // true
console.log(frase.includes("Python")); // false
Detalle útil

includes distingue mayúsculas y minúsculas. En buscadores suele combinarse con toLowerCase().

🟨 startsWith y endsWith​

Estos métodos permiten comprobar si un texto empieza o termina con un fragmento concreto. Se usan mucho en web para validar URLs, detectar extensiones (.jpg, .png), comprobar prefijos (https://) o formatos simples.

startsWith y endsWith
let url = "https://ejemplo.com";

console.log(url.startsWith("https://")); // true
console.log(url.endsWith(".com")); // true
Ejemplo típico

Comprobar si un archivo subido termina en .png o .jpg antes de aceptarlo.

🟧 indexOf​

indexOf devuelve la posición (índice) donde aparece por primera vez un texto dentro de otro. Si no lo encuentra devuelve -1. Es un método clásico, y aunque hoy se use mucho includes, indexOf sigue siendo útil cuando también interesa saber dónde aparece.

indexOf
let frase = "hola hola";
console.log(frase.indexOf("hola")); // 0
console.log(frase.indexOf("adios")); // -1
Idea clave
  • includes responde “¿está o no está”
  • indexOf responde “¿está y en qué posición aparece”

4️⃣ Extraer y modificar strings​

Además de buscar texto, en web se necesita constantemente extraer partes (por ejemplo, un código o un fragmento de URL) y modificar contenido (limpiar texto, reemplazar separadores, transformar formatos). Estos métodos permiten hacerlo de forma directa.

🟩 slice​

slice extrae una parte del string sin modificar el original. Es útil para cortar prefijos, quedarte con una parte concreta o separar un texto en dos trozos (por ejemplo, antes y después de cierta posición).

slice
let texto = "JavaScript";
console.log(texto.slice(0, 4)); // "Java"
console.log(texto.slice(4));    // "Script"
Cómo leerlo

slice(inicio, fin) incluye inicio pero no incluye fin.

🟨 replace y replaceAll​

Estos métodos permiten sustituir texto dentro de un string. Se usan para limpiar formatos, cambiar separadores, corregir partes del contenido o normalizar datos.

  • replace cambia solo la primera coincidencia
  • replaceAll cambia todas las coincidencias
replace
let mensaje = "Hola mundo";
console.log(mensaje.replace("mundo", "clase")); // "Hola clase"
replaceAll
let repetido = "a-a-a";
console.log(repetido.replaceAll("-", ",")); // "a,a,a"
Nota

replace solo cambia la primera coincidencia. Para todas, usar replaceAll o expresiones regulares.

🟧 split y join​

split y join se usan muchísimo porque conectan strings con arrays:

  • split convierte un string en un array (separando por un delimitador)
  • join convierte un array en un string (uniendo con un separador)

Esto es clave en web para procesar listas que vienen como texto, CSV simples, tags, rutas o parámetros.

split
let lista = "rojo,verde,azul";
let colores = lista.split(",");
console.log(colores); // ["rojo","verde","azul"]
join
let colores2 = ["rojo", "verde", "azul"];
console.log(colores2.join(" | ")); // "rojo | verde | azul"
Caso típico
  • split(",") para convertir un input tipo “tags” en array
  • join(", ") para volver a mostrarlo en pantalla como texto

5️⃣ Template literals (texto dinámico)​

Los template literals (strings con backticks ` `) son una forma moderna de crear texto en JavaScript. En web se usan muchísimo porque permiten insertar variables dentro del texto sin tener que concatenar con +, y además facilitan escribir texto multilínea.

Dentro del string puedes “inyectar” variables o incluso expresiones completas.

Interpolar variables
let producto = "Camiseta";
let precio = 19.99;

let salida = `Producto: ${producto}
Precio: ${precio} euros`;

console.log(salida);
en este ejemplo
  • Todo lo que esté dentro de ${ ... } se evalúa como JavaScript
  • El resultado se convierte a texto y se coloca dentro del string final
  • El salto de línea se mantiene porque el texto está escrito en varias líneas

6️⃣ Números: conceptos básicos​

En JavaScript, casi todos los números que se usan a diario pertenecen al tipo number. Lo importante es que number es un tipo de coma flotante (IEEE 754, doble precisión). En la práctica esto significa dos cosas:

  • JavaScript no tiene un tipo separado para enteros y decimales: todo es number
  • algunos decimales no se representan exactamente y pueden aparecer errores de precisión (lo verás en el apartado de decimales)

En web esto es clave porque muchos datos vienen como texto y hay que convertirlos bien para evitar errores.

🟩 Conversión a número​

Cuando un valor viene de un input (input.value), de una URL o de un JSON, normalmente llega como string. Si intentas operar directamente, puedes provocar concatenación en lugar de suma.

Ejemplo de problema típico: concatenación
let a = "10";
let b = "5";

console.log(a + b); // "105" (concatena, no suma)

Por eso, antes de operar, conviene convertir.

1. Number() (conversión “estricta” a número)​

Number() intenta convertir el string completo a número.

  • si el string es numérico, devuelve un number
  • si no se puede convertir, devuelve NaN
Number()
let textoPrecio = "12.50";
let precio = Number(textoPrecio);

console.log(precio);        // 12.5
console.log(typeof precio); // "number"
Number() con valores no válidos
console.log(Number("123"));     // 123
console.log(Number("12.9"));    // 12.9
console.log(Number("20px"));    // NaN (no puede convertir el string completo)
Cuándo usar Number()

Cuando quieres asegurarte de que el dato es numérico “de verdad” y no aceptas texto mezclado como "20px".

2. parseInt() y parseFloat() (conversión “parcial”)​

Estas funciones leen el número desde el inicio del string:

  • parseInt lee un entero tiene dos parámetros (el valor a convertir y la base numérica 2-36)
  • parseFloat lee un decimal
  • si el texto empieza por un número, lo extraen aunque luego haya letras
parseInt y parseFloat
console.log(parseInt("20px", 10));   // 20
console.log(parseFloat("3.14km"));   // 3.14
¿Para qué sirve esto en web?

Es útil cuando un dato viene con unidades o formato, por ejemplo "20px", "15.5rem" o "3.2km".

Ojo

parseInt("12.9") devuelve 12. No redondea, corta.

7️⃣ Operaciones y métodos útiles de números​

🟩 Comprobar si algo es número​

En JavaScript, no todo lo que “parece” un número lo es. Además, hay dos casos habituales:

  • comprobar si un valor es numérico (o se puede convertir a número)
  • comprobar si un valor es exactamente NaN

Por eso conviene distinguir entre:

  • isNaN(valor) ✅ convierte antes (a veces da resultados “raros”)
  • Number.isNaN(valor) ✅ no convierte, solo devuelve true si el valor ya es NaN
isNaN vs Number.isNaN
console.log(isNaN("hola"));          // true  (convierte "hola" -> NaN)
console.log(Number.isNaN("hola"));   // false (no convierte)

console.log(isNaN("123"));           // false (convierte "123" -> 123)
console.log(isNaN("123px"));         // true  (convierte -> NaN)

console.log(Number.isNaN(NaN));      // true
console.log(isNaN(NaN));             // true
Regla práctica (frontend)

Si quieres validar un input que debe ser número:

  1. conviértelo con Number(...)
  2. comprueba con Number.isNaN(...)
Validar que un input sea número
let valor = "12.5"; // simulando input.value

let n = Number(valor);

if (Number.isNaN(n)) {
  console.log("No es un número válido");
} else {
  console.log("Número válido:", n);
}
Ojo con strings vacíos

Number("") devuelve 0. Por eso, si el dato viene de un input, conviene validar también el trim().

Evitar que '' cuente como 0
let valor = "   ";

if (valor.trim() === "") {
  console.log("Campo vacío");
} else {
  let n = Number(valor);
  console.log(Number.isNaN(n) ? "No es número" : "OK");
}

🟨 Redondeos: Math​

En web se redondea constantemente, sobre todo cuando se trabaja con:

  • precios y totales de carrito
  • porcentajes (descuentos, progreso)
  • posiciones o tamaños calculados (coordenadas, píxeles)
  • paginación (calcular cuántas páginas hacen falta)

JavaScript ofrece varias funciones en Math para redondear, y es importante elegir bien porque no hacen lo mismo:

  • Math.round(x) redondea al entero más cercano
  • Math.floor(x) “baja” al entero inferior (corta hacia abajo)
  • Math.ceil(x) “sube” al entero superior (corta hacia arriba)
Math.round, floor, ceil
console.log(Math.round(3.5)); // 4
console.log(Math.floor(3.9)); // 3
console.log(Math.ceil(3.1));  // 4
Ejemplos típicos
  • floor: convertir un índice o posición a entero sin pasarte
  • ceil: calcular páginas necesarias, por ejemplo ceil(totalItems / itemsPorPagina)
  • round: mostrar un número “bonito” al usuario

🟧 Limitar decimales: toFixed​

Cuando se muestran números en pantalla, muchas veces se necesita limitar decimales, por ejemplo:

  • precios con 2 decimales
  • porcentajes con 1 o 2 decimales
  • medias o estadísticas

toFixed(n) devuelve el número con n decimales, pero hay un detalle clave: devuelve un string, porque está pensado para “formato de salida”, no para cálculos.

toFixed
let total = 10 / 3;
console.log(total.toFixed(2)); // "3.33"

Si después quieres seguir calculando con ese valor, entonces conviertes a número:

Convertir toFixed a número
let total = 10 / 3;
let redondeado = Number(total.toFixed(2));
console.log(redondeado); // 3.33
Idea clave
  • toFixed es perfecto para mostrar
  • para cálculos, normalmente se trabaja con el number original y solo se formatea al final

🟥 Mínimo y máximo​

Estas funciones se usan mucho en frontend para controlar valores dentro de límites.

  • Math.min y Math.max ayudan a obtener extremos de varios valores
  • también se usan para limitar valores, por ejemplo “que nunca baje de 0 ni suba de 100”
Math.min y Math.max
console.log(Math.min(5, 2, 9)); // 2
console.log(Math.max(5, 2, 9)); // 9
Math.random
let aleatorio = Math.random(); // 0 a 1
console.log(aleatorio);

8️⃣ Problemas típicos con decimales (importante)​

En JavaScript (y en la mayoría de lenguajes), los números decimales se guardan en memoria usando un sistema llamado coma flotante. El detalle importante es que muchos decimales “normales” en base 10 no se pueden representar exactamente en binario, igual que 1/3 no se puede representar exactamente en decimal (0.3333... infinito).

Por eso, a veces aparecen resultados “raros” al sumar o restar decimales, aunque matemáticamente parezcan simples.

Precisión decimal
console.log(0.1 + 0.2); // 0.30000000000000004

🧠 ¿Por qué pasa esto?

  • 0.1 y 0.2 no se almacenan como “0.1 exacto” y “0.2 exacto”
  • se almacenan como aproximaciones binarias muy cercanas
  • cuando las sumas, el resultado también es una aproximación y puede mostrar un pequeño error

Esto no suele ser un problema en la mayoría de cálculos visuales, pero sí es crítico en dinero (precios, descuentos, totales), porque un error de 0.00000000004 puede acabar afectando a redondeos, comparaciones o validaciones.

Ejemplo de bug típico

Si comparas directamente decimales, puede fallar:

Comparación peligrosa
console.log(0.1 + 0.2 === 0.3); // false

Por eso, para comparar decimales, normalmente se usa tolerancia o se redondea antes.

Solución práctica en web

Para precios, se suele trabajar en céntimos (enteros) y luego convertir a euros solo para mostrar. La idea es:

  • guardar 1,99 € como 199
  • guardar 2,50 € como 250
  • operar siempre con enteros (sin errores de coma flotante)
  • al final, dividir entre 100 y formatear
Precios en céntimos
let precio1 = 199; // 1.99 euros
let precio2 = 250; // 2.50 euros

let totalCentimos = precio1 + precio2; // 449
let totalEuros = (totalCentimos / 100).toFixed(2);

console.log(totalEuros); // "4.49"

9️⃣ Números aleatorios (Math.random)​

En web se usan números aleatorios para cosas como:

  • elegir un mensaje al azar
  • generar un número de verificación simple (no seguro)
  • barajar opciones de un juego
  • seleccionar un elemento aleatorio de una lista
Importante

Math.random no es seguro para cosas de seguridad (passwords, tokens, etc.). Para eso se usa crypto.getRandomValues (se verá más adelante si hace falta).

🟩 Math.random() básico​

Math.random() devuelve un número decimal entre 0 (incluido) y 1 (excluido).

Math.random básico
let r = Math.random();
console.log(r); // 0 <= r < 1

🟨 Entero aleatorio en un rango (fórmula clave)​

Para un entero entre min y max (incluidos):

Entero aleatorio entre min y max
function enteroAleatorio(min, max) {
  return Math.floor(Math.random() * (max - min + 1)) + min;
}

console.log(enteroAleatorio(1, 10));
Explicación rápida
  • Math.random() genera 0..1
  • se escala al tamaño del rango con * (max - min + 1)
  • Math.floor lo convierte a entero
  • + min lo desplaza al rango correcto

🟦 Ejemplo típico: número aleatorio del 1 al 6 (dado)​

Dado
function tirarDado() {
  return Math.floor(Math.random() * 6) + 1;
}

console.log(tirarDado());

🟧 Elegir un elemento aleatorio de un array​

Elemento aleatorio de un array
let frases = ["hola", "buenas", "que tal", "bienvenido"];

function elementoAleatorio(lista) {
  let i = Math.floor(Math.random() * lista.length);
  return lista[i];
}

console.log(elementoAleatorio(frases));

📚 Ejercicios​

Misión: CriptoDetective - El Código Enigma

Has sido contratado por la Agencia de Inteligencia de Scriptonia. Tu misión es limpiar datos, descifrar mensajes y asegurar que la base de datos de sospechosos esté en perfecto orden.

🕵️ Misión 1: Registro de Intrusos​

Un sospechoso ha dejado su nombre en el sistema, pero está lleno de espacios innecesarios.

  • Objetivo: Toma este texto: " Agente_K ".
  • Requisito: Elimina los espacios y muestra cuántos caracteres tiene el nombre limpio.
  • Pista: Usa trim() y length.

📛 Misión 2: El Formateador de Identidades​

Los nombres de los agentes deben estar normalizados: la primera letra en mayúscula y el resto en minúsculas.

  • Objetivo: Convierte cualquier nombre (ej: "mArTa") a "Marta".
  • Requisito: Debes hacerlo en una sola línea de código (o encadenando métodos).
  • Pista: Usa toLowerCase(), slice() y toUpperCase().

🚫 Misión 3: Palabras Prohibidas​

Tenemos una lista de palabras que no pueden aparecer en los mensajes secretos.

  • Objetivo: Comprueba si el mensaje "El servidor fallará a medianoche" contiene la palabra prohibida "fallará".
  • Requisito: Muestra true o false en consola.
  • Pista: Usa el método includes().

🌐 Misión 4: Extracción de Servidores​

De un enlace de base de datos, necesitamos saber a qué servidor pertenece.

  • Objetivo: De la URL "servidor01-db-madrid", extrae solo el nombre del servidor (todo lo que hay antes del primer guion).
  • Requisito: El resultado debe ser "servidor01".
  • Pista: Usa split("-") e índices.

🔑 Misión 5: Generador de Pines Temporales​

Necesitamos generar un código PIN de 4 dígitos para los agentes de campo.

  • Objetivo: Crea una función que devuelva un número entero aleatorio entre 1000 y 9999.
  • Pista: Usa Math.random() y Math.floor().

💶 Misión 6: El Conversor de Divisas Interno​

Tenemos que pagar a los informantes en euros, pero los datos vienen de una API con muchos decimales.

  • Objetivo: Toma el pago de 1245.6789 euros y redondéalo a exactamente 2 decimales para el recibo.
  • Requisito: El resultado debe ser el string "1245.68".
  • Pista: Usa toFixed(2).

📂 Misión 7: Verificador de Evidencias​

Solo podemos abrir archivos que sean imágenes seguras.

  • Objetivo: Comprueba si el archivo "sospechoso_visto_en_metro.jpg" termina con la extensión .jpg.
  • Pista: Usa endsWith().

🧹 Misión 8: Limpieza de Pruebas​

Alguien ha intentado sabotear un informe cambiando los espacios por guiones bajos.

  • Objetivo: Toma el texto "El_sospechoso_huyó_por_la_derecha" y cambia todos los guiones bajos por espacios.
  • Pista: Usa replaceAll().

📝 Misión 9: Informe de Misión Dinámico​

Debes generar un mensaje automático para la central.

  • Objetivo: Usa variables para agente, caso y estado.
  • Requisito: Crea un string que diga: "El Agente [X] informa que el caso [Y] está en estado: [Z]".
  • Pista: Usa Template Literals (backticks).

📊 Misión 10: La Cuota de Precisión​

Para finalizar la misión, debemos calcular cuántas páginas de informe necesitamos imprimir.

  • Objetivo: Si un informe tiene 45 secciones y en cada página caben 10 secciones, calcula cuántas páginas enteras necesitamos (siempre redondeando hacia arriba).
  • Requisito: El resultado debe ser 5.
  • Pista: Usa Math.ceil().
