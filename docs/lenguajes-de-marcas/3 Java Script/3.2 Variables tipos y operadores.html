3.2 Variables, Tipos y Operadores

En cualquier programa web, casi todo se reduce a tres ideas que se repiten una y otra vez: guardar datos, saber qué tipo de datos se tienen, y operar con ellos para tomar decisiones o construir resultados. En este apartado vamos a trabajar en estos conceptos.

1️⃣ Variables en JavaScript​

En JavaScript, las variables son contenedores que almacenan valores. Pueden contener desde números y cadenas de texto hasta objetos complejos. Declarar variables correctamente es fundamental para garantizar un código legible, eficiente y sin errores.

JavaScript ofrece tres formas principales de declarar variables: var, let y const. Cada una tiene características específicas relacionadas con el ámbito (scope), reasignación y uso recomendado.

🟩 var​

var es la forma más antigua de declarar variables en JavaScript. Aunque sigue funcionando, su uso está desaconsejado en proyectos modernos debido a ciertos problemas con su comportamiento.

Características de var​

  1. Ámbito de función:

    • Una variable declarada con var es accesible en toda la función donde se declara, incluso fuera de bloques específicos.
    • No respeta el ámbito de bloque {}.

  2. Reasignable y redeclarable:

    • Puedes reasignar y redeclarar variables declaradas con var.
Ejemplo de uso de var
var x = 10;
if (true) {
    var x = 20; // Sobrescribe la misma variable
    console.log(x); // 20
}
console.log(x); // 20 (acceso fuera del bloque)
Problema con var

En manuales modernos se prioriza let y const. var aparece en código antiguo y tiene un comportamiento distinto con el ámbito.

El alcance de función puede causar confusión y errores no deseados, especialmente en estructuras como bucles.

Ejemplo de error típico con var
for (var i = 0; i < 3; i++) {
    setTimeout(() => console.log(i), 1000);
}
// Resultado: 3, 3, 3 (porque `i` es global al bucle)

🟨 let​

let es una forma más moderna de declarar variables y está diseñada para evitar los problemas asociados con var.

Características de let​

  1. Ámbito de bloque:

    • La variable solo es accesible dentro del bloque {} donde se declara.

  2. Reasignable, pero no redeclarable en el mismo ámbito:

    • Puedes cambiar el valor de una variable let, pero no puedes declararla de nuevo en el mismo ámbito.
Ejemplo de uso de let
let y = 10;
if (true) {
    let y = 20; // Nueva variable en este bloque
    console.log(y); // 20
}
console.log(y); // 10 (la variable original permanece intacta)

Uso en bucles​

let funciona mejor en bucles debido a su ámbito de bloque.

Ejemplo correcto con let:

for (let i = 0; i < 3; i++) {
    setTimeout(() => console.log(i), 1000);
}
// Resultado: 0, 1, 2 (cada iteración tiene su propio ámbito)

🟧 const​

const se utiliza para declarar constantes, es decir, valores que no pueden ser reasignados. Es la mejor opción para valores que no cambian a lo largo de la ejecución del programa.

Características de const​

  1. Ámbito de bloque:

    • Igual que let, la constante solo es accesible dentro del bloque donde se declara.

  2. No reasignable:

    • Una vez asignado un valor a una constante, no se puede cambiar.

  3. No redeclarable:

    • Similar a let, no puedes declarar otra constante con el mismo nombre en el mismo ámbito.
Ejemplo de uso de const
const z = 30;
console.log(z); // 30
// z = 40; // Error: no se puede reasignar
Uso con objetos y arrays

Aunque no se puede reasignar un objeto o array declarado con const, sus propiedades internas pueden modificarse.

Ejemplo
const persona = { nombre: "Ana", edad: 30 };
persona.edad = 31; // Se permite cambiar propiedades
console.log(persona.edad); // 31

const numeros = [1, 2, 3];
numeros.push(4); // Se permite modificar el contenido
console.log(numeros); // [1, 2, 3, 4]

🟥 Diferencias entre var, let y const​

DeclaraciónÁmbitoReasignableRedeclarableUso recomendado
varFunciónSíSíEvitar su uso
letBloqueSíNoUsar para variables mutables
constBloqueNoNoUsar para constantes

✅ Buenas prácticas​

Sigue estos consejos...
  1. Evita el uso de var: var está obsoleto en la mayoría de las prácticas modernas y puede causar errores difíciles de depurar.
  2. Usa nombres descriptivos: Los nombres de las variables deben reflejar su propósito.
  3. Agrupa declaraciones relacionadas: Coloca las declaraciones de variables al principio del bloque donde se usarán.

2️⃣ Tipos de Datos​

En JavaScript, los tipos de datos representan las diversas formas en las que se pueden almacenar y manipular valores. Estos tipos se dividen en primitivos y complejos. ✨ Entender cómo funcionan y se comportan es fundamental para escribir código eficiente y libre de errores.

🟩 Clasificación de Tipos de Datos​

  1. Primitivos:

    • Representan un único valor inmutable.
    • Se almacenan directamente en la memoria.

  2. Complejos:

    • Representan colecciones de valores o estructuras más elaboradas.
    • Se almacenan por referencia.

🟨 Tipos Primitivos​

Los tipos primitivos son básicos y simples. JavaScript tiene los siguientes tipos de datos primitivos:

TipoDescripciónEjemplo
NumberRepresenta números (enteros o decimales).42, 3.14, NaN
StringCadenas de texto delimitadas por comillas.'Hola', "Mundo"
BooleanValores lógicos (verdadero o falso).true, false
UndefinedVariable declarada pero no inicializada.let x;
NullRepresenta un valor intencionadamente vacío.let x = null;
SymbolIdentificadores únicos e inmutables.Symbol('id')
BigIntNúmeros mayores que el límite de Number.12345678901234567890n

1. Number 🔢​

El tipo Number se utiliza para representar números enteros o decimales. También incluye valores especiales como Infinity, -Infinity y NaN (Not a Number).

Ejemplo
let entero = 42;
let decimal = 3.14;
let infinito = Infinity;
let noEsNumero = "hola" * 2; // NaN
console.log(entero, decimal, infinito, noEsNumero);

2. String 📝​

Una cadena de texto es una secuencia de caracteres encerrada entre comillas simples ('), dobles (") o invertidas (```).

Ejemplo
let saludo = "Hola";
let nombre = 'Juan';
let mensaje = `Hola, ${nombre}`; // Template literal
console.log(saludo, nombre, mensaje);
Características importantes
  • Los template literals (usando comillas invertidas) permiten interpolación de variables y expresiones.

3. Boolean ✔️❌​

Los valores booleanos solo pueden ser true o false. Son útiles para representar estados lógicos o resultados de comparaciones.

Ejemplo
let activo = true;
let inactivo = false;
console.log(5 > 3); // true
console.log(5 < 3); // false

4. Undefined 🕳️​

Indica que una variable ha sido declarada pero no se le ha asignado un valor.

Ejemplo
let x;
console.log(x); // undefined

5. Null ⛔​

Representa la ausencia intencional de un valor. Es diferente de undefined, ya que null se asigna explícitamente.

Ejemplo
let y = null;
console.log(y); // null

6. Symbol 🔑​

Introducido en ES6, Symbol se utiliza para crear identificadores únicos.

Ejemplo
let id1 = Symbol("id");
let id2 = Symbol("id");
console.log(id1 === id2); // false

7. BigInt 🧮​

BigInt es un tipo de dato para números enteros extremadamente grandes.

Ejemplo
let numeroGrande = 123456789012345678901234567890n;
console.log(numeroGrande);

🟧 Tipos Complejos​

Los tipos complejos permiten representar estructuras más elaboradas y almacenar colecciones de datos. En JavaScript, los tipos complejos incluyen:

TipoDescripciónEjemplo
ObjectColección de pares clave-valor.{nombre: "Ana", edad: 30}
ArrayLista ordenada de valores.[1, 2, 3]
FunctionBloque de código ejecutable.function saludar() { return "Hola"; }
DateRepresenta una fecha y hora.new Date()
RegExpExpresiones regulares./\d+/

1. Object 🧳​

Los objetos son colecciones de pares clave-valor. Se utilizan para estructurar datos complejos.

Ejemplo
let persona = {
    nombre: "Juan",
    edad: 25,
    activo: true
};
console.log(persona.nombre); // "Juan"
persona.edad = 26; // Modifica una propiedad

2. Array 📋​

Los arrays son listas ordenadas de valores, donde cada elemento tiene un índice numérico.

Ejemplo
let numeros = [1, 2, 3, 4];
console.log(numeros[0]); // 1
numeros.push(5); // Agrega un elemento
console.log(numeros); // [1, 2, 3, 4, 5]

3. Function 🛠️​

Una función es un bloque de código reutilizable.

Ejemplo
function sumar(a, b) {
    return a + b;
}
console.log(sumar(2, 3)); // 5

4. Date 🗓️​

El tipo Date se utiliza para trabajar con fechas y horas.

Ejemplo
let hoy = new Date();
console.log(hoy.toISOString()); // Fecha actual en formato ISO

5. RegExp 🔍​

Las expresiones regulares son patrones utilizados para buscar o manipular cadenas.

Ejemplo
let patron = /\d+/;
console.log(patron.test("123")); // true
console.log(patron.test("abc")); // false

🟥 El operador typeof​

El operador typeof se utiliza para verificar el tipo de dato de una variable.

Ejemplo
let numero = 42;
let texto = "Hola";
let booleano = true;

console.log(typeof numero); // "number"
console.log(typeof texto); // "string"
console.log(typeof booleano); // "boolean"

🟪 Conversión de tipos​

JavaScript permite convertir entre diferentes tipos de datos.

Conversión a número 🔢​

let texto = "123";
console.log(Number(texto)); // 123

Conversión a cadena 🖋️​

let numero = 42;
console.log(String(numero)); // "42"

Conversión a booleano ✅​

let valor = 0;
console.log(Boolean(valor)); // false

3️⃣ Operadores en JavaScript​

Los operadores son fundamentales en cualquier lenguaje de programación 🧮, ya que permiten realizar cálculos, comparaciones, combinaciones lógicas y otras operaciones sobre los datos. En JavaScript, los operadores están organizados en varias categorías según su funcionalidad. 🚀

🟩 Categorías de operadores​

  1. Aritméticos ➕ ➖: Para realizar operaciones matemáticas.
  2. Asignación 📝: Para asignar valores a variables.
  3. Comparación 🔍: Para comparar valores.
  4. Lógicos 🤔: Para combinar o invertir valores booleanos.
  5. Ternario ❓: Operador condicional en una sola línea.
  6. De cadena 🧵: Para trabajar con texto.
  7. Especiales 🌟: Incluyen el operador de coalescencia nula (??), el operador de propagación (...), entre otros.

🟨 Operadores aritméticos​

Se utilizan para realizar operaciones matemáticas básicas. ⚙️

OperadorDescripciónEjemploResultado
+Suma ➕5 + 38
-Resta ➖10 - 46
*Multiplicación ✖️2 * 36
/División ➗10 / 25
%Módulo (resto) 🪙10 % 31
**Exponenciación 🌟2 ** 38
++Incremento 📈let a = 1; a++;2
--Decremento 📉let b = 2; b--;1
Ejemplo práctico
let a = 10;
let b = 5;

console.log(a + b); // 15
console.log(a - b); // 5
console.log(a * b); // 50
console.log(a / b); // 2
console.log(a % b); // 0
console.log(2 ** 3); // 8
Sobre ++ y --
  • Si se usan antes de la variable (++a o --a), el incremento o decremento ocurre antes de evaluar la expresión (preincremento).
  • Si se usan después (a++ o a--), la expresión se evalúa antes del cambio (postincremento).
Ejemplo de pre y postincremento
let a = 1;
let b = 1;

// PREINCREMENTO: primero suma, luego devuelve el valor
const resultadoPre = ++a; // a pasa a 2, y resultadoPre vale 2

// POSTINCREMENTO: primero devuelve el valor, luego suma
const resultadoPost = b++; // resultadoPost vale 1, y luego b pasa a 2

console.log("a:", a); // 2
console.log("resultadoPre:", resultadoPre); // 2
console.log("b:", b); // 2
console.log("resultadoPost:", resultadoPost); // 1

🟧 Operadores de asignación​

Estos operadores asignan un valor a una variable. Algunos combinan asignación y operación matemática. 🔗

OperadorDescripciónEjemploResultado
=Asignación simplex = 5x = 5
+=Asignación con sumax += 3x = x + 3
-=Asignación con restax -= 2x = x - 2
*=Asignación con multiplicaciónx *= 4x = x * 4
/=Asignación con divisiónx /= 5x = x / 5
%=Asignación con módulox %= 2x = x % 2
Ejemplo práctico
let x = 10;
x += 5; // x = 15
x *= 2; // x = 30
console.log(x); // 30

🟥 Operadores de comparación​

Estos operadores comparan dos valores y devuelven un resultado booleano (true o false). ✅❌

OperadorDescripciónEjemploResultado
==Igualdad débil (convierte tipos)5 == '5'true
===Igualdad estricta (sin conversión)5 === '5'false
!=Desigualdad débil5 != '5'false
!==Desigualdad estricta5 !== '5'true
>Mayor que10 > 5true
<Menor que3 < 7true
>=Mayor o igual5 >= 5true
<=Menor o igual4 <= 4true
Ejemplo práctico
console.log(5 == "5");  // true (igualdad débil)
console.log(5 === "5"); // false (igualdad estricta)
console.log(10 > 5);    // true
console.log(10 <= 5);   // false

🟪 Operadores lógicos​

Estos operadores se usan para combinar o invertir valores booleanos. 🔄

OperadorDescripciónEjemploResultado
&&AND lógico (y)true && falsefalse
|| OR lógico (o)true || falsetrue
!NOT lógico (no)!truefalse
Ejemplo práctico
let a = true;
let b = false;

console.log(a && b); // false
console.log(a || b); // true
console.log(!a);     // false

🟦 Truthy y falsy​

En JavaScript, cuando se evalúa una condición (por ejemplo en un if) no siempre se usa un booleano literal. El lenguaje convierte el valor a booleano de forma automática y lo interpreta como:

  • Truthy: se comporta como true
  • Falsy: se comporta como false

Esto es clave en web porque muchos valores vienen de inputs, estados y variables que no son booleanos.

Valores falsy (los más importantes)​

Estos valores siempre se interpretan como false en condiciones:

  • false
  • 0
  • "" (cadena vacía)
  • null
  • undefined
  • NaN
Ejemplo falsy en if
let texto = "";

if (texto) {
  console.log("Entra");
} else {
  console.log("No entra"); // ✅
}
Regla rápida

Si no está en la lista de falsy, entonces es truthy. Por ejemplo: "0", "false", [] y {} son truthy.

Conversión rápida a booleano​

Boolean() y doble negación
console.log(Boolean("hola")); // true
console.log(Boolean(""));     // false

console.log(!!"hola"); // true
console.log(!!0);      // false

&& y || no siempre devuelven true o false​

Aunque se llamen operadores lógicos, en JavaScript suelen devolver uno de los operandos:

  • A && B devuelve el primer falsy o el último valor si todo es truthy
  • A || B devuelve el primer truthy o el último valor si todo es falsy
Short-circuit con valores no booleanos
console.log("hola" && 123); // 123
console.log("" && 123);     // ""  (se para en el primer falsy)

console.log(null || "defecto"); // "defecto"
console.log("ok" || "defecto"); // "ok"
Idea clave
  • Usa || si quieres un default cuando el valor sea falsy en general
  • Usa ?? si solo quieres un default cuando falte el valor (null o undefined)

🟫 Operador ternario​

El operador ternario es una forma abreviada de escribir una condición if...else. ✂️

✍️ Sintaxis del operador ternario
condición ? valor_si_verdadero : valor_si_falso;
Ejemplo práctico
let edad = 20;
let mensaje = edad >= 18 ? "Eres mayor de edad" : "Eres menor de edad";
console.log(mensaje); // "Eres mayor de edad"

🟩 Operadores especiales​

1. Operador de coalescencia nula (??)​

Devuelve el primer operando que no sea null ni undefined.

Ejemplo práctico
let nombre = null;
let nombrePorDefecto = "Usuario";
console.log(nombre ?? nombrePorDefecto); // "Usuario"

2. Operador de propagación (...)​

Se utiliza para descomponer arrays u objetos.

Ejemplo con arrays 🔢
let numeros = [1, 2, 3];
let nuevosNumeros = [...numeros, 4, 5];
console.log(nuevosNumeros); // [1, 2, 3, 4, 5]
Ejemplo con objetos 🧱
let persona = { nombre: "Juan", edad: 30 };
let personaActualizada = { ...persona, ciudad: "Madrid" };
console.log(personaActualizada); // { nombre: "Juan", edad: 30, ciudad: "Madrid" }

✅ Buenas prácticas con operadores​

Sigue estos consejos...
  1. Usar igualdad estricta (===) siempre que sea posible: Evita errores relacionados con la conversión automática de tipos.
  2. Priorizar claridad sobre brevedad: Aunque el operador ternario es útil, evita usarlo en condiciones muy largas o complejas.
  3. Combina operadores lógicos sabiamente: En JavaScript, no todos los operadores se evalúan en el orden en el que se leen. El lenguaje aplica una precedencia (prioridad) parecida a la de las matemáticas: por ejemplo, multiplicar y dividir se hace antes que sumar y restar.

Para evitar dudas y errores, se recomienda:

  • Usar paréntesis cuando una expresión tenga varias operaciones
  • Si una condición mezcla && y ||, poner paréntesis aunque funcione sin ellos
console.log(1 + 2 * 3);     // 7  (primero 2*3, luego +1)
console.log((1 + 2) * 3);   // 9  (primero (1+2), luego *3)

const a = true;
const b = false;
const c = true;

// Sin paréntesis puede ser confuso
console.log(a || b && c);       // true (&& se evalúa antes que ||)

// Mejor: expresión explícita
console.log(a || (b && c));     // true
console.log((a || b) && c);     // true
