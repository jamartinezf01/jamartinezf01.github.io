3.7 DOM

Introducción al DOM​

¡El DOM (Document Object Model) 🌐 es una interfaz de programación para documentos HTML y XML. Permite a los desarrolladores 👩‍💻👨‍💻 acceder y manipular los contenidos 📄, la estructura 🧱 y los estilos 🎨 de un documento de manera dinámica. Es la base para la interacción entre JavaScript y el navegador 🌍.

El DOM representa el documento como un árbol de nodos 🌳, donde cada nodo corresponde a una parte del documento, como un elemento HTML 🏷️, un atributo 🔑 o un contenido de texto 📝.

¿Qué es el DOM? ❓​

El DOM es una representación en memoria 💾 del contenido de un documento web. Cuando un navegador carga una página web:

1️⃣ El navegador analiza el HTML y lo convierte en un árbol de nodos 🌲.
2️⃣ Este árbol puede ser manipulado mediante JavaScript 🖥️ para actualizar dinámicamente el contenido o la estructura de la página 🔄.

Características principales:

CaracterísticaDescripción
Estructura jerárquica 🌳El DOM organiza el documento como un árbol, con nodos padres, hijos y hermanos.
Interactividad 🖱️Permite modificar la estructura, el contenido y los estilos del documento.
Estándar del W3C 🏢Es una especificación definida por el World Wide Web Consortium (W3C).

Relación entre HTML, CSS y el DOM 🌐​

El DOM es el punto de conexión 🔗 entre las tecnologías web principales:

  • HTML define la estructura del contenido 🏗️.
  • CSS se utiliza para los estilos visuales 🎨.
  • JavaScript manipula el DOM para agregar interactividad ⚡.

Por ejemplo, cuando cambias el color de un elemento con CSS 🎨, el DOM actualiza su representación visual en el navegador 🖥️. Cuando usas JavaScript para cambiar el texto de un párrafo ✏️, el DOM modifica el nodo correspondiente y refleja el cambio en la interfaz.

DOM vs. BOM 🤔​

Es importante distinguir entre dos conceptos clave:

ConceptoDescripción
DOM 📄Interfaz que permite manipular la estructura y contenido del documento.
BOM 🌐(Browser Object Model) Permite interactuar con el navegador (ventanas, URL, historial).

Estructura Jerárquica del DOM 🏗️​

El documento HTML se convierte en un árbol de nodos 🌲, donde cada etiqueta es un nodo 🏷️ y cada texto o atributo es un nodo hijo. Por ejemplo, el siguiente código HTML:

<!DOCTYPE html>
<html>
  <head>
    <title>Mi página</title>
  </head>
  <body>
    <h1>Hola, DOM</h1>
    <p>Este es un párrafo.</p>
  </body>
</html>

Ventajas del DOM ✅​

1️⃣ Interactividad Dinámica: Puedes modificar elementos sin recargar la página (por ejemplo, añadir nuevos botones o actualizar contenido).
2️⃣ Accesibilidad: Ofrece métodos y propiedades estándar para manipular cualquier nodo del documento.
3️⃣ Flexibilidad: Compatible con documentos HTML y XML.

Ejemplo Básico del DOM 📚​

Un ejemplo sencillo para entender cómo JavaScript interactúa con el DOM:

HTML:

<!DOCTYPE html>
<html>
  <body>
    <h1 id="titulo">Bienvenido</h1>
    <button id="cambiarTexto">Cambiar Texto</button>

    <script src="script.js"></script>
  </body>
</html>

JavaScript (script.js):

// Seleccionar el nodo del título
const titulo = document.getElementById('titulo');
// Selecciona el nodo botón
const boton = document.getElementById('cambiarTexto');
// Se asigna al boton un escuchador de eventos (evento click en este caso)
boton.addEventListener('click', () => {
  // Cambiar el texto cuando se haga clic en el botón
  titulo.textContent = "¡Hola, DOM!";
});

Cuando el usuario hace clic en el botón 🖱️, el DOM actualiza dinámicamente el contenido del h1 sin necesidad de recargar la página 🔄.
