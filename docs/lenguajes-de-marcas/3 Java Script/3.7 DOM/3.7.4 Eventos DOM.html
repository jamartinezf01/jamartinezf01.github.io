3.7.4 Eventos DOM

Los eventos en el DOM son interacciones o sucesos que ocurren dentro de una página web. Estas interacciones pueden ser disparadas por el usuario, como un clic o el envío de un formulario, o por el navegador, como la carga completa del documento. Los eventos son esenciales para crear páginas web dinámicas e interactivas, ya que permiten responder a las acciones de los usuarios.

1️⃣ ¿Qué es un evento?​

Un evento es una señal que indica que algo ocurrió. Algunos ejemplos de eventos comunes incluyen:

  • Hacer clic en un botón (click)
  • Pasar el ratón (mouse) sobre un elemento (mouseover)
  • Escribir texto en un campo (input)
  • Cargar completamente una página (DOMContentLoaded)

Cuando ocurre un evento, el navegador genera un objeto evento que contiene información relevante, como el tipo de evento, el elemento que lo disparó y el momento en que ocurrió.

2️⃣ Tipos de eventos en el DOM​

En el DOM existen diversos tipos de eventos que permiten interactuar con los elementos de una página web. Estos eventos se agrupan en categorías según su naturaleza y propósito: eventos del ratón, del teclado, de formularios, de carga, entre otros. A continuación, se muestran las categorías más importantes y sus eventos más comunes.

🟩 Eventos del ratón (mouse)​

Estos eventos se generan a partir de interacciones con el ratón, como clics, movimientos o cambios en su posición.

EventoDescripciónEjemplo de uso
clickSe dispara cuando el usuario hace clicBotones, enlaces
dblclickSe dispara al hacer doble clicAbrir un elemento en detalle
mouseoverSe dispara cuando el ratón pasa sobre un elementoMostrar un tooltip
mouseoutSe dispara cuando el ratón sale de un elementoOcultar un tooltip
mousedownSe dispara cuando el botón del ratón es presionadoIniciar un arrastre
mouseupSe dispara cuando se suelta el botón del ratónFinalizar un arrastre
mousemoveSe dispara cuando el ratón se mueve sobre un elementoEfectos visuales al mover el ratón
contextmenuSe dispara al hacer clic derechoMenú contextual personalizado
wheelSe dispara al mover la rueda del ratónscroll personalizado
Ejemplo práctico: mostrar coordenadas del ratón
<div id="zona" style="width: 300px; height: 200px; border: 1px solid black;">
  Mueve el ratón aquí
</div>

<script>
  const zona = document.getElementById("zona");

  zona.addEventListener("mousemove", (evento) => {
    zona.textContent = `Coordenadas: X=${evento.clientX}, Y=${evento.clientY}`;
  });
</script>

🟨 Eventos del teclado​

Los eventos del teclado se disparan al interactuar con teclas físicas del teclado.

EventoDescripciónEjemplo de uso
keydownSe dispara cuando se presiona una teclaDetectar combinaciones de teclas (Ctrl + S)
keyupSe dispara al soltar una teclaVerificar qué tecla fue liberada
keypress(Obsoleto) Similar a keydownSe recomienda keydown o keyup
Ejemplo práctico: detectar teclas específicas
<input type="text" id="campo" placeholder="Escribe algo" />

<script>
  const campo = document.getElementById("campo");

  campo.addEventListener("keydown", (evento) => {
    console.log(`Tecla pulsada: ${evento.key}`);

    if (evento.key === "Enter") {
      console.log("Se pulsó Enter");
    }
  });
</script>

🟧 Eventos de formularios​

Estos eventos están relacionados con la interacción del usuario con formularios: campos de texto, envío, foco, etc.

EventoDescripciónEjemplo de uso
submitSe dispara al enviar un formularioValidar antes del envío
changeSe dispara al cambiar el valor de un campoDetectar cambios en selects
inputSe dispara mientras se escribe en un campoSugerencias/buscadores en vivo
focusSe dispara cuando un campo gana el focoResaltar el campo activo
blurSe dispara cuando un campo pierde el focoValidar al salir del campo
Ejemplo práctico: validar un formulario
<form id="formulario">
  <input type="text" id="nombre" placeholder="Nombre" />
  <button type="submit">Enviar</button>
</form>

<script>
  const formulario = document.getElementById("formulario");

  formulario.addEventListener("submit", (evento) => {
    const nombre = document.getElementById("nombre").value;

    if (!nombre) {
      evento.preventDefault(); // Evita el envío
      alert("Por favor, completa el campo de nombre.");
    }
  });
</script>

🟥 Eventos de carga​

Estos eventos se disparan cuando se cargan o fallan recursos de la página.

EventoDescripciónEjemplo de uso
DOMContentLoadedDOM listo (estructura cargada)Ejecutar scripts cuando el HTML ya está listo
loadTodo cargado (incluye imágenes, CSS, etc.)Confirmar recursos listos
unloadAl abandonar la página (obsoleto en muchos casos)Guardar datos antes de salir
errorCuando un recurso falla al cargarManejar errores de imágenes
Ejemplo práctico: detectar carga y error en una imagen
<img src="imagen.jpg" id="imagen" alt="Imagen" />

<script>
  const imagen = document.getElementById("imagen");

  imagen.addEventListener("load", () => {
    console.log("La imagen se cargó correctamente.");
  });

  imagen.addEventListener("error", () => {
    console.log("Error al cargar la imagen.");
  });
</script>

🟪 Otros tipos de eventos​

  1. Drag and Drop (arrastrar y soltar)

    • drag, dragstart, dragend, drop
    • Usados para mover elementos visualmente dentro de una página.

  2. Eventos de medios (audio/video)

    • play, pause, ended, timeupdate
    • Relacionados con elementos de audio y vídeo.

  3. Eventos de ventana

    • resize: al cambiar el tamaño de la ventana
    • scroll: al desplazarse por la página
Ejemplo práctico: detectar desplazamiento (scroll)
<div style="height: 2000px; background: linear-gradient(white, gray);">
  Desplázate para ver el efecto.
</div>

<script>
  window.addEventListener("scroll", () => {
    console.log(`Desplazamiento: ${window.scrollY}px`);
  });
</script>
tip

Hay muchos más. Puedes ojear la lista completa en la documentación de Mozilla Developer.

3️⃣ Modelo de eventos del DOM (captura, destino y burbujeo)​

El modelo de eventos del DOM describe cómo se generan y cómo se propagan los eventos en un documento. Cuando ocurre un evento, este sigue un recorrido en el árbol DOM que consta de tres fases:

  1. Fase de captura: el evento se propaga desde el nodo raíz (document) hacia abajo, pasando por cada nodo hasta llegar al objetivo.
  2. Fase de destino: el evento alcanza el nodo objetivo donde ocurrió.
  3. Fase de burbujeo: el evento se propaga hacia arriba desde el nodo objetivo hasta el nodo raíz.

Representación del flujo​

Si haces clic en el botón:

  1. En la fase de captura, el evento viaja: document → html → body → div → button
  2. En la fase de destino, el evento interactúa directamente con el button
  3. En la fase de burbujeo, el evento viaja de regreso: button → div → body → html → document

🟩 Fase de burbujeo (bubbling)​

La burbuja ocurre después de que el evento interactúa con el nodo objetivo. En esta fase, el evento asciende por el árbol DOM.

Ejemplo práctico
<div id="contenedor">
  <button id="boton">Haz clic aquí</button>
</div>

<script>
  const contenedor = document.getElementById("contenedor");
  const boton = document.getElementById("boton");

  contenedor.addEventListener("click", () => {
    console.log("Evento en contenedor (burbujeo)");
  });

  boton.addEventListener("click", () => {
    console.log("Evento en botón");
  });
</script>
en este ejemplo

Salida esperada al hacer clic en el botón

  1. Se ejecuta el evento del botón
  2. El evento burbujea y se ejecuta el del contenedor

🟨 Fase de captura (capturing)​

La captura ocurre antes de que el evento alcance el nodo objetivo. En esta fase, el evento viaja desde el nodo raíz hasta el nodo donde ocurrió.

Para manejar un evento en la fase de captura, se debe pasar true como tercer argumento en addEventListener().

Ejemplo práctico
<div id="contenedor">
  <button id="boton">Haz clic aquí</button>
</div>

<script>
  const contenedor = document.getElementById("contenedor");
  const boton = document.getElementById("boton");

  contenedor.addEventListener(
    "click",
    () => {
      console.log("Evento en contenedor (captura)");
    },
    true
  );

  boton.addEventListener("click", () => {
    console.log("Evento en botón");
  });
</script>
en este ejemplo

Salida esperada al hacer clic en el botón

  1. Se ejecuta primero el evento en el contenedor (captura)
  2. Luego, se ejecuta el evento en el botón

🟧 Diferencias entre burbujeo y captura​

CaracterísticaBurbujeo (bubbling)Captura (capturing)
Orden de propagaciónDesde el nodo objetivo hacia arribaDesde el nodo raíz hacia el nodo objetivo
ConfiguraciónFase por defecto en addEventListenerDebe habilitarse explícitamente con true
Uso comúnManejar eventos en elementos internosInterceptar eventos antes de llegar al objetivo
¿Por qué es importante entender el modelo de eventos?
  1. Delegación de eventos: gestionar eventos de muchos elementos con un único manejador
  2. Control de propagación: detener burbujeo con stopPropagation()
  3. Evitar conflictos: entender el flujo evita que eventos se “pisen” entre sí

4️⃣ Manejo de eventos​

El manejo de eventos en el DOM se realiza principalmente a través del método addEventListener(). Este método permite asociar funciones manejadoras a un elemento para que respondan a eventos específicos. Por otro lado, removeEventListener() se utiliza para eliminar esos manejadores.

🟩 addEventListener()​

Descripción​

  • Permite asociar un manejador de eventos a un elemento
  • Es flexible, ya que permite manejar múltiples eventos en el mismo elemento
  • Puede escuchar eventos en las fases de captura o burbujeo
✍️ Sintaxis
elemento.addEventListener(evento, manejador, opciones);
en este ejemplo
  • evento: tipo de evento (click, mouseover, etc.)
  • manejador: función que se ejecuta cuando ocurra el evento
  • opciones: (opcional) objeto o booleano para configurar fase y comportamiento
Ejemplo básico: escuchar un clic
<button id="boton">Haz clic</button>

<script>
  const boton = document.getElementById("boton");

  boton.addEventListener("click", () => {
    console.log("Botón pulsado");
  });
</script>

Agregar múltiples manejadores al mismo elemento​

<button id="boton">Haz clic</button>

<script>
  const boton = document.getElementById("boton");

  boton.addEventListener("click", () => {
    console.log("Primer manejador ejecutado");
  });

  boton.addEventListener("click", () => {
    console.log("Segundo manejador ejecutado");
  });
</script>
en este ejemplo

Salida al hacer clic

  1. Primer manejador ejecutado
  2. Segundo manejador ejecutado

Escuchar eventos en diferentes fases​

<div id="contenedor">
  <button id="boton">Haz clic</button>
</div>

<script>
  const contenedor = document.getElementById("contenedor");
  const boton = document.getElementById("boton");

  contenedor.addEventListener(
    "click",
    () => {
      console.log("Evento en contenedor (captura)");
    },
    true
  );

  boton.addEventListener("click", () => {
    console.log("Evento en botón (burbujeo)");
  });
</script>

🟨 removeEventListener()​

Descripción​

  • Elimina un manejador previamente asociado con addEventListener()
  • Requiere una referencia exacta a la función manejadora usada al registrar el evento
✍️ Sintaxis
elemento.removeEventListener(evento, manejador, opciones);
Ejemplo: eliminar un evento
<button id="boton">Haz clic</button>

<script>
  const boton = document.getElementById("boton");

  function manejarClic() {
    console.log("Evento activado");
  }

  boton.addEventListener("click", manejarClic);

  setTimeout(() => {
    boton.removeEventListener("click", manejarClic);
    console.log("Manejador eliminado");
  }, 5000);
</script>
en este ejemplo
  1. Durante los primeros 5 segundos, al hacer clic se mostrará Evento activado
  2. Después de 5 segundos, el manejador se elimina y el botón ya no responde al clic
Importante

Si registras el evento con una función anónima, luego no podrás eliminarlo con removeEventListener porque no tendrás la referencia exacta.

🟧 Opciones adicionales en addEventListener​

El tercer argumento de addEventListener puede ser un objeto con las siguientes opciones:

  • capture: true para captura, false para burbujeo (por defecto)
  • once: si es true, el manejador se ejecuta una vez y se elimina solo
  • passive: si es true, indica que el manejador no llamará a preventDefault() (mejora rendimiento en scroll, por ejemplo)
Ejemplo con once
<button id="boton">Haz clic</button>

<script>
  const boton = document.getElementById("boton");

  boton.addEventListener(
    "click",
    () => {
      console.log("Este evento se ejecuta solo una vez");
    },
    { once: true }
  );
</script>

🟥 Manejadores con parámetros​

Si necesitas pasar parámetros a la función manejadora, puedes usar una función flecha como “envoltorio”:

<button id="boton">Haz clic</button>

<script>
  const boton = document.getElementById("boton");

  function manejarClic(mensaje) {
    console.log(mensaje);
  }

  boton.addEventListener("click", () => {
    manejarClic("Hola, evento con parámetros");
  });
</script>

🟪 Manejadores dinámicos (añadir o quitar según una condición)​

Es posible añadir o eliminar eventos basados en condiciones.

<button id="boton">Haz clic</button>

<script>
  const boton = document.getElementById("boton");
  let activado = true;

  function manejarClic() {
    console.log("Evento activado");
  }

  // Activamos el manejador inicialmente
  boton.addEventListener("click", manejarClic);

  // Ejemplo: activar/desactivar el manejador con una condición
  // (en este caso, alternando una variable)
  setInterval(() => {
    if (activado) {
      boton.removeEventListener("click", manejarClic);
      console.log("Manejador eliminado");
      activado = false;
    } else {
      boton.addEventListener("click", manejarClic);
      console.log("Manejador añadido");
      activado = true;
    }
  }, 5000);
</script>
en este ejemplo
  • Durante 5 segundos, el evento está activo
  • Luego se elimina, y a los 5 siguientes vuelve a añadirse
info

La idea clave es que añadir y quitar eventos requiere siempre la misma referencia a la función (manejarClic).

🟦 Delegación de eventos (muy importante en listas dinámicas)​

Cuando tienes muchos elementos similares (por ejemplo, muchos <li>), en vez de poner un listener en cada uno, puedes poner un único listener en el contenedor y detectar qué elemento disparó el evento.

Esto funciona gracias al burbujeo.

<ul id="lista">
  <li data-id="1">Elemento 1</li>
  <li data-id="2">Elemento 2</li>
  <li data-id="3">Elemento 3</li>
</ul>

<script>
  const lista = document.getElementById("lista");

  lista.addEventListener("click", (evento) => {
    const li = evento.target.closest("li");
    if (!li) return;

    console.log("Has pulsado:", li.textContent);
    console.log("ID:", li.dataset.id);
  });
</script>
tip

closest("li") es muy útil porque permite que el clic funcione incluso si dentro del <li> hay spans, iconos u otros elementos anidados.

5️⃣ Propiedades y métodos del objeto Event​

El objeto evento (Event) es una herramienta poderosa que proporciona información y métodos para interactuar con el evento que se ha disparado. Este objeto es generado automáticamente por el navegador cada vez que ocurre un evento y se pasa como parámetro a la función manejadora.

A continuación, se detallan las propiedades y métodos más útiles del objeto evento, junto con explicaciones y ejemplos prácticos.

🟩 Propiedades del objeto evento​

type​

Devuelve el tipo de evento que se ha disparado: click, keydown, submit, etc.

document.addEventListener("click", (evento) => {
  console.log(`Tipo de evento: ${evento.type}`);
});

target​

Devuelve el elemento del DOM que disparó el evento (donde ocurrió exactamente).

<ul id="lista">
  <li>Elemento 1</li>
  <li>Elemento 2</li>
</ul>

<script>
  const lista = document.getElementById("lista");

  lista.addEventListener("click", (evento) => {
    console.log(`Elemento pulsado: ${evento.target.textContent}`);
  });
</script>
  • Uso típico: delegación de eventos.
target.closest(selector) (muy útil)​

closest() permite obtener el primer ancestro (o el propio elemento) que cumpla un selector CSS. Sirve para que un clic funcione aunque el target sea un <span>, un icono o una imagen dentro del elemento que interesa.

  • Devuelve el elemento encontrado o null si no existe
  • Sube por el DOM desde target hacia sus padres
lista.addEventListener("click", (evento) => {
  const li = evento.target.closest("li");
  if (!li) return;

  console.log("Has pulsado el li con ID:", li.dataset.id);
});
tip

closest() es ideal en delegación de eventos para “subir” desde el elemento exacto pulsado hasta la tarjeta, botón o <li> que realmente se quiere manejar.

currentTarget​

Devuelve el elemento al que se le asignó el listener.

lista.addEventListener("click", (evento) => {
  console.log(`Evento asignado a: ${evento.currentTarget.id}`);
});
target vs currentTarget
  • target: el elemento exacto donde ocurrió el clic (puede ser un <span> dentro de un botón)
  • currentTarget: el elemento que tiene el listener (por ejemplo, el <button>)

defaultPrevented​

Indica si la acción por defecto ha sido prevenida mediante preventDefault().

const enlace = document.querySelector("a");

enlace.addEventListener("click", (evento) => {
  evento.preventDefault();
  console.log(`¿Se evitó la acción predeterminada? ${evento.defaultPrevented}`);
});

timeStamp​

Devuelve la marca de tiempo (en milisegundos) en la que ocurrió el evento.

document.addEventListener("click", (evento) => {
  console.log(`Evento disparado en: ${evento.timeStamp} ms`);
});

key y code (eventos de teclado)​

  • key: tecla pulsada (Enter, a, Escape)
  • code: código físico (KeyA, Enter)
document.addEventListener("keydown", (evento) => {
  console.log(`Tecla: ${evento.key}, Código: ${evento.code}`);
});

🟨 Métodos del objeto evento​

preventDefault()​

Evita la acción predeterminada asociada al evento.

<a href="https://example.com" id="enlace">Ir a Example</a>

<script>
  const enlace = document.getElementById("enlace");

  enlace.addEventListener("click", (evento) => {
    evento.preventDefault();
    console.log("Acción predeterminada evitada");
  });
</script>

stopPropagation()​

Detiene la propagación del evento hacia otros manejadores en el árbol DOM.

<div id="contenedor">
  <button id="boton">Haz clic</button>
</div>

<script>
  const contenedor = document.getElementById("contenedor");
  const boton = document.getElementById("boton");

  contenedor.addEventListener("click", () => {
    console.log("Evento en contenedor");
  });

  boton.addEventListener("click", (evento) => {
    evento.stopPropagation();
    console.log("Evento en botón");
  });
</script>

Salida esperada al hacer clic en el botón

  • Solo se ejecuta el evento del botón

stopImmediatePropagation()​

Detiene la propagación del evento y evita que otros manejadores asociados al mismo elemento se ejecuten.

const boton = document.getElementById("boton");

boton.addEventListener("click", () => {
  console.log("Primer manejador");
});

boton.addEventListener("click", (evento) => {
  evento.stopImmediatePropagation();
  console.log("Segundo manejador detenido");
});

boton.addEventListener("click", () => {
  console.log("Tercer manejador"); // No se ejecutará
});

composedPath()​

Devuelve un array con todos los elementos que forman el recorrido del evento en el árbol DOM.

document.addEventListener("click", (evento) => {
  console.log(evento.composedPath());
});

Salida: array con los nodos desde el objetivo hasta document.

🟧 Ejemplo completo​

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8" />
  <title>Propiedades y Métodos del Objeto Evento</title>
</head>
<body>
  <div id="contenedor">
    <button id="boton">Haz clic</button>
    <a href="https://example.com" id="enlace">Ir a Example</a>
  </div>

  <script>
    const contenedor = document.getElementById("contenedor");
    const boton = document.getElementById("boton");
    const enlace = document.getElementById("enlace");

    boton.addEventListener("click", (evento) => {
      console.log(`Tipo de evento: ${evento.type}`);
      console.log(`Elemento objetivo: ${evento.target.tagName}`);
    });

    enlace.addEventListener("click", (evento) => {
      evento.preventDefault();
      console.log("Navegación evitada");
    });

    contenedor.addEventListener("click", () => {
      console.log("Evento en contenedor");
    });

    boton.addEventListener("click", (evento) => {
      evento.stopPropagation();
      console.log("Evento en botón, propagación detenida");
    });
  </script>
</body>
</html>

✅ Buenas prácticas​

  1. Usa addEventListener en lugar de atributos HTML como onclick
  2. Prefiere delegación de eventos en listas y contenido dinámico
  3. Usa preventDefault() en formularios y enlaces cuando la lógica sea JS
  4. Evita añadir listeners dentro de otros listeners sin control (puede duplicarlos)
  5. Si vas a eliminar un evento, no uses funciones anónimas al registrarlo
  6. Si mezclas captura y burbujeo, documenta el porqué para evitar confusiones
