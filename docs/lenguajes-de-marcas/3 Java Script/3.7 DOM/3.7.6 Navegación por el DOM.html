3.7.6 Navegación por el DOM

La navegación por el DOM permite moverse por el árbol de nodos de una página para localizar elementos relacionados y manipular su contenido o su estructura.

En el DOM existen nodos de distintos tipos:

  • Elementos (<div>, <p>, <button>, etc.)
  • Texto (incluye espacios y saltos de línea entre etiquetas)
  • Comentarios (<!-- ... -->)
  • Otros (menos comunes en el día a día)

Por eso hay dos familias de propiedades:

  • Generales (parentNode, childNodes, firstChild, etc.) → incluyen texto y comentarios
  • Específicas de elementos (parentElement, children, firstElementChild, etc.) → ignoran lo que no sea elemento y son más “limpias” para trabajar

Resumen rápido de propiedades​

PropiedadQué devuelveIncluye texto/comentarios
parentNodeNodo padre (cualquier tipo)✅
parentElementPadre si es elemento❌ (solo elemento)
childNodesLista de nodos hijos (NodeList)✅
childrenHijos que son elementos (HTMLCollection)❌
firstChild / lastChildPrimer/último hijo (cualquier tipo)✅
firstElementChild / lastElementChildPrimer/último hijo elemento❌
previousSibling / nextSiblingHermano anterior/siguiente (cualquier tipo)✅
previousElementSibling / nextElementSiblingHermano anterior/siguiente elemento❌
Recomendación general

En desarrollo web, para evitar sorpresas con espacios y saltos de línea, suele ser mejor usar las versiones Element (children, parentElement, nextElementSibling, etc.)

1️⃣ Propiedades generales (incluyen nodos de texto y comentarios)​

Estas propiedades son útiles cuando necesitas trabajar con todos los nodos, pero pueden confundir si hay espacios en el HTML.

🟩 parentNode​

Devuelve el nodo padre del nodo actual. Si el nodo no tiene padre (por ejemplo, document), devuelve null.

<div id="contenedor">
  <p id="parrafo">Hola, DOM</p>
</div>
<script>
  const parrafo = document.getElementById("parrafo");
  console.log(parrafo.parentNode); // Nodo <div id="contenedor">
</script>

🟨 childNodes​

Devuelve un NodeList con todos los nodos hijos:

  • elementos
  • comentarios
  • nodos de texto (incluyendo espacios y saltos de línea)
<div id="contenedor">
  <p>Hola</p>
  <!-- Comentario -->
</div>
<script>
  const contenedor = document.getElementById("contenedor");
  console.log(contenedor.childNodes); // NodeList [#text, <p>, #text, #comment, #text] (según el HTML)
</script>
Importante

Es muy habitual que aparezcan #text debido a saltos de línea y espacios del HTML. Por eso, muchas veces childNodes[0] no es el primer elemento, sino un nodo de texto.

🟧 firstChild y lastChild​

Devuelven el primer/último nodo hijo, aunque sea texto o comentario.

<div id="contenedor">
  <p>Hola</p>
  <!-- Comentario -->
</div>
<script>
  const contenedor = document.getElementById("contenedor");
  console.log(contenedor.firstChild); // puede ser #text si hay salto de línea antes del <p>
  console.log(contenedor.lastChild);  // puede ser #text si hay salto de línea al final
</script>
Consejo

Si lo que quieres es el primer/último elemento, usa firstElementChild y lastElementChild.

🟥 previousSibling y nextSibling​

Devuelven el hermano anterior/siguiente, incluyendo texto y comentarios.

<div>
  <p id="parrafo1">Párrafo 1</p>
  <!-- Comentario -->
  <p id="parrafo2">Párrafo 2</p>
</div>
<script>
  const parrafo2 = document.getElementById("parrafo2");
  console.log(parrafo2.previousSibling); // puede ser #text o #comment según el HTML
  console.log(parrafo2.nextSibling);     // puede ser #text si hay espacios/saltos
</script>

2️⃣ Propiedades específicas de elementos (ignoran texto y comentarios)​

Estas propiedades son las más usadas en web porque devuelven solo elementos HTML.

🟩 parentElement​

Igual que parentNode, pero solo devuelve el padre si es un elemento.

<div id="contenedor">
  <p id="parrafo">Hola, DOM</p>
</div>
<script>
  const parrafo = document.getElementById("parrafo");
  console.log(parrafo.parentElement); // Nodo <div id="contenedor">
</script>

🟨 children​

Devuelve un HTMLCollection con los hijos que son elementos, excluyendo nodos de texto y comentarios.

<div id="contenedor">
  <p>Párrafo 1</p>
  <p>Párrafo 2</p>
</div>
<script>
  const contenedor = document.getElementById("contenedor");
  console.log(contenedor.children); // HTMLCollection [<p>, <p>]
</script>
NodeList vs HTMLCollection
  • childNodes → NodeList (incluye nodos de texto/comentarios)
  • children → HTMLCollection (solo elementos)

Ambas son colecciones “tipo array”, pero no son arrays reales.

🟧 firstElementChild y lastElementChild​

Devuelven el primer/último hijo que sea elemento.

<div id="contenedor">
  <p>Primero</p>
  <p>Último</p>
</div>
<script>
  const contenedor = document.getElementById("contenedor");
  console.log(contenedor.firstElementChild); // <p>Primero</p>
  console.log(contenedor.lastElementChild);  // <p>Último</p>
</script>

🟪 previousElementSibling y nextElementSibling​

Devuelven el hermano anterior/siguiente que sea elemento.

<div>
  <p id="parrafo1">Párrafo 1</p>
  <p id="parrafo2">Párrafo 2</p>
</div>
<script>
  const parrafo2 = document.getElementById("parrafo2");
  console.log(parrafo2.previousElementSibling); // <p id="parrafo1">
  console.log(parrafo2.nextElementSibling);     // null (no hay más hermanos)
</script>

3️⃣ Ejemplo completo de navegación​

Este ejemplo mezcla propiedades generales y de elementos para ver la diferencia en la práctica.

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <title>Navegación por el DOM</title>
</head>
<body>
  <div id="contenedor">
    <h1>Título</h1>
    <p id="parrafo1">Párrafo 1</p>
    <p id="parrafo2">Párrafo 2</p>
    <!-- Comentario -->
  </div>

  <script>
    const contenedor = document.getElementById("contenedor");
    const parrafo1 = document.getElementById("parrafo1");

    // Propiedades generales (incluyen texto/comentarios)
    console.log(parrafo1.parentNode);      // <div id="contenedor">
    console.log(contenedor.childNodes);    // Incluye #text y #comment
    console.log(contenedor.firstChild);    // Puede ser #text según el HTML

    // Propiedades de elementos (más limpias)
    console.log(contenedor.children);      // HTMLCollection [<h1>, <p>, <p>]
    console.log(parrafo1.nextElementSibling); // <p id="parrafo2">

    // Navegación combinada
    console.log(contenedor.firstElementChild.nextElementSibling); // <p id="parrafo1">
  </script>
</body>
</html>

✅ Buenas prácticas al navegar por el DOM​

Sigue estos consejos...

  1. Prioriza propiedades Element (children, parentElement, nextElementSibling) para evitar nodos #text.

  2. Si te aparece null, revisa si estás en el borde del árbol (sin padre, sin hermano, etc.).

  3. Si necesitas recorrer hijos, suele ser más cómodo convertir a array:

    • Array.from(contenedor.children)

  4. Evita depender de posiciones como children[0] si el HTML puede cambiar. Mejor usar selectores (querySelector) cuando sea posible.
