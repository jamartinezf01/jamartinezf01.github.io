3.9 Almacenamiento en el navegador

En desarrollo web es muy común necesitar guardar datos en el navegador para que no se pierdan al recargar la página. Por ejemplo:

  • recordar si el usuario ha aceptado cookies
  • guardar preferencias (tema oscuro, idioma)
  • mantener un carrito de la compra
  • almacenar borradores de formularios
  • guardar un token de sesión (con cuidado)

En este punto se ven las opciones más usadas en frontend: localStorage, sessionStorage y cookies. Además, se introduce IndexedDB como alternativa cuando necesitamos guardar muchos datos o datos más complejos.

1️⃣ Conceptos clave antes de empezar​

Antes de usar cualquier almacenamiento, conviene tener claras estas ideas:

  • Persistencia: si los datos se mantienen o se pierden al cerrar el navegador
  • Ámbito: si el dato es por pestaña, por navegador, por dominio
  • Formato: qué tipo de datos se pueden guardar
  • Seguridad: qué datos es peligroso guardar en el navegador
Importante

Nunca guardes datos sensibles “de verdad” en el almacenamiento del navegador
Por ejemplo, contraseñas o datos personales críticos
En frontend se guarda solo lo necesario y con sentido

2️⃣ localStorage​

localStorage permite guardar datos de forma persistente: permanecen incluso si cierras el navegador y vuelves más tarde.

  • clave y valor son strings
  • se accede por dominio (cada web tiene su propio almacenamiento)

🟩 Guardar y leer datos​

Guardar y leer en localStorage
localStorage.setItem("tema", "oscuro")

const tema = localStorage.getItem("tema")
console.log(tema) // "oscuro"

🟨 Borrar datos​

Borrar una clave y limpiar todo
localStorage.removeItem("tema")
localStorage.clear()

🟧 Guardar objetos y arrays con JSON​

Como localStorage solo guarda texto, para guardar objetos se usa:

  • JSON.stringify() al guardar
  • JSON.parse() al recuperar
Guardar y recuperar un objeto
const usuario = { nombre: "Ana", rol: "admin" }

localStorage.setItem("usuario", JSON.stringify(usuario))

const usuarioGuardado = JSON.parse(localStorage.getItem("usuario"))
console.log(usuarioGuardado.nombre) // "Ana"
Recomendación

Siempre que recuperes JSON, controla el caso en el que la clave no exista (null)

Recuperar con valor por defecto
const texto = localStorage.getItem("config")
const config = texto ? JSON.parse(texto) : { idioma: "es" }

console.log(config)

3️⃣ sessionStorage​

sessionStorage funciona igual que localStorage, pero los datos duran solo mientras exista la pestaña:

  • si cierras la pestaña, se borra
  • recargar la página NO lo borra
  • cada pestaña mantiene su sesión (no se comparte entre pestañas)
Ejemplo con sessionStorage
sessionStorage.setItem("pasoWizard", "2")

console.log(sessionStorage.getItem("pasoWizard")) // "2"
Caso típico

Guardar el estado de un formulario por pasos (wizard) o una navegación temporal

4️⃣ Diferencias entre localStorage y sessionStorage​

CaracterísticalocalStoragesessionStorage
PersistenciaSe mantiene al cerrar el navegadorSe borra al cerrar la pestaña
ÁmbitoDominioPestaña

Uso típicoPreferencias, datos persistentesEstado temporal, procesos en curso

5️⃣ Cookies (visión práctica en frontend)​

Las cookies son un mecanismo antiguo, pero siguen siendo importantes porque:

  • se envían automáticamente al servidor en cada petición (si aplica)
  • permiten control de caducidad
  • son clave para sesiones tradicionales (backend)

En frontend, se pueden leer y escribir con document.cookie, aunque la API es menos cómoda.

🟩 Crear una cookie simple​

Crear cookie simple
document.cookie = "idioma=es; path=/"

🟨 Cookie con caducidad​

Cookie con fecha de expiración
const fecha = new Date()
fecha.setDate(fecha.getDate() + 7)

document.cookie = `bannerCookies=ok; expires=${fecha.toUTCString()}; path=/`

🟧 Leer cookies​

document.cookie devuelve un string con todas las cookies.

Leer cookies
console.log(document.cookie)

Ejemplo básico para obtener una cookie por nombre:

Leer una cookie concreta
function obtenerCookie(nombre) {
  const cookies = document.cookie.split("; ")
  const encontrada = cookies.find((c) => c.startsWith(nombre + "="))
  return encontrada ? encontrada.split("=")[1] : null
}

console.log(obtenerCookie("idioma"))
Importante

Desde JS no puedes crear cookies con HttpOnly (esas solo las pone el servidor) Si una cookie es HttpOnly, JS no la puede leer, y eso es bueno para seguridad

6️⃣ Storage events (sincronizar cambios entre pestañas)​

Cuando cambias localStorage en una pestaña, las otras pestañas del mismo dominio pueden enterarse con el evento storage.

  • solo salta en otras pestañas, no en la misma donde se cambió
  • útil para sincronizar logout, tema, preferencias, etc.
Escuchar cambios de localStorage en otras pestañas
window.addEventListener("storage", (event) => {
  console.log("Clave:", event.key)
  console.log("Valor nuevo:", event.newValue)
  console.log("Valor antiguo:", event.oldValue)
})

7️⃣ IndexedDB (introducción)​

Cuando localStorage se queda corto (por ejemplo, muchos datos o necesidad de búsquedas), se usa IndexedDB:

  • base de datos en el navegador
  • asíncrona
  • admite objetos grandes
  • más compleja de usar
Cuándo pensar en IndexedDB
  • caché de muchas respuestas de API
  • guardar contenido offline
  • apps tipo “notas”, “lectores”, “catálogos”

En este manual se deja como concepto, porque suele requerir una sección propia.

8️⃣ Ejemplo práctico: modo oscuro persistente​

🟩 HTML​

Botón de tema
<button id="btn-tema">Cambiar tema</button>

🟨 JS​

Guardar preferencia de tema
const boton = document.getElementById("btn-tema")

function aplicarTema(tema) {
  document.documentElement.dataset.tema = tema
}

const temaGuardado = localStorage.getItem("tema") || "claro"
aplicarTema(temaGuardado)

boton.addEventListener("click", () => {
  const temaActual = document.documentElement.dataset.tema
  const nuevo = temaActual === "claro" ? "oscuro" : "claro"

  aplicarTema(nuevo)
  localStorage.setItem("tema", nuevo)
})

✅ Buenas prácticas de almacenamiento en navegador​

Sigue estos consejos...
  1. Guarda solo lo necesario y evita datos sensibles
  2. Usa localStorage para preferencias y datos que deban persistir
  3. Usa sessionStorage para estados temporales por pestaña
  4. Usa JSON.stringify y JSON.parse para arrays y objetos
  5. Controla el caso null cuando una clave no existe
  6. Evita guardar “megadatos” en localStorage (para eso, IndexedDB)
