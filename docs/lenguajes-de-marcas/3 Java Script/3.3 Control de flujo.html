3.3 Control de Flujo

En este punto se empieza a dar estructura al código. Ya no se trata solo de declarar datos u operar con ellos, sino de decidir qué camino seguir según una condición y repetir acciones cuando sea necesario. A partir de aquí, JavaScript se convierte en una herramienta capaz de construir lógica real para una página web, preparando el terreno para trabajar con funciones, listas de datos y, más adelante, con el DOM y los eventos.

1️⃣ Estructuras Condicionales​

Las condicionales permiten que un programa tome decisiones basadas en condiciones específicas. En esta sección, desarrollaremos las estructuras condicionales más comunes en JavaScript: if...else, switch y el operador ternario.

🟩 if, else if y else​

La estructura condicional básica en JavaScript es el bloque if. Puedes usar else if para manejar múltiples condiciones y else como una opción predeterminada.

✍️ Sintaxis de la estructura if
if (condición) {
    // Código a ejecutar si la condición es verdadera
} else if (otraCondición) {
    // Código a ejecutar si otra condición es verdadera
} else {
    // Código a ejecutar si ninguna condición es verdadera
}
Ejemplo de uso de if...else if...else
let temperatura = 25;

if (temperatura < 10) {
    console.log("Hace frío 🥶");
} else if (temperatura >= 10 && temperatura <= 30) {
    console.log("El clima es templado 🌤️");
} else {
    console.log("Hace calor 🥵");
}

🟨 Operador Ternario​

El operador ternario es una forma compacta de escribir un if...else cuando solo hay una condición simple que evaluar.

✍️ Sintaxis del operador ternario
condición ? expresión_si_verdadera : expresión_si_falsa;
Ejemplo de uso del operador ternario
let edad = 20;
let mensaje = edad >= 18 ? "Mayor de edad 🎉" : "Menor de edad 🚸";
console.log(mensaje); // "Mayor de edad 🎉"

🟧 switch​

El switch es una estructura condicional utilizada para evaluar un valor contra múltiples casos posibles, ejecutando el bloque correspondiente al primer caso que coincida.

✍️ Sintaxis de la estructura switch-case
switch (expresión) {
    case valor1:
        // Código a ejecutar si expresión === valor1
        break;
    case valor2:
        // Código a ejecutar si expresión === valor2
        break;
    default:
        // Código a ejecutar si no coincide ningún caso
}
Ejemplo de uso de switch...case
let dia = "sábado";

switch (dia) {
    case "lunes":
        console.log("Inicio de la semana 📅");
        break;
    case "viernes":
        console.log("Fin de la semana 🎉");
        break;
    case "sábado":
    case "domingo":
        console.log("Es fin de semana 🏖️");
        break;
    default:
        console.log("Día normal 📆");
}

🟥 Comparativa entre if...else y switch​

Ambas estructuras permiten tomar decisiones, pero hay casos específicos donde una es más adecuada que la otra.

Aspectoif...elseswitch
CondicionesPermite evaluar expresiones complejas.Evalúa una sola expresión contra múltiples casos.
LegibilidadPuede volverse difícil de leer con muchas condiciones.Más claro para comparar un solo valor con múltiples casos.
FlexibilidadMás flexible, admite operadores lógicos y comparaciones complejas.Menos flexible, ideal para valores exactos.

Ejemplo comparativo​

Con if...else:

let nivel = "intermedio";

if (nivel === "principiante") {
    console.log("Curso básico");
} else if (nivel === "intermedio") {
    console.log("Curso intermedio");
} else if (nivel === "avanzado") {
    console.log("Curso avanzado");
} else {
    console.log("Nivel no reconocido");
}

Con switch:

let nivel = "intermedio";

switch (nivel) {
    case "principiante":
        console.log("Curso básico");
        break;
    case "intermedio":
        console.log("Curso intermedio");
        break;
    case "avanzado":
        console.log("Curso avanzado");
        break;
    default:
        console.log("Nivel no reconocido");
}

✅ Buenas Prácticas​

Sigue estos consejos...

  1. Prefiere switch para múltiples valores de un mismo tipo:

    • Cuando comparas un solo valor contra muchos posibles, switch es más legible.

  2. Simplifica condiciones con operadores lógicos:

    if (edad >= 18 && edad <= 65) {
        console.log("Edad laboral");
    }

  3. Evita anidar demasiados bloques if:

    • Usa funciones o estructuras como switch para mejorar la claridad.

  4. Usa siempre default en un switch:

    • Asegúrate de manejar casos no esperados.

2️⃣ Bucles e Iteraciones​

Los bucles son estructuras fundamentales en programación, ya que permiten ejecutar un bloque de código varias veces, dependiendo de una condición o un conjunto de valores. En JavaScript, los bucles incluyen: for, while, do...while, for...in y for...of.

🟩 for​

El bucle for se utiliza cuando se conoce de antemano cuántas veces debe repetirse el código. Es ideal para iteraciones con un número definido.

✍️ Sintaxis de la estructura for
for (inicialización; condición; actualización) {
    // Código a ejecutar en cada iteración
}
  1. Inicialización: Se ejecuta una vez al inicio del bucle. Usualmente se utiliza para declarar e inicializar una variable de control. 🏁
  2. Condición: Se evalúa antes de cada iteración. Si es true, se ejecuta el bloque; si es false, el bucle termina. ✅
  3. Actualización: Se ejecuta al final de cada iteración para modificar la variable de control. 🔄
Ejemplo de uso del bucle for
for (let i = 0; i < 5; i++) {
    console.log(`Iteración: ${i}`);
}
en este ejemplo
  1. let i = 0: Inicialización. 🏁
  2. i < 5: Condición (el bucle se ejecuta mientras esta condición sea verdadera). ✅
  3. i++: Actualización (incrementa el valor de i en cada iteración). 🔄

Salida:

Iteración: 0
Iteración: 1
Iteración: 2
Iteración: 3
Iteración: 4

🟨 while​

El bucle while repite un bloque de código mientras la condición sea verdadera. Es ideal cuando no se sabe exactamente cuántas iteraciones se necesitan.

✍️ Sintaxis de la estructura while
while (condición) {
    // Código a ejecutar mientras la condición sea verdadera
}
Ejemplo de uso del bucle while
let contador = 0;

while (contador < 5) {
    console.log(`Contador: ${contador}`);
    contador++;
}
en este ejemplo
  1. contador < 5: Condición (el bucle se ejecuta mientras esta condición sea verdadera). ✅
  2. contador++: Incrementa el valor de contador en cada iteración. 🔄

Salida:

Contador: 0
Contador: 1
Contador: 2
Contador: 3
Contador: 4

🟧 do...while​

El bucle do...while es similar a while, pero garantiza que el bloque de código se ejecute al menos una vez, ya que la condición se evalúa al final.

✍️ Sintaxis de la estructura do...while
do {
    // Código a ejecutar
} while (condición);
Ejemplo de uso del bucle do...while
let numero = 0;

do {
    console.log(`Número: ${numero}`);
    numero++;
} while (numero < 3);
en este ejemplo
  1. El bloque de código se ejecuta una vez antes de evaluar la condición. ✔️
  2. numero < 3: Condición que se evalúa después de cada iteración. ✅

Salida:

Número: 0
Número: 1
Número: 2

🟥 for...in​

El bucle for...in se utiliza para iterar sobre las propiedades de un objeto. Devuelve las claves (propiedades) del objeto en cada iteración.

✍️ Sintaxis de la estructura for...in
for (let clave in objeto) {
    // Código a ejecutar
}
Ejemplo de uso del bucle for...in
let persona = { nombre: "Juan", edad: 30, ciudad: "Madrid" };

for (let propiedad in persona) {
    console.log(`${propiedad}: ${persona[propiedad]}`);
}
Salida del código anterior
nombre: Juan
edad: 30
ciudad: Madrid

🟪 for...of​

El bucle for...of se utiliza para iterar sobre objetos iterables como arrays, cadenas de texto, mapas y conjuntos. Devuelve los valores en cada iteración.

✍️ Sintaxis de la estructura for...of
for (let valor of iterable) {
    // Código a ejecutar
}
Ejemplo de uso del bucle for...of
let numeros = [10, 20, 30];

for (let numero of numeros) {
    console.log(numero);
}
Salida del código anterior
10
20
30

🟦 Declaraciones Especiales en Bucles​

1. break 🛑​

Termina el bucle inmediatamente, sin importar si la condición aún es verdadera.

Ejemplo de uso del bucle for con break
for (let i = 0; i < 10; i++) {
    if (i === 5) {
        break; // Detiene el bucle cuando i es 5
    }
    console.log(i);
}
Salida del código anterior
0
1
2
3
4

2. continue ⏭️​

Salta a la siguiente iteración del bucle, ignorando el resto del código en la iteración actual.

Ejemplo de uso de continue
for (let i = 0; i < 10; i++) {
    if (i % 2 === 0) {
        continue; // Salta los números pares
    }
    console.log(i);
}
Salida del código anterior
1
3
5
7
9

✅ Buenas Prácticas​

Sigue estos consejos...

  1. Usa el bucle adecuado:

    • for: Cuando conoces el número exacto de iteraciones. 🧮
    • while o do...while: Cuando el número de iteraciones depende de una condición. 🕹️
    • for...in: Para objetos. 🗂️
    • for...of: Para iterables. 📜

  2. Evita bucles infinitos 🔄:

    • Asegúrate de que las condiciones del bucle cambien en cada iteración. 🚫🔄

  3. Usa break y continue con moderación ⚖️:

    • Aunque son útiles, un uso excesivo puede dificultar la lectura del código. 👀

📚 Ejercicios​

Misión Odisea: Control de Flujo

Bienvenido, Cadete. Eres la IA de la nave espacial "Odisea". Tu misión es asegurar que todos los sistemas funcionen correctamente para llegar a Marte a salvo.

🚀 Misión 1: La Cuenta Atrás​

El despegue es inminente. Necesitamos iniciar la secuencia de lanzamiento.

  • Objetivo: Crear un bucle que cuente desde 10 hasta 0.
  • Requisito: Al llegar a 0, debe imprimir "¡Despegue! 🚀".
  • Pista: Usa un bucle for decremental.

⛽ Misión 2: Repostaje en Órbita​

Necesitamos llenar los tanques de combustible antes de abandonar la órbita terrestre.

  • Objetivo: Simular el llenado del tanque.
  • Datos: Empiezas con combustible = 0.
  • Requisito: Mientras el combustible sea menor a 100, añade cantidades (por ejemplo 10 o 20) e imprime el nivel actual.
  • Pista: Usa un bucle while.

🧭 Misión 3: Sistema de Navegación​

El piloto automático necesita comprobar el rumbo.

  • Objetivo: Traducir las coordenadas del compás.
  • Datos: Tienes una variable rumbo que puede ser 'N', 'S', 'E', 'O'.
  • Requisito:
    • 'N': Imprime "Rumbo al Norte".
    • 'S': Imprime "Rumbo al Sur".
    • 'E': Imprime "Rumbo al Este".
    • 'O': Imprime "Rumbo al Oeste".
    • Cualquier otra cosa: "Rumbo desconocido, abortar 🚨".
  • Pista: Debes usar la estructura switch.

⚖️ Misión 4: Estabilizador de Gravedad​

Al entrar en hipervelocidad, la gravedad artificial fluctúa.

  • Objetivo: Aumentar la gravedad hasta un nivel seguro.
  • Requisito: La gravedad empieza en 0. Auméntala de 1 en 1.
  • Condición: El bucle debe ejecutarse al menos una vez y detenerse cuando la gravedad sea mayor o igual a 3.
  • Pista: Usa do...while.

☄️ Misión 5: Campo de Asteroides​

Estamos atravesando un cinturón de asteroides. El radar escanea 10 sectores.

  • Objetivo: Escanear sectores del 1 al 10.
  • Problema: El sector 7 tiene interferencias magnéticas extremas.
  • Requisito: Si el sector es el 7, salta ese escaneo y no imprimas nada para ese número. Para el resto, imprime "Sector X despejado".
  • Pista: Usa continue.

🌡️ Misión 6: Parada de Emergencia​

¡Alerta! El motor se está sobrecalentando.

  • Objetivo: Monitorizar la temperatura en un bucle "infinito" (o de muchas repeticiones, ej: 100).
  • Requisito: La temperatura sube en cada iteración. Si supera 100 grados, detén el bucle inmediatamente y muestra un mensaje de alerta.
  • Pista: Usa break.

📦 Misión 7: Inventario de Carga​

Antes de aterrizar en Marte, revisemos los suministros críticos.

  • Objetivo: Revisar la lista de carga.
  • Datos: Tienes una lista de suministros: ["Oxígeno", "Agua", "Comida", "Trajes"].
  • Requisito: Recorre la lista de suministros uno a uno e imprime "Revisado: [Suministro]".
  • Pista: Usa for...of.

👽 Misión 8: Simbología Alienígena​

Hemos encontrado extraños símbolos en una cueva. Parecen patrones matemáticos.

  • Objetivo: Generar un patrón triangular de asteriscos.
  • Requisito: Usa dos bucles for anidados para imprimir 5 líneas.
    • Línea 1: "*"
    • Línea 2: "**"
    • ...
    • Línea 5: "*****"
  • Pista: Un bucle controla las líneas (i), otro imprime los caracteres (j) concatenando en una variable string.

🔐 Misión 9: Códigos de Seguridad​

La puerta de la base está bloqueada. Necesitamos resolver un acertijo matemático para entrar.

  • Objetivo: Encontrar números primos para desbloquear la seguridad.
  • Requisito:
    • Busca y muestra en consola los primeros 3 números primos que sean mayores que 10.
    • Debes hacer un bucle que recorra números (11, 12, 13...) y verifique si son primos.
  • Pista: Un número primo solo es divisible por 1 y por sí mismo. Necesitarás un bucle para probar divisores o una bandera (isPrime).

🚩 Misión 10: Aterrizaje en Marte​

¡Llegamos! Pero el sistema de aterrizaje automático falla. Debes programar la lógica de frenado de emergencia.

  • Objetivo: Simular el descenso y lograr un aterrizaje suave.
  • Datos Iniciales: altura = 200 metros, velocidad = 20 m/s.
  • Reglas de la Física (Simulada):
    • En cada paso del bucle (segundo), la altura disminuye según la velocidad actual.
    • Debes actualizar la velocidad: si la velocidad es mayor a 10, activa los frenos (resta 3 a la velocidad). Si es 10 o menos, la gravedad acelera la nave (suma 1 a la velocidad).
  • Condición de Parada: El bucle se detiene cuando la altura llega a 0 (o menos).
  • Resultado:
    • Si al tocar suelo (altura <= 0) la velocidad es menor de 15: Imprime "¡Aterrizaje exitoso! 🏆".
    • Si es mayor o igual a 15: Imprime "¡Crash! La nave se ha estrellado 💥".
  • Pista: Un while controla la altura. Dentro, usa if para modificar la velocidad y muestra los datos en cada paso para ver qué ocurre.
