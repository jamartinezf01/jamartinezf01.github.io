2.6 Selectores Avanzados

Los selectores avanzados permiten apuntar a elementos por su relacion en el arbol DOM o por atributos. Reducen clases innecesarias y hacen el CSS mas expresivo.

1️⃣ Combinadores (relaciones en el DOM)​

Los combinadores describen cómo se relacionan dos selectores en el HTML: >descendencia, filiación directa y relación entre hermanos. Saber elegir el >combinador adecuado evita reglas demasiado genéricas o frágiles.

SintaxisSelecciona…EjemploExplicacion
A BB dentro de A (cualquier nivel).card pTodos los <p> dentro de .card.
A > BB hijo directo de Aul > liSolo <li> que son hijos inmediatos del ul.
A + BB hermano inmediatamente posterior a Ah2 + pEl primer <p> justo despues de cada <h2>.
A ~ BB hermanos posteriores a A.tag ~ .tagTodos los .tag que vienen despues del primero.
  • CSS
  • HTML
  • Representación
.menu li { padding: .25rem .5rem; }         /* A B */
.menu > li.destacado { font-weight: 800; }  /* A > B */
.menu + p { color: #db5f5f; }             /* A + B */
.menu ~ p { font-style: italic; }           /* A ~ B */
<ul class="menu">
  <li>Inicio</li>
  <li class="destacado">Cursos</li>
  <li>Blog</li>
</ul>
<p>Parrafo 1</p>
<p>Parrafo 2</p>

en este ejemplo

  • .menu li { ... } → Descendiente (A B): aplica a todo li dentro de un elemento con clase .menu, sin importar la profundidad.

  • .menu > li.destacado { ... } → Hijo directo (A > B): aplica solo al li con clase .destacado que sea hijo inmediato de .menu (no a nietos).

  • .menu + p { ... } → Hermano adyacente (A + B): aplica al primer p que va justo después de .menu y comparte el mismo padre.

  • .menu ~ p { ... } → Hermanos generales (A ~ B): aplica a todos los p posteriores a .menu con el mismo padre (no solo al primero).

2️⃣ Selectores de atributo​

Seleccionan elementos por la presencia o el contenido de un atributo. Son >esenciales para formularios, enlaces y convenciones como BEM, además de permitir >filtros muy precisos sin añadir clases.

En CSS, estos atributos se indican rodeando de corchetes [ ] al atributo en cuestión.

SintaxisCoincidenciaEjemploUso tipico
[attr]Atributo presenteimg[alt]Accesibilidad.
[attr="x"]Igual exactoinput[type="password"]Campos concretos.
[attr^="x"]Empieza pora[href^="https://"]Enlaces externos.
[attr$="x"]Termina enimg[src$=".svg"]Formato de archivo.
[attr*="x"]Contiene[class*="btn-"]Variantes BEM.
[attr~="x"]Contiene palabra[title~="pro"]Etiquetas por palabra.
[attr ="x"]x o x-...[lang="es"]Idiomas.
  • CSS
  • HTML
  • Representación
a[href^="mailto:"] { color: teal; }
a[href^="https://"]::after { content: " ↗"; }
input[type="password"] { letter-spacing: .15em; }
<a href="https://ejemplo.com">Web</a>
<a href="mailto:yo@ejemplo.com">Correo</a>
<input type="text" name="usuario">
<input type="password" name="pass">

en este ejemplo

Explicación breve:

  • a[href^="mailto:"] { color: teal; } Selector de atributo con prefijo (^=): aplica a enlaces cuyo href empieza por mailto: (enlaces de correo). Les pone el color teal.

  • a[href^="https://"]::after { content: " ↗"; } Selector de atributo con prefijo + pseudoelemento ::after: a los enlaces que empiezan por https:// les añade un icono/flecha después del texto del enlace. ::after necesita siempre content.

  • input[type="password"] { letter-spacing: .15em; } Selector de atributo exacto (=): aplica a los <input> cuyo type es password y espacia ligeramente los caracteres para mejorar legibilidad de los puntos/asteriscos.

3️⃣ Selectores logicos modernos​

Estos selectores simplifican reglas largas, agrupan alternativas y excluyen >casos sin duplicar CSS. También ayudan a controlar la especificidad de forma >consciente.

SelectorPara que sirveNota de especificidadEjemplo
:is(lista)Agrupar alternativasToma la especificidad mas alta de la lista:is(h2,p,a,button){margin:.5rem 0}
:where(lista)Igual que :is pero sin “peso”Especificidad 0:where(h2,p,ul,ol){margin:0}
:not(lista)Excluir coincidenciasSolo suma la especificidad de su argumentobutton:not(.primario){opacity:.8}

4️⃣ Selector relacional​

:has() Permite seleccionar un elemento si dentro (o en relacion) hay >cierta coincidencia. Ideal para formularios, tarjetas o acordeones.

  • CSS
  • HTML
  • Representación
.campo:has(input:invalid){ border:2px solid crimson; background:#fff5f6; }
.campo:has(input:valid) .error{ display:none; }
<label class="campo">
  <span>Email</span>
  <input type="email" required>
  <small class="error">Obligatorio</small>
</label>

en este ejemplo

Explicación breve:

  • .campo:has(input:invalid){ ... } :has() selecciona el contenedor .campo si dentro tiene un <input> inválido (:invalid). Así, cuando el input falla la validación HTML, el contenedor se resalta con borde rojo y fondo rosado.

  • .campo:has(input:valid) .error{ display:none; } Si dentro de .campo hay un <input> válido (:valid), entonces el elemento hijo .error se oculta. Resultado: el aviso solo aparece mientras el campo no cumpla la validación.

info

Tambien funciona con combinadores: .item:has(+ .item.activo) selecciona .item si su hermano siguiente esta activo.

5️⃣ Pseudoclases​

Las pseudoclases representan estados dinámicos (interacción, validación) o estructurales (posición entre hermanos). No requieren modificar el HTML y favorecen componentes accesibles y mantenibles.

🟩 Estructura/posicion​

Permiten seleccionar elementos por su orden o tipo dentro de su padre, creando patrones (zebra, cabeceras, últimas filas) sin añadir clases extras ni JavaScript.

PseudoclaseSignificadoEjemplo
:first-child / :last-childPrimer/ultimo hijo.lista li:first-child{...}
:only-childUnico hijo.tarjeta:only-child{...}
:emptySin hijos (ni texto)p:empty{display:none}
:nth-child(an+b)Patron por indiceli:nth-child(odd) (impares)
:nth-of-type()Como nth-child pero por tipop:nth-of-type(2)
  • CSS
  • HTML
  • Representación
.lista li:nth-child(odd){ background:#f4f4f4; }
.lista li:nth-child(3n){ color: steelblue; }
.lista li:first-child{ border-radius:8px 8px 0 0; }
.lista li:last-child{ border-radius:0 0 8px 8px; }
    <ul class="lista">
        <li>1</li>
        <li>2</li>
        <li>3</li>
        <li>4</li>
        <li>5</li>
    </ul>

en este ejemplo

Explicación breve:

  • .lista li:nth-child(odd){ ... } Impares: aplica a los li en posiciones 1, 3, 5, … y les pone fondo #f4f4f4.

  • .lista li:nth-child(3n){ ... } Cada 3: aplica a los li en posiciones 3, 6, 9, … y les pone color steelblue.

  • .lista li:first-child{ ... } Primer hijo: redondea las esquinas superiores del primer li (8px 8px 0 0).

  • .lista li:last-child{ ... } Último hijo: redondea las esquinas inferiores del último li (0 0 8px 8px).

nota

Un mismo li puede recibir varios estilos si coincide con más de un selector (por ejemplo, ser impar y además múltiplo de 3). La cascada y la especificidad decidirán el resultado final.

🟨 Interaccion y enfoque​

Modelan la experiencia al pasar el ratón, hacer clic o navegar con teclado. Son clave para accesibilidad: el foco visible y coherente mejora la usabilidad para todas las personas.

PseudoclaseUsoEjemplo
:hoverPuntero encima.btn:hover{background:#f0f0f0}
:activeDurante el click.btn:active{transform:translateY(1px)}
:focusElemento con focoa:focus{outline:2px solid}
:focus-visibleFoco “visible” (accesible).btn:focus-visible{outline:3px solid dodgerblue}
:focus-withinSi algun hijo tiene foco.card:focus-within{box-shadow:0 0 0 2px #cdf}

🟧 Formularios y validacion​

Exponen estados nativos de los controles (requerido, válido, fuera de rango, marcado). Permiten validar y comunicar errores solo con CSS, guiando al usuario en tiempo real.

PseudoclaseCuando aplicaEjemplo
:required / :optionalCampo requerido / opcionalinput:required{border-left:4px solid orange}
:valid / :invalidPasa/No pasa validacioninput:invalid{border-color:crimson}
:in-range / :out-of-rangeDentro/Fuera del rangoinput:out-of-range{background:#fff0f0}
:checked / :indeterminateCheckbox/Radio marcado/indet.input:checked + span{font-weight:700}
:placeholder-shownPlaceholder visibleinput:placeholder-shown{font-style:italic}
:read-only / :read-writeEditabilidadinput:read-only{opacity:.6}
  • CSS
  • HTML
  • Representación
.f input:valid{ border-color: seagreen; }
.f input:invalid{ border-color: crimson; }
.f input:out-of-range{ background:#fff0f0; }
.f input:placeholder-shown{ font-style: italic; }
<form class="f">
  <input type="email" placeholder="tu@correo.com" required>
  <input type="number" min="1" max="10" value="11">
  <label><input type="checkbox" checked><span> Acepto</span></label>
</form>

en este ejemplo

Explicación breve:

  • .f input:valid { border-color: seagreen; } El input válido (cumple tipo/requisitos como required, pattern, min/max, etc.) muestra el borde verde.

  • .f input:invalid { border-color: crimson; } El input inválido (no cumple validación HTML) muestra el borde rojo.

  • .f input:out-of-range { background: #fff0f0; } Para campos numéricos o con rango, si el valor está fuera de min/max, se pinta el fondo rosado.

  • .f input:placeholder-shown { font-style: italic; } Mientras el placeholder está visible (el campo está vacío), el texto del input se ve en cursiva.

🟥 Idioma, raiz y destino de URL​

Abarcan casos especiales del documento: estilos globales desde :root, ajustes por idioma con :lang() y navegación interna con :target para crear interacciones sin JavaScript.

PseudoclaseUsoEjemplo
:rootNodo raiz del documento:root{ color-scheme: light dark; }
:lang(xx)Contenido por idioma:lang(es){ quotes:"«" "»"; }
:targetElemento cuyo id coincide con el #hash#detalles:target{ outline:3px solid rebeccapurple }

6️⃣ Pseudoelementos​

Los pseudoelementos permiten dar estilo a porciones como la primera letra/linea, el marcador de lista o insertar contenido decorativo. Son fundamentales para microdetalles tipográficos y señales visuales.

PseudoelementoPara queNota
::before / ::afterContenido decorativoRequieren content.
::first-linePrimera linea del bloqueDepende del ancho.
::first-letterInicial capitularTipico en articulos.
::selectionSeleccion de textoColores al seleccionar.
::markerViñetas de listasPersonaliza bullets.
::placeholderPlaceholder de inputsEstilo del texto fantasma.
::file-selector-buttonBoton de <input type="file">Estilizado nativo.

Ejemplos

/* Badges con before/after */
.btn{
  position:relative; padding:.5rem 1rem .5rem 2rem; border-radius:999px;
}
.btn::before{
  content:"★"; position:absolute; left:.6rem; top:50%; transform:translateY(-50%);
}
.btn::after{
  content:" Nuevo "; margin-left:.5rem; background:gold; border-radius:.5rem; padding:0 .3rem;
}

/* Capitulacion */
.intro::first-letter{ font-size:250%; float:left; line-height:1; padding-right:.25rem; }

/* Seleccion de texto */
.texto::selection{ background:gold; color:black; }

/* Bullets personalizados */
.lista li::marker{ content:"→ "; font-weight:700; }

/* Placeholder y boton de archivo */
.input::placeholder{ color:#999; font-style:italic; }
.file::file-selector-button{ padding:.4rem .8rem; border:1px solid #ccc; }

📊 Resumen​

  • Relaciones: A B, A > B, A + B, A ~ B
  • Atributo: [a], [a="x"], [a^="x"], [a$="x"], [a*="x"], [a~="x"], [a|="x"]
  • Lógicos: :is(), :where(), :not(), :has()
  • Estructura: :first/last/only-child, :nth-child(), :of-type(), :empty
  • Interaccion: :hover, :active, :focus, :focus-visible, :focus-within
  • Formularios: :checked, :required/:optional, :valid/:invalid, :in/out-of-range, :placeholder-shown, :read-only/:read-write
  • Varios: :root, :lang(), :target
  • Pseudoelementos: ::before, ::after, ::first-line, ::first-letter, ::selection, ::marker, ::placeholder, ::file-selector-button
