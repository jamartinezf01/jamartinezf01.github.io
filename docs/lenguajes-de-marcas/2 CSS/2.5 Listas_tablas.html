2.5 Listas y Tablas

En esta sección veremos cómo dar estilo a listas (ul, ol, li) y tablas (table, thead, tbody, tfoot, tr, th, td) con CSS, cubriendo propiedades, atajos, personalizaciones avanzadas y patrones responsivos. Se incluyen ejemplos en CSS, HTML y fragmentos en Java para generar el marcado desde código.

1️⃣ Listas​

🟩 Propiedades básicas de listas​

Propiedad¿Qué hace?Valores comunes / ejemplo
list-style-typeDefine el tipo de viñeta o numeración.disc, circle, square, decimal, lower-roman, upper-alpha, none
list-style-positionColoca la viñeta dentro o fuera de la caja del li.outside (por defecto), inside
list-style-imageUsa una imagen como viñeta.url("img/bullet.svg")
list-style (shorthand)Atajo que combina type, position e image.list-style: square inside;
padding-left / margin-leftControlan la sangría de los elementos.padding-left: 1.25rem;
  • CSS
  • HTML
  • Representación
/* Estilos globales para listas no ordenadas y ordenadas */
ul, ol {
  margin: 1rem 0;
  padding-left: 1.25rem;
}

/* Tipos de viñetas y numeración */
.lista-disc   { list-style-type: disc; }
.lista-square { list-style-type: square; }
.lista-circle { list-style-type: circle; }
.lista-dec    { list-style-type: decimal; }
.lista-romano { list-style-type: upper-roman; }
.lista-none   { list-style: none; padding-left: 0; }

/* Posición de la viñeta */
.viñeta-inside  { list-style-position: inside; }
.viñeta-outside { list-style-position: outside; }

/* Imagen como viñeta */
.lista-imagen {
  list-style-image: url("img/arrow.png");
  /* Alternativa robusta con ::marker más abajo */
}

🟨 Personalización moderna con ::marker​

::marker permite dar estilo directo al marcador (viñeta/número) del li.

  • CSS
  • HTML
  • Representación
/* Cambiar color, tipografía y contenido del marcador */
ul.personalizada li::marker {
  color: #0d6efd;
  font-weight: 700;
}

ol.semanas li::marker {
  color: #16a34a;
  font-feature-settings: "tnum"; /* números tabulares */
}

/* Reemplazar marcador con emoji o texto (solo listas con list-style-type distinto de none) */
ul.iconos {
  list-style-type: "✔  "; /* fallback simple */
}

<ul class="personalizada">
  <li>Tarea prioritaria</li>
  <li>Tarea secundaria</li>
</ul>

<ol class="semanas">
  <li>Semana 1</li>
  <li>Semana 2</li>
</ol>

<ul class="iconos">
  <li>Accesible</li>
  <li>Ligero</li>
  <li>Performante</li>
</ul>

2️⃣ Tablas​

🟩 Anatomía de una tabla​

Semántica

Usar caption para el título, scope="col|row" en th para accesibilidad, y separar thead/tbody/tfoot ayuda a estilos y sticky headers.

🟨 Propiedades clave​

Propiedad¿Para qué sirve?Ejemplo
border-collapseColapsar bordes (collapse) o mantenerlos separados (separate).border-collapse: collapse;
border-spacingEspacio entre celdas (modo separate).border-spacing: 0 .5rem;
table-layoutControla el algoritmo de layout.table-layout: fixed;
widthAnchura total de la tabla.width: 100%;
text-alignAlineación horizontal del contenido.text-align: left/right/center;
vertical-alignAlineación vertical de celdas.vertical-align: middle;
caption-sidePosición del caption.caption-side: top/bottom;
position: stickyCabecera fija al hacer scroll.en th con top: 0;

🟧 Patrón general de tabla estilizada​

  • CSS
  • HTML
  • Representación
/* Tabla base */
.tabla--simple {
  width: 100%;
  border-radius: 6px;
  border-collapse:separate;
  border-spacing: 5px;
  font-size: 0.95rem;
  color: #1e293b;
}

/* Cabecera azul */
.tabla--simple thead th {
  background-color: #3b82f6;      /* azul intenso */
  color: #fff;
  font-weight: 700;
  text-align: center;
  padding: 0.8rem;
  border-right: 1px solid rgba(255, 255, 255, 0.3);
}
.tabla--simple thead th:last-child{ border-right: none; }
.tabla--simple thead th:first-child{ border-top-left-radius: 6px; }
.tabla--simple thead th:last-child { border-top-right-radius: 6px; }

/* Celdas del cuerpo */
.tabla--simple td {
  border: 1px solid #e5e7eb;
  padding: 0.7rem 0.9rem;
  vertical-align: middle;
  background-color: #fff;
}

/* Alineación numérica */
.tabla--simple .num{
  text-align: right;
  font-variant-numeric: tabular-nums;
}

/* --- Sticky header --- */
.tabla--sticky thead th{
  position: sticky;
  top: 0;            /* pega al borde superior del contenedor */
  z-index: 2;        /* sobre las celdas al hacer scroll */
}

✅ Buenas prácticas​

buenas practicas en listas
  • Usa ::marker cuando quieras solo cambiar el marcador; usa list-style: none + ::before para marcadores complejos.
  • Mantén coherente la sangría (padding-left) entre diferentes listas.
  • En menús, elimina viñetas y usa flex o grid con gap.
buenas prácticas en tablas
  • Prefiere border-collapse: collapse; para tablas de datos tradicionales.
  • Usa table-layout: fixed; cuando tengas muchas filas y anchos predecibles (mejor rendimiento).
  • Sticky header (position: sticky; top: 0;) mejora la lectura en scroll.
  • Para móviles, empieza por scroll horizontal. Considera “apilado” solo si aporta claridad.
  • Añade caption y scope en th para mejorar accesibilidad.
