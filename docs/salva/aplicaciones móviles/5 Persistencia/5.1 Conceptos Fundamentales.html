5.1 Conceptos Fundamentales

📱 La persistencia de datos en Android: Clave para un rendimiento superior​

La persistencia de datos es el corazón de una aplicación robusta. ¿Por qué? Porque garantiza que la información importante se conserve más allá de reinicios, cierres inesperados o cambios de configuración. Esto asegura que tus usuarios disfruten de una experiencia consistente y fluida.

❓ ¿Por qué es crucial persistir datos?​

🧠 La volatilidad de la memoria RAM:
Los datos almacenados en RAM desaparecen en casos como:

  • 🚪 Cierre de la aplicación: Por el usuario o inactividad prolongada.
  • 🔄 Rotación de pantalla: Cambios en la configuración del dispositivo.
  • 📉 Optimización del sistema: Liberación de memoria.
  • 🔋 Reinicio o apagado: Pérdida de datos al apagar el dispositivo.

🗂️ Persistencia: Consiste en almacenar datos en medios no volátiles para que estén disponibles incluso después de reiniciar la app o el dispositivo.

🛠️ Opciones de almacenamiento en Android​

Aquí tienes los principales métodos de almacenamiento, ilustrados para facilitar su comprensión:

1️⃣ SharedPreferences

  • 📝 Almacena configuraciones simples como: "tema oscuro = true".
  • ✔️ Ideal para datos básicos como cadenas, números o booleanos.

2️⃣ Almacenamiento interno

  • 🔐 Espacio privado de la aplicación, inaccesible para otras apps.
  • ✔️ Perfecto para datos sensibles.

3️⃣ Almacenamiento externo

  • 📂 Espacio compartido visible para otras aplicaciones y usuarios.
  • ✔️ Útil para fotos, videos o documentos.

4️⃣ SQLite

  • 📊 Base de datos relacional integrada en Android.
  • ✔️ Gestiona datos complejos y estructurados.

5️⃣ Room

  • 📚 Biblioteca que simplifica SQLite con menos código y más seguridad.
  • ✔️ La mejor opción para grandes volúmenes de datos.

6️⃣ Content Providers

  • 🔄 Intercambio estándar de datos entre aplicaciones.
  • ✔️ Ejemplo: compartir contactos o archivos multimedia.

7️⃣ Servicios en la nube

  • ☁️ Firebase, AWS, etc., para sincronización entre dispositivos.
  • ✔️ Datos accesibles desde cualquier lugar.

⏳ El ciclo de vida y la persistencia de datos​

🕒 Puntos clave en el ciclo de vida:

  • onPause() / onStop(): Momento perfecto para guardar cambios.
  • onDestroy(): Guarda datos críticos antes de que sea tarde.
🎯 Tips de buenas prácticas:
  • Guarda datos clave tan pronto como sea posible.
  • Evita operaciones pesadas en el hilo principal.
  • Usa hilos de fondo o herramientas como Room para manejar bases de datos.

🧠 Cómo elegir la mejor estrategia​

🎯 Usa esta tabla como guía para seleccionar el método de almacenamiento ideal:

📐 Criterio🚀 Recomendación
🔢 Volumen de datosSharedPreferences para datos pequeños; Room o SQLite para grandes volúmenes.
🛠️ EstructuraDatos simples: Archivos o preferencias. Datos complejos: Bases de datos.
🔁 Frecuencia de accesoRoom, optimizado para lecturas/escrituras frecuentes.
🤝 ComparticiónContent Providers o almacenamiento externo para datos compartidos.
🔒 SeguridadAlmacenamiento interno o cifrado para datos sensibles.
