Soporte OffLine

Firestore proporciona un modo offline que permite a las aplicaciones seguir funcionando sin conexión a Internet. Cuando un dispositivo pierde la conexión, Firestore almacena los datos en caché y sincroniza los cambios cuando se restablece la conexión.

Beneficios del soporte offline:
  • Permite acceder a datos sin conexión.
  • Guarda cambios localmente y los sincroniza automáticamente cuando vuelve la conexión.
  • Mejora el rendimiento al evitar consultas repetitivas a Firestore.
  • Reduce el consumo de red en dispositivos móviles.

1. Habilitar el Soporte Offline en Firestore​

El modo offline está activado por defecto en Firestore, pero podemos confirmarlo o activarlo manualmente con la siguiente configuración en nuestra aplicación:

🔹 Ejemplo: Habilitar la persistencia offline

FirebaseFirestore db = FirebaseFirestore.getInstance();

// Habilitar caché persistente
FirebaseFirestoreSettings settings = new FirebaseFirestoreSettings.Builder()
        .setPersistenceEnabled(true) // Activa el almacenamiento en caché
        .build();

db.setFirestoreSettings(settings);
Explicación:
  • setPersistenceEnabled(true) permite almacenar datos localmente.
  • Firestore usa un caché local para almacenar datos y sincronizarlos cuando haya conexión.

2. Consultar Datos desde el Caché Local​

Cuando Firestore está en modo offline, primero intenta recuperar datos del caché local antes de hacer una consulta en línea.

🔹 Ejemplo: Consultar datos desde caché

db.collection("peliculas").get(Source.CACHE)
    .addOnSuccessListener(queryDocumentSnapshots -> {
        for (DocumentSnapshot document : queryDocumentSnapshots) {
            Log.d("Firestore", "Datos desde caché: " + document.getData());
        }
    })
    .addOnFailureListener(e -> Log.w("Firestore", "Error al obtener datos offline", e));
Explicación:
  • Source.CACHE fuerza la consulta a usar solo los datos almacenados localmente.
  • Si el caché no tiene datos, la consulta fallará.

🔹 Fuentes de datos disponibles en Firestore:

FuenteDescripción
Source.DEFAULTUsa primero caché, si no hay datos consulta Firestore.
Source.CACHEObtiene los datos solo desde caché (offline).
Source.SERVERObtiene los datos solo desde Firestore (requiere conexión).

3. Detectar el Estado de Conexión​

Podemos verificar si la aplicación está conectada a Firestore en modo offline o en línea.

🔹 Ejemplo: Detectar conexión

db.collection("peliculas").addSnapshotListener((queryDocumentSnapshots, e) -> {
    if (e != null) {
        Log.w("Firestore", "Error en la escucha", e);
        return;
    }

    if (queryDocumentSnapshots != null && !queryDocumentSnapshots.isEmpty()) {
        Log.d("Firestore", "Datos recibidos desde: " +
                (queryDocumentSnapshots.getMetadata().isFromCache() ? "CACHÉ" : "SERVIDOR"));
    }
});
Explicación:
  • queryDocumentSnapshots.getMetadata().isFromCache() indica si los datos provienen del caché local o de Firestore en línea.
  • Útil para mostrar mensajes como "Modo offline activado" en la UI.

4. Sincronización Automática cuando Vuelve la Conexión​

Cuando el dispositivo recupera la conexión, Firestore sincroniza automáticamente los cambios almacenados en caché con la base de datos en la nube.

🔹 Ejemplo: Detectar cuándo Firestore sincroniza datos

db.collection("peliculas").addSnapshotListener((queryDocumentSnapshots, e) -> {
    if (queryDocumentSnapshots != null) {
        if (!queryDocumentSnapshots.getMetadata().hasPendingWrites()) {
            Log.d("Firestore", "Cambios sincronizados con el servidor.");
        } else {
            Log.d("Firestore", "Cambios pendientes de sincronización.");
        }
    }
});
Explicación:
  • hasPendingWrites() indica si hay cambios en caché que aún no han sido enviados al servidor.
  • Nos permite avisar al usuario cuando sus cambios se han guardado en Firestore.

5. Limpiar la Caché Manualmente​

Si la aplicación tiene problemas de espacio o datos desactualizados, podemos borrar el caché de Firestore.

🔹 Ejemplo: Limpiar caché local

db.clearPersistence()
    .addOnSuccessListener(aVoid -> Log.d("Firestore", "Caché borrado con éxito"))
    .addOnFailureListener(e -> Log.w("Firestore", "Error al limpiar caché", e));
Explicación:
  • clearPersistence() borra todos los datos almacenados en caché.
  • Solo funciona si no hay listeners activos en Firestore.
Advertencia:
  • Al limpiar la caché, los datos se perderán si el usuario está offline.
  • Se recomienda solo en casos donde sea necesario actualizar datos obsoletos.

6. Uso del Modo Offline con LiveData y MVVM​

Si usamos MVVM y LiveData, Firestore se puede integrar para obtener datos en tiempo real, incluso offline.

🔹 Ejemplo: ViewModel con Firestore y LiveData

public class PeliculasViewModel extends ViewModel {
    private MutableLiveData<List<Pelicula>> peliculasLiveData = new MutableLiveData<>();
    private FirebaseFirestore db = FirebaseFirestore.getInstance();

    public PeliculasViewModel() {
        db.collection("peliculas")
            .addSnapshotListener((queryDocumentSnapshots, e) -> {
                if (e != null) {
                    return;
                }

                List<Pelicula> listaPeliculas = new ArrayList<>();
                for (DocumentSnapshot document : queryDocumentSnapshots) {
                    Pelicula pelicula = document.toObject(Pelicula.class);
                    listaPeliculas.add(pelicula);
                }
                peliculasLiveData.setValue(listaPeliculas);
            });
    }

    public LiveData<List<Pelicula>> getPeliculas() {
        return peliculasLiveData;
    }
}
Explicación:
  • addSnapshotListener() obtiene datos en tiempo real y desde caché cuando no hay conexión.
  • LiveData permite que los datos se reflejen automáticamente en la UI.
