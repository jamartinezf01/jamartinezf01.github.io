Ejemplo Tablón de Anuncios

En este tutorial crearemos la aplicación "talón de anuncios", realizada previamente con el servicio Realtime Database, empleando Firestore...

1. Creación del Proyecto y Configuración Inicial​

  1. Crear un proyecto en Android Studio:
    • Selecciona Empty Activity (o Basic Activity).
    • Asegúrate de que el lenguaje sea Java.
    • Indica un Package name apropiado (p. ej. com.ejemplo.tablon).
    • Mínimo SDK: 28 (o el que prefieras).
  2. Activar el soporte para ViewBinding: en el build.gradle (Módulo: app), dentro de android {}, añade: 
    buildFeatures {
        viewBinding true
    }
  3. Configura tu proyecto para emplear Autentication y Firestore del modo visto en apartados anteriores
  4. Sincroniza el proyecto para que se apliquen los cambios.

2. Estructura del Proyecto (MVVM y Paquetes)​

A continuación se describe una estructura para una aplicación Android (Java) que cumple los siguientes requisitos:

  1. Un Activity de Registro donde el usuario se registra con Email y Password.
  2. Una Activity Principal con un Fragment:
    • TablonFragment: Muestra los anuncios y permite publicar uno nuevo.
  3. Se utiliza un solo Repositorio y un solo ViewModel que serán compartidos.
  4. Arquitectura básica MVVM (aunque simplificada).

La idea principal es la misma que en el caso de implementación anterior:

  • TablonRepository: Accede a Firebase (Auth + Realtime Database).
  • TablonViewModel: Provee métodos y LiveData para autenticación y anuncios.
  • RegistroActivity: Pantalla de registro de usuario.
  • MainActivity: Hospeda un Fragment (TablonFragment) y comparte el mismo TablonViewModel. Esto permitirá agregar más fragments en el futuro.

Una posible estructura de paquetes sería la siguiente...

com.ejemplo.tablon
 ┣ model
 ┃ ┣ Anuncio.java
 ┃ ┗ TablonRepository.java
 ┣ ui
 ┃ ┣ TablonViewModel.java
 ┃ ┣ TablonFragment.java
 ┃ ┗ AnunciosAdapter.java
 ┣ MainActivity.java
 ┗ RegistroActivity.java

Puedes ajustar la organización a tu preferencia, pero esta forma te ayuda a mantener el Repositorio, el ViewModel y las Activities/Fragments separados.

Reutilización de Código
  • Para la actividad RegstroActivity y su layOut emplearemos el código visto en el Ejemplo de Firebase Autentication.
  • Para la actividad MainActivity y su layOut emplearemos el código visto en el Ejemplo de Firebase RealTime Database.
  • Todo el pakage UI queda sin alterarse, gracias al modelo MVVM que separa la implementación de la UI del modelo de datos.
  • Para el modelo reutilizaremos la clase Anuncio

3. Modificaciones en el modelo​

Del proyecto original con Realtime deberemos modificar el Repositorio que es el que mantiene la lógica de comunicación con la base de datos. Para ello deberemos emplear la lógica de Firestore.

com/ejemplo/tablon/model/TablonRepository.java
package com.agora.tablonfirestore.model;

import androidx.annotation.NonNull;

import com.google.firebase.auth.FirebaseAuth;
import com.google.firebase.auth.FirebaseUser;
import com.google.firebase.database.*;

import com.google.firebase.firestore.*;

import java.util.ArrayList;
import java.util.Calendar;
import java.util.List;

public class TablonRepository {

    public interface AuthCallback {
        void onSuccess();
        void onFailure(String error);
    }

    public interface AnunciosCallback {
        void onSuccess(List<Anuncio> lista);
        void onFailure(String error);
    }

    private final FirebaseAuth mAuth;
    private final FirebaseFirestore db;
    private final CollectionReference anunciosRef;

    public TablonRepository() {
        mAuth = FirebaseAuth.getInstance();
        db = FirebaseFirestore.getInstance();
        anunciosRef = db.collection("anuncios"); // Referencia a la colección de Firestore
    }

    public FirebaseUser getUsuarioActual() {
        return mAuth.getCurrentUser();
    }

    // Publicar un anuncio
    public void publicarAnuncio(String contenido, AuthCallback callback) {
        FirebaseUser user = getUsuarioActual();
        if (user == null) {
            callback.onFailure("No hay usuario autenticado");
            return;
        }

        // Generar un ID automático en Firestore
        DocumentReference newAnuncioRef = anunciosRef.document();

        Anuncio anuncio = new Anuncio(newAnuncioRef.getId(), contenido, System.currentTimeMillis(), user.getEmail());

        newAnuncioRef.set(anuncio)
                .addOnSuccessListener(aVoid -> callback.onSuccess())
                .addOnFailureListener(e -> callback.onFailure(e.getMessage()));
    }

    // Leer todos los anuncios ordenados por fecha (del más reciente al más antiguo)
    public void leerAnuncios(AnunciosCallback callback) {
        anunciosRef.orderBy("fecha", Query.Direction.DESCENDING)
                .limit(50) // Limita a los últimos 50 anuncios
                .addSnapshotListener((querySnapshot, e) -> {
                    if (e != null) {
                        callback.onFailure(e.getMessage());
                        return;
                    }

                    List<Anuncio> lista = new ArrayList<>();
                    if (querySnapshot != null) {
                        for (DocumentSnapshot doc : querySnapshot.getDocuments()) {
                            Anuncio an = doc.toObject(Anuncio.class);
                            if (an != null) lista.add(an);
                        }
                    }
                    callback.onSuccess(lista);
                });
    }

    // Eliminar un anuncio
    public void eliminarAnuncio(String id, AuthCallback callback) {
        FirebaseUser user = getUsuarioActual();
        if (user == null) {
            callback.onFailure("No hay usuario autenticado");
            return;
        }

        anunciosRef.document(id).delete()
                .addOnSuccessListener(aVoid -> callback.onSuccess())
                .addOnFailureListener(e -> callback.onFailure(e.getMessage()));
    }

    // Búsqueda básica de anuncios por contenido
    public void buscarAnuncios(String texto, AnunciosCallback callback) {
        anunciosRef.whereEqualTo("contenido", texto)
                .get()
                .addOnSuccessListener(querySnapshot -> {
                    List<Anuncio> resultados = new ArrayList<>();
                    for (DocumentSnapshot doc : querySnapshot.getDocuments()) {
                        Anuncio an = doc.toObject(Anuncio.class);
                        if (an != null) resultados.add(an);
                    }
                    callback.onSuccess(resultados);
                })
                .addOnFailureListener(e -> callback.onFailure(e.getMessage()));
    }


    //Consulta más compleja: Buscar anuncios de un usuario en los últimos 7 días
    public void buscarAnunciosRecientesDeUsuario(String emailUsuario, AnunciosCallback callback) {
        // Obtener el timestamp de hace 7 días
        Calendar calendar = Calendar.getInstance();
        calendar.add(Calendar.DAY_OF_YEAR, -7);
        long fechaLimite = calendar.getTimeInMillis();

        // Firestore query: filtrar por usuario y fecha
        anunciosRef
                .whereEqualTo("uidAutor", emailUsuario) // Buscar solo los anuncios de este usuario
                .whereGreaterThan("fecha", fechaLimite) // Solo anuncios de los últimos 7 días
                .orderBy("fecha", Query.Direction.DESCENDING) // Ordenar por fecha, de más reciente a más antiguo
                .get()
                .addOnSuccessListener(querySnapshot -> {
                    List<Anuncio> resultados = new ArrayList<>();
                    for (DocumentSnapshot doc : querySnapshot.getDocuments()) {
                        Anuncio an = doc.toObject(Anuncio.class);
                        if (an != null) resultados.add(an);
                    }
                    callback.onSuccess(resultados);
                })
                .addOnFailureListener(e -> callback.onFailure(e.getMessage()));
    }
}
¿Qué cambios se han hecho?

  1. Base de datos:

    • DatabaseReference → Reemplazado por CollectionReference en Firestore.
    • addValueEventListener() → Reemplazado por addSnapshotListener() para lectura en tiempo real.

  2. Publicación de anuncios:

    • En lugar de push().getKey(), Firestore genera automáticamente un ID con document().getId().

  3. Eliminación de anuncios:

    • Se usa document(id).delete() en lugar de removeValue().

  4. Búsqueda:

    • Se usa whereEqualTo("contenido", texto).get() en lugar de orderByChild().startAt().endAt().
info

Las funciones buscarAnuncios() y buscarAnunciosRecientesDeUsuario() se han creado a modo de ejemplos de consultas aunque no se empleará en este proyecto.

4. Reglas de Seguridad en FIrestore​

Para asegurar tu nueva aplicación en Firestore, necesitas definir Reglas de Seguridad en Firebase Console (Sección: Firestore Database → Rules). Estas reglas deben: ✅ Proteger los datos para que solo los usuarios autenticados puedan leer y escribir.
✅ Evitar que los usuarios eliminen o editen anuncios de otros.
✅ Permitir la lectura pública si es necesario o restringirla según necesidades.

Reglas básicas​

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    // Reglas para la colección "anuncios"
    match /anuncios/{anuncioId} {
      
      // Solo los usuarios autenticados pueden leer los anuncios
      allow read: if request.auth != null;
      
      // Solo los usuarios autenticados pueden publicar anuncios
      allow create: if request.auth != null;
      
      // Solo el usuario que creó el anuncio puede modificarlo o eliminarlo
      allow update, delete: if request.auth != null && request.auth.token.email == resource.data.uidAutor;
    }
  }
}
Explicación de las reglas

1️⃣ allow read: if request.auth != null;

  • Solo los usuarios autenticados pueden ver los anuncios.
  • ❌ Si un usuario no está autenticado, Firestore rechazará la consulta.

2️⃣ allow create: if request.auth != null;

  • Solo los usuarios autenticados pueden publicar anuncios.

3️⃣ allow update, delete: if request.auth.token.email == resource.data.uidAutor;

  • Un usuario solo puede editar o eliminar su propio anuncio.
  • Compara el email del usuario autenticado (request.auth.token.email) con el email del autor (resource.data.uidAutor).

Opción más abierta: Permitir lectura pública​

Si quieres que cualquiera pueda ver los anuncios sin autenticación, pero solo los creadores puedan escribir o modificar:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    match /anuncios/{anuncioId} {
      
      // Cualquiera puede leer los anuncios (incluso sin autenticarse)
      allow read: if true;

      // Solo usuarios autenticados pueden escribir
      allow create: if request.auth != null;
      
      // Solo el usuario que creó el anuncio puede modificarlo o eliminarlo
      allow update, delete: if request.auth != null && request.auth.token.email == resource.data.uidAutor;
    }
  }
}
info

Útil si tu app es pública y cualquiera debe ver los anuncios sin iniciar sesión.
