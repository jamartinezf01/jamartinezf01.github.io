Operaciones CRUD

Una de las características más importantes de Firestore es su capacidad para realizar operaciones CRUD (Crear, Leer, Actualizar y Eliminar) de manera eficiente y escalable. Estas operaciones permiten gestionar los datos dentro de la base de datos y son esenciales para cualquier aplicación que requiera almacenamiento persistente.Estas operaciones nos permitirán...

✅ Crear documentos en colecciones, con y sin ID automático.
✅ Leer documentos individuales y colecciones completas.
✅ Actualizar datos específicos sin sobrescribir el documento completo.
✅ Eliminar documentos o campos específicos de la base de datos.

Estas operaciones se ejecutan de forma asíncrona y permiten interactuar con Firestore de manera eficiente, aprovechando su sistema de indexación y sincronización en tiempo real.

1. Crear documentos en Firestore​

En Firestore, los datos se organizan en colecciones, y dentro de estas colecciones, cada documento representa una entidad con información estructurada en forma de pares clave-valor (similar a un objeto JSON). Un documento no puede contener otros documentos, pero puede referenciar subcolecciones.

Crear un documento con un ID automático​

Cuando agregamos un documento sin especificar un ID, Firestore genera un identificador único aleatorio. Esto es útil cuando no necesitamos un control estricto sobre los identificadores.

🔹 Ejemplo...

// Instancia de Firestore
FirebaseFirestore db = FirebaseFirestore.getInstance();

// Crear un mapa con los datos del documento
Map<String, Object> pelicula = new HashMap<>();
pelicula.put("titulo", "Inception");
pelicula.put("director", "Christopher Nolan");
pelicula.put("año", 2010);
pelicula.put("genero", "Ciencia Ficción");

// Agregar el documento a la colección "peliculas"
db.collection("peliculas")
    .add(pelicula)
    .addOnSuccessListener(documentReference -> {
        Log.d("Firestore", "Documento agregado con ID: " + documentReference.getId());
    })
    .addOnFailureListener(e -> {
        Log.w("Firestore", "Error al agregar documento", e);
    });
Explicación:
  • Se obtiene una instancia de FirebaseFirestore.
  • Se crea un Map<String, Object> con los datos de la película.
  • Se usa .add(pelicula) para insertar el documento en la colección "peliculas".
  • Firestore genera un ID único automáticamente.
  • Si la operación es exitosa, el ID del documento se muestra en el log.

Crear un documento con un ID definido manualmente​

Si queremos que nuestro documento tenga un identificador específico (por ejemplo, un nombre legible o un ID basado en una clave primaria), podemos usar document(ID).set(datos) en lugar de add().

🔹 Ejemplo

// Instancia de Firestore
FirebaseFirestore db = FirebaseFirestore.getInstance();

// Crear los datos de la película
Map<String, Object> pelicula = new HashMap<>();
pelicula.put("titulo", "Interstellar");
pelicula.put("director", "Christopher Nolan");
pelicula.put("año", 2014);
pelicula.put("genero", "Ciencia Ficción");

// Guardar el documento con un ID específico
db.collection("peliculas")
    .document("interstellar") // ID definido manualmente
    .set(pelicula)
    .addOnSuccessListener(aVoid -> Log.d("Firestore", "Documento creado con ID específico"))
    .addOnFailureListener(e -> Log.w("Firestore", "Error al crear documento", e));
Explicación:
  • Se usa .document("interstellar") para definir el ID del documento.
  • .set(pelicula) guarda los datos en Firestore bajo el ID "interstellar".
Ventajas de usar un ID manual

✔️ Facilita la búsqueda directa si el ID es significativo.
✔️ Evita duplicados si usamos una clave única (por ejemplo, un email o un código).
✔️ Permite una mejor organización en bases de datos estructuradas.

Precaución

Si el documento ya existe con el mismo ID, se sobrescribirá al usar set(). Para evitarlo, debemos manejar los conflictos adecuadamente.

Prevenir sobrescritura​

Si queremos que Firestore solo cree el documento si no existe previamente, podemos usar .set() con SetOptions.merge().

🔹 Ejemplo

import com.google.firebase.firestore.SetOptions;

// Instancia de Firestore
FirebaseFirestore db = FirebaseFirestore.getInstance();

// Datos de la película
Map<String, Object> pelicula = new HashMap<>();
pelicula.put("titulo", "Tenet");
pelicula.put("director", "Christopher Nolan");
pelicula.put("año", 2020);
pelicula.put("genero", "Ciencia Ficción");

// Agregar documento sin sobrescribir datos existentes
db.collection("peliculas")
    .document("tenet")
    .set(pelicula, SetOptions.merge()) // Evita sobrescribir campos existentes
    .addOnSuccessListener(aVoid -> Log.d("Firestore", "Documento actualizado sin sobrescribir"))
    .addOnFailureListener(e -> Log.w("Firestore", "Error al actualizar documento", e));
Explicación:

SetOptions.merge() evita que los campos existentes en el documento se borren. Solo se actualizan o agregan los nuevos.

Crear documentos dentro de una subcolección​

Los documentos no pueden contener otros documentos directamente, pero sí pueden tener subcolecciones.

🔹 Ejemplo

// Datos de una reseña dentro de la subcolección "reseñas"
Map<String, Object> reseña = new HashMap<>();
reseña.put("usuario", "Juan Pérez");
reseña.put("comentario", "Excelente película");
reseña.put("puntuacion", 5);

// Agregar una reseña a la película "inception"
db.collection("peliculas")
    .document("inception")
    .collection("reseñas")  // Subcolección
    .add(reseña)
    .addOnSuccessListener(documentReference -> {
        Log.d("Firestore", "Reseña agregada con ID: " + documentReference.getId());
    })
    .addOnFailureListener(e -> {
        Log.w("Firestore", "Error al agregar reseña", e);
    });

2. Leer documentos en Firestore​

Firestore permite leer datos almacenados en documentos individuales o colecciones completas.

Leer un documento por su ID​

Si conocemos el ID de un documento, podemos obtenerlo directamente usando get().

🔹 Ejemplo

// Instancia de Firestore
FirebaseFirestore db = FirebaseFirestore.getInstance();

// Obtener el documento con ID "inception" en la colección "peliculas"
db.collection("peliculas").document("inception")
    .get()
    .addOnSuccessListener(documentSnapshot -> {
        if (documentSnapshot.exists()) {
            String titulo = documentSnapshot.getString("titulo");
            String director = documentSnapshot.getString("director");
            Long año = documentSnapshot.getLong("año");

            Log.d("Firestore", "Título: " + titulo + ", Director: " + director + ", Año: " + año);
        } else {
            Log.d("Firestore", "El documento no existe");
        }
    })
    .addOnFailureListener(e -> Log.w("Firestore", "Error al obtener documento", e));
Explicación:
  • Se usa .document("inception") para acceder al documento con ese ID.
  • .get() recupera los datos almacenados.
  • documentSnapshot.getString("titulo") obtiene valores de los campos.
  • Se verifica si el documento existe antes de acceder a sus datos.

Leer todos los documentos de una colección​

Para obtener todos los documentos de una colección, usamos get() sobre la colección.

🔹 Ejemplo

db.collection("peliculas")
    .get()
    .addOnSuccessListener(queryDocumentSnapshots -> {
        for (DocumentSnapshot document : queryDocumentSnapshots) {
            String titulo = document.getString("titulo");
            String director = document.getString("director");
            Long año = document.getLong("año");

            Log.d("Firestore", "Título: " + titulo + ", Director: " + director + ", Año: " + año);
        }
    })
    .addOnFailureListener(e -> Log.w("Firestore", "Error al obtener documentos", e));
Explicación:
  • .collection("peliculas").get() obtiene todos los documentos de la colección.
  • Se itera sobre cada DocumentSnapshot para extraer los datos.

Leer un documento dentro de una subcolección​

Si un documento tiene una subcolección, podemos acceder a ella con collection().

🔹 Ejemplo

db.collection("peliculas").document("inception").collection("reseñas")
    .get()
    .addOnSuccessListener(queryDocumentSnapshots -> {
        for (DocumentSnapshot document : queryDocumentSnapshots) {
            String usuario = document.getString("usuario");
            String comentario = document.getString("comentario");
            Long puntuacion = document.getLong("puntuacion");

            Log.d("Firestore", "Usuario: " + usuario + ", Comentario: " + comentario + ", Puntuación: " + puntuacion);
        }
    })
    .addOnFailureListener(e -> Log.w("Firestore", "Error al obtener reseñas", e));
Explicación:
  • document("inception").collection("reseñas") accede a la subcolección "reseñas".
  • Se itera sobre los documentos dentro de la subcolección.

Filtrar documentos con consultas​

Firestore permite aplicar filtros con whereEqualTo(), whereGreaterThan(), etc.

🔹 Ejemplo: Obtener películas de un género específico

db.collection("peliculas")
    .whereEqualTo("genero", "Ciencia Ficción")
    .get()
    .addOnSuccessListener(queryDocumentSnapshots -> {
        for (DocumentSnapshot document : queryDocumentSnapshots) {
            Log.d("Firestore", "Película: " + document.getString("titulo"));
        }
    })
    .addOnFailureListener(e -> Log.w("Firestore", "Error al filtrar documentos", e));

🔹 Ejemplo: Obtener películas estrenadas después del 2010

db.collection("peliculas")
    .whereGreaterThan("año", 2010)
    .get()
    .addOnSuccessListener(queryDocumentSnapshots -> {
        for (DocumentSnapshot document : queryDocumentSnapshots) {
            Log.d("Firestore", "Película: " + document.getString("titulo") + ", Año: " + document.getLong("año"));
        }
    })
    .addOnFailureListener(e -> Log.w("Firestore", "Error en la consulta", e));
Explicación:
  • whereEqualTo("genero", "Ciencia Ficción") filtra solo películas de ese género.
  • whereGreaterThan("año", 2010) obtiene películas estrenadas después de 2010.

Escuchar cambios en tiempo real​

Si queremos que los datos se actualicen automáticamente cuando cambien en Firestore, usamos addSnapshotListener().

🔹 Ejemplo

db.collection("peliculas").document("inception")
    .addSnapshotListener((documentSnapshot, e) -> {
        if (e != null) {
            Log.w("Firestore", "Error al escuchar cambios", e);
            return;
        }

        if (documentSnapshot != null && documentSnapshot.exists()) {
            String titulo = documentSnapshot.getString("titulo");
            String director = documentSnapshot.getString("director");
            Log.d("Firestore", "Datos actualizados -> Título: " + titulo + ", Director: " + director);
        }
    });
Explicación:
  • La aplicación escucha cambios en el documento "inception".
  • Si el documento se modifica, la función se ejecuta automáticamente.

3. Actualizar documentos en Firestore​

Actualizar documentos en Firestore nos permite modificar datos sin necesidad de eliminarlos o reemplazarlos por completo. Existen varias formas de realizar una actualización:

Sobrescribir un documento completamente (set())​

Cuando usamos set() en un documento existente, Firestore sobrescribe todos los datos previos con los nuevos. Si el documento no existe, lo crea.

🔹 Ejemplo

FirebaseFirestore db = FirebaseFirestore.getInstance();

// Nuevo conjunto de datos (sobreescribe todo)
Map<String, Object> pelicula = new HashMap<>();
pelicula.put("titulo", "Interstellar");
pelicula.put("director", "Christopher Nolan");
pelicula.put("año", 2014);
pelicula.put("genero", "Ciencia Ficción");

// Sobrescribir completamente el documento con ID "inception"
db.collection("peliculas").document("inception")
    .set(pelicula)
    .addOnSuccessListener(aVoid -> Log.d("Firestore", "Documento sobrescrito con éxito"))
    .addOnFailureListener(e -> Log.w("Firestore", "Error al sobrescribir documento", e));
Explicación:
  • .set(pelicula) reemplaza todo el contenido del documento "inception".
  • Si faltan campos en el nuevo mapa, los valores anteriores se perderán.
peligro

Si el documento contenía otros campos no incluidos en pelicula, se eliminarán.

Actualizar solo campos específicos (update())​

Si solo queremos modificar ciertos campos sin afectar los demás, usamos update().

🔹 Ejemplo

db.collection("peliculas").document("inception")
    .update("año", 2020)  // Modifica solo el campo "año"
    .addOnSuccessListener(aVoid -> Log.d("Firestore", "Campo actualizado"))
    .addOnFailureListener(e -> Log.w("Firestore", "Error al actualizar campo", e));
Explicación:
  • Solo se actualiza el campo "año", manteniendo el resto del documento intacto.
  • Si el documento no existe, la operación fallará.

Añadir nuevos campos sin sobrescribir (set() con merge())​

Si queremos agregar nuevos campos sin eliminar los existentes, usamos SetOptions.merge().

🔹 Ejemplo

import com.google.firebase.firestore.SetOptions;

Map<String, Object> nuevaInfo = new HashMap<>();
nuevaInfo.put("duracion", 148);  // Nuevo campo: duración en minutos

db.collection("peliculas").document("inception")
    .set(nuevaInfo, SetOptions.merge())  // Agrega sin borrar los datos existentes
    .addOnSuccessListener(aVoid -> Log.d("Firestore", "Campo agregado sin sobrescribir"))
    .addOnFailureListener(e -> Log.w("Firestore", "Error al agregar campo", e));
Explicación:
  • .set(nuevaInfo, SetOptions.merge()) fusiona los nuevos datos con los existentes.
  • No elimina los campos previos, solo agrega o actualiza los especificados.

Incrementar valores numéricos (FieldValue.increment())​

Si queremos sumar o restar valores numéricos, usamos FieldValue.increment().

🔹 Ejemplo

import com.google.firebase.firestore.FieldValue;

db.collection("peliculas").document("inception")
    .update("visualizaciones", FieldValue.increment(1))  // Incrementa en 1
    .addOnSuccessListener(aVoid -> Log.d("Firestore", "Visualizaciones incrementadas"))
    .addOnFailureListener(e -> Log.w("Firestore", "Error al incrementar visualizaciones", e));
Explicación:
  • Si el campo no existe, se crea automáticamente con el valor 1.
  • Se usa FieldValue.increment(valor) para sumar o restar valores sin necesidad de obtener el documento primero.

Actualizar arrays (FieldValue.arrayUnion() y FieldValue.arrayRemove())​

Firestore permite agregar o eliminar elementos en arrays sin sobrescribirlos.

🔹 Ejemplo: Agregar un nuevo actor al array actores

db.collection("peliculas").document("inception")
    .update("actores", FieldValue.arrayUnion("Leonardo DiCaprio"))
    .addOnSuccessListener(aVoid -> Log.d("Firestore", "Actor agregado"))
    .addOnFailureListener(e -> Log.w("Firestore", "Error al agregar actor", e));

🔹 Ejemplo: Eliminar un actor del array actores

db.collection("peliculas").document("inception")
    .update("actores", FieldValue.arrayRemove("Leonardo DiCaprio"))
    .addOnSuccessListener(aVoid -> Log.d("Firestore", "Actor eliminado"))
    .addOnFailureListener(e -> Log.w("Firestore", "Error al eliminar actor", e));
Explicación:
  • FieldValue.arrayUnion() agrega valores al array sin duplicados.
  • FieldValue.arrayRemove() elimina un valor específico del array.

4. Eliminar documentos​

Firestore permite eliminar tanto documentos completos como campos específicos dentro de un documento. Sin embargo, es importante notar que la eliminación de un documento no elimina sus subcolecciones. Estas seguirán existiendo en la base de datos hasta que se eliminen manualmente.

Eliminar un documento completo​

Para eliminar un documento por su ID, usamos .delete() sobre la referencia al documento.

🔹 Ejemplo

FirebaseFirestore db = FirebaseFirestore.getInstance();

db.collection("peliculas").document("inception")
    .delete()
    .addOnSuccessListener(aVoid -> Log.d("Firestore", "Documento eliminado con éxito"))
    .addOnFailureListener(e -> Log.w("Firestore", "Error al eliminar documento", e));
Explicación:
  • document("inception") accede al documento con el ID "inception".
  • .delete() lo elimina por completo de Firestore.
  • Si el documento no existe, Firestore no generará un error, simplemente no hará nada.
Importante:
  • Las subcolecciones del documento eliminado seguirán existiendo en Firestore.
  • Si necesitas eliminar una subcolección junto con el documento, debes hacerlo manualmente recorriendo sus elementos y eliminándolos.

Eliminar un campo dentro de un documento​

Si solo queremos borrar un campo dentro del documento pero sin eliminarlo completamente, podemos usar FieldValue.delete().

🔹 Ejemplo

db.collection("peliculas").document("inception")
    .update("genero", FieldValue.delete())
    .addOnSuccessListener(aVoid -> Log.d("Firestore", "Campo eliminado con éxito"))
    .addOnFailureListener(e -> Log.w("Firestore", "Error al eliminar campo", e));
Explicación:
  • .update("genero", FieldValue.delete()) borra únicamente el campo "genero".
  • Los demás campos del documento permanecen intactos.
  • Si el campo no existe, Firestore simplemente ignora la operación.

Eliminar todos los documentos de una colección​

Firestore no permite eliminar una colección directamente, por lo que debemos recorrer cada documento y eliminarlo uno por uno.

🔹 Ejemplo

db.collection("peliculas")
    .get()
    .addOnSuccessListener(queryDocumentSnapshots -> {
        for (DocumentSnapshot document : queryDocumentSnapshots) {
            db.collection("peliculas").document(document.getId()).delete();
        }
        Log.d("Firestore", "Todos los documentos eliminados");
    })
    .addOnFailureListener(e -> Log.w("Firestore", "Error al eliminar documentos", e));
Explicación:
  • Se obtiene todos los documentos de la colección "peliculas".
  • Se recorren los documentos y se eliminan uno por uno con .delete().
  • No se eliminan las subcolecciones automáticamente.
Consideraciones importantes:
  • Si hay muchos documentos en la colección, la operación puede ser lenta.
  • Para evitar problemas de rendimiento, se recomienda eliminar documentos en lotes.

Eliminar un documento y todas sus subcolecciones​

Firestore no elimina automáticamente las subcolecciones cuando se borra un documento. Si queremos eliminar un documento y todas sus subcolecciones, debemos recorrerlas manualmente.

🔹 Ejemplo

void eliminarDocumentoYSubcolecciones(String coleccion, String documentoId) {
    FirebaseFirestore db = FirebaseFirestore.getInstance();
    
    // Obtener subcolecciones del documento
    db.collection(coleccion).document(documentoId).collection("reseñas")
        .get()
        .addOnSuccessListener(queryDocumentSnapshots -> {
            for (DocumentSnapshot subdocumento : queryDocumentSnapshots) {
                db.collection(coleccion).document(documentoId)
                    .collection("reseñas").document(subdocumento.getId()).delete();
            }

            // Eliminar el documento principal después de borrar las subcolecciones
            db.collection(coleccion).document(documentoId)
                .delete()
                .addOnSuccessListener(aVoid -> Log.d("Firestore", "Documento y subcolecciones eliminados"))
                .addOnFailureListener(e -> Log.w("Firestore", "Error al eliminar documento", e));
        });
}

// Llamada a la función
eliminarDocumentoYSubcolecciones("peliculas", "inception");
Explicación:
  • Primero, obtenemos los documentos de la subcolección "reseñas" dentro del documento "inception".
  • Eliminamos cada documento de la subcolección uno por uno.
  • Finalmente, eliminamos el documento "inception".
