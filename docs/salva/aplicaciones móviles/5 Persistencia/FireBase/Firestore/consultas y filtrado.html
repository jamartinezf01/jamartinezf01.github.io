Consultas y Filtrado

Firestore ofrece un potente sistema de consultas que permite recuperar documentos de manera eficiente, aplicando filtros, ordenaciones y paginación. A diferencia de bases de datos SQL, Firestore no utiliza consultas con JOIN, sino que organiza la información en colecciones y documentos, permitiendo realizar búsquedas flexibles a través de su API. Mediante consultas y filtrado podremos...

✅ Obtener todos los documentos de una colección de forma sencilla.
✅ Filtrar datos utilizando condiciones como whereEqualTo(), whereGreaterThan(), whereLessThanOrEqualTo(), entre otros.
✅ Ordenar los resultados mediante orderBy(), permitiendo organizar los datos en orden ascendente o descendente.
✅ Combinar múltiples condiciones en una misma consulta para obtener resultados más precisos.
✅ Implementar paginación con limit() y startAfter() para mejorar el rendimiento cuando manejamos grandes volúmenes de datos.

Estas técnicas son esenciales para optimizar el rendimiento y la escalabilidad de una aplicación que usa Firestore como base de datos.

1. Obtener todos los documentos de una colección​

La forma más básica de obtener datos en Firestore es traer todos los documentos de una colección con get().

🔹 Ejemplo

FirebaseFirestore db = FirebaseFirestore.getInstance();

db.collection("peliculas")
    .get()
    .addOnSuccessListener(queryDocumentSnapshots -> {
        for (DocumentSnapshot document : queryDocumentSnapshots) {
            Log.d("Firestore", "ID: " + document.getId() + " -> " + document.getData());
        }
    })
    .addOnFailureListener(e -> Log.w("Firestore", "Error al obtener documentos", e));
Explicación:
  • .collection("peliculas").get() obtiene todos los documentos de la colección "peliculas".
  • Se recorren los documentos y se imprimen sus datos.
Cuidado:

Si la colección tiene muchos documentos, esta consulta puede afectar el rendimiento y aumentar los costos de Firestore.

2. Filtrar datos con condiciones (whereEqualTo(), whereGreaterThan(), etc.)​

Firestore permite filtrar datos con condiciones en los campos de los documentos.

Filtrar con whereEqualTo()​

Para obtener documentos que coincidan con un valor específico en un campo:

db.collection("peliculas")
    .whereEqualTo("genero", "Ciencia Ficción")
    .get()
    .addOnSuccessListener(queryDocumentSnapshots -> {
        for (DocumentSnapshot document : queryDocumentSnapshots) {
            Log.d("Firestore", "Película: " + document.getString("titulo"));
        }
    })
    .addOnFailureListener(e -> Log.w("Firestore", "Error al filtrar documentos", e));
Explicación:
  • whereEqualTo("genero", "Ciencia Ficción") obtiene solo las películas del género "Ciencia Ficción".

Filtrar con whereGreaterThan() y whereLessThanOrEqualTo()​

Podemos obtener documentos con valores mayores, menores o iguales a un valor dado.

db.collection("peliculas")
    .whereGreaterThan("año", 2010) // Películas después de 2010
    .get()
    .addOnSuccessListener(queryDocumentSnapshots -> {
        for (DocumentSnapshot document : queryDocumentSnapshots) {
            Log.d("Firestore", "Película: " + document.getString("titulo") + ", Año: " + document.getLong("año"));
        }
    })
    .addOnFailureListener(e -> Log.w("Firestore", "Error en la consulta", e));
Explicación:
  • whereGreaterThan("año", 2010) filtra solo las películas estrenadas después de 2010.

Operadores disponibles:​

OperadorDescripciónEjemplo de uso
whereEqualTo()Coincidencia exacta con un valor específico..whereEqualTo("genero", "Acción")
whereNotEqualTo()Filtra documentos donde el campo no sea igual al valor dado..whereNotEqualTo("genero", "Terror")
whereGreaterThan()Filtra documentos donde el campo sea mayor que un valor..whereGreaterThan("año", 2000)
whereGreaterThanOrEqualTo()Filtra documentos donde el campo sea mayor o igual a un valor..whereGreaterThanOrEqualTo("año", 2010)
whereLessThan()Filtra documentos donde el campo sea menor que un valor..whereLessThan("año", 2000)
whereLessThanOrEqualTo()Filtra documentos donde el campo sea menor o igual a un valor..whereLessThanOrEqualTo("año", 1999)
whereArrayContains()Filtra documentos donde un array contenga un valor específico..whereArrayContains("actores", "Leonardo DiCaprio")
whereArrayContainsAny()Filtra documentos donde un array contenga al menos uno de varios valores..whereArrayContainsAny("actores", Arrays.asList("Tom Hardy", "Brad Pitt"))
whereIn()Filtra documentos donde el valor del campo coincida con cualquiera de una lista dada..whereIn("genero", Arrays.asList("Acción", "Ciencia Ficción"))
whereNotIn()Filtra documentos donde el valor del campo no coincida con ninguno de una lista dada..whereNotIn("genero", Arrays.asList("Romance", "Drama"))
Notas importantes sobre los operadores:
  • whereNotEqualTo() y whereNotIn() no se pueden usar en la misma consulta con otros filtros de whereEqualTo().
  • whereArrayContains() solo funciona para buscar elementos dentro de arrays.
  • whereIn() y whereArrayContainsAny() permiten máximo 10 valores en la lista.

3. Ordenación de datos con orderBy()​

Firestore permite ordenar resultados por un campo específico con orderBy().

🔹 Ejemplo: Ordenar películas por año de estreno

db.collection("peliculas")
    .orderBy("año") // Orden ascendente
    .get()
    .addOnSuccessListener(queryDocumentSnapshots -> {
        for (DocumentSnapshot document : queryDocumentSnapshots) {
            Log.d("Firestore", "Película: " + document.getString("titulo") + ", Año: " + document.getLong("año"));
        }
    })
    .addOnFailureListener(e -> Log.w("Firestore", "Error en la ordenación", e));
Explicación:
  • orderBy("año") ordena las películas de más antigua a más reciente.

🔹 Orden descendente (DESC)

.orderBy("año", Query.Direction.DESCENDING)
Reglas de Firestore:

Si usas whereGreaterThan(), debes incluir orderBy() en el mismo campo.
Por ejemplo:

.whereGreaterThan("año", 2010)
.orderBy("año")

4. Consultas combinadas (múltiples condiciones)​

Firestore permite combinar varias condiciones en una misma consulta.

🔹 Ejemplo: Películas de ciencia ficción después del año 2000

db.collection("peliculas")
    .whereEqualTo("genero", "Ciencia Ficción")
    .whereGreaterThan("año", 2000)
    .get()
    .addOnSuccessListener(queryDocumentSnapshots -> {
        for (DocumentSnapshot document : queryDocumentSnapshots) {
            Log.d("Firestore", "Película: " + document.getString("titulo") + ", Año: " + document.getLong("año"));
        }
    })
    .addOnFailureListener(e -> Log.w("Firestore", "Error en la consulta", e));
Explicación:
  • .whereEqualTo("genero", "Ciencia Ficción") filtra solo películas de ese género.
  • .whereGreaterThan("año", 2000) filtra solo películas posteriores al año 2000.
Limitaciones:
  • No se pueden usar dos condiciones en distintos campos sin índices compuestos.
  • Si necesitas múltiples condiciones, debes crear índices en Firebase Console.

5. Paginación de resultados con limit() y startAfter()​

Firestore permite paginar resultados para evitar cargar demasiados documentos a la vez.

🔹 Ejemplo: Obtener solo los primeros 3 resultados

db.collection("peliculas")
    .orderBy("año")
    .limit(3)
    .get()
    .addOnSuccessListener(queryDocumentSnapshots -> {
        for (DocumentSnapshot document : queryDocumentSnapshots) {
            Log.d("Firestore", "Película: " + document.getString("titulo") + ", Año: " + document.getLong("año"));
        }
    })
    .addOnFailureListener(e -> Log.w("Firestore", "Error en la consulta paginada", e));
Explicación:
  • .limit(3) devuelve solo los primeros 3 documentos.

🔹 Ejemplo: Obtener la siguiente página de resultados Para obtener la siguiente página, usamos startAfter().

// Último documento de la página actual
DocumentSnapshot ultimoDocumento = ...;

db.collection("peliculas")
    .orderBy("año")
    .startAfter(ultimoDocumento) // Continúa desde el último documento de la página anterior
    .limit(3)
    .get()
    .addOnSuccessListener(queryDocumentSnapshots -> {
        for (DocumentSnapshot document : queryDocumentSnapshots) {
            Log.d("Firestore", "Película: " + document.getString("titulo") + ", Año: " + document.getLong("año"));
        }
    });
Explicación:
  • startAfter(ultimoDocumento) salta los documentos ya mostrados.
  • Se usa limit(3) para traer la siguiente página.
