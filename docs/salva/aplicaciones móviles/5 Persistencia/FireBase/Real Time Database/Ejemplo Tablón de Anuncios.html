Ejemplo Tablón de Anuncios

En este tutorial crearemos una aplicación Android que implemente un talón de anuncios donde se puedan postear anuncios que sean vistos por todos los usuarios y solo puedan ser borrados por el usuario que posteó el anuncio..

Creación del Proyecto y Configuración Inicial​

  1. Crear un proyecto en Android Studio:
    • Selecciona Empty Activity (o Basic Activity).
    • Asegúrate de que el lenguaje sea Java.
    • Indica un Package name apropiado (p. ej. com.ejemplo.tablon).
    • Mínimo SDK: 28 (o el que prefieras).
  2. Activar el soporte para ViewBinding: en el build.gradle (Módulo: app), dentro de android {}, añade: 
    buildFeatures {
        viewBinding true
    }
  3. Configura tu proyecto para emplear Autentication y RealTime Database del modo visto en apartados anteriores
  4. Sincroniza el proyecto para que se apliquen los cambios.

Estructura del Proyecto (MVVM y Paquetes)​

A continuación se describe una estructura para una aplicación Android (Java) que cumple los siguientes requisitos:

  1. Un Activity de Registro donde el usuario se registra con Email y Password.
  2. Una Activity Principal con un Fragment:
    • TablonFragment: Muestra los anuncios y permite publicar uno nuevo.
  3. Se utiliza un solo Repositorio y un solo ViewModel que serán compartidos.
  4. Arquitectura básica MVVM (aunque simplificada).

La idea principal es:

  • TablonRepository: Accede a Firebase (Auth + Realtime Database).
  • TablonViewModel: Provee métodos y LiveData para autenticación y anuncios.
  • RegistroActivity: Pantalla de registro de usuario.
  • MainActivity: Hospeda un Fragment (TablonFragment) y comparte el mismo TablonViewModel. Esto permitirá agregar más fragments en el futuro.

Una posible estructura de paquetes sería la siguiente...

com.ejemplo.tablon
 ┣ model
 ┃ ┣ Anuncio.java
 ┃ ┗ TablonRepository.java
 ┣ ui
 ┃ ┣ TablonViewModel.java
 ┃ ┣ TablonFragment.java
 ┃ ┗ AnunciosAdapter.java
 ┣ MainActivity.java
 ┗ RegistroActivity.java

Puedes ajustar la organización a tu preferencia, pero esta forma te ayuda a mantener el Repositorio, el ViewModel y las Activities/Fragments separados.

2. El Repositorio Único: TablonRepository​

Esta clase se encarga de todas las operaciones contra Firebase.

com/ejemplo/tablon/model/TablonRepository.java
package com.agora.tablon.model;

import androidx.annotation.NonNull;

import com.google.firebase.auth.FirebaseAuth;
import com.google.firebase.auth.FirebaseUser;
import com.google.firebase.database.*;

import java.util.ArrayList;
import java.util.List;

public class TablonRepository {

    // Interfaz para manejar respuestas de autenticación
    public interface AuthCallback {
        void onSuccess();  // Se ejecuta cuando la operación es exitosa
        void onFailure(String error); // Se ejecuta cuando hay un error
    }

    // Interfaz para manejar respuestas al obtener anuncios
    public interface AnunciosCallback {
        void onSuccess(List<Anuncio> lista);  // Se ejecuta cuando se obtienen los anuncios correctamente
        void onFailure(String error); // Se ejecuta cuando hay un error al obtener los anuncios
    }

    // Instancia de autenticación de Firebase para manejar usuarios
    private final FirebaseAuth mAuth;

    // Referencia a la base de datos en Firebase donde se guardan los anuncios
    private final DatabaseReference anunciosRef;

    // Constructor de la clase
    public TablonRepository() {
        // Habilita la persistencia local de Firebase para que los datos se almacenen en caché y funcionen sin conexión
        FirebaseDatabase.getInstance().setPersistenceEnabled(true);

        // Obtiene la instancia de autenticación de Firebase
        mAuth = FirebaseAuth.getInstance();

        // Obtiene la referencia a la ubicación "anuncios" en la base de datos
        anunciosRef = FirebaseDatabase.getInstance().getReference("anuncios");

        // Mantiene sincronizados los datos entre Firebase y la caché local
        anunciosRef.keepSynced(true);
    }

    // Obtiene el usuario actualmente autenticado
    public FirebaseUser getUsuarioActual() {
        return mAuth.getCurrentUser();
    }

    // Publicar un anuncio en Firebase
    public void publicarAnuncio(String contenido, AuthCallback callback) {
        FirebaseUser user = getUsuarioActual();

        // Verifica si hay un usuario autenticado antes de permitir la publicación
        if (user == null) {
            callback.onFailure("No hay usuario autenticado");
            return;
        }

        // Genera una clave única para el nuevo anuncio
        String key = anunciosRef.push().getKey();

        // Si no se pudo generar la clave, devuelve un error
        if (key == null) {
            callback.onFailure("No se pudo generar clave para el anuncio");
            return;
        }

        // Crea un objeto Anuncio con los datos
        Anuncio anuncio = new Anuncio(key, contenido, System.currentTimeMillis(), user.getEmail());

        // Guarda el anuncio en Firebase en la ruta "anuncios/key"
        anunciosRef.child(key).setValue(anuncio)
                .addOnCompleteListener(task -> {
                    if (task.isSuccessful()) {
                        callback.onSuccess(); // Si se guarda correctamente, notifica éxito
                    } else {
                        // Manejo de errores en caso de fallo
                        if (task.getException() != null) {
                            callback.onFailure(task.getException().getMessage());
                        } else {
                            callback.onFailure("Error desconocido al publicar");
                        }
                    }
                });
    }

    // Leer los anuncios de Firebase
    public void leerAnuncios(AnunciosCallback callback) {
        // Obtiene los últimos 50 anuncios ordenados por fecha
        anunciosRef.orderByChild("fecha").limitToLast(50)
                .addValueEventListener(new ValueEventListener() {
                    @Override
                    public void onDataChange(@NonNull DataSnapshot snapshot) {
                        List<Anuncio> lista = new ArrayList<>();
                        
                        // Recorre los datos obtenidos de Firebase y los convierte en objetos Anuncio
                        for (DataSnapshot ds : snapshot.getChildren()) {
                            Anuncio an = ds.getValue(Anuncio.class);
                            if (an != null) lista.add(an);
                        }

                        // Invierte la lista para mostrar los anuncios más recientes primero
                        Collections.reverse(lista);
                        callback.onSuccess(lista);
                    }

                    @Override
                    public void onCancelled(@NonNull DatabaseError error) {
                        callback.onFailure(error.getMessage()); // Notifica error si la consulta es cancelada
                    }
                });
    }

    // Eliminar un anuncio de Firebase
    public void eliminarAnuncio(String id, AuthCallback callback) {
        FirebaseUser user = getUsuarioActual();

        // Verifica si hay un usuario autenticado antes de permitir la eliminación
        if (user == null) {
            callback.onFailure("No hay usuario autenticado");
            return;
        }

        // Elimina el anuncio con el ID dado de la base de datos
        anunciosRef.child(id).removeValue()
                .addOnCompleteListener(task -> {
                    if (task.isSuccessful()) {
                        callback.onSuccess(); // Notifica éxito si la eliminación es correcta
                    } else {
                        // Manejo de errores en caso de fallo
                        if (task.getException() != null) {
                            callback.onFailure(task.getException().getMessage());
                        } else {
                            callback.onFailure("Error desconocido al eliminar");
                        }
                    }
                });
    }

    // Búsqueda de anuncios por contenido en Firebase
    public void buscarAnuncios(String texto, AnunciosCallback callback) {
        // Realiza una consulta en Firebase buscando anuncios cuyo contenido comience con el texto dado
        Query query = anunciosRef.orderByChild("contenido")
                .startAt(texto)
                .endAt(texto + "\uf8ff"); // "\uf8ff" es un carácter especial que permite buscar coincidencias con prefijo

        query.addValueEventListener(new ValueEventListener() {
            @Override
            public void onDataChange(@NonNull DataSnapshot snapshot) {
                List<Anuncio> resultados = new ArrayList<>();

                // Recorre los resultados y los almacena en una lista
                for (DataSnapshot ds : snapshot.getChildren()) {
                    Anuncio an = ds.getValue(Anuncio.class);
                    if (an != null) resultados.add(an);
                }
                
                callback.onSuccess(resultados); // Devuelve los resultados al callback
            }

            @Override
            public void onCancelled(@NonNull DatabaseError error) {
                callback.onFailure(error.getMessage()); // Notifica error si la consulta es cancelada
            }
        });
    }
}

Características clave del uso de Firebase
  • Persistencia habilitada: FirebaseDatabase.getInstance().setPersistenceEnabled(true);
    • Permite que los datos sigan estando disponibles sin conexión.
  • Sincronización automática: anunciosRef.keepSynced(true);
    • Asegura que la base de datos local y la remota estén siempre en sincronía.
  • Uso de push() para claves únicas:
    • Cada anuncio obtiene un identificador único generado por Firebase, evitando conflictos.
  • Eventos en tiempo real con addValueEventListener:
    • Cada vez que hay un cambio en la base de datos, la interfaz se actualiza automáticamente.
info

La función buscarAnuncios() se ha creado a modo de ejemplo de consulta simple aunque no se empleará en este proyecto.

El ViewModel Único: TablonViewModel​

Utilizamos LiveData y métodos que internamente llaman al Repositorio. Así, cualquier Fragment o Activity que obtenga este ViewModel puede compartir el mismo estado.

com/ejemplo/tablon/ui/TablonViewModel.java
package com.ejemplo.tablon.ui;

import androidx.lifecycle.LiveData;
import androidx.lifecycle.MutableLiveData;
import androidx.lifecycle.ViewModel;

import com.ejemplo.tablon.model.TablonRepository;
import com.ejemplo.tablon.model.Anuncio;
import com.google.firebase.auth.FirebaseUser;

import java.util.List;

public class TablonViewModel extends ViewModel {

    // Instancia del repositorio para gestionar datos en Firebase
    private final TablonRepository repositorio;

    // LiveData para manejar la lista de anuncios y errores
    private final MutableLiveData<List<Anuncio>> anunciosLive = new MutableLiveData<>();
    private final MutableLiveData<String> anunciosErrorLive = new MutableLiveData<>();

    // Constructor: Inicializa el repositorio
    public TablonViewModel() {
        repositorio = new TablonRepository();
    }

    /**
     * Obtiene el usuario actualmente autenticado en Firebase.
     * Se usa para verificar si un usuario está logueado y quién es.
     */
    public FirebaseUser getUsuarioActual() {
        return repositorio.getUsuarioActual();
    }

    // ========== MÉTODOS PARA EXPONER LiveData A LA VISTA ==========

    // Permite que la UI observe la lista de anuncios
    public LiveData<List<Anuncio>> getAnunciosLive() { return anunciosLive; }

    // Permite que la UI observe errores al obtener anuncios
    public LiveData<String> getAnunciosErrorLive() { return anunciosErrorLive; }

    // ========== MÉTODOS DE MANEJO DE ANUNCIOS ==========

    /**
     * Publica un nuevo anuncio en Firebase.
     * Si la operación es exitosa, recarga la lista de anuncios.
     */
    public void publicarAnuncio(String contenido) {
        repositorio.publicarAnuncio(contenido, new TablonRepository.AuthCallback() {
            @Override
            public void onSuccess() {
                leerAnuncios(); // Recargar la lista de anuncios al publicar uno nuevo
            }

            @Override
            public void onFailure(String error) {
                anunciosErrorLive.setValue(error); // Notifica el error a la UI
            }
        });
    }

    /**
     * Lee los anuncios desde Firebase y los almacena en LiveData.
     * Permite que la UI se actualice automáticamente cuando haya cambios en la base de datos.
     */
    public void leerAnuncios() {
        repositorio.leerAnuncios(new TablonRepository.AnunciosCallback() {
            @Override
            public void onSuccess(List<Anuncio> lista) {
                anunciosLive.setValue(lista); // Actualiza la lista de anuncios en la UI
            }

            @Override
            public void onFailure(String error) {
                anunciosErrorLive.setValue(error); // Guarda el error en caso de fallo
            }
        });
    }

    /**
     * Elimina un anuncio de Firebase si el usuario es el creador.
     * Luego de la eliminación, recarga la lista de anuncios.
     */
    public void eliminarAnuncio(String id) {
        repositorio.eliminarAnuncio(id, new TablonRepository.AuthCallback() {
            @Override
            public void onSuccess() {
                leerAnuncios(); // Recargar la lista de anuncios después de eliminar
            }

            @Override
            public void onFailure(String error) {
                anunciosErrorLive.setValue(error); // Notifica el error a la UI
            }
        });
    }
}


Notas:

Este ViewModel no interactúa directamente con Firebase, sino que delega las operaciones a TablonRepository, que sí maneja la comunicación con Firebase Authentication y Firebase Realtime. Database.

Interfaz de Usuario​

1. Fragmento Tablón​

TablonFragment representa la interfaz donde se muestran y gestionan los anuncios en la aplicación.

  • Muestra un RecyclerView con los anuncios publicados.
  • Permite a los usuarios publicar nuevos anuncios.
  • Observa cambios en la base de datos de Firebase en tiempo real.
  • Implementa swipe para eliminar anuncios, pero solo si el usuario es el autor.
com/ejemplo/tablon/ui/TablonFragment.java
package com.ejemplo.tablon.register;

import androidx.appcompat.app.AppCompatActivity;
import androidx.lifecycle.ViewModelProvider;

import android.os.Bundle;
import android.view.View;
import android.widget.Toast;

import com.ejemplo.tablon.databinding.ActivityRegistroBinding;
import com.ejemplo.tablon.ui.TablonViewModel;

public class TablonFragment extends Fragment {

    // ViewBinding para acceder a los elementos del layout sin usar findViewById()
    private FragmentTablonBinding binding;

    // ViewModel para interactuar con los datos de los anuncios
    private TablonViewModel tablonVM;

    // Adaptador para gestionar la lista de anuncios en el RecyclerView
    private AnunciosAdapter adapter;

    @Nullable
    @Override
    public View onCreateView(@NonNull LayoutInflater inflater,
                             @Nullable ViewGroup container,
                             @Nullable Bundle savedInstanceState) {
        // Infla el layout usando ViewBinding
        binding = FragmentTablonBinding.inflate(inflater, container, false);
        return binding.getRoot(); // Retorna la vista raíz del fragmento
    }

    @Override
    public void onViewCreated(@NonNull View view, @Nullable Bundle savedInstanceState) {
        super.onViewCreated(view, savedInstanceState);

        // Obtenemos una instancia del ViewModel compartido para gestionar los anuncios
        tablonVM = new ViewModelProvider(requireActivity()).get(TablonViewModel.class);

        // Inicializamos el adaptador con una lista vacía para el RecyclerView
        adapter = new AnunciosAdapter(new ArrayList<>());

        // Configuración del RecyclerView
        binding.recyclerAnuncios.setLayoutManager(new LinearLayoutManager(requireContext())); // Layout en forma de lista vertical
        binding.recyclerAnuncios.setAdapter(adapter); // Asigna el adaptador al RecyclerView

        // Observamos cambios en la lista de anuncios desde el ViewModel
        tablonVM.getAnunciosLive().observe(getViewLifecycleOwner(), anuncios -> {
            adapter.setLista(anuncios); // Cuando hay cambios, se actualiza el RecyclerView
        });

        // Observamos errores en la carga de anuncios
        tablonVM.getAnunciosErrorLive().observe(getViewLifecycleOwner(), error -> {
            if (error != null) {
                Toast.makeText(getContext(), "Error: " + error, Toast.LENGTH_SHORT).show(); // Muestra un mensaje de error
            }
        });

        // Botón para publicar un nuevo anuncio
        binding.btnPublicar.setOnClickListener(v -> {
            // Obtiene el contenido del EditText
            String contenido = binding.etContenido.getText().toString().trim();
            if (!contenido.isEmpty()) {
                // Publica el anuncio usando el ViewModel
                tablonVM.publicarAnuncio(contenido);
                binding.etContenido.setText(""); // Limpia el campo después de publicar
            } else {
                Toast.makeText(getContext(), "Escribe algo antes de publicar", Toast.LENGTH_SHORT).show();
            }
        });

        // ===================== GESTO DE SWIPE PARA ELIMINAR ANUNCIOS =====================

        // Agregar funcionalidad de swipe (deslizamiento) para eliminar anuncios
        new ItemTouchHelper(new ItemTouchHelper.SimpleCallback(0, ItemTouchHelper.LEFT) {
            @Override
            public boolean onMove(@NonNull RecyclerView recyclerView,
                                  @NonNull RecyclerView.ViewHolder viewHolder,
                                  @NonNull RecyclerView.ViewHolder target) {
                return false; // No permitimos arrastrar elementos, solo swipe
            }

            @Override
            public void onSwiped(@NonNull RecyclerView.ViewHolder viewHolder, int direction) {
                // Obtener la posición del anuncio que se deslizó
                int posicion = viewHolder.getAdapterPosition();
                Anuncio anuncio = adapter.getAnuncioEnPosicion(posicion);

                // Verificamos si el usuario autenticado es el autor del anuncio
                if (tablonVM.getUsuarioActual() != null &&
                        anuncio.getUidAutor().equals(tablonVM.getUsuarioActual().getEmail())) {

                    // Mostrar un cuadro de diálogo para confirmar la eliminación
                    new AlertDialog.Builder(requireContext())
                            .setTitle("Eliminar anuncio")
                            .setMessage("¿Estás seguro de que deseas eliminar este anuncio?")
                            .setPositiveButton("Sí", (dialog, which) -> {
                                tablonVM.eliminarAnuncio(anuncio.getId()); // Eliminar anuncio de Firebase
                            })
                            .setNegativeButton("Cancelar", (dialog, which) -> {
                                adapter.notifyItemChanged(posicion); // Restaurar el elemento si se cancela la eliminación
                            })
                            .show();
                } else {
                    // Si el usuario no es el autor, no se permite eliminar el anuncio
                    Toast.makeText(requireContext(), "No puedes eliminar este anuncio", Toast.LENGTH_SHORT).show();
                    adapter.notifyItemChanged(posicion); // Restaurar el elemento
                }
            }
        }).attachToRecyclerView(binding.recyclerAnuncios); // Vincular la funcionalidad de swipe al RecyclerView

        // ===================== CARGAR ANUNCIOS DESDE FIREBASE =====================
        tablonVM.leerAnuncios(); // Se llama al iniciar para obtener los anuncios desde Firebase
    }
}

2. AnunciosAdapter​

AnunciosAdapter es un adaptador para RecyclerView que se encarga de mostrar una lista de anuncios en la interfaz de usuario.

  • Muestra una lista de anuncios en tarjetas (card views) dentro de un RecyclerView.
  • Permite actualizar la lista de anuncios en tiempo real.
  • Convierte la fecha de cada anuncio a un formato legible.
  • Permite obtener un anuncio en una posición específica (útil para eliminación con swipe).
java/com/ejemplo/tablon/ui/AnunciosAdapter.java
package com.ejemplo.tablon.ui;

import android.view.*;
import androidx.annotation.NonNull;
import androidx.recyclerview.widget.RecyclerView;

import com.ejemplo.tablon.databinding.ItemAnuncioBinding;
import com.ejemplo.tablon.model.Anuncio;

import java.text.SimpleDateFormat;
import java.util.Date;
import java.util.List;
import java.util.Locale;

public class AnunciosAdapter extends RecyclerView.Adapter<AnunciosAdapter.AnuncioViewHolder> {

    // Lista de anuncios que se mostrarán en el RecyclerView
    private List<Anuncio> lista;

    // Constructor del adaptador: recibe una lista de anuncios
    public AnunciosAdapter(List<Anuncio> lista) {
        this.lista = lista;
    }

    /**
     * Método para actualizar la lista de anuncios en el RecyclerView.
     * Se usa cuando hay cambios en los datos (Firebase envía actualizaciones).
     */
    public void setLista(List<Anuncio> nuevaLista) {
        this.lista = nuevaLista;
        notifyDataSetChanged(); // Notifica al RecyclerView que los datos han cambiado
    }

    /**
     * Infla el layout de cada elemento del RecyclerView.
     */
    @NonNull
    @Override
    public AnuncioViewHolder onCreateViewHolder(@NonNull ViewGroup parent, int viewType) {
        // Usamos ViewBinding para inflar el layout de cada anuncio
        ItemAnuncioBinding binding = ItemAnuncioBinding.inflate(LayoutInflater.from(parent.getContext()), parent, false);
        return new AnuncioViewHolder(binding);
    }

    /**
     * Asigna los datos de un anuncio a la vista en una posición específica.
     */
    @Override
    public void onBindViewHolder(@NonNull AnuncioViewHolder holder, int position) {
        // Obtenemos el anuncio en la posición actual
        Anuncio an = lista.get(position);

        // Mostrar el contenido del anuncio
        holder.binding.textoContenido.setText(an.getContenido());

        // Mostrar el usuario que publicó el anuncio (podría ser UID o email)
        holder.binding.textoUsuario.setText("Usuario: " + an.getUidAutor());

        // Formatear y mostrar la fecha del anuncio
        String fechaFormateada = formatearFecha(an.getFecha());
        holder.binding.textoFecha.setText(fechaFormateada);
    }

    /**
     * Devuelve el número total de anuncios en la lista.
     */
    @Override
    public int getItemCount() {
        return lista.size();
    }

    /**
     * ViewHolder: Clase interna que representa la vista de cada elemento de la lista.
     */
    static class AnuncioViewHolder extends RecyclerView.ViewHolder {
        private final ItemAnuncioBinding binding;

        // Constructor: Asigna la vista inflada al ViewHolder
        public AnuncioViewHolder(@NonNull ItemAnuncioBinding binding) {
            super(binding.getRoot());
            this.binding = binding;
        }
    }

    /**
     * Convierte el timestamp de un anuncio en un formato de fecha legible.
     */
    private String formatearFecha(long timestamp) {
        // Convierte el timestamp en un objeto Date
        Date date = new Date(timestamp);

        // Define el formato de fecha deseado
        SimpleDateFormat sdf = new SimpleDateFormat("dd/MM/yyyy HH:mm", Locale.getDefault());

        // Retorna la fecha en formato legible
        return "Fecha: " + sdf.format(date);
    }

    /**
     * Obtiene un anuncio en una posición específica del RecyclerView.
     * Se usa para eliminar anuncios con swipe.
     */
    public Anuncio getAnuncioEnPosicion(int posicion) {
        return lista.get(posicion);
    }
}

Reglas de Seguridad en Realtime Database​

En la consola de Firebase → Realtime Database → Reglas, podrías poner:

{
  "rules": {
    "anuncios": {
      ".read": "auth != null",  
      ".write": "auth != null"
    }
  }
}

De esta forma, solo usuarios autenticados pueden leer/escribir en anuncios. Si alguien no está autenticado, la base de datos denegará la operación.
