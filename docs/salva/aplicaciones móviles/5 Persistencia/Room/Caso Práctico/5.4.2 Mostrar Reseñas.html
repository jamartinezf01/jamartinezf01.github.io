Mostrar Lista de Reseñas

En esta primera versión del proyecto, vamos a implementar el RecyclerView para mostrar la lista de reseñas. vamos a diseñar y configurar la base de datos local de nuestra aplicación utilizando Room. Esto nos permitirá guardar y recuperar las películoas o series que el usuario agregue.

¿Qué vamos a hacer? 🤔

  1. Crear la clase de Entidad para representar las recetas.
  2. Definir el DAO (Data Access Object) para interactuar con la base de datos.
  3. Configurar la clase de Base de Datos.
  4. Inplementar el repositorio que hará de intermediario con el ViewModel
  5. Implementar una clase ViewModel para exponer los datos del repositorio en la Vista
  6. Crearemos una vista mediante un fragmento que nos muestre un ReciclerView con todas las reseñas
  7. Implementaremos la vista principal en la MainActivity mediante un NavHostFragment que nos permitirá insertar posteriormente diferentes fragmentos.

Creación del modelo​

1. Crear la entidad Pelicula​

El modelo comienza con la definición de la Entidad, que describe la estructura de la tabla en la base de datos Room. En este caso, cada crítica de una película o serie será representada como un registro en la tabla tabla_peliculas.

  1. Crear la clase Pelicula:

    • Ve a la carpeta app/java/com.tu.paquete.
    • Crea un paquete llamado Model (esto ayuda a mantener el proyecto organizado).
    • Dentro de este paquete, crea una nueva clase llamada Pelicula.

  2. Definir la clase Pelicula:

Model/Pelicula.java
package com.agora.criticascine.Model; // sustituye esta línea por tu package

import androidx.room.Entity;
import androidx.room.PrimaryKey;

// Esta clase representa una película o serie en la base de datos.
@Entity(tableName = "tabla_peliculas") // Define el nombre de la tabla en Room.
public class Pelicula {

    @PrimaryKey(autoGenerate = true) // Clave primaria que se genera automáticamente.
    private int id;

    private String nombre; // Nombre de la película o serie.
    private String descripcion; // Breve descripción o comentario.
    private float valoracion; // Valoración numérica, por ejemplo, de 0 a 5.

    // Constructor de la clase.
    public Pelicula(String nombre, String descripcion, float valoracion) {
        this.nombre = nombre;
        this.descripcion = descripcion;
        this.valoracion = valoracion;
    }

    // Métodos para obtener y modificar el ID.
    public int getId() {
        return id;
    }

    public void setId(int id) {
        this.id = id;
    }

    // Métodos para obtener y modificar el nombre.
    public String getNombre() {
        return nombre;
    }

    public void setNombre(String nombre) {
        this.nombre = nombre;
    }

    // Métodos para obtener y modificar la descripción.
    public String getDescripcion() {
        return descripcion;
    }

    public void setDescripcion(String descripcion) {
        this.descripcion = descripcion;
    }

    // Métodos para obtener y modificar la valoración.
    public float getValoracion() {
        return valoracion;
    }

    public void setValoracion(float valoracion) {
        this.valoracion = valoracion;
    }
}
Explicación
  • Anotación @Entity:
    • Declara que esta clase corresponde a una tabla de la base de datos Room.
    • El atributo tableName define el nombre de la tabla: tabla_peliculas.
  • Atributos:
    • id: Es la clave primaria, se genera automáticamente con autoGenerate = true.
    • nombre: Almacena el nombre de la película o serie.
    • descripcion: Almacena una breve sinopsis o comentario.
    • valoracion: Guarda una valoración numérica (puede ser un número flotante, como 4.5).

2. Crear el DAO PeliculaDAO​

El DAO es una interfaz que define las operaciones que se pueden realizar en la base de datos, como insertar, actualizar, eliminar y consultar datos. Room genera automáticamente el código necesario para estas operaciones a partir de las anotaciones en el DAO.

  1. Crear la interfaz PeliculaDAO:

    • En el mismo paquete Model, crea una nueva interfaz llamada PeliculaDAO.

  2. Definir las operaciones del DAO:

    • Añade los métodos para interactuar con la base de datos:
Model/PeliculaDAO.java
package com.agora.criticascine.Model;

import androidx.lifecycle.LiveData;
import androidx.room.Dao;
import androidx.room.Delete;
import androidx.room.Insert;
import androidx.room.Query;
import androidx.room.Update;

import java.util.List;

// Esta interfaz define las operaciones para interactuar con la tabla "tabla_peliculas".
@Dao
public interface PeliculaDAO {

    // Inserta una nueva película en la tabla.
    @Insert
    void insertar(Pelicula pelicula);

    // Actualiza una película existente en la tabla.
    @Update
    void actualizar(Pelicula pelicula);

    // Elimina una película de la tabla.
    @Delete
    void eliminar(Pelicula pelicula);

    // Consulta todas las películas de la tabla, ordenadas por nombre.
    @Query("SELECT * FROM tabla_peliculas ORDER BY nombre ASC")
    LiveData<List<Pelicula>> obtenerTodasLasPeliculas();

    // Consulta una película específica por su nombre.
    @Query("SELECT * FROM tabla_peliculas WHERE nombre = :nombre")
    Pelicula obtenerPeliculaPorNombre(int nombre);

    // Elimina todas las películas de la tabla.
    @Query("DELETE FROM tabla_peliculas")
    void eliminarTodasLasPeliculas();
}
Explicación
  • @Dao: define la interfaz que interactúa con la base de datos.
  • @Insert: Añade una nueva película a la tabla.Room se encarga de generar el código necesario para esta operación.
  • @Update: Actualiza los datos de una película existente.
  • @Delete: Elimina una película específica de la tabla.
  • @Query:
    • obtenerTodasLasPeliculas: Consulta todas las películas almacenadas y las devuelve como una lista observable (LiveData<List<Pelicula>>). La lista se ordena alfabéticamente por el nombre.
    • obtenerPeliculaPorNombre: Consulta una película específica basándose en su atributo nombre. :nombre es un parámetro vinculado (bind parameter), de forma que Room reemplaza automáticamente :nombre con el valor pasado al método.
    • eliminarTodasLasPeliculas: Elimina todos los registros de la tabla.
info

Se han incluído, a modo de ejemplo, más métodos de los que nuestra aplicación necesitará en realidad. En tus aplicaciones solo debes implementar los métodos CRUD realmente necesarios.

peligro

En las consultas debes especificar el nombre de la tabla y las columnas, no el nombre de la clase y los atributos. Si utilizaste tableName para modificar el nombre de la tabla, como es nuestro caso, deberás utilizar "tabla_peliculas" en la consulta y no "Pelicula" (nombre de la clase).

3. Configurar la Base de Datos Room PeliculasDatabase​

La Base de Datos Room es el componente central de Room que gestiona el acceso a las tablas y DAOs. Aquí configuraremos la clase que representará nuestra base de datos para interactuar con la tabla tabla_peliculas.

  1. Crear la clase PeliculasDatabase:

    • En el mismo paquete Model, crea una nueva clase llamada PeliculasDatabase.

  2. Configurar la base de datos:

    • Añade las anotaciones y métodos necesarios para que Room la reconozca.
Model/PeliculasDatabase.java
package com.agora.criticascine.Model;

import androidx.room.Database;
import androidx.room.Room;
import androidx.room.RoomDatabase;
import android.content.Context;

import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

// Clase que representa la base de datos Room para la aplicación.
@Database(entities = {Pelicula.class}, version = 1, exportSchema = false)
public abstract class PeliculasDatabase extends RoomDatabase {

    // Define el DAO que interactuará con la base de datos.
    public abstract PeliculaDAO peliculaDAO();

    // Instancia única de la base de datos para evitar múltiples inicializaciones.
    private static volatile PeliculasDatabase instancia;

    // Método para obtener la instancia de la base de datos.
    public static PeliculasDatabase obtenerInstancia(final Context context) {
        if (instancia == null) { // Verifica si ya existe una instancia.
            synchronized (PeliculasDatabase.class) {
                if (instancia == null) { // Doble verificación para mayor seguridad.
                    instancia = Room.databaseBuilder(
                                    context.getApplicationContext(),
                                    PeliculasDatabase.class,
                                    "base_datos_peliculas"
                            )
                            .fallbackToDestructiveMigration() // Permite recrear la base en caso de cambios en el esquema.
                            .build();
                }
            }
        }
        return instancia;
    }
}
Explicación
  • Anotación @Database:
    • entities: Lista las entidades que estarán en la base de datos, en este caso solo Pelicula.
    • version: Versión de la base de datos, importante para migraciones.
    • exportSchema: Define si Room debe exportar el esquema (opcional, aquí desactivado).
  • Método abstracto peliculaDAO():
    • Room generará automáticamente el código para implementar este método, proporcionando el DAO PeliculaDAO.
  • Room.databaseBuilder crea la base de datos:. Sus parámetros son
    • Contexto de aplicación.
    • La clase que representa la base de datos (PeliculasDatabase.class).
    • El nombre del archivo SQLite (base_datos_peliculas.db).
    • fallbackToDestructiveMigration(): Elimina y recrea la base de datos si se detectan cambios en el esquema que no tienen una migración definida. Ideal para desarrollo, pero en producción se recomienda definir migraciones.
Patrón Singleton
  • Por qué usar un Singleton:
    • Garantiza que solo exista una única instancia de la base de datos en toda la aplicación.
    • Evita problemas de acceso simultáneo desde múltiples hilos.

4. Implementar el Repositorio PeliculasRepositorio​

El Repositorio es una capa que actúa como intermediaria entre el ViewModel y la base de datos. Su función principal es abstraer el origen de los datos y centralizar la lógica para acceder a ellos, lo que facilita cambios futuros, como añadir una fuente de datos remota.

  1. Crear la clase PeliculasRepositorio:

    • En el paquete modelo, crea una nueva clase llamada PeliculasRepositorio.

  2. Implementar la lógica del Repositorio:

    • Define métodos que interactúen con el DAO y expongan los datos al ViewModel.
Model/PeliculasRepositorio.java
package com.agora.criticascine.Model;

import android.app.Application;

import androidx.lifecycle.LiveData;

import java.util.List;
import java.util.concurrent.Executor;
import java.util.concurrent.Executors;

public class PeliculasRepositorio {

    // DAO para interactuar con la base de datos.
    private final PeliculaDAO peliculaDAO;

    // Define un Executor para manejar tareas en segundo plano.
    private final Executor databaseExecutor;

    public PeliculasRepositorio(Application aplicacion) {
        // Obtiene la instancia de la base de datos y el DAO.
        PeliculasDatabase baseDeDatos = PeliculasDatabase.obtenerInstancia(aplicacion);
        peliculaDAO = baseDeDatos.peliculaDAO();
        // Inicializamos el Executor.
        databaseExecutor = Executors.newSingleThreadExecutor();
    }

    // Método para insertar una película en la base de datos.
    public void insertar(Pelicula pelicula) {
        databaseExecutor.execute(() -> peliculaDAO.insertar(pelicula));
    }

    // Método para actualizar una película en la base de datos.
    public void actualizar(Pelicula pelicula) {
        databaseExecutor.execute(() -> peliculaDAO.actualizar(pelicula));
    }

    // Método para eliminar una película de la base de datos.
    public void eliminar(Pelicula pelicula) {
        databaseExecutor.execute(() -> peliculaDAO.eliminar(pelicula));
    }

    // Método para eliminar todas las películas.
    public void eliminarTodasLasPeliculas() {
        databaseExecutor.execute(peliculaDAO::eliminarTodasLasPeliculas);
    }

    // Método para obtener todas las películas como LiveData.
    public LiveData<List<Pelicula>> obtenerTodasLasPeliculas() {
        return peliculaDAO.obtenerTodasLasPeliculas();
    }
}
Explicación:
  • Referencias clave:
    • peliculaDAO: Se usa para acceder a los métodos del DAO.
    • todasLasPeliculas: Representa una lista observable de todas las películas.
  • Acceso a la base de datos en segundo plano:
    • PeliculasDatabase.databaseWriteExecutor se utiliza para ejecutar operaciones en un hilo de fondo, evitando bloquear el hilo principal.
  • Métodos públicos:
    • insertar, actualizar, eliminar, eliminarTodasLasPeliculas: Llaman a los métodos del DAO en un hilo de fondo.
    • obtenerTodasLasPeliculas: Proporciona un LiveData observable que actualiza automáticamente la vista cuando los datos cambian.

2. Crear el Adapter del ReciclerView PeliculasAdapter​

El Adaptador es responsable de vincular los datos de la lista con las vistas definidas en el Layout XML. Configuraremos el adaptador para que utilice el ViewHolder y maneje la interacción con los datos.

En el paquete UI, crea una clase llamada PeliculasAdapter.

UI/PeliculasAdapter.java
package com.agora.criticascine.UI;

import android.view.LayoutInflater;
import android.view.ViewGroup;

import androidx.annotation.NonNull;
import androidx.recyclerview.widget.RecyclerView;

import com.agora.criticascine.Model.Pelicula;
import com.agora.criticascine.databinding.ItemPeliculaBinding;

import java.util.ArrayList;
import java.util.List;


// Adaptador para gestionar la lista de películas en el RecyclerView.
public class PeliculasAdapter extends RecyclerView.Adapter<PeliculasAdapter.PeliculasViewHolder> {

    private List<Pelicula> listaPeliculas = new ArrayList<>(); // Lista de películas.

    @NonNull
    @Override
    public PeliculasViewHolder onCreateViewHolder(@NonNull ViewGroup parent, int viewType) {
        // Inflamos el layout de la tarjeta (item_pelicula.xml).
        ItemPeliculaBinding binding = ItemPeliculaBinding.inflate(LayoutInflater.from(parent.getContext()), parent, false);
        return new PeliculasViewHolder(binding);
    }

    @Override
    public void onBindViewHolder(@NonNull PeliculasViewHolder holder, int position) {
        // Obtenemos la película en la posición actual.
        Pelicula peliculaActual = listaPeliculas.get(position);

        // Configuramos los datos en las vistas.
        holder.binding.textoNombre.setText(peliculaActual.getNombre());
        holder.binding.textoDescripcion.setText(peliculaActual.getDescripcion());
        holder.binding.barraValoracion.setRating(peliculaActual.getValoracion());
    }

    @Override
    public int getItemCount() {
        // Devuelve el número de elementos en la lista.
        return listaPeliculas.size();
    }

    // Método para actualizar la lista de películas.
    public void establecerPeliculas(List<Pelicula> peliculas) {
        this.listaPeliculas = peliculas;
        notifyDataSetChanged(); // Notifica al adaptador que los datos han cambiado.
    }

    // Clase interna ViewHolder para vincular las vistas de cada tarjeta.
    static class PeliculasViewHolder extends RecyclerView.ViewHolder {

        private final ItemPeliculaBinding binding;

        public PeliculasViewHolder(@NonNull ItemPeliculaBinding binding) {
            super(binding.getRoot());
            this.binding = binding;
        }
    }
}

3. Crear el fragmento FragmentListaPeliculas y su Layout​

Crearemos el fragmento principal que inicializará el RecyclerView, conectará el adaptador con el ViewModel, y observará los datos de la lista de películas para actualizarlos automáticamente en la interfaz.

En el paquete UI, crea un nuevo Fragment llamado FragmentListaPeliculas. Además del archivo .java se generará automáticamente en la carpeta res/layout un archivo llamado fragment_lista_peliculas.xml que es el Layout del Fragment .

UI/PeliculasAdapter.java
package com.agora.criticascine.UI;

import android.os.Bundle;
import android.view.LayoutInflater;
import android.view.View;
import android.view.ViewGroup;

import androidx.annotation.NonNull;
import androidx.annotation.Nullable;
import androidx.fragment.app.Fragment;
import androidx.lifecycle.ViewModelProvider;
import androidx.recyclerview.widget.LinearLayoutManager;
import androidx.recyclerview.widget.RecyclerView;

import com.agora.criticascine.R;

public class FragmentListaPeliculas extends Fragment {

    private PeliculasViewModel peliculasViewModel; // Referencia al ViewModel.
    private PeliculasAdapter adaptador; // Adaptador para el RecyclerView.
    private FragmentListaPeliculasBinding binding; // Objeto para ViewBinding

    @Nullable
    @Override
    public View onCreateView(@NonNull LayoutInflater inflater, @Nullable ViewGroup container, @Nullable Bundle savedInstanceState) {
        // Inflamos el diseño del fragmento.
        binding = FragmentListaPeliculasBinding.inflate(getLayoutInflater());
        return binding.getRoot();
    }

    @Override
    public void onViewCreated(@NonNull View view, @Nullable Bundle savedInstanceState) {
        super.onViewCreated(view, savedInstanceState);

        // Inicializamos el RecyclerView.
        RecyclerView recyclerView = view.findViewById(R.id.recycler_peliculas);
        recyclerView.setLayoutManager(new LinearLayoutManager(getContext())); // Diseño en lista vertical.

        // Obtenemos una instancia del ViewModel.
        peliculasViewModel = new ViewModelProvider(this).get(PeliculasViewModel.class);

        // Configuramos el adaptador del RecyclerView.
        adaptador = new PeliculasAdapter();
        recyclerView.setAdapter(adaptador);

        // Observamos los datos de la lista de películas.
        peliculasViewModel.obtenerTodasLasPeliculas().observe(getViewLifecycleOwner(), peliculas -> {
            // Actualizamos los datos del adaptador cuando cambien en el ViewModel.
            adaptador.establecerPeliculas(peliculas);
        });
    }
}

Modificación de la Actividad Principal​

Ya hemos comentado que esta aplicación será una aplicación de tipo Single Activity por lo que tendremos que adaptar la actividad principal para que, mediante el uso de un NavHostFragment, sea capaz de motrar diferentes fragmentos según nuestras necesidades. Para ello deberemos crear un grafo de navegación y modificar el Layout por defecto del MainActivity.

1. Crear un grafo de navegación​

El grafo de navegación es necesario para implementar un NavhostFragment. En este caso el grafo será un grafo simple y solo tendrá un destino que es el fragmento de la lista de elículas.

Añade un nuevo Recurso Android de tipo Navigation llamado nav_graph.xml en la carpeta res/navigation

res/navigation/nav_graph.xml
<?xml version="1.0" encoding="utf-8"?>
<navigation xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    xmlns:tools="http://schemas.android.com/tools"
    android:id="@+id/nav_graph"
    app:startDestination="@id/fragmentListaPeliculas">

    <fragment
        android:id="@+id/fragmentListaPeliculas"
        android:name="com.agora.criticascine.UI.FragmentListaPeliculas"
        android:label="Lista de Películas"
        tools:layout="@layout/fragment_lista_peliculas" />
</navigation>

2. Modifica el Layout del MainActivity​

Modifica el archivo activity_main.xml en la carpeta res/layout.

res/layout/activity_main.xml
<?xml version="1.0" encoding="utf-8"?>
<androidx.constraintlayout.widget.ConstraintLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    xmlns:tools="http://schemas.android.com/tools"
    android:id="@+id/main"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    tools:context=".MainActivity">

    <androidx.fragment.app.FragmentContainerView
        xmlns:android="http://schemas.android.com/apk/res/android"
        android:id="@+id/nav_host_fragment"
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        android:name="androidx.navigation.fragment.NavHostFragment"
        app:defaultNavHost="true"
        app:navGraph="@navigation/nav_graph" />

</androidx.constraintlayout.widget.ConstraintLayout>
