5.3.1 Entidad (@Entity)

En Android Room, una Entidad es un componente fundamental que define una tabla en la base de datos. 📊 Cada instancia de la clase entidad corresponde a una fila de dicha tabla. Room utiliza anotaciones 📝 para mapear la estructura de la clase a la estructura de la tabla en SQLite.

¿Qué es una Entidad? 🤔​

Una entidad es una clase de Java o Kotlin anotada con @Entity. Esta clase define:

  • El nombre de la tabla 🏷️.
  • Las columnas de la tabla 📋, que corresponden a los campos de la clase.
  • Las claves primarias y foráneas 🔑, esenciales para mantener la integridad de los datos.

Cómo crear una Entidad 🛠️​

1. Anotar la clase con @Entity​

La anotación @Entity marca una clase como una tabla en la base de datos. Si no se especifica el nombre de la tabla, Room usará el nombre de la clase por defecto.

@Entity(tableName = "users")
public class User {
    ...
}

2. Definir una clave primaria​

Cada tabla debe tener una clave primaria única, que se define con la anotación @PrimaryKey. Room no permite tablas sin claves primarias.

@PrimaryKey
public int id;

3. Definir columnas con @ColumnInfo (opcional)​

Puedes personalizar el nombre de las columnas con la anotación @ColumnInfo. Si no la usas, Room tomará el nombre de los campos de la clase.

@ColumnInfo(name = "first_name")
public String firstName;

4. Ignorar campos no necesarios​

Si un campo de la clase no debe ser incluido en la tabla, usa la anotación @Ignore.

@Ignore
public String tempData; // Este campo no será parte de la tabla.

Ejemplo completo de una entidad 📘​

Aquí se define una tabla users con columnas para un ID único, nombre y apellido. Además, se incluye un campo temporal que no forma parte de la tabla.

import androidx.room.ColumnInfo;
import androidx.room.Entity;
import androidx.room.Ignore;
import androidx.room.PrimaryKey;

@Entity(tableName = "users")
public class User {

    @PrimaryKey(autoGenerate = true) // La clave primaria se genera automáticamente.
    public int id;

    @ColumnInfo(name = "first_name") // El nombre de la columna será "first_name".
    public String firstName;

    @ColumnInfo(name = "last_name") // El nombre de la columna será "last_name".
    public String lastName;

    @Ignore // Este campo no será incluido en la tabla.
    public String tempData;

    // Constructor principal
    public User(String firstName, String lastName) {
        this.firstName = firstName;
        this.lastName = lastName;
    }
}

Anotaciones de Entidad 📝​

1. @Entity​

  • Define una clase como entidad.
  • Parámetros comunes:
    • tableName: Nombre personalizado de la tabla.
    • indices: Define índices para mejorar las consultas.
    • primaryKeys: Define claves primarias compuestas.

2. @PrimaryKey​

  • Declara un campo como clave primaria.
  • Parámetros comunes:
    • autoGenerate: Si es true, Room genera automáticamente valores únicos.

3. @ColumnInfo​

  • Define el nombre de la columna y sus propiedades.
  • Parámetros comunes:
    • name: Nombre personalizado de la columna.
    • typeAffinity: Especifica el tipo de dato SQLite (por ejemplo, TEXT, INTEGER).

4. @Ignore​

  • Excluye un campo de la tabla.

Índices en las Entidades 🔍​

Los índices se utilizan para optimizar consultas. Puedes definirlos usando el parámetro indices en @Entity.

@Entity(
    tableName = "users",
    indices = {@Index(value = "first_name", unique = true)}
)
public class User {
    @PrimaryKey(autoGenerate = true)
    public int id;

    @ColumnInfo(name = "first_name")
    public String firstName;

    @ColumnInfo(name = "last_name")
    public String lastName;
}
nota
  • Se crea un índice para la columna first_name.
  • El índice es único, lo que significa que no puede haber dos filas con el mismo first_name.

Claves primarias compuestas 🔗​

Si necesitas que tu tabla tenga una clave primaria compuesta (varios campos como clave primaria), puedes usar el parámetro primaryKeys en @Entity.

@Entity(
    tableName = "user_preferences",
    primaryKeys = {"userId", "preferenceKey"}
)
public class UserPreference {
    public int userId;
    public String preferenceKey;
    public String preferenceValue;
}

Relaciones entre Entidades 🤝​

En Room, puedes establecer relaciones entre tablas mediante claves foráneas. Esto se hace utilizando el parámetro foreignKeys en @Entity.

@Entity(
    tableName = "orders",
    foreignKeys = @ForeignKey(
        entity = User.class,
        parentColumns = "id",
        childColumns = "userId",
        onDelete = ForeignKey.CASCADE
    )
)
public class Order {
    @PrimaryKey(autoGenerate = true)
    public int orderId;

    public int userId; // Clave foránea que se relaciona con la tabla "users".
}

Restricciones de las Entidades 🚫​

restricciones
  1. Campos obligatorios:
    • Todos los campos no anulables deben tener un valor.
  2. Tipos compatibles:
    • Room admite tipos de datos básicos (int, String, boolean, etc.) y algunas clases específicas como Date.
    • Para otros tipos, necesitas un TypeConverter.
  3. Sin herencia:
    • Las clases de entidad no pueden heredar de otras clases.
