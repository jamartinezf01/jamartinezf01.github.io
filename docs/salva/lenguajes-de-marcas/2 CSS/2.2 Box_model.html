2.2 Box Model

El box model (Modelo de caja) es un conjunto de reglas que define cómo se calcula el tamaño y el espacio que ocupa cada elemento en la página. CSS trata cada elemento de su documento HTML como una “caja" con ciertas propiedades como: contenido, padding, border y margin...

Dominarlo evita “saltos” de diseño, ensanchamientos inesperados y problemas de alineación.

Tipos de Cajas​

El "modelo de caja CSS" cada elemento HTML representado en la pantalla es un cuadro y puede ser de dos tipos: elemento de "bloque" y elemento “de línea"

ejemplo
  • CSS
  • HTML
  • Representación
body{
    font-size: 18px;
}

h1, p {
    background-color: rgb(210, 210, 210);
}

h1{
    padding-bottom: 70px;
}

em, strong{
    background-color: aqua;
}
<html lang='es'>
  <head>

    <title>Modelo de Cajas</title>
    <link rel="stylesheet" href="modelo-caja.css">
  </head>
  <body>

    <h2>Cabecera mas pequeña</h2>
    <p >Los párrafos también son bloques. <em>Sin embargo</em>, los elementos &lt;em&gt; y &lt;strong&gt;
       no lo son, son elementos <strong>de línea</strong>.</p>
    <p>Los elementos de bloque definen el flujo del docuemnto HTML, mientras que los elementos de línea no. <span>esto es un Span</span></p>
  </body>
</html>
cajas genéricas

Los elementos <div> y <span>. Son contenedores que no tienen ningún efecto en la estructura semántica de un documento HTML pero nos permiten agregar estilos a secciones arbitrarias de una web.

  • <div> se comporta como una caja tipo bloque.
  • <span>se comporta como una como una caja tipo línea.

Elementos tipo bloque​

Los elementos de bloque siempre aparecen debajo del elemento de bloque anterior. Este es el flujo "natural" o "estático" de un documento HTML cuando un navegador web lo representa.

El ancho de los elementos de bloque se establece automáticamente en función del ancho de su contenedor principal. En el ejemplo anterior, nuestros bloques son siempre del ancho de la ventana del navegador.

La altura predeterminada de los elementos de bloque se basa en su contenido. Por ejemplo, cuando se estrecha la ventana del navegador, <h1> se divide en dos líneas y su altura se ajusta en consecuencia.

cajas sin contenido

Si un elemento bloque no tiene contenido se deberá indicar su altura de forma explícita, de lo contrario el elemento no será visible.

Elementos tipo linea​

Los elementos de línea no afectan el espaciado vertical. No sirven para determinar el diseño, sino para diseñar elementos dentro de un bloque.

El ancho de los elementos de línease basa en el contenido, no en el ancho del elemento principal.

La propiedad display​

Cada etiqueta html tiene asignado un tipo de caja por defecto, pero esto puede cambiarse mediante la propiedad display.

Esta porpiedad tiene tres valores posibles:

  • block: ocupa todo el ancho disponible; respeta width, height y márgenes verticales.
  • inline: fluye en línea; no respeta width/height; los márgenes verticales no desplazan la línea.
  • inline-block: en línea pero respeta width/height.
haz este cambio en el ejemplo anterior y observa lo que ocurre
El tipo de los elementos pasa de línea a bloque
em, strong{
    background-color: aqua;
    display: block;
}

el resultado se verá algo así...

Atención

En inline, width/height no afectan al layout. Si necesitas medidas, usa inline-block o block.

Partes de la caja​

En CSS las cajas se componen de 4 partes principales:

  • Contenido: el contenido elemento.
  • Relleno: el espacio entre el contenido el borde.
  • Borde: la línea entre el relleno y el margen
  • Margen: el espacio entre la caja y las cajas circundantes.

En conjunto, esto es todo lo que un navegador necesita para representar la caja de un elemento.El contenido es lo que se recoge en el documento HTML y es lo único que tiene valor semántico. El resto son puramente de presentación, por lo que están definidos por reglas CSS.

  • CSS
  • HTML
  • Representación
.box {
  width: 300px;                  /* contenido (por defecto) */
  padding: 16px;                 /* espacio interno */
  border: 4px solid #0ea5e9;     /* borde alrededor del padding */
  margin: 20px;                  /* espacio externo con otros elementos */
  background: #f8fafc;           /* se pinta en el área de contenido+padding */
}
<div class="box">Caja de ejemplo</div>
nota
  • El background ocupa contenido + padding (no el margin).
  • Si usas outline en vez border este se dibuja por fuera y no ocupa espacio.
Orden (de dentro a fuera)

content → padding → border → margin.

Dimensiones de las Cajas​

Por defecto, el tamaño de las cajas está limitado por su contenido (una caja sin contenido no tiene representación gráfica). En esta sección veremos cómo asignar un tamaño a los elementos de manera explícita en vez de que estos tengan un tamaño predeterminado.

Propiedades width y height​

Las propiedades width y height permiten indicar respectivamente (en el modopor defecto) el ancho y el alo de una caja de forma explícita. A la hora de indicar las dimensiones de un elemento se deben tener en cuenta dos aspectos:

  • Por defecto width/height solo miden el contenido.
  • El tamaño total ocupado suma padding + border (y además se separa por margin).
.total {
  width: 300px;          /* contenido */
  height: 200px;         /* contenido */
  padding: 20px;         /* +40px (izq+der) */
  border: 2px solid;     /* +4px  (izq+der) */
  margin: 10px;          /* espacio externo (no suma al ancho de la caja) */
}
/* Ancho total renderizado = 300 + 40 + 4 = 344px (sin contar margin) */
/* Alto total renderizado = 200 + 40 + 4 = 244px (sin contar margin) */
problema

Diseñas una tarjeta “de 300px”, pero con padding/border termina midiendo más.
Solución recomendada: modificar la propiedad box-sizing de su valor por defecto a box-sizing: border-box;.

La propiedad box-sizing​

La propiedad box-sizing nos permite asignar el tamaño al total de la caja. Puede tener dos valores:

  • box-sizing: content-box; (por defecto). El tamaño NO incluye ni padding ni border
  • box-sizing: border-box;. El tamaño SI incluye ni padding ni border
  • content-box (por defecto)
  • border-box (UI moderno)
Ancho total = 300 + 40 + 8 = 348px (sin contar margin)
.cb {
  box-sizing: content-box; /* default */
  width: 300px; padding: 20px; border: 4px solid;
}
/* Ancho total = 300 + 40 + 8 = 348px (sin contar margin) */
Ancho total = 300px (padding y border van “por dentro”)
.bb {
  box-sizing: border-box;
  width: 300px; padding: 20px; border: 4px solid;
}
/* Ancho total = 300px (padding y border van “por dentro”) */
Regla práctica (reset global)

En interfaces modernas, usa border-box para maquetar más fácil.

Por ello, para tener un comportamiento predecible de todos los elementos de tu página WEB es recomendable que, al principio de tu archivo CSS, añadas la siguiente regla.

* { box-sizing: border-box;}

Desbordamiento overflow​

La propiedad overflow controla qué pasa si el contenido se desborda (no cabe) en el ancho/alto del elemento. Sus valores principales son los siguientes:

ValorComportamiento
visible (por defecto)El contenido se sale de la caja (no se recorta).
hiddenRecorta el contenido que se sale (no hay scroll).
scrollSiempre muestra barras de scroll (aunque no hagan falta).
autoAparecen barras de scroll solo si hace falta.
  • CSS
  • HTML
.caja {
  width: 210px;
  height: 80px;
  border: 1px solid #cbd5e1;
  border-radius: 8px;
  padding: .5rem;
  margin: 50px;
}
.visible { overflow: visible; }
.hidden  { overflow: hidden; }
.auto    { overflow: auto; }
    <div class="caja visible">
        <h2>visible: el texto puede salir de la caja…</h2>
    </div>
    <div class="caja hidden">
        <h2>hidden: En este caso, el texto que no cabe se recorta…</h2>
    </div>
    <div class="caja auto">
        <h2>auto: En este caso, aparece una barra de scroll si hace falta…</h2>
    </div>
Ejes independientes

Puedes usar overflow-x (horizontal) y overflow-y (vertical) de forma separada.

Util

overflow: auto es la opción más práctica para listas, paneles y cajas con contenido variable.

Límite de dimensiones​

En ocasiones, por cuestiones de diseño o responsividad necesitaremos limitar las dimensiones, tanto máximas como mínimas, de cualquier elemento de tipo bloque. Para ello emplearemos las siguientes propiedades:

max-width / max-heightMáximo permitidoEvitar que se disparen de tamañomax-width: 640px; · max-height: 40vh;
realiza este ejemplo para comprobar su funcionamiento

  • HTML

  border-radius: 12px;
  overflow: auto;           /* si rebasa max-height, muestra scroll */
}

Relleno padding​

Padding es el espacio interior entre el contenido de una caja y su borde. la propiedad padding permite modificar este espacio.

Unidades: admite px, rem, em, %, etc.

No admite valores negativos.

Existen varios métodos de declarar esta propiedad

Formas de declarar padding​

FormaSintaxisAfecta aEjemploComentario
Shorthandpadding: V;todos los ladospadding: 16px;Misma separacion por cada lado.
2 valorespadding: V1 V2;V1: top/bottom · V2: left/rightpadding: 8px 16px;Patrón vertical / horizontal.
3 valorespadding: V1 V2 V3;V1: top · V2: left/right · V3: bottompadding: 8px 16px 12px;Menos usado, pero útil.
4 valorespadding: T R B L;top, right, bottom, leftpadding: 8px 16px 12px 4px;Control total por lado.

Propiedades individuales​

También podemos indicar el padding de cada lado de la caja de manera individual.

Margenes margin​

Los márgenes definen el espacio fuera del borde de un elemento o, mejor dicho, el espacio entre una caja y las cajas que la rodean. la propiedad margin permite modificar este espacio.

Unidades: admite px, rem, em, %, etc.

Sí admite valores negativos. Desplaza los elemntos solapándolos (emplear con cudado)

Existen varios métodos de declarar esta propiedad:

Formas de declarar margin​

3 valoresmargin: V1 V2 V3;\top left/right bottom\margin: 8px 16px 12px;
4 valoresmargin: T R B L;top, right, bottom, left`margin: 8px 16px 12px 4px;

Propiedades individuales​

Margen automático​

Centrar los elementos en el centro de la pantalla es una tarea dificil si no manejamos Layouts modernos. Sin embargo, la manera clasica de centrar un elemento es dando el valor auto a los márgenes izquierdo y derecho de un elemento.

Requisitos:

  • El elemento debe comportarse como bloque (display: block o inline-block).
  • Debe tener un ancho definido (width o max-width < ancho del contenedor).
  • Se usa margin-left: auto; margin-right: auto; (o margin: 0 auto;).
  • CSS
  • HTML
  • Representación
.caja {
  width: 320px;                 /* ancho < contenedor */
  margin: 0 auto;               /* izquierda y derecha automáticas */
  padding: 1rem;
  border: 1px solid #cbd5e1; border-radius: 12px;
}
<div class="contenedor">
  <div class="caja">Centrada</div>
</div>

Colapso de márgenes​

Los márgenes verticales entre elementos de bloque pueden colapsar.

Entre bloques del mismo nivel se elegirá el margen mayor
.a { margin-bottom: 50px; background: #f1f5f9; }
.b { margin-top: 25px;   background: #f1f5f9; }
/* El espacio final entre .a y .b será 25px (no 75px) */
Diagnóstico

Si ves “espacios que desaparecen” entre bloques, sospecha de colapso de márgenes.

Soluciones posibles

Entre las posibles soluciones al colapso de márgenes las mas empleadas son 2:

  • Crea un contenedor vacío con un padding de 1px entre los elementos a separar.
añade esto
<div style='padding-top: 1px'></div>
  • Emplea Layouts modernos (Flex o Grid) (Opción recomendada)

Bordes border​

La propiedad border define el borde que rodea a la caja de un elemento (alrededor de su padding y contenido). Un borde se compone de tres piezas: grosor, estilo y color.

Definición y sintaxis​

Shorthand

Ancho / Estilo / Color
/* border: <width> <style> <color>;  (orden flexible) */
border: 2px solid #0ea5e9;

Subpropiedades (todas las caras):

Declaración Individual de cada elemento del borde
border-width: 2px;
border-style: solid;
border-color: #0ea5e9;

Por lado:

Declaración Individual de las propiedades de cada lado
border-top:    4px dashed red;
border-right:  2px solid #999;
border-bottom: 6px double #0ea5e9;
border-left:   0; /* sin borde en el lado izquierdo */

Sombras box-shadow​

box-shadow dibuja sombras alrededor de la caja de un elemento (siguen el contorno del borde y respetan border-radius). No afecta al layout (tamaño/posición) y puedes declarar varias sombras separadas por comas.

Sintaxis​

Sintaxis básica de la propiedad box-shadow
/* Sintaxis (orden flexible de <color>): 
   [inset] <offset-x> <offset-y> <blur-radius?> <spread-radius?> <color?> */
selector {
  box-shadow: 0 8px 24px -8px rgba(0,0,0,.2);
}
  • offset-x / offset-y: desplazamiento horizontal/vertical.
  • blur-radius (opcional): desenfoque (no puede ser negativo).
  • spread-radius (opcional): expande/contrae la sombra (admite negativos).
  • color (opcional): por defecto usa currentColor.
  • inset (opcional): convierte la sombra en interna.
  • none: sin sombra.
  • Múltiples sombras: s1, s2, s3; (la primera se pinta encima).

Parámetros​

ParámetroTipoDescripciónEjemplo
insetkeywordSombra internainset 0 2px 4px rgba(0,0,0,.2)
offset-xlongitudDesplaza en x0, 4px, -6px
offset-ylongitudDesplaza en y2px, 10px
blur-radiuslongitudDifuminado (≥ 0)12px
spread-radiuslongitudExpansión (+) o contracción (–)-6px, 8px
colorcolorColor de la sombrargba(0,0,0,.15)
