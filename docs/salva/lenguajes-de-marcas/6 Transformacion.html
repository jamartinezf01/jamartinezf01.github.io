UT6 - Transformación XML

1. ¿Qué es la transformación de XML?​

El formato XML es muy útil para almacenar y organizar datos estructurados, pero no siempre es el más adecuado para su visualización o procesamiento en otros entornos. Por este motivo, es común transformar un XML a otros formatos según las necesidades: mostrar los datos en una página web, generar un informe en texto, exportarlos a una hoja de cálculo o adaptarlos a una estructura diferente.

El proceso de transformación permite reutilizar los datos almacenados en XML y adaptarlos a diferentes sistemas y aplicaciones.

Formatos de salida más comunes al transformar XML con XSLT​

Formato de salidaUso comúnEjemplo
HTMLMostrar los datos en una web con estilo visual. Permite navegar y comprender fácilmente la información.Tabla con predicción por días, mostrando fecha, estado del cielo, temperaturas y viento.
Texto planoGenerar informes simples, listados o avisos automáticos legibles sin formato. Útil para resúmenes rápidos.Informe meteorológico: Día: 05/04 - Máxima: 24°C - Estado: Despejado - Viento: 15 km/h
CSV / ExcelExportar los datos como tabla para su análisis en hojas de cálculo o carga en otras herramientas.Fecha,Estado,Máxima,Mínima\n05/04, Despejado,24,10\n06/04,Nuboso,22,12
Otro XMLAdaptar la estructura del XML original a otro esquema compatible con otros sistemas o herramientas.Convertir <dia> en <prediccionDia> con etiquetas como <tempMax> o <descripcionCielo>.

Nota: Aunque el texto plano no es un formato estructurado como HTML o CSV, es útil para generar mensajes, informes simples o resúmenes fácilmente legibles por personas.

2. ¿Por qué transformar documentos XML?​

XML estructura los datos, pero no se preocupa de cómo se muestran ni de a quién van dirigidos.

Las principales razones para transformar un documento XML son:

✅ Mejorar la visualización

  • Mostrar los datos de forma agradable y entendible para personas.
  • Ejemplo: convertir un XML en una tabla HTML para una web.

⚙️ Facilitar la integración con otros sistemas

  • Adaptar estructuras XML para otras aplicaciones (por ejemplo, otra web, una app o una API).

📓 Generar diferentes salidas a partir de una sola fuente

  • HTML, PDF, Excel, informes...
  • Automatizar el formato según el uso.

3. Soluciones para representar documentos XML​

🔸 XML + CSS (Cascading Style Sheets)​

  • Ventajas:
    • Separa los datos (XML) del estilo (CSS).
    • Permite definir colores, tipografía, y disposición visual.
    • Mejora la legibilidad.
  • Limitaciones:
    • Solo aplica formato visual, no cambia estructuras.
    • No puede acceder a ciertos datos (por ejemplo, atributos de elementos vacíos).
  • Resultado
  • XML
  • CSS

🔸 XML + XSL (Extensible Stylesheet Language)​

  • Permite transformar completamente el contenido:
    • Generar HTML, CSV u otro XML.
    • Aplicar condiciones, bucles, filtros, etc.
  • Muy utilizado cuando:
    • Se requiere estructura y presentación.
    • Se necesitan distintas salidas con los mismos datos.
  • Resultado
  • XML
  • XSL
  • CSS

Estándares XSL

Para transformar documentos XML de forma eficaz, es necesario conocer los principales estándares que permiten manipular, presentar o adaptar su contenido. Estas tecnologías trabajan de forma conjunta y cada una cumple un papel específico en el proceso de transformación.

ESTÁNDARDESCRIPCIÓN
XMLLenguaje base para estructurar los datos
XSLConjunto de lenguajes diseñados para trabajar con documentos XML
XSLTTransforma XML en otros formatos (HTML, texto, otro XML, etc.)
XPathSelecciona partes del XML que se quieren procesar o transformar
XSL-FOAplica formato para documentos impresos o PDF (menos usado actualmente)

4. Comparativa visual: XML con CSS vs. XML con XSLT​

AspectoXML con CSSXML con XSLTObservaciones
Formato generalBloques verticalesTabla estructuradaCSS no permite reorganizar los datos en formato tabular.
Título📍 Predicción para Cáceres (añadido con attr(localidad))“Predicción para Cáceres” en <h2>Ambos acceden al atributo localidad, pero con XSLT se inserta como contenido real.
Separación por díaCada día en un bloque visual distintoCada día es una filaCSS da formato visual, pero no puede agrupar ni repetir estructuras.
IconosIconos manuales insertados en ::beforeIconos dinámicos según el contenidoCon XSLT se puede condicionar el icono según los datos (if / choose).
Descripción del cielo❌ No se muestra✅ “Despejado”, “Intervalos nubosos”…CSS no puede acceder a atributos como @descripcion. XSLT sí.
Estilo de cabeceraNo tieneCabecera en azul oscuro con texto blancoCSS no puede crear nuevas secciones como <thead>.
Orden de elementosFijo según el orden del XML originalOrden personalizado con XSLTXSLT permite reordenar, filtrar y agrupar información.
Color de fondoFondo azul claro en bloquesBlanco en celdas, alternancia con grisAmbas personalizaciones son posibles, pero CSS lo aplica por bloques y XSLT lo transforma en HTML.
Accesibilidad del contenidoSolo nombres de etiquetas visiblesTexto descriptivo visible en pantallaXSLT puede generar contenido a partir de atributos ocultos.
Flexibilidad para web real❌ Limitada: solo visual✅ Total: se genera HTML completoCon XSLT el XML se convierte en HTML y puede adaptarse a cualquier diseño.
