3.7.4 Eventos DOM

Los eventos en el DOM son interacciones o sucesos que ocurren dentro de una página web. Estas interacciones pueden ser disparadas por el usuario, como un clic o el envío de un formulario, o por el navegador, como la carga completa del documento. Los eventos son esenciales para crear páginas web dinámicas e interactivas, ya que permiten responder a las acciones de los usuarios.

¿Qué es un evento?​

Un evento es una señal que indica que algo ocurrió. Algunos ejemplos de eventos comunes incluyen:

  • Hacer clic en un botón (click).
  • Pasar el mouse sobre un elemento (mouseover).
  • Escribir texto en un campo (input).
  • Cargar completamente una página (DOMContentLoaded).

Cuando ocurre un evento, el navegador genera un objeto evento que contiene información relevante, como el tipo de evento, el elemento que lo disparó y el momento en que ocurrió.

Tipos de Eventos en el DOM​

En el DOM existen diversos tipos de eventos que permiten interactuar con los elementos de una página web. Estos eventos se agrupan en categorías según su naturaleza y propósito: eventos del mouse, del teclado, de formularios, de carga, entre otros. A continuación, exploraremos las categorías más importantes y sus eventos más comunes.

1. Eventos del Mouse​

Estos eventos se generan a partir de interacciones con el mouse, como clics, movimientos o cambios en su posición.

EventoDescripciónEjemplo de Uso
clickSe dispara cuando el usuario hace clic.Botones, enlaces.
dblclickSe dispara al hacer doble clic.Abrir un elemento en detalle al hacer doble clic.
mouseoverSe dispara cuando el mouse pasa sobre un elemento.Mostrar un tooltip.
mouseoutSe dispara cuando el mouse sale de un elemento.Ocultar un tooltip.
mousedownSe dispara cuando el botón del mouse es presionado.Iniciar un evento de arrastre.
mouseupSe dispara cuando se suelta el botón del mouse.Finalizar un evento de arrastre.
mousemoveSe dispara cuando el mouse se mueve sobre un elemento.Crear efectos visuales al mover el mouse.
contextmenuSe dispara al hacer clic derecho.Mostrar un menú contextual personalizado.

Ejemplo Práctico: Mostrar Coordenadas del Mouse​

<div id="zona" style="width: 300px; height: 200px; border: 1px solid black;">
  Mueve el mouse aquí
</div>
<script>
  const zona = document.getElementById('zona');
  zona.addEventListener('mousemove', (evento) => {
    zona.textContent = `Coordenadas: X=${evento.clientX}, Y=${evento.clientY}`;
  });
</script>

2. Eventos del Teclado​

Los eventos del teclado se disparan al interactuar con teclas físicas del teclado.

EventoDescripciónEjemplo de Uso
keydownSe dispara cuando se presiona una tecla.Detectar combinaciones de teclas (e.g., Ctrl + S).
keyupSe dispara al soltar una tecla.Verificar qué tecla fue liberada.
keypress(Obsoleto) Similar a keydown.Se recomienda keydown o keyup.

Ejemplo Práctico: Detectar Teclas Específicas​

<input type="text" id="campo" placeholder="Escribe algo">
<script>
  const campo = document.getElementById('campo');
  campo.addEventListener('keydown', (evento) => {
    console.log(`Tecla presionada: ${evento.key}`);
    if (evento.key === 'Enter') {
      console.log('Se presionó Enter');
    }
  });
</script>

3. Eventos de Formularios​

Estos eventos están relacionados con la interacción del usuario con elementos de formularios, como campos de texto, botones de envío, etc.

EventoDescripciónEjemplo de Uso
submitSe dispara al enviar un formulario.Validar los datos antes del envío.
changeSe dispara al cambiar el valor de un campo.Detectar cambios en listas desplegables.
inputSe dispara cuando se escribe en un campo.Mostrar sugerencias mientras se escribe.
focusSe dispara cuando un campo gana el foco.Resaltar el campo en el que está escribiendo el usuario.
blurSe dispara cuando un campo pierde el foco.Validar el contenido de un campo al salir.

Ejemplo Práctico: Validar un Formulario​

<form id="formulario">
  <input type="text" id="nombre" placeholder="Nombre">
  <button type="submit">Enviar</button>
</form>
<script>
  const formulario = document.getElementById('formulario');
  formulario.addEventListener('submit', (evento) => {
    const nombre = document.getElementById('nombre').value;
    if (!nombre) {
      evento.preventDefault(); // Evita el envío
      alert('Por favor, completa el campo de nombre.');
    }
  });
</script>

4. Eventos de Carga​

Estos eventos se disparan cuando se cargan o descargan recursos en la página.

EventoDescripciónEjemplo de Uso
DOMContentLoadedSe dispara cuando el DOM está completamente cargado.Ejecutar scripts cuando la estructura está lista.
loadSe dispara cuando todos los recursos están cargados.Verificar que imágenes o scripts externos estén listos.
unloadSe dispara al abandonar la página.Guardar datos antes de salir (obsoleto en muchos casos).
errorSe dispara cuando un recurso no se carga correctamente.Manejar errores de carga de imágenes.

Ejemplo Práctico: Ejecutar Código al Cargar la Página​

<img src="imagen.jpg" id="imagen" alt="Imagen">
<script>
  const imagen = document.getElementById('imagen');
  imagen.addEventListener('load', () => {
    console.log('La imagen se cargó correctamente.');
  });
  imagen.addEventListener('error', () => {
    console.log('Error al cargar la imagen.');
  });
</script>

5. Otros Tipos de Eventos​

  1. Eventos de Drag and Drop (Arrastrar y Soltar):

    • drag, dragstart, dragend, drop.
    • Usados para mover elementos visualmente dentro de una página.

  2. Eventos de Medios:

    • play, pause, ended, timeupdate.
    • Relacionados con elementos de audio y video.

  3. Eventos de Ventana:

    • resize: Se dispara al cambiar el tamaño de la ventana.
    • scroll: Se dispara al desplazarse por la página.

Ejemplo Práctico: Detectar Desplazamiento​

<div style="height: 2000px; background: linear-gradient(white, gray);">
  Desplázate para ver el efecto.
</div>
<script>
  window.addEventListener('scroll', () => {
    console.log(`Desplazamiento: ${window.scrollY}px`);
  });
</script>
tip

Hay muchos más. Puedes ojear la lista completa en la documentación de Mozilla Developer.

Modelo de Eventos del DOM​

El modelo de eventos del DOM describe cómo se generan y propagan los eventos en un documento. Cuando ocurre un evento, este sigue un recorrido en el árbol DOM que consta de tres fases:

  1. Fase de captura: El evento se propaga desde el nodo raíz (document) hacia abajo, pasando por cada nodo hasta llegar al objetivo.
  2. Fase de destino: El evento alcanza el nodo objetivo donde ocurrió.
  3. Fase de burbujeo: El evento se propaga hacia arriba desde el nodo objetivo hasta el nodo raíz.

Representación del flujo:​

Si haces clic en el botón:

  1. En la fase de captura, el evento viaja: document → html → body → div → button.
  2. En la fase de destino, el evento interactúa directamente con el button.
  3. En la fase de burbujeo, el evento viaja de regreso: button → div → body → html → document.

1. Fase de Bubbling​

La burbuja ocurre después de que el evento interactúa con el nodo objetivo. En esta fase, el evento asciende por el árbol DOM.

  • Ejemplo práctico:
<div id="contenedor">
  <button id="boton">Haz clic aquí</button>
</div>
<script>
  const contenedor = document.getElementById('contenedor');
  const boton = document.getElementById('boton');

  // Evento en el contenedor
  contenedor.addEventListener('click', () => {
    console.log('Evento en contenedor (burbujeo)');
  });

  // Evento en el botón
  boton.addEventListener('click', () => {
    console.log('Evento en botón');
  });
</script>
  • Salida esperada al hacer clic en el botón:
    1. Se ejecuta el evento del botón.
    2. El evento "burbujea" y se ejecuta el del contenedor.

2. Fase de Capturing​

La captura ocurre antes de que el evento alcance el nodo objetivo. En esta fase, el evento viaja desde el nodo raíz hasta el nodo donde ocurrió.

  • Para manejar un evento en la fase de captura, se debe pasar true como tercer argumento en addEventListener().

  • Ejemplo práctico:

<div id="contenedor">
  <button id="boton">Haz clic aquí</button>
</div>
<script>
  const contenedor = document.getElementById('contenedor');
  const boton = document.getElementById('boton');

  // Evento en el contenedor durante la fase de captura
  contenedor.addEventListener(
    'click',
    () => {
      console.log('Evento en contenedor (captura)');
    },
    true // Habilitar la fase de captura
  );

  // Evento en el botón
  boton.addEventListener('click', () => {
    console.log('Evento en botón');
  });
</script>
  • Salida esperada al hacer clic en el botón:
    1. Se ejecuta primero el evento en el contenedor (captura).
    2. Luego, se ejecuta el evento en el botón.

Diferencias entre Burbujeo y Captura​

CaracterísticaBurbujeo (Bubbling)Captura (Capturing)
Orden de PropagaciónDesde el nodo objetivo hacia arriba.Desde el nodo raíz hacia el nodo objetivo.
ConfiguraciónEs la fase por defecto en addEventListener.Debe habilitarse explícitamente con true.
Uso ComúnManejar eventos en elementos internos.Manejar eventos antes de llegar al objetivo.
¿Por qué es importante entender el modelo de eventos?
  1. Delegación de eventos: Permite gestionar eventos de múltiples elementos con un único manejador (veremos esto más adelante).
  2. Control de propagación: Puedes detener la propagación de eventos con stopPropagation().
  3. Evitar conflictos: Entender el flujo evita que eventos en diferentes elementos interfieran entre sí.

Manejo de Eventos​

El manejo de eventos en el DOM se realiza principalmente a través del método addEventListener(). Este método permite asociar funciones manejadoras a un elemento para que respondan a eventos específicos. Por otro lado, removeEventListener() se utiliza para eliminar esos manejadores.

addEventListener()​

Descripción:​

  • addEventListener() permite asociar un manejador de eventos a un elemento.
  • Es flexible, ya que permite manejar múltiples eventos en el mismo elemento.
  • Puede escuchar eventos en las fases de captura o burbujeo.

Sintaxis:​

elemento.addEventListener(evento, manejador, opciones);
  • evento: El tipo de evento a escuchar (por ejemplo, click, mouseover).
  • manejador: La función que se ejecutará cuando ocurra el evento.
  • opciones: (Opcional) Un objeto o un valor booleano para especificar si el evento se maneja en la fase de captura o burbujeo.

Ejemplo Básico: Escuchar un Evento de Clic​

<button id="boton">Haz clic</button>
<script>
  const boton = document.getElementById('boton');
  boton.addEventListener('click', () => {
    console.log('Botón clickeado');
  });
</script>

Al hacer clic en el botón, se muestra en la consola el mensaje: Botón clickeado.

Agregar Múltiples Manejadores al Mismo Elemento​

<button id="boton">Haz clic</button>
<script>
  const boton = document.getElementById('boton');

  boton.addEventListener('click', () => {
    console.log('Primer manejador ejecutado');
  });

  boton.addEventListener('click', () => {
    console.log('Segundo manejador ejecutado');
  });
</script>

Salida al hacer clic:

  1. Primer manejador ejecutado.
  2. Segundo manejador ejecutado.

Escuchar Eventos en Diferentes Fases​

<div id="contenedor">
  <button id="boton">Haz clic</button>
</div>
<script>
  const contenedor = document.getElementById('contenedor');
  const boton = document.getElementById('boton');

  // Manejador en la fase de captura
  contenedor.addEventListener(
    'click',
    () => {
      console.log('Evento en contenedor (captura)');
    },
    true
  );

  // Manejador en la fase de burbujeo
  boton.addEventListener('click', () => {
    console.log('Evento en botón (burbujeo)');
  });
</script>

removeEventListener()​

Descripción:​

  • Este método elimina un manejador previamente asociado con addEventListener().
  • Requiere una referencia exacta a la función manejadora que se usó en addEventListener().

Sintaxis:​

elemento.removeEventListener(evento, manejador, opciones);

Ejemplo: Eliminar un Evento​

<button id="boton">Haz clic</button>
<script>
  const boton = document.getElementById('boton');

  function manejarClic() {
    console.log('Evento activado');
  }

  // Asociar el manejador
  boton.addEventListener('click', manejarClic);

  // Eliminar el manejador después de 5 segundos
  setTimeout(() => {
    boton.removeEventListener('click', manejarClic);
    console.log('Manejador eliminado');
  }, 5000);
</script>
  1. Durante los primeros 5 segundos, al hacer clic en el botón se mostrará: Evento activado.
  2. Después de 5 segundos, el manejador se elimina y el botón ya no responde al clic.

Opciones Adicionales en addEventListener​

El tercer argumento de addEventListener puede ser un objeto con las siguientes opciones:

  • capture: Indica si el evento debe manejarse en la fase de captura (true) o burbujeo (false, por defecto).
  • once: Si es true, el manejador se ejecutará solo una vez y luego se eliminará automáticamente.
  • passive: Si es true, indica que el manejador nunca llamará a preventDefault(). Esto puede mejorar el rendimiento en eventos como scroll.

Ejemplo con Opciones​

<button id="boton">Haz clic</button>
<script>
  const boton = document.getElementById('boton');

  boton.addEventListener(
    'click',
    () => {
      console.log('Este evento se ejecuta solo una vez');
    },
    { once: true } // El evento se elimina automáticamente tras ejecutarse
  );
</script>

Al hacer clic en el botón, el mensaje se muestra una vez y luego el manejador se elimina.

Manejadores con Parámetros​

Si necesitas pasar parámetros a la función manejadora, puedes usar una función anónima o una función flecha como envoltorio:

<button id="boton">Haz clic</button>
<script>
  const boton = document.getElementById('boton');

  function manejarClic(mensaje) {
    console.log(mensaje);
  }

  boton.addEventListener('click', () => {
    manejarClic('Hola, evento con parámetros');
  });
</script>

2.5 Manejadores Dinámicos​

Es posible añadir o eliminar eventos basados en condiciones:

<button id="boton">Haz clic</button>
<script>
  const boton = document.getElementById('boton');
  let activado = true;

  function manejarClic() {
    console.log('Evento activado');
  }

  boton.addEventListener('click', () => {
    if (activado) {
      boton.addEventListener('click', manejarClic);
      activado = false;
    } else {
      boton.removeEventListener('click', manejarClic);
      console.log('Manejador eliminado');
      activado = true;
    }
  });
</script>
  • La primera vez que haces clic, el evento se activa.
  • Al siguiente clic, el manejador se elimina y ya no responde.

Propiedades y Métodos del Objeto Evento​

El objeto evento (Event) es una herramienta poderosa que proporciona información y métodos para interactuar con el evento que se ha disparado. Este objeto es generado automáticamente por el navegador cada vez que ocurre un evento y se pasa como parámetro a la función manejadora.

A continuación, se detallan las propiedades y métodos más útiles del objeto evento, junto con explicaciones y ejemplos prácticos.

1. Propiedades del Objeto Evento​

type​

  • Descripción: Devuelve el tipo de evento que se ha disparado, como click, keydown, submit, etc.

  • Ejemplo:

document.addEventListener('click', (evento) => {
  console.log(`Tipo de evento: ${evento.type}`); // "click"
});

target​

  • Descripción: Devuelve el elemento del DOM que disparó el evento.

  • Ejemplo:

<ul id="lista">
  <li>Elemento 1</li>
  <li>Elemento 2</li>
</ul>
<script>
  const lista = document.getElementById('lista');
  lista.addEventListener('click', (evento) => {
    console.log(`Elemento clickeado: ${evento.target.textContent}`);
  });
</script>
  • Uso típico: Delegación de eventos.

currentTarget​

  • Descripción: Devuelve el elemento al que se le asignó el manejador de eventos (puede ser diferente de target en delegación de eventos).

  • Ejemplo:

lista.addEventListener('click', (evento) => {
  console.log(`Evento asignado a: ${evento.currentTarget.id}`); // "lista"
});

bubbles​

  • Descripción: Indica si el evento puede propagarse en la fase de burbujeo (true o false).

  • Ejemplo:

document.addEventListener('click', (evento) => {
  console.log(`¿Burbujea? ${evento.bubbles}`); // "true" para eventos como click
});

defaultPrevented​

  • Descripción: Indica si la acción predeterminada del evento ha sido prevenida mediante preventDefault().

  • Ejemplo:

const enlace = document.querySelector('a');
enlace.addEventListener('click', (evento) => {
  evento.preventDefault();
  console.log(`¿Se evitó la acción predeterminada? ${evento.defaultPrevented}`); // true
});

timeStamp​

  • Descripción: Devuelve la marca de tiempo (en milisegundos) en que ocurrió el evento.

  • Ejemplo:

document.addEventListener('click', (evento) => {
  console.log(`Evento disparado en: ${evento.timeStamp} ms`);
});

key y code (Eventos de Teclado)​

  • Descripción:

    • key: Devuelve la tecla presionada (por ejemplo, Enter, a, Esc).
    • code: Devuelve el código físico de la tecla (por ejemplo, KeyA, Enter).

  • Ejemplo:

document.addEventListener('keydown', (evento) => {
  console.log(`Tecla: ${evento.key}, Código: ${evento.code}`);
});

2. Métodos del Objeto Evento​

preventDefault()​

  • Descripción: Evita la acción predeterminada asociada al evento.

  • Ejemplo:

<a href="https://example.com" id="enlace">Ir a Example</a>
<script>
  const enlace = document.getElementById('enlace');
  enlace.addEventListener('click', (evento) => {
    evento.preventDefault(); // Evita que el enlace navegue
    console.log('Acción predeterminada evitada');
  });
</script>

stopPropagation()​

  • Descripción: Detiene la propagación del evento hacia otros manejadores en el árbol DOM.

  • Ejemplo:

<div id="contenedor">
  <button id="boton">Haz clic</button>
</div>
<script>
  const contenedor = document.getElementById('contenedor');
  const boton = document.getElementById('boton');

  contenedor.addEventListener('click', () => {
    console.log('Evento en contenedor');
  });

  boton.addEventListener('click', (evento) => {
    evento.stopPropagation(); // Detiene la propagación
    console.log('Evento en botón');
  });
</script>

Salida esperada al hacer clic en el botón:

  • Solo se ejecuta el evento del botón.

stopImmediatePropagation()​

  • Descripción: Detiene la propagación del evento y evita que otros manejadores asociados al mismo elemento se ejecuten.

  • Ejemplo:

const boton = document.getElementById('boton');
boton.addEventListener('click', () => {
  console.log('Primer manejador');
});
boton.addEventListener('click', (evento) => {
  evento.stopImmediatePropagation(); // Detiene todos los manejadores en este elemento
  console.log('Segundo manejador detenido');
});
boton.addEventListener('click', () => {
  console.log('Tercer manejador'); // No se ejecutará
});

composedPath()​

  • Descripción: Devuelve un array con todos los elementos que forman el recorrido del evento en el árbol DOM.

  • Ejemplo:

document.addEventListener('click', (evento) => {
  console.log(evento.composedPath());
});

Salida: Un array con los nodos desde el objetivo hasta document.

3. Ejemplo​

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <title>Propiedades y Métodos del Objeto Evento</title>
</head>
<body>
  <div id="contenedor">
    <button id="boton">Haz clic</button>
    <a href="https://example.com" id="enlace">Ir a Example</a>
  </div>
  <script>
    const contenedor = document.getElementById('contenedor');
    const boton = document.getElementById('boton');
    const enlace = document.getElementById('enlace');

    // Mostrar información básica del evento
    boton.addEventListener('click', (evento) => {
      console.log(`Tipo de evento: ${evento.type}`);
      console.log(`Elemento objetivo: ${evento.target.tagName}`);
    });

    // Prevenir acción predeterminada
    enlace.addEventListener('click', (evento) => {
      evento.preventDefault();
      console.log('Navegación evitada');
    });

    // Detener propagación
    contenedor.addEventListener('click', () => {
      console.log('Evento en contenedor');
    });

    boton.addEventListener('click', (evento) => {
      evento.stopPropagation();
      console.log('Evento en botón, propagación detenida');
    });
  </script>
</body>
</html>
