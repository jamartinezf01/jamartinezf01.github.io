3.7.1 Estructura del DOM

El DOM organiza un documento HTML o XML como un árbol de nodos. Cada parte del documento, ya sea un elemento, un atributo, un texto o incluso un comentario, se representa como un nodo dentro de este árbol. Esta estructura jerárquica permite navegar, manipular y modificar dinámicamente el contenido y la estructura del documento.

Tipos de nodos en el DOM​

El DOM clasifica cada parte de un documento HTML o XML en nodos. Los nodos son los componentes básicos de la estructura del árbol del DOM y se dividen en varios tipos, dependiendo de su función en el documento.

Tipo de NodoDescripciónEjemplo
Elemento (Element)Representa las etiquetas HTML o XML.<div>, <p>, <h1>
Atributo (Attr)Representa los atributos de los elementos.class="container"
Texto (Text)Representa el contenido de texto dentro de un elemento."Hola, mundo" en <p>Hola</p>
Comentario (Comment)Representa los comentarios del documento.<!-- Comentario -->
Documento (Document)Representa el nodo raíz del árbol DOM.document

Nodo de Elemento (Element)​

Los nodos de elemento representan las etiquetas HTML o XML en el documento. Son los nodos más comunes y forman la estructura básica del árbol del DOM.

  • Ejemplo de un nodo de elemento:
<div class="contenedor"></div>

  • Propiedades importantes:

    • tagName: Devuelve el nombre de la etiqueta, como DIV, H1, etc.
    • attributes: Devuelve una colección de los atributos del elemento.

  • Acciones comunes:

    • Modificar atributos: setAttribute(), getAttribute().
    • Navegar entre nodos hijos: children, firstElementChild, lastElementChild.

  • Ejemplo en JavaScript:

const elemento = document.querySelector('.contenedor');
console.log(elemento.tagName); // "DIV"
elemento.setAttribute('id', 'nuevoId'); // Añadir un atributo
console.log(elemento.id); // "nuevoId"

Nodo de Atributo (Attr)​

Los nodos de atributo representan los atributos de los elementos HTML o XML. Aunque están asociados a nodos de elemento, son objetos independientes en el DOM.

  • Ejemplo de un nodo de atributo:
<input type="text" placeholder="Escribe aquí">

  • Propiedades importantes:

    • name: Nombre del atributo (por ejemplo, type o placeholder).
    • value: Valor del atributo (por ejemplo, text o Escribe aquí).

  • Ejemplo en JavaScript:

const input = document.querySelector('input');
console.log(input.getAttribute('placeholder')); // "Escribe aquí"
input.setAttribute('placeholder', 'Nuevo texto'); // Modifica el valor del atributo
console.log(input.placeholder); // "Nuevo texto"

Nodo de Texto (Text)​

Los nodos de texto representan el contenido de texto dentro de los elementos. Son hijos directos de nodos de elemento y contienen solo texto (sin etiquetas).

  • Ejemplo de un nodo de texto:
<p>Este es un párrafo.</p>

Aquí, "Este es un párrafo." es un nodo de texto que es hijo del nodo <p>.

  • Propiedades importantes:

    • nodeValue o textContent: Devuelven o modifican el contenido del nodo de texto.

  • Ejemplo en JavaScript:

const parrafo = document.querySelector('p');
console.log(parrafo.textContent); // "Este es un párrafo."
parrafo.textContent = "Texto modificado."; // Cambia el contenido del texto
console.log(parrafo.textContent); // "Texto modificado."

Nodo de Comentario (Comment)​

Los nodos de comentario representan los comentarios en el documento. Son útiles para documentar el código HTML, pero no afectan la estructura visible de la página.

  • Ejemplo de un nodo de comentario:
<!-- Esto es un comentario -->

  • Propiedades importantes:

    • nodeValue: Contiene el texto del comentario.

  • Ejemplo en JavaScript:

const comentario = document.createComment('Comentario generado con JavaScript');
document.body.appendChild(comentario); // Añade el comentario al final del body
console.log(comentario.nodeValue); // "Comentario generado con JavaScript"

Nodo de Documento (Document)​

El nodo de documento es el nodo raíz del árbol DOM. Representa todo el documento y actúa como punto de entrada para interactuar con cualquier parte del DOM.

  • Propiedades importantes:

    • documentElement: Devuelve el nodo raíz del documento (normalmente <html>).
    • body: Devuelve el nodo <body> del documento.
    • head: Devuelve el nodo <head> del documento.

  • Ejemplo en JavaScript:

console.log(document.documentElement); // Muestra el nodo <html>
console.log(document.body); // Muestra el nodo <body>
console.log(document.head); // Muestra el nodo <head>

Otros Tipos de Nodos​

Además de los nodos principales mencionados, el DOM incluye otros tipos menos comunes:

Tipo de NodoDescripciónEjemplo
Nodo de Documento Fragmentado (DocumentFragment)Representa un contenedor vacío que puede contener nodos temporales. Es útil para mejorar el rendimiento al manipular el DOM.N/A
Nodo de Documento Tipo (DocumentType)Representa el tipo de documento, como <!DOCTYPE html>.document.doctype
Nodo de Notación (Notation)Representa una notación declarada en DTD (usado en XML).N/A

El nodo raíz: document​

El nodo raíz del DOM es el nodo document, que representa todo el documento HTML o XML cargado en el navegador. Es el punto de entrada principal para acceder y manipular cualquier elemento del árbol DOM. Este nodo es único por documento y actúa como la raíz jerárquica desde la cual se pueden navegar y modificar los demás nodos.

¿Qué es el Nodo Document?​

El nodo document es una instancia de la interfaz Document, que extiende la interfaz más genérica Node. Sirve como una representación programática del contenido del archivo HTML o XML y permite a los desarrolladores acceder a la estructura, el contenido y los estilos del documento.

Cuando un navegador carga una página, convierte el código HTML en un árbol de nodos y lo asocia al nodo raíz document.

Propiedades del Nodo Document​

El nodo document proporciona varias propiedades importantes que facilitan el acceso a diferentes partes del documento:

PropiedadDescripción
documentElementDevuelve el nodo raíz del documento, normalmente el elemento <html>.
headDevuelve el nodo <head> del documento.
bodyDevuelve el nodo <body> del documento.
titleDevuelve o establece el título del documento.
URLDevuelve la URL completa del documento cargado.
cookiePermite leer o escribir cookies asociadas al documento.
formsDevuelve una colección de todos los formularios (<form>) del documento.
imagesDevuelve una colección de todas las imágenes (<img>) del documento.
linksDevuelve una colección de todos los enlaces (<a>) con atributos href.
scriptsDevuelve una colección de todos los elementos <script>.

Acceso a Elementos Básicos​

// Acceder al nodo raíz del documento
console.log(document.documentElement); // <html>...</html>

// Acceder al nodo <head> y su contenido
console.log(document.head); // <head>...</head>

// Acceder al nodo <body>
console.log(document.body); // <body>...</body>

// Obtener el título del documento
console.log(document.title); // "Ejemplo Nodo Document"

// Modificar el título del documento
document.title = "Nuevo Título";
console.log(document.title); // "Nuevo Título"

Propiedades comunes de los nodos​

Todos los nodos en el DOM comparten un conjunto de propiedades que permiten interactuar con ellos de manera uniforme. Estas propiedades son útiles para acceder a la información básica de un nodo, navegar por el árbol del DOM y realizar manipulaciones.

Propiedades Clave​

PropiedadDescripciónEjemplo
nodeTypeDevuelve el tipo de nodo (un número que indica si es un elemento, texto, comentario, etc.).1 para elementos, 3 para texto, 8 para comentarios.
nodeNameDevuelve el nombre del nodo, como el nombre de la etiqueta para elementos.DIV, P, #text.
nodeValueDevuelve o establece el contenido de un nodo (aplicable solo a nodos de texto y comentarios)."Hola, DOM" para nodos de texto.
parentNodeReferencia al nodo padre en el árbol DOM.El nodo <body> es el padre de <h1>.
childNodesDevuelve una colección (NodeList) de todos los nodos hijos, incluidos nodos de texto y comentarios.Nodos hijos de un <div>.
firstChildDevuelve el primer nodo hijo.Un nodo de texto, comentario o elemento.
lastChildDevuelve el último nodo hijo.Un nodo de texto, comentario o elemento.
nextSiblingDevuelve el siguiente nodo hermano en el mismo nivel jerárquico.El siguiente nodo después de un <h1>.
previousSiblingDevuelve el nodo hermano anterior en el mismo nivel jerárquico.El nodo anterior antes de un <h1>.

Propiedad: nodeType​

La propiedad nodeType identifica el tipo de un nodo mediante un valor numérico.

Valor de nodeTypeDescripciónEjemplo
1Nodo de elemento<div>
2Nodo de atributoclass="container"
3Nodo de texto"Hola, mundo"
8Nodo de comentario<!-- Comentario -->
9Nodo de documentodocument

Ejemplo en JavaScript:

const nodo = document.querySelector('h1');
console.log(nodo.nodeType); // 1 (Nodo de elemento)

Propiedad: nodeName​

La propiedad nodeName devuelve el nombre del nodo. Para nodos de elemento, es el nombre de la etiqueta en mayúsculas; para nodos de texto, es #text; y para comentarios, es #comment.

Ejemplo en JavaScript:

const nodo = document.querySelector('h1');
console.log(nodo.nodeName); // "H1"

const texto = nodo.firstChild;
console.log(texto.nodeName); // "#text"

Propiedad: nodeValue​

La propiedad nodeValue contiene el contenido del nodo. Es útil principalmente para nodos de texto y comentarios.

  • Para nodos de texto: Devuelve el contenido textual.
  • Para nodos de comentario: Devuelve el texto del comentario.
  • Para nodos de elemento: Devuelve null.

Ejemplo en JavaScript:

const comentario = document.createComment('Este es un comentario');
console.log(comentario.nodeValue); // "Este es un comentario"

const texto = document.createTextNode('Hola, DOM');
console.log(texto.nodeValue); // "Hola, DOM"

Propiedades Relacionadas con la Navegación​

Estas propiedades permiten navegar por el árbol DOM desde un nodo específico:

  1. parentNode: Devuelve el nodo padre del nodo actual. Si el nodo no tiene padre (como document), devuelve null.
const h1 = document.querySelector('h1');
console.log(h1.parentNode); // Nodo <body>
  1. childNodes: Devuelve una lista de nodos hijos, incluidos nodos de texto y comentarios.
const body = document.body;
console.log(body.childNodes); // NodeList con todos los nodos hijos de <body>
  1. firstChild y lastChild: Devuelven el primer y último nodo hijo respectivamente.
console.log(body.firstChild); // Primer nodo hijo (puede ser texto)
console.log(body.lastChild);  // Último nodo hijo
  1. nextSibling y previousSibling: Permiten acceder al nodo hermano siguiente o anterior.
const h1 = document.querySelector('h1');
console.log(h1.nextSibling); // Nodo que sigue a <h1>
console.log(h1.previousSibling); // Nodo antes de <h1>

Ejemplo​

Considere el siguiente HTML:

<body>
  <h1>Hola, DOM</h1>
  <p>Este es un párrafo.</p>
  <!-- Comentario -->
</body>

Código JavaScript:

// Acceder al nodo <h1>
const h1 = document.querySelector('h1');

// Propiedad nodeType
console.log(h1.nodeType); // 1 (Elemento)

// Propiedad nodeName
console.log(h1.nodeName); // "H1"

// Propiedad parentNode
console.log(h1.parentNode); // Nodo <body>

// Propiedad childNodes
console.log(h1.childNodes); // NodeList que incluye un nodo de texto

// Propiedad firstChild
console.log(h1.firstChild.nodeValue); // "Hola, DOM"

// Propiedad nextSibling
console.log(h1.nextSibling.nodeName); // "P" (El párrafo es el nodo hermano siguiente)
