3.6 Funciones en JavaScript

Las funciones son bloques de código reutilizables que se pueden ejecutar en cualquier momento. En JavaScript, las funciones son un concepto central, ya que permiten estructurar 🏗️, modularizar 🧩 y reutilizar ♻️ el código, facilitando la legibilidad 👓 y el mantenimiento 🔧 de las aplicaciones.

¿Qué es una Función? 📚​

Una función es un bloque de código que realiza una tarea específica o calcula un valor. Se puede definir una vez y ejecutar tantas veces como sea necesario 🔄.

Sintaxis Básica 💡​

function nombreFuncion(parámetros) {
    // Código a ejecutar
    return resultado; // Opcional
}

Tipos de Declaración de Funciones 📂​

1. Funciones Declaradas 📝​

Las funciones declaradas son las más comunes y tienen el beneficio de ser hoisted 🚀 (pueden ser utilizadas antes de su declaración en el código).

Ejemplo:

function saludar(nombre) {
    return `Hola, ${nombre}`;
}

console.log(saludar("Juan")); // "Hola, Juan"

🔑 Características:

  • ✅ Se pueden utilizar antes de declararlas.
  • 🔄 Ideales para funciones generales y reutilizables.

2. Funciones Anónimas 🤐​

Son funciones que no tienen nombre y se asignan a una variable o se pasan como argumento.

Ejemplo:

const saludar = function(nombre) {
    return `Hola, ${nombre}`;
};

console.log(saludar("Ana")); // "Hola, Ana"

3. Funciones Flecha (Arrow Functions) ⚡​

Introducidas en ES6, las funciones flecha son una forma más compacta de escribir funciones. Además, no tienen su propio contexto this 🤯, lo que las hace ideales para funciones anidadas.

🔧 Sintaxis:

const nombreFuncion = (parámetros) => {
    // Código a ejecutar
    return resultado;
};

Ejemplo:

const sumar = (a, b) => a + b;

console.log(sumar(3, 4)); // 7

🔑 Características:

  • ✂️ Más concisas.
  • 📌 No tienen su propio this.

Parámetros y Valores de Retorno 🔎​

1. Parámetros 🛠️​

Los parámetros son variables que se definen en la declaración de la función y se utilizan como entrada.

Ejemplo:

function multiplicar(a, b) {
    return a * b;
}

console.log(multiplicar(2, 3)); // 6

2. Valores Predeterminados 🎯​

Puedes asignar valores predeterminados a los parámetros para manejar casos donde no se proporcionen argumentos.

Ejemplo:

function saludar(nombre = "Visitante") {
    return `Hola, ${nombre}`;
}

console.log(saludar()); // "Hola, Visitante"

3. El Operador rest 📦​

El operador rest (...) permite pasar un número variable de argumentos como un array 🛍️.

Ejemplo:

function sumar(...numeros) {
  let total = 0;
  for (let numero of numeros) {
    total += numero;
  }
  return total;
}

console.log(sumar(1, 2, 3));      // 6
console.log(sumar(10, 20));      // 30
console.log(sumar(4, 5, 6, 7));  // 22
¿Cómo Funciona?

Al escribir function sumar(...numeros), estamos diciendo: “la función sumar puede recibir cualquier cantidad de argumentos, y todos ellos se almacenarán en un array llamado numeros”. Por dentro, podemos iterar sobre numeros con un bucle para realizar las operaciones deseadas.

4. Valores de Retorno (return) 🔙​

Las funciones pueden devolver un valor utilizando la palabra clave return. Si no se incluye, la función devuelve undefined por defecto.

Ejemplo:

function cuadrado(x) {
    return x * x;
}

console.log(cuadrado(4)); // 16

5. Return Implícito​

El return implícito es una característica de las funciones flecha (arrow functions) en JavaScript que te permite omitir la palabra clave return cuando el bloque de la función se reduce a una sola expresión.

En otras palabras, si escribes una arrow function sin llaves ({}) y con una sola expresión, JavaScript interpretará ese valor como el valor de retorno de la función. Por ejemplo:

// Ejemplo con "return implícito"
const sumar = (a, b) => a + b; 

console.log(sumar(2, 3)); // 5

Aquí, no has escrito return a + b, pero JavaScript entiende que el resultado de a + b es lo que se retorna.

En cambio, si usas llaves, ya no hay return implícito y debes poner la palabra return explícitamente:

// Ejemplo SIN "return implícito"
const sumar = (a, b) => {
  return a + b; 
};

console.log(sumar(2, 3)); // 5

Funciones Especiales: setTimeout, setInterval y clearInterval ⏱️​

JavaScript proporciona funciones especiales para trabajar con la temporización de tareas. Estas funciones son particularmente útiles para ejecutar código después de un retraso (setTimeout) ⏳ o de forma repetida (setInterval) 🔄. También se puede detener la ejecución de un intervalo utilizando clearInterval 🚫.

1. setTimeout ⏳​

La función setTimeout permite ejecutar un bloque de código o función después de un período de tiempo especificado.

🛠️ Sintaxis​

setTimeout(función, tiempoEnMilisegundos, argumentosOpcionales);
  • función: La función o el bloque de código que se ejecutará.
  • tiempoEnMilisegundos: Tiempo de espera antes de ejecutar el código (en milisegundos) ⏰.
  • argumentosOpcionales: Argumentos que se pasarán a la función cuando sea ejecutada.

📋 Ejemplo Básico​

console.log("Iniciando...");

setTimeout(() => {
    console.log("Ejecutado después de 2 segundos");
}, 2000);

console.log("Fin del código");

📤 Salida:

Iniciando...
Fin del código
Ejecutado después de 2 segundos

🔧 Uso con Funciones​

function mostrarMensaje(mensaje) {
    console.log(mensaje);
}

setTimeout(mostrarMensaje, 3000, "Este mensaje se muestra después de 3 segundos");

2. setInterval 🔁​

La función setInterval ejecuta repetidamente una función o bloque de código en intervalos de tiempo establecidos 🕒.

🛠️ Sintaxis​

setInterval(función, tiempoEnMilisegundos, argumentosOpcionales);
info
  • función: La función o el bloque de código que se ejecutará repetidamente.
  • tiempoEnMilisegundos: Intervalo entre cada ejecución ⏱️.
  • argumentosOpcionales: Argumentos que se pasarán a la función.

📋 Ejemplo Básico​

let contador = 0;

const intervalo = setInterval(() => {
    contador++;
    console.log(`Contador: ${contador}`);
}, 1000);

📤 Salida (cada segundo):

Contador: 1
Contador: 2
Contador: 3
...

3. clearInterval 🚫​

La función clearInterval detiene un intervalo previamente configurado con setInterval 🛑. Es necesario guardar el identificador devuelto por setInterval para poder cancelarlo.

🛠️ Sintaxis​

clearInterval(identificador);

📋 Ejemplo​

let contador = 0;

const intervalo = setInterval(() => {
    contador++;
    console.log(`Contador: ${contador}`);
    
    if (contador === 5) {
        clearInterval(intervalo);
        console.log("Intervalo detenido");
    }
}, 1000);

📤 Salida:

Contador: 1
Contador: 2
Contador: 3
Contador: 4
Contador: 5
Intervalo detenido

Comparativa entre setTimeout y setInterval ⚖️​

⏰ Función🔎 Descripción👩‍💻 Ejemplo de uso
setTimeoutEjecuta una función después de un tiempo especificado una sola vez.Mostrar un mensaje tras 5 segundos.
setIntervalEjecuta una función repetidamente con un intervalo de tiempo especificado.Contador en tiempo real.
clearIntervalDetiene un intervalo configurado con setInterval.Detener un contador cuando alcanza un valor.

Parámetros Adicionales​

Las funciones setTimeout y setInterval permiten ejecutar una función después de un período de tiempo o a intervalos regulares, respectivamente. Además del tiempo en milisegundos, ambas funciones aceptan parámetros adicionales que pueden ser pasados como argumentos a la función que se ejecutará.

setTimeout ⌛​

setTimeout(función, tiempo, [argumento1, argumento2, ...]);

Ejemplo Práctico:

function mostrarMensaje(nombre, mensaje) {
  console.log(`Hola, ${nombre}. ${mensaje}`);
}

// Usamos setTimeout con parámetros adicionales
setTimeout(mostrarMensaje, 2000, "Juan", "Bienvenido a nuestra plataforma.");

setInterval 🔁​

setInterval(función, tiempo, [argumento1, argumento2, ...]);

Ejemplo Práctico:

function contarHasta(numero) {
  console.log(`Contador: ${numero}`);
}

// Usamos setInterval con un argumento adicional
let contador = 1;
const intervalo = setInterval(contarHasta, 1000, contador);

// Incrementar el contador en cada iteración
const incrementar = setInterval(() => {
  contador++;
  if (contador > 5) {
    clearInterval(intervalo); // Detenemos el intervalo después de 5 iteraciones
    clearInterval(incrementar);
  }
}, 1000);

💡 Recomendaciones​

ojo con esto...

  1. ⚠️ Evita intervalos infinitos no controlados:

    • Siempre considera cuándo detener un setInterval para evitar un uso excesivo de recursos.
    let contador = 0;
    const intervalo = setInterval(() => {
        contador++;
        if (contador > 10) clearInterval(intervalo);
    }, 1000);

  2. 🗂️ Gestiona identificadores correctamente:

    • Almacena los identificadores devueltos por setTimeout o setInterval en una variable para detenerlos si es necesario.

  3. 🔍 Combina con condicionales:

    • Usa condicionales dentro de los intervalos para decidir cuándo detener o cambiar la ejecución.

Buenas Prácticas con Funciones 👍🏼​

sigue estos consejos...

  1. Mantén las funciones cortas:

    • Una función debe realizar una sola tarea específica.

  2. Nombra las funciones descriptivamente:

    • Usa nombres que indiquen claramente lo que hace la función.
    function calcularPromedio(numeros) {
        // Código
    }

  3. Evita funciones anidadas complejas:

    • Divide el código en funciones más pequeñas y reutilizables.

  4. Usa valores predeterminados:

    • Facilita el manejo de casos donde los parámetros no se proporcionan.

  5. Prefiere funciones puras:

    • Evita modificar variables externas al bloque de la función.
