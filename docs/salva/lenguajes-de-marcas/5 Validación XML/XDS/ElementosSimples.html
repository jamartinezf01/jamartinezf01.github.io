Elementos Simples

Los elementos simples son aquellos que contienen únicamente datos de texto y no pueden tener subelementos ni atributos complejos (aunque sí pueden tener atributos simples). Estos elementos se definen en el XSD utilizando el elemento <xs:element> y se asocian a un tipo simple, ya sea uno predefinido (como xs:string, xs:int, xs:date, etc.) o uno definido por el usuario con restricciones específicas.

1. Declaración y Características​

La declaración básica de un elemento simple se realiza con los atributos name y type. Por ejemplo:

<xs:element name="Nombre" type="xs:string"/>
info
  • name: Define el nombre del elemento.
  • type: Indica el tipo de dato simple que se espera (en este caso, una cadena de texto).

2. Atributos fixed y default​

En la declaración de un elemento (o atributo), se pueden especificar valores fijos o por defecto:

  • fixed: Indica que el valor debe ser exactamente el especificado.
  • default: Establece un valor predeterminado en caso de que el elemento (o atributo) no esté presente en el documento XML.

Elemento con atributo fixed​

En este ejemplo, el elemento Pais debe tener siempre el valor "España". Si se incluye en el XML, solo se acepta ese valor:

<xs:element name="Pais" type="xs:string" fixed="España"/>

Uso en XML:

<Pais xmlns="http://www.ejemplo.org/datos">España</Pais>

Elemento con atributo default​

Aquí, el elemento Moneda tiene un valor por defecto "EUR". Si el XML no lo especifica, se asume "EUR":

<xs:element name="Moneda" type="xs:string" default="EUR"/>

Uso en XML (cuando se omite):

<Transaccion xmlns="http://www.ejemplo.org/datos">
    <Moneda/> <!-- Se asume "EUR" por defecto -->
</Transaccion>

3. Tipos de Datos Predefinidos​

Los tipos de datos predefinidos en XML Schema (XSD) son un conjunto de tipos integrados que vienen definidos en la especificación de XML Schema. Estos tipos sirven para describir y validar el contenido de los elementos y atributos de un documento XML sin necesidad de definir nuevos tipos desde cero. Al usarlos, se garantiza que los datos se ajusten a formatos y restricciones estandarizadas, lo que facilita la interoperabilidad y la validación.

Algunos de los Tipos de Datos Predefinidos más comunes son...

TipoDescripciónEjemplo de Valor
xs:stringCadena de texto."Hola mundo"
xs:integerNúmeros enteros.42
xs:decimalNúmeros decimales.3.1416
xs:booleanValores lógicos: true o false.true
xs:dateFecha en formato ISO (YYYY-MM-DD).2025-12-31
xs:timeHora en formato ISO (HH:MM:SS).14:30:00
xs:dateTimeFecha y hora combinadas en formato ISO.2025-12-31T14:30:00
xs:durationIntervalo de tiempo.P2Y6M5DT12H35M30S
xs:base64BinaryDatos binarios codificados en base64.SGVsbG8=
xs:anyURIIdentificador de recurso (URI).http://www.ejemplo.org

Fuente X3C (https://www.w3.org/TR/xmlschema-2/)

A continuación, se muestra un fragmento de un XSD que declara elementos utilizando algunos de los tipos predefinidos:

<xs:schema 
    xmlns:xs="http://www.w3.org/2001/XMLSchema"
    targetNamespace="http://www.ejemplo.org/datos"
    xmlns="http://www.ejemplo.org/datos"
    elementFormDefault="qualified">

  <!-- Elemento para un nombre de persona -->
  <xs:element name="Nombre" type="xs:string"/>
  
  <!-- Elemento para la edad -->
  <xs:element name="Edad" type="xs:integer"/>
  
  <!-- Elemento para la fecha de nacimiento -->
  <xs:element name="FechaNacimiento" type="xs:date"/>
  
  <!-- Elemento para indicar si está activo -->
  <xs:element name="Activo" type="xs:boolean"/>

</xs:schema>
En este ejemplo:
  • El elemento <Nombre> se define con el tipo xs:string.
  • El elemento <Edad> se define con el tipo xs:integer.
  • El elemento <FechaNacimiento> se define con el tipo xs:date.
  • El elemento <Activo> se define con el tipo xs:boolean.

Cada uno de estos elementos se valida automáticamente para asegurar que el contenido del XML se ajuste a la definición del tipo predefinido.

4. Tipos Personalizados​

En XML Schema (XSD) es posible definir tipos simples personalizados que heredan de un tipo predefinido (como xs:string o xs:integer) y a los cuales se les pueden aplicar restricciones específicas mediante facetas. Estas restricciones permiten controlar aspectos como la longitud de una cadena, el formato de los datos o el rango permitido en valores numéricos. A continuación, se describen las principales facetas y se muestra un ejemplo para cada una de ellas.

Las facetas permiten limitar y validar el contenido de los elementos simples. Algunas de las facetas más utilizadas son:

FacetaDescripciónEjemplo
lengthLongitud fija de la cadena<xs:length value="10"/>
minLengthLongitud mínima de la cadena<xs:minLength value="1"/>
maxLengthLongitud máxima de la cadena<xs:maxLength value="50"/>
patternExpresión regular que debe cumplir el valor<xs:pattern value="[A-Za-z ]+"/>
enumerationConjunto de valores permitidos<xs:enumeration value="Rojo"/>
minInclusiveValor mínimo permitido (incluyente)<xs:minInclusive value="0"/>
minExclusiveValor mínimo permitido (excluyente)<xs:minExclusive value="0"/>
maxInclusiveValor máximo permitido (incluyente)<xs:maxInclusive value="100"/>
maxExclusiveValor máximo permitido (excluyente)<xs:maxExclusive value="100"/>
totalDigitsNúmero total de dígitos permitidos<xs:totalDigits value="5"/>
fractionDigitsNúmero máximo de dígitos a la derecha del punto decimal<xs:fractionDigits value="2"/>
whiteSpaceCómo trata espacios en blanco, tabulaciones, saltos de línea...<xs:whiteSpace value="preserve"/>

length​

  • Descripción: Define la longitud exacta permitida para una cadena.
  • Ejemplo: Se requiere que una cadena tenga exactamente 8 caracteres.
<xs:simpleType name="CadenaConLength">
  <xs:restriction base="xs:string">
    <xs:length value="8"/>
  </xs:restriction>
</xs:simpleType>

✅ Valores válidos: "ABCDEFGH", "12345678"
❌ Valores inválidos: "ABC", "ABCDEFGHI", ""

minLength​

  • Descripción: Define la longitud mínima permitida para una cadena.
<xs:simpleType name="CadenaConMinLength">
  <xs:restriction base="xs:string">
    <xs:minLength value="3"/>
  </xs:restriction>
</xs:simpleType>

✅ Valores válidos: "abc", "hello", "1234"
❌ Valores inválidos: "", "a", "ab"

maxLength​

  • Descripción: Establece la longitud máxima permitida para una cadena.
<xs:simpleType name="CadenaConMaxLength">
  <xs:restriction base="xs:string">
    <xs:maxLength value="10"/>
  </xs:restriction>
</xs:simpleType>

✅ Valores válidos: "12345", "Hola", "ABCDEFGHIJ"
❌ Valores inválidos: "ABCDEFGHIJK", "1234567890123"

enumeration​

  • Descripción: Define un conjunto fijo de valores permitidos.
<xs:simpleType name="Color">
  <xs:restriction base="xs:string">
    <xs:enumeration value="Rojo"/>
    <xs:enumeration value="Verde"/>
    <xs:enumeration value="Azul"/>
  </xs:restriction>
</xs:simpleType>

✅ Valores válidos: "Rojo", "Verde", "Azul"
❌ Valores inválidos: "Amarillo", "rojo" (diferencia en mayúsculas), "azul " (espacio extra)

minInclusive​

  • Descripción: Valor mínimo permitido (incluyente).
<xs:simpleType name="NumeroConMinInclusive">
  <xs:restriction base="xs:integer">
    <xs:minInclusive value="10"/>
  </xs:restriction>
</xs:simpleType>

✅ Valores válidos: 10, 15, 100
❌ Valores inválidos: 9, 0, -5

maxInclusive​

  • Descripción: Valor máximo permitido (incluyente).
<xs:simpleType name="NumeroConMaxInclusive">
  <xs:restriction base="xs:integer">
    <xs:maxInclusive value="100"/>
  </xs:restriction>
</xs:simpleType>

✅ Valores válidos: 50, 100, 0
❌ Valores inválidos: 101, 200

totalDigits​

  • Descripción: Número total de dígitos permitidos.
<xs:simpleType name="NumeroTotalDigitos">
  <xs:restriction base="xs:decimal">
    <xs:totalDigits value="5"/>
  </xs:restriction>
</xs:simpleType>

✅ Valores válidos: 12345, 999.9, 12.34
❌ Valores inválidos: 123456, 99999.9

fractionDigits​

  • Descripción: Número máximo de dígitos a la derecha del punto decimal.
<xs:simpleType name="NumeroConDecimales">
  <xs:restriction base="xs:decimal">
    <xs:fractionDigits value="2"/>
  </xs:restriction>
</xs:simpleType>

✅ Valores válidos: 123.45, 99.99, 12.3, 10.00
❌ Valores inválidos: 123.456, 99.999

pattern​

  • Descripción: Expresión regular que deben cumplir los valores.

Ejemplo 1: Solo letras mayúsculas​

<xs:simpleType name="SoloMayusculas">
  <xs:restriction base="xs:string">
    <xs:pattern value="[A-Z]+"/>
  </xs:restriction>
</xs:simpleType>

✅ Valores válidos: "HELLO", "WORLD"
❌ Valores inválidos: "hello", "Hello123", "HELLO WORLD"

Ejemplo 2: Código postal de 5 dígitos​

<xs:simpleType name="CodigoPostal">
  <xs:restriction base="xs:string">
    <xs:pattern value="\d{5}"/>
  </xs:restriction>
</xs:simpleType>

✅ Valores válidos: "12345", "98765"
❌ Valores inválidos: "1234", "123456", "12A45"

recuerda

Las expresiones regulares usadas en pattern siguen reglas específicas. Aquí algunos elementos clave:

SímboloSignificado
.Cualquier carácter excepto nueva línea
\dCualquier dígito (0-9)
\wCualquier letra, número o guion bajo (a-z, A-Z, 0-9, _)
[abc]Uno de los caracteres especificados (a, b o c)
[a-z]Cualquier letra minúscula (a hasta z)
[A-Z]Cualquier letra mayúscula (A hasta Z)
[0-9]Cualquier número (0 a 9)
{n}Exactamente n repeticiones
{n,}Al menos n repeticiones
{n,m}Entre n y m repeticiones
+Al menos una repetición
*Cero o más repeticiones
?Cero o una repetición

whiteSpace​

  • Descripción: Controla el tratamiento de los espacios en blanco en una cadena.
    • preserve: Conserva los espacios tal como aparecen en la entrada.
    • replace: Reemplaza todos los espacios en blanco por espacios simples.
    • collapse: Elimina espacios al inicio y al final, y reduce múltiples espacios internos a uno solo.
<xs:simpleType name="TextoConWhiteSpace">
  <xs:restriction base="xs:string">
    <xs:whiteSpace value="collapse"/>
  </xs:restriction>
</xs:simpleType>

✅ Valores válidos (colapsando espacios):
Entrada: " Hola mundo " → Se almacena como "Hola mundo"
Entrada: "Hola " → Se almacena como "Hola"
Entrada: " XML Schema " → Se almacena como "XML Schema"

❌ Valores inválidos:
Este tipo de faceta no invalida valores, sino que modifica la forma en que se almacenan. Si se usa collapse, una entrada con espacios innecesarios será ajustada automáticamente.
