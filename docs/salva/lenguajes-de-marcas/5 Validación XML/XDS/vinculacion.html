Vinculación

Para asociar un documento XML a su correspondiente esquema XSD debemos vincularlo, de modo que las herramientas de validación sepan qué reglas utilizar para verificar la estructura y el contenido del XML.

1. Atributos clave para la vinculación​

Para vincular un XML a su esquema XSD, se utilizan principalmente dos atributos dentro de la etiqueta de apertura de la raíz del documento XML:

  • xmlns:xsi: Declara el namespace necesario para usar atributos relacionados con la validación (definidos por W3C). Normalmente se define así: 
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
  • xsi:schemaLocation: Indica la ubicación física (ruta o URL) del archivo XSD que define el namespace correspondiente. La forma habitual de uso es: 
    xsi:schemaLocation="NAMESPACE RUTA_O_URL_XSD"
    nota

    Se especifica primero el namespace y luego la ruta o URL del archivo XSD.

Ejemplo básico de vinculación​

<?xml version="1.0" encoding="UTF-8"?>
<DocumentoEjemplo
    xmlns="http://www.ejemplo.org/misdatos"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://www.ejemplo.org/misdatos misdatos.xsd">

    <Elemento>Valor</Elemento>
</DocumentoEjemplo>
En este ejemplo:
  • El espacio de nombres principal del documento es http://www.ejemplo.org/misdatos.
  • A través de xmlns:xsi, se habilitan los atributos de instancia de esquema de XML (schemaLocation, noNamespaceSchemaLocation, etc.).
  • Con xsi:schemaLocation, vinculamos ese mismo namespace (http://www.ejemplo.org/misdatos) al archivo misdatos.xsd, el cual contiene la definición de los elementos y reglas de validación.

2. schemaLocation vs noNamespaceSchemaLocation​

Además de schemaLocation, existe la variante noNamespaceSchemaLocation. Ambas se usan para indicar la ubicación de uno o varios esquemas, pero difieren en la forma en que manejan el espacio de nombres.

AtributoUso principalEjemplo
xsi:schemaLocationSe usa cuando el XML sí está asociado a uno o más namespaces.xsi:schemaLocation="http://www.ejemplo.org/misdatos misdatos.xsd"
xsi:noNamespaceSchemaLocationSe usa cuando el XML no posee un espacio de nombres (no usa atributos xmlns).xsi:noNamespaceSchemaLocation="archivosinnamespace.xsd"
info
  • Si tu XML no tiene un namespace definido (es decir, no incluyes xmlns="..."), deberías utilizar noNamespaceSchemaLocation.
  • Si tu XML sí tiene un namespace (como en la mayoría de los casos empresariales), emplearás schemaLocation, indicando la asociación entre el namespace y el XSD.

3. Ejemplo comparativo​

A continuación se muestra la diferencia en la cabecera de un XML con namespace (arriba) y uno sin namespace (abajo).

XML con namespace:​

<DocumentoConNS 
    xmlns="http://www.ejemplo.org/conNamespace"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://www.ejemplo.org/conNamespace conNamespace.xsd">
    <!-- Contenido -->
</DocumentoConNS>

XML sin namespace:​

<DocumentoSinNS
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:noNamespaceSchemaLocation="sinNamespace.xsd">
    <!-- Contenido -->
</DocumentoSinNS>
nota

Observa que en el primer ejemplo (DocumentoConNS) existe un xmlns="...", mientras que en el segundo (DocumentoSinNS) no se define un namespace por defecto.

4. Errores comunes a evitar​

Evita estos errores
  1. Namespace mal escrito:
    Usar un namespace diferente al indicado en el XSD (o tener una errata) provoca que el validador no asocie el esquema.
  2. Ruta incorrecta del XSD:
    Poner mal el nombre del archivo o la ruta genera un error al intentar cargar el esquema.
  3. Falta de coincidencia entre el namespace y la declaración en el XSD:
    El targetNamespace en el XSD debe coincidir con el namespace en el XML.
  4. Falta de declaración de xmlns:xsi:
    Sin la declaración xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance", el atributo xsi:schemaLocation carece de validez.
