5.2 XSD

Un XSD (siglas de XML Schema Definition) es un lenguaje que permite describir la estructura y los tipos de datos de un documento XML. En otras palabras, un XSD especifica:

  1. Qué elementos pueden aparecer en el documento.
  2. Cómo se relacionan esos elementos (si tienen elementos hijos, en qué orden, cuántas veces pueden repetirse, etc.).
  3. Qué atributos pueden llevar y de qué tipo son.
  4. Qué restricciones adicionales se aplican a su contenido (por ejemplo, un número mínimo o máximo de caracteres, un patrón concreto, un rango de valores…).

De esta manera, los documentos XML que se ajusten a las reglas definidas en un XSD se consideran “válidos” con respecto a ese esquema.

¿En qué se diferencia de un DTD?​

  • Sintaxis XML: A diferencia de los DTD (Document Type Definition), un XSD está escrito en la propia sintaxis de XML, lo que facilita la lectura por parte de herramientas y la ampliación del esquema.
  • Tipado de datos: Los XSD permiten definir tipos de datos numéricos, fechas, booleanos, etc., dando la posibilidad de realizar validaciones más complejas (por ejemplo, “que este valor sea un entero y esté entre 1 y 100”).
  • Espacios de nombres: Ofrece un soporte completo para Namespaces, pudiendo combinar diferentes vocabularios XML sin ambigüedades.

Ejemplo sencillo de XSD​

<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema">
    <xs:element name="libros">
        <xs:complexType>
            <xs:sequence>
                <xs:element name="libro" maxOccurs="unbounded">
                    <xs:complexType>
                        <xs:sequence>
                            <xs:element name="titulo" type="xs:string"/>
                            <xs:element name="autor" type="xs:string"/>
                            <xs:element name="paginas" type="xs:integer"/>
                        </xs:sequence>
                    </xs:complexType>
                </xs:element>
            </xs:sequence>
        </xs:complexType>
    </xs:element>
</xs:schema>
info
  • xs:schema: Es el elemento raíz del XSD.
  • xs:element: Declara un elemento, en este caso, libros.
  • xs:complexType y xs:sequence: Indican que libros es un tipo complejo que contiene una secuencia de subelementos.
  • maxOccurs="unbounded": Significa que puede haber tantos <libro> como se desee.
  • xs:string y xs:integer: Tipos de datos predefinidos por XML Schema.

Un documento XML que cumpla estas reglas será válido: deberá tener un elemento raíz <libros> que contenga uno o varios <libro>, y cada <libro> deberá llevar exactamente <titulo>, <autor> y <paginas> (siendo <paginas> un entero).

Ventajas principales de usar XSD​

ventajas XSD
  1. Validaciones más avanzadas: Permite usar restricciones sofisticadas (expresiones regulares, rangos numéricos, longitudes de cadena, etc.).
  2. Extensibilidad: Al estar escrito en XML, es más fácil de integrar y mantener en conjunto con otras herramientas que procesan XML.
  3. Modularidad: Se puede dividir un gran esquema en varios archivos y combinarlos (por ejemplo, usando xs:import o xs:include).
