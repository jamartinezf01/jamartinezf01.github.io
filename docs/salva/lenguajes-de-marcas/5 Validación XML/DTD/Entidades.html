Entidades

Las entidades en un DTD (Document Type Definition) permiten definir identificadores que el procesador XML sustituirá o usará en distintos lugares, ya sea dentro del contenido del documento o para hacer referencia a archivos externos. Suelen emplearse para:

  1. Reutilizar fragmentos de texto (por ejemplo, una frase que aparece muchas veces).
  2. Incluir recursos externos (imágenes, audio, video…), especialmente cuando necesitas declararlos de forma centralizada y asignarles una notación.

Existen distintos tipos de entidades en un DTD, que veremos a continuación.

ventajas
  • Facilitan la reutilización de texto o recursos.
  • Proporcionan una forma centralizada de administrar referencias externas (imágenes, etc.).
  • Modularizan el DTD cuando hay mucho contenido repetido.

1. Entidades Generales​

Las entidades generales se pueden invocar (usar) en el contenido del documento XML. Se declaran con la sintaxis:

<!ENTITY nombre "contenido de la entidad">
nota

o con SYSTEM / PUBLIC en caso de enlaces a ficheros externos.

Entidades generales internas (analizables)​

  • Se declaran con <!ENTITY nombre "texto">.
  • Al usarlas en el XML, se escribe &nombre;, y el procesador XML reemplaza &nombre; por el texto declarado.

Ejemplo de DTD

<!ENTITY sponsor "Patrocinador SuperCorp">

Uso en el XML

<info>
  ¡Gracias a &sponsor; por hacer posible este evento!
</info>
info

En tiempo de procesamiento, &sponsor; se sustituye por Patrocinador SuperCorp.

Entidades generales externas (analizables o no analizables)​

Se utilizan para indicar que la entidad hace referencia a un recurso externo (un fichero, por ejemplo). Aquí se suelen ver dos grandes posibilidades:

  • Analizables (parsed): Para incluir texto o trozos XML ubicados en un archivo externo.
  • No analizables (unparsed): Para archivos binarios como imágenes, videos, etc.

Entidades generales externas analizables​

<!ENTITY descripcion SYSTEM "descripcion.txt">
info
  • “descripcion.txt” podría contener texto que se insertará como si fuera parte del documento.
  • Se invoca con &descripcion; en el XML.

Entidades generales externas no analizables (unparsed)​

Se declaran con NDATA para indicar que no se van a analizar como XML o texto sino que son datos binarios o de otro tipo.

<!ENTITY logo SYSTEM "images/logo.png" NDATA png>
<!NOTATION png SYSTEM "image/png">
info
  • NDATA png indica la notación que describe el formato de la entidad (en este caso, png y su MIME type "image/png").
  • Generalmente, un atributo de tipo ENTITY en el XML puede hacer referencia a logo.

Ejemplo de uso en el XML

<imagen src="logo"/>

Si se declara:

<!ATTLIST imagen src ENTITY #IMPLIED>
nota

El procesador sabe que src hace referencia a la entidad logo.

2. Entidades Paramétricas​

Estas entidades se definen con % y solo se usan dentro del propio DTD, no en el contenido del XML. Sirven para “incluir” fragmentos de la definición del DTD en múltiples partes, o para condicionar partes del DTD.

<!ENTITY % definicion_comun "
    <!ELEMENT cabecera (titulo, autor)>
    <!ELEMENT titulo (#PCDATA)>
    <!ELEMENT autor (#PCDATA)>
">
%definicion_comun;
info
  • Se declara %definicion_comun con una serie de definiciones de elementos.
  • Luego, con %definicion_comun;, se “pega” ese contenido donde se invoque.
tip

Las entidades paramétricas son muy útiles cuando el DTD es extenso y necesitas repetición o modularidad.

3. Ejemplo de Uso Global​

Imagina un DTD interno que define entidades para un documento XML sobre un evento de e-sports:

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE evento [
  <!-- NOTACIONES -->
  <!NOTATION png SYSTEM "image/png">
  <!NOTATION jpg SYSTEM "image/jpeg">

  <!-- ENTIDADES GENERALES INTERNAS -->
  <!ENTITY sponsor "GigaSponsor Inc.">
  <!ENTITY premio  "Tarjeta de regalo de 100$">

  <!-- ENTIDAD GENERAL EXTERNA NO ANALIZABLE (unparsed) -->
  <!ENTITY banner SYSTEM "images/banner.jpg" NDATA jpg>

  <!-- ELEMENTOS --> 
  <!ELEMENT evento (info, premios)>
  <!ELEMENT info (#PCDATA)>
  <!ELEMENT premios (#PCDATA)>

  <!-- ATRIBUTOS -->
  <!ATTLIST premios
    galeria ENTITY #IMPLIED
  >
]>
<evento>
  <info>
    ¡Bienvenidos al gran torneo patrocinado por &sponsor;!
  </info>
  <premios galeria="banner">
    El ganador se llevará &premio; y una medalla exclusiva.
  </premios>
</evento>
  1. Entidades generales internas:
    • sponsor: Se sustituye en el XML por “GigaSponsor Inc.”
    • premio: Se sustituye por “Tarjeta de regalo de 100$”.
  2. Entidad general externa no analizable:
    • banner apunta a images/banner.jpg, se declara NDATA jpg para especificar su notación.
    • El elemento <premios> tiene un atributo galeria de tipo ENTITY, cuyo valor “banner” se refiere a esta entidad.

El procesador, al encontrar &sponsor; o &premio;, hace la sustitución de texto. Y sabe que galeria="banner" se refiere a un recurso externo (banner.jpg).
