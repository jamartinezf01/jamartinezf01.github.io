Atributos

En un DTD , los atributos se definen mediante la declaración <!ATTLIST>. Esto permite controlar las propiedades que pueden tener los elementos del XML, sus tipos (por ejemplo, si son identificadores únicos, listas enumeradas, etc.) y si son obligatorios u opcionales.

1. Estructura de <!ATTLIST>​

La forma general de declarar atributos en un DTD es:

<!ATTLIST nombre_elemento
    nombre_atributo tipo_de_atributo modo_de_uso
>
info
  • nombre_elemento: El elemento al que se le van a asignar atributos.
  • nombre_atributo: El nombre del atributo.
  • tipo_de_atributo: Expresa qué tipo de valor puede tomar (ej. CDATA, ID, enumeraciones, etc.).
  • modo_de_uso: Indica si es obligatorio, opcional o tiene un valor por defecto.

2. Modos de Declaración de Atributos​

Atributos en DTD pueden tener distintos “modos” que determinan cómo o cuándo se usan. Aquí una tabla resumen:

ModoSignificadoEjemplo en DTD
#REQUIREDEl atributo debe aparecer siempre en el elemento. Si no está presente en el XML, el documento no será válido.<br/> <!ATTLIST artista nombre CDATA #REQUIRED>
#IMPLIEDEl atributo es opcional: puede aparecer o no.<br/> <!ATTLIST escenario tipo (principal|secundario) #IMPLIED>
#FIXEDEl atributo tendrá siempre el mismo valor. Si en el XML se intenta poner otro valor, no será válido.<br/> <!ATTLIST festival version CDATA #FIXED "1.0">
"valor"El atributo es opcional, y si no se especifica, se asigna automáticamente el valor indicado como defecto.<br/> <!ATTLIST pase acceso (VIP|general) "general">
nota

Cuando simplemente ponemos "cadena" entre comillas, estamos indicando el valor por defecto del atributo si no se especifica nada en el XML.

Ejemplo​

 <!-- Declaración de elementos -->
    <!ELEMENT festival (artista+, escenario+, programa)>
    <!ELEMENT artista (nombre, genero)>
    <!ELEMENT escenario (nombre, ubicacion?)>
    <!ELEMENT programa (evento+)>
    <!ELEMENT evento (#PCDATA)>
    <!ELEMENT nombre (#PCDATA)>
    <!ELEMENT genero (#PCDATA)>
    <!ELEMENT ubicacion (#PCDATA)>

    <!-- Declaración de atributos -->

    <!-- festival -->
    <!ATTLIST festival
        id           ID        #REQUIRED
        edicion      CDATA     #REQUIRED
        tipoEntrada  (VIP|general) "general"
        version      CDATA     #FIXED "1.0"
    >

    <!-- artista -->
    <!ATTLIST artista
        id_artista   ID              #REQUIRED
        popularidad  CDATA           #IMPLIED
        foto         ENTITY          #IMPLIED
    >

    <!-- escenario -->
    <!ATTLIST escenario
        id_escenario ID              #REQUIRED
        tipo         (principal|secundario) #IMPLIED
    >

3. Tipos de Atributos más Comunes​

En la siguiente tabla se muestran los tipos de atributos que con mayor frecuencia se utilizan en DTD:

Tipo de AtributoDescripciónEjemplo DTD
CDATACualquier cadena de caracteres.<!ATTLIST artista nombre CDATA #REQUIRED>
IDIdentificador único en todo el documento. No puede repetirse ni en el mismo elemento ni en ningún otro.<!ATTLIST artista id ID #REQUIRED>
IDREF / IDREFSReferencia(s) a un atributo ID definido en algún elemento del documento. Sirve para enlazar elementos entre sí (ej. un artista que referencia un escenario por su ID).<!ATTLIST escenario refEscenario IDREF #IMPLIED>
(opción1|opción2...)Atributo enumerado, solo puede tomar uno de los valores indicados.<!ATTLIST pase acceso (VIP|general) "general">
NMTOKEN / NMTOKENSCadena sin espacios ni caracteres raros. Si son varios tokens, separados por espacios.<!ATTLIST festival keywords NMTOKENS #IMPLIED>
NOTATIONPara indicar que el valor del atributo es el nombre de una notación declarada (<!NOTATION ...>). Se usa para manejar formatos no XML (imágenes, vídeos...).<!ATTLIST poster formato NOTATION (jpg|png) #REQUIRED>
ENTITY / ENTITIESPara enlazar a entidades declaradas dentro del DTD (por ejemplo, un archivo externo de imagen).<!ATTLIST artista foto ENTITY #IMPLIED>

A continuación se explica cada tipo de atributo en DTD y se proporciona un pequeño ejemplo de uso (tanto la parte del DTD como un fragmento XML ilustrativo).

Atributo de tipo ID​

Un atributo de tipo ID funciona como un identificador único para un elemento dentro del documento XML. Nadie más puede tener el mismo valor para un atributo ID.

Ejemplo​

  • DTD
  • XML
<!ELEMENT jugador (nombre, ranking)>
<!ATTLIST jugador
    id_jugador   ID   #REQUIRED
>
<!ELEMENT nombre (#PCDATA)>
<!ELEMENT ranking (#PCDATA)>
<jugador id_jugador="J001">
    <nombre>ProGamer</nombre>
    <ranking>1</ranking>
</jugador>
Explicación:
  • id_jugador="J001" identifica de manera única a este <jugador>.

Atributo de tipo IDREF​

Un atributo de tipo IDREF apunta a (o hace referencia a) un atributo ID definido en algún otro elemento dentro del mismo documento XML.

Ejemplo​

  • DTD
  • XML
<!ELEMENT partida (jugador1, jugador2)>
<!ATTLIST partida
    ganador   IDREF   #IMPLIED
>
<!ELEMENT jugador1 (#PCDATA)>
<!ELEMENT jugador2 (#PCDATA)>
<partida ganador="J002">
    <jugador1>J001</jugador1>
    <jugador2>J002</jugador2>
</partida>
Explicación:
  • ganador="J002" hace referencia a algún elemento <jugador> que tenga id_jugador="J002".
  • Es imprescindible que ese ID exista en el documento. De lo contrario, la validación fallaría.

Atributo de tipo IDREFS​

Es igual que IDREF, pero puede contener varios identificadores separados por espacios. Cada identificador debe existir en algún atributo de tipo ID.

Ejemplo​

  • DTD
  • XML
<!ELEMENT equipo (nombre)>
<!ATTLIST equipo
    miembros   IDREFS   #IMPLIED
>
<!ELEMENT nombre (#PCDATA)>
<equipo miembros="J001 J002 J005">
    <nombre>Team Alpha</nombre>
</equipo>
Explicación:
  • miembros="J001 J002 J005" indica que este equipo hace referencia (en la misma partida o documento) a tres jugadores distintos (todos deben existir con un id_jugador igual a esos valores).

Atributo Enumerado (opciones fijas)​

El valor del atributo solo puede ser uno de los que aparezcan en la lista delimitada por ( y ) y separada por barras |.

Ejemplo​

  • DTD
  • XML
<!ELEMENT consola (#PCDATA)>
<!ATTLIST consola
    marca (Sony|Microsoft|Nintendo) "Sony"
>
<consola marca="Nintendo">Switch</consola>
Explicación:
  • marca solo puede tomar los valores Sony, Microsoft o Nintendo.
  • Si no se especifica nada, por defecto se asigna "Sony".

Atributo de tipo CDATA​

Puede contener cualquier cadena de caracteres. Es el tipo más abierto y se emplea para texto libre.

Ejemplo​

  • DTD
  • XML
<!ELEMENT sala (#PCDATA)>
<!ATTLIST sala
    ubicacion CDATA   #IMPLIED
>
<sala ubicacion="Planta 2, pasillo A">Zona de juego retro</sala>
Explicación:
  • ubicacion="Planta 2, pasillo A" es texto libre (con espacios, comas, etc.).
  • #IMPLIED indica que es opcional.

Atributo NMTOKEN / NMTOKENS​

  • NMTOKEN: Una cadena sin espacios ni caracteres especiales (solo letras, dígitos, puntos, guiones...).
  • NMTOKENS: Varios NMTOKENs separados por espacios.

Ejemplo​

  • DTD
  • XML
<!ELEMENT torneo (#PCDATA)>
<!ATTLIST torneo
    hashtag NMTOKEN  #IMPLIED
>
<torneo hashtag="SummerCup2023">
    Torneo de verano
</torneo>
Explicación:
  • hashtag no puede contener espacios; “SummerCup2023” es un valor válido para un NMTOKEN. (Si quisiéramos varios hashtags, usaríamos NMTOKENS, e.g. hashtag="SummerCup2023 CosplaySpecial".)

Atributo NOTATION​

Este tipo de atributo se utiliza cuando el valor debe ser el nombre de una notación declarada con <!NOTATION ...>. Se usa a menudo para asociar tipos de archivos (imágenes, vídeos) con su MIME type.

Ejemplo​

  • DTD
  • XML
<!NOTATION jpg SYSTEM "image/jpeg">
<!NOTATION png SYSTEM "image/png">

<!ELEMENT poster EMPTY>
<!ATTLIST poster
    formato NOTATION (jpg|png) #REQUIRED
    src     CDATA              #REQUIRED
>
<poster formato="png" src="cartel.png"/>
Explicación:
  • formato="png" remite a la notación png declarada como <!NOTATION png SYSTEM "image/png">.
  • Esto ayuda a la aplicación a saber que es un archivo PNG.

Atributo ENTITY / ENTITIES​

Permite referirse a entidades declaradas en el DTD.

  • ENTITY: Apunta a una sola entidad.
  • ENTITIES: Puede apuntar a varias entidades (separadas por espacios).

Ejemplo​

  • DTD
  • XML
<!ENTITY logo SYSTEM "images/logo.jpg" NDATA jpg>
<!ENTITY trofeo SYSTEM "images/trofeo.png" NDATA png>
<!NOTATION jpg SYSTEM "image/jpeg">
<!NOTATION png SYSTEM "image/png">

<!ELEMENT premios (#PCDATA)>
<!ATTLIST premios
    galeria  ENTITIES   #IMPLIED
>
<premios galeria="logo trofeo">
    Entrega de premios a los ganadores
</premios>
Explicación:
  • galeria="logo trofeo" hace referencia a dos entidades definidas (logo y trofeo).
  • Cada entidad apunta a un archivo externo (logo.jpg, trofeo.png).

4. Atributos especiales​

En XML, hay dos atributos especiales que no necesitan ser declarados en el DTD o en el XML Schema, porque forman parte de las recomendaciones del W3C y están reconocidos de manera universal: xml:lang y xml:space. Aun así, en un entorno DTD, se pueden incluir declaraciones para que queden registradas y el validador acepte su uso. A continuación, te explico su propósito.

Atributo xml:lang​

xml:lang se utiliza para indicar el idioma del contenido (texto) y de los posibles atributos de un elemento. Afecta también de manera recursiva a todos sus subelementos, salvo que alguno de ellos defina un nuevo xml:lang.

Ejemplo de XML​

<articulo xml:lang="es">
  <titulo>Competición de Videojuegos</titulo>
  <contenido>Nuevo evento para jóvenes...</contenido>
</articulo>
info
  • xml:lang="es" indica que el texto dentro de <articulo> está en español.
  • Principalmente, sirve para que los programas (por ejemplo, navegadores web, indexadores, traductores automáticos, lectores de pantalla, etc.) sepan en qué idioma está el contenido y puedan realizar ajustes o procesamientos adecuados (seleccionar diccionarios de corrección ortográfica, activar reglas tipográficas, etc.).

Ejemplo de DTD​

Aunque xml:lang no necesita declaración para que funcione, si se quiere ser explícito en un DTD, se haría algo como:

<!ATTLIST articulo
    xml:lang CDATA #IMPLIED
>

De este modo, el validador reconocerá que articulo puede tener un atributo xml:lang.

Atributo xml:space​

xml:space se utiliza para indicar cómo tratar los espacios en blanco, tabulaciones y saltos de línea presentes en el contenido de un elemento. Sus valores más comunes son:

  • xml:space="default": Indica que los espacios en blanco pueden ser normalizados según la aplicación o el parser (por ejemplo, convertir saltos de línea en un solo espacio, ignorar algunos espacios repetidos, etc.).
  • xml:space="preserve": Indica que se deben mantener todos los espacios, tabulaciones y saltos de línea tal cual aparecen en el código fuente.

Ejemplo de XML​

<descripcion xml:space="preserve">
   La primera línea.
   La segunda línea (tab).
        <contenido extra="info" />
</descripcion>
info

Si un procesador respeta xml:space="preserve", no eliminará ni reconfigurará esos saltos de línea y espacios al presentar o procesar el texto.

Ejemplo de DTD​

Al igual que xml:lang, también puede declararse opcionalmente en un DTD:

<!ATTLIST descripcion
    xml:space (default|preserve) #IMPLIED
>

Esto aclara para el validador que xml:space puede tomar uno de esos dos valores.
