Notaciones

Las notaciones en DTD permiten indicar el formato o tipo de datos de archivos externos que no son analizados por el parser XML (por ejemplo, imágenes, vídeos, documentos binarios). Sirven como un mecanismo de identificación, para que las aplicaciones sepan cómo manejar un recurso que no es estrictamente texto XML.

1. ¿Por qué existen las notaciones?​

Cuando un documento XML hace referencia a archivos binarios (imágenes, PDFs, audio, etc.), el procesador XML no interpreta ese contenido internamente (se dice que son no analizables o unparsed). Entonces, las notaciones proporcionan una forma de declarar el tipo de dichos archivos.

Por ejemplo, si declaramos <!NOTATION png SYSTEM "image/png">, le estamos diciendo al procesador (y a cualquier aplicación que use el XML) que cuando se haga referencia a la notación png, el archivo es de tipo image/png (un PNG).

2. Declaración de Notaciones​

En un DTD, se declaran con la sintaxis:

<!NOTATION nombre_notacion SYSTEM "tipo_MIME">

o

<!NOTATION nombre_notacion PUBLIC "identificador_publico">

Donde:

  • SYSTEM suele indicar una ruta o un identificador local del tipo MIME.
  • PUBLIC se usa con un identificador público estándar o registrado.

Ejemplo de Notaciones​

<!NOTATION png SYSTEM "image/png">
<!NOTATION jpg SYSTEM "image/jpeg">
<!NOTATION gif SYSTEM "image/gif">
info

Aquí, definimos tres notaciones (png, jpg y gif), cada una asociada a su respectivo tipo MIME.

3. ¿Cómo se usan en XML?​

Para que un elemento o atributo pueda indicar la notación, se suele declarar un atributo de tipo NOTATION en el DTD. Por ejemplo:

<!ELEMENT imagen EMPTY>
<!ATTLIST imagen
    formato NOTATION (png|jpg|gif) #REQUIRED
    src     CDATA                  #REQUIRED
>

Con esto, en el XML podemos escribir:

<imagen formato="png" src="foto.png"/>
info
  • formato="png" apunta a la notación declarada <!NOTATION png SYSTEM "image/png">.
  • De esta forma, un procesador o aplicación que lea el XML sabe que foto.png es un archivo PNG y, por lo tanto, debe manejarlo como tal.

4. Notaciones y Entidades No Analizables​

Las notaciones suelen trabajar de la mano de entidades generales externas no analizables. Un ejemplo:

<!NOTATION png SYSTEM "image/png">
<!ENTITY logo SYSTEM "img/logo.png" NDATA png>
info
  1. Declaramos la notación png con SYSTEM "image/png".
  2. Declaramos una entidad logo con SYSTEM "img/logo.png" y NDATA png, lo que significa que logo es un recurso (archivo) no analizable, cuyo formato es png.
  3. En el XML, podemos usar un atributo ENTITY (por ejemplo, <imagen src="logo"/>) para hacer referencia a ese recurso.

5. Notación Privada vs. Pública​

  • SYSTEM: Se usa con una referencia local o con un tipo MIME.
  • PUBLIC: Se emplea cuando hay un identificador público estándar (por ejemplo, "-//W3C//NOTATION GIF 1.0//EN"). 
    <!NOTATION gif PUBLIC "-//W3C//NOTATION GIF 1.0//EN">

En la práctica, SYSTEM "image/png" es más habitual si solo necesitas especificar el MIME type.
