4.5 Espacios de Nombres

Los espacios de nombres en XML (Namespaces) evitan conflictos cuando diferentes documentos XML contienen elementos con el mismo nombre, pero con significados diferentes. Se utilizan principalmente en:

  • Intercambio de datos entre sistemas que pueden tener estructuras XML similares.
  • Combinación de múltiples documentos XML en un solo archivo.
  • Uso de estándares XML como SOAP, RDF y XSLT.

📌 Ejemplo del Problema (sin namespace):
Si combinamos información de Pokémon y Entrenadores, ambos podrían tener elementos <nombre>:

<pokemon>
    <nombre>Pikachu</nombre>
</pokemon>

<entrenador>
    <nombre>Ash Ketchum</nombre>
</entrenador>

❌ Confusión: ¿El <nombre> hace referencia a un Pokémon o a un Entrenador?

✅ Solución: Usar Espacios de Nombres (Namespaces) para diferenciarlos.

Declaración de Espacios de Nombres (xmlns)​

La declaración de un espacio de nombres se realiza con el atributo xmlns="URL", donde la URL no es un enlace real, sino un identificador único. Por ejemplo...

<?xml version="1.0" encoding="UTF-8"?>
<pokedex xmlns="http://ejemplo.com/pokemon">
    <pokemon>
        <nombre>Pikachu</nombre>
        <tipo>Eléctrico</tipo>
    </pokemon>
</pokedex>
Explicación:
  • xmlns="http://ejemplo.com/pokemon" → Define un espacio de nombres por defecto para <pokedex> y todos sus elementos internos.
  • Sin prefijos, todos los elementos dentro del <pokedex> pertenecen automáticamente a este espacio de nombres.

Uso de Prefijos en Espacios de Nombres​

Cuando combinamos múltiples fuentes XML, es mejor usar prefijos para diferenciar los datos. Mira este ejemplo:

<?xml version="1.0" encoding="UTF-8"?>
<datos xmlns:p="http://ejemplo.com/pokemon"
       xmlns:e="http://ejemplo.com/entrenadores">
       
    <p:pokemon>
        <p:nombre>Pikachu</p:nombre>
        <p:tipo>Eléctrico</p:tipo>
    </p:pokemon>

    <e:entrenador>
        <e:nombre>Ash Ketchum</e:nombre>
        <e:region>Kanto</e:region>
    </e:entrenador>

</datos>
Explicación:
  • Espacio de Nombres para Pokémon: xmlns:p="http://ejemplo.com/pokemon"
  • Espacio de Nombres para Entrenadores: xmlns:e="http://ejemplo.com/entrenadores"
  • Se usan prefijos (p: y e:) para diferenciar los elementos.
Ventaja:

Se pueden combinar diferentes estructuras XML sin confusión.

Namespace por Defecto vs. Namespace con Prefijos​

XML permite definir espacios de nombres por defecto (sin prefijos) o con prefijos explícitos.

📌 Namespace por Defecto (sin prefijo)​

<pokedex xmlns="http://ejemplo.com/pokemon">
    <pokemon>
        <nombre>Pikachu</nombre>
        <tipo>Eléctrico</tipo>
    </pokemon>
</pokedex>

✅ Todos los elementos dentro de <pokedex> usan automáticamente el namespace de Pokémon.

📌 Namespace con Prefijos​

<pokedex xmlns:p="http://ejemplo.com/pokemon">
    <p:pokemon>
        <p:nombre>Pikachu</p:nombre>
        <p:tipo>Eléctrico</p:tipo>
    </p:pokemon>
</pokedex>

✅ El prefijo p: deja claro que los elementos pertenecen al espacio de nombres de Pokémon.

Combinación de Múltiples Espacios de Nombres​

Si un documento XML combina Pokémon, Entrenadores y Movimientos, podemos definir varios espacios de nombres. Por ejemplo así...

<?xml version="1.0" encoding="UTF-8"?>
<datos xmlns:p="http://ejemplo.com/pokemon"
       xmlns:e="http://ejemplo.com/entrenadores"
       xmlns:m="http://ejemplo.com/movimientos">

    <p:pokemon id="025">
        <p:nombre>Pikachu</p:nombre>
        <p:tipo>Eléctrico</p:tipo>
        <m:movimientos>
            <m:movimiento>Impactrueno</m:movimiento>
            <m:movimiento>Rayo</m:movimiento>
        </m:movimientos>
    </p:pokemon>

    <e:entrenador>
        <e:nombre>Ash Ketchum</e:nombre>
        <e:region>Kanto</e:region>
        <e:pokemon_favorito ref="p:025" />
    </e:entrenador>

</datos>
Explicación:
  • xmlns:p="http://ejemplo.com/pokemon" → Prefijo p: para Pokémon.
  • xmlns:e="http://ejemplo.com/entrenadores" → Prefijo e: para Entrenadores.
  • xmlns:m="http://ejemplo.com/movimientos" → Prefijo m: para Movimientos.
  • <e:pokemon_favorito ref="p:025" /> → Hace referencia a Pikachu usando su espacio de nombres.
Beneficio:

Podemos diferenciar los datos fácilmente, asegurando que los sistemas entiendan qué información pertenece a qué categoría.
