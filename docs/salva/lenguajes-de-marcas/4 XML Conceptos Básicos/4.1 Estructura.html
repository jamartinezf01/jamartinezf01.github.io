4.1 Estructura de un Documento

Estructura General 📄​

Un documento XML bien formado debe seguir una estructura jerárquica que se asemeja a un árbol 🌳, en donde cada nodo representa un elemento que puede contener atributos, texto o más elementos.

Un documento XML estándar se divide en las siguientes partes:

  1. Declaración XML 📢
  2. Comentarios y Procesamiento 📝
  3. Elemento Raíz 🌱
  4. Elementos Secundarios y Contenido 📂
  5. Secciones CDATA (Opcional) 🔒
  6. Instrucciones de Procesamiento (Opcional) ⚙️

A continuación, explicamos cada una en detalle.

1. Declaración XML 📢​

La declaración XML es la primera línea de un documento XML y proporciona información sobre la versión del estándar XML utilizado y la codificación de caracteres.

  • Ejemplo de Declaración XML
<?xml version="1.0" encoding="UTF-8"?>
Componentes de la Declaración XML
  • version="1.0" → 📌 Especifica la versión de XML utilizada. Actualmente, la versión más común es 1.0.
  • encoding="UTF-8" → 🔠 Define la codificación de caracteres del documento. UTF-8 es el más recomendado porque soporta una amplia gama de caracteres internacionales.
Reglas para la Declaración XML

✅ Debe ser la primera línea del documento.
✅ Es opcional, pero su inclusión es una buena práctica.
✅ Si se incluye, debe seguir el formato correcto: <?xml version="X.Y" encoding="ENCODING"?>.

2. Comentarios y Procesamiento 📝​

Los comentarios en XML permiten incluir notas dentro del código sin afectar la estructura o el contenido.

  • Ejemplo de Comentario en XML
<!-- Este es un comentario en XML -->
Reglas de los Comentarios

✅ Se escriben entre <!-- y -->.
❌ No pueden anidarse (<!-- Esto <!-- está mal --> -->).
✅ No afectan la interpretación del documento.

Además, en XML también existen instrucciones de procesamiento ⚙️ que indican cómo un procesador XML debe manejar el documento.

  • Ejemplo de Instrucción de Procesamiento
<?xml-stylesheet type="text/xsl" href="estilo.xsl"?>
nota

📌 Esta instrucción indica que el documento debe utilizar la hoja de estilos estilo.xsl.

3. Elemento Raíz 🌱​

Todo documento XML debe contener un único elemento raíz que envuelve a todos los demás elementos.

  • Ejemplo de Elemento Raíz
<pokedex>
    <pokemon>
        <nombre>Pikachu</nombre>
        <tipo>Eléctrico</tipo>
    </pokemon>
</pokedex>
nota

Aquí, <pokedex> es el elemento raíz y contiene otros elementos dentro.

Reglas del Elemento Raíz

✅ Debe haber solo un elemento raíz.
✅ Todos los demás elementos deben estar contenidos dentro del elemento raíz.
✅ Su nombre debe ser significativo y representar la estructura del contenido.

4. Elementos Secundarios y Contenido 📂​

Dentro del elemento raíz, un documento XML puede contener múltiples niveles de elementos secundarios 📄, organizando la información en una estructura jerárquica.

  • Ejemplo con Elementos Secundarios
<pokedex>
    <pokemon id="001">
        <nombre>Bulbasaur</nombre>
        <tipo>Planta</tipo>
        <tipo>Veneno</tipo>
    </pokemon>
</pokedex>
En este ejemplo:
  • <pokedex> es el elemento raíz 🌳.
  • <pokemon> es un elemento secundario ⚡.
  • <nombre>, <tipo> son subelementos de <pokemon> 🏷️.
Reglas de los Elementos

✅ Deben abrirse y cerrarse correctamente: <nombre>Bulbasaur</nombre>.
✅ Pueden contener texto, atributos o más elementos.
✅ Deben anidarse correctamente para mantener la jerarquía.

5. Secciones CDATA (Opcional) 🔒​

Las secciones CDATA permiten incluir texto sin que el procesador XML lo interprete como código XML.

  • Ejemplo de CDATA
<descripcion>
    <![CDATA[Pikachu es un Pokémon de tipo <Eléctrico> con ataques como Rayo y Trueno.]]>
</descripcion>
nota

📌 Aquí, <Eléctrico> no se interpreta como una etiqueta XML gracias a CDATA.

Cuándo Usar CDATA

✅ Cuando se necesita incluir caracteres reservados como <, > o &.
✅ Para insertar fragmentos de código HTML dentro de XML sin que generen errores.

6. Instrucciones de Procesamiento (Opcional) ⚙️​

Las instrucciones de procesamiento en XML proporcionan información a los programas que procesan el documento.

  • Ejemplo de Instrucción de Procesamiento
<?xml-stylesheet type="text/css" href="estilos.css"?>
nota

📌 Esto indica que el documento debe utilizar un archivo CSS para definir su presentación.

Casos de Uso de Instrucciones de Procesamiento

✅ Aplicar hojas de estilo (XSLT, CSS).
✅ Incluir instrucciones específicas para procesadores XML.
✅ Definir configuraciones personalizadas.

<?xml-stylesheet type="text/xsl" href="pokedex.xsl"?>
<!-- Pokédex de Pokémon iniciales de la primera generación -->
<pokedex>
    <pokemon id="001">
        <nombre>Bulbasaur</nombre>
        <tipo>Planta</tipo>
        <tipo>Veneno</tipo>
        <habilidades>
            <habilidad>Espesura</habilidad>
            <habilidad oculta="true">Clorofila</habilidad>
        </habilidades>
       <descripcion>
            <![CDATA[Bulbasaur < Ivysaur < Venusaur en evolución.]]>
        </descripcion>
    </pokemon>

    <pokemon id="004">
        <nombre>Charmander</nombre>
        <tipo>Fuego</tipo>
        <habilidades>
            <habilidad>Mar Llamas</habilidad>
            <habilidad oculta="true">Poder Solar</habilidad>
        </habilidades>
        <descripcion>
            <![CDATA[Charmander < Charmeleon < Charizard en evolución.]]>
        </descripcion>
    </pokemon>

    <pokemon id="007">
        <nombre>Squirtle</nombre>
        <tipo>Agua</tipo>
        <habilidades>
            <habilidad>Torrente</habilidad>
            <habilidad oculta="true">Cura Lluvia</habilidad>
        </habilidades>
          <descripcion>
            <![CDATA[Squirtle < Wartortle < Blastoise en evolución.]]>
        </descripcion>
    </pokemon>

    <pokemon id="025">
        <nombre>Pikachu</nombre>
        <tipo>Eléctrico</tipo>
        <habilidades>
            <habilidad>Electricidad Estática</habilidad>

        </habilidades>
        <descripcion>
            <![CDATA[Pikachu < Raichu en evolución.]]>
        </descripcion>
    </pokemon>
</pokedex>
