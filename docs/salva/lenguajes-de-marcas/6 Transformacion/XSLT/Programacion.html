Programar plantillas

Una vez comprendida la estructura general de las hojas de estilo y cómo se genera HTML, es momento de aprender a programar el contenido de una plantilla: cómo insertar valores, recorrer nodos, aplicar condiciones, generar atributos, etc.

Todo esto se hace dentro de la plantilla principal, normalmente la que se aplica al nodo raíz (<xsl:template match="/">).

FICHEROS EJEMPLOS

En esta sección utilizaremos el ejemplo que te puedes descargar en 👉 este enlace para explicar la programación con XSLT.

El archivo ACTIVIDAD TIENDA.rar contiene:

  • directorio css con la hoja de estilos css
  • directorio img con las imágenes necesarias para desarrollar el ejemplo
  • fichero tienda.html es la referencia de cómo tiene que quedar la transformación.
  • 3 ficheros .xml que serán los datos de entrada de nuestra transformación.

1. Mostrar contenido: <xsl:value-of>​

La instrucción <xsl:value-of> se utiliza para mostrar el contenido de un nodo del XML.
Es la forma más sencilla de insertar datos dinámicos en el HTML generado desde una plantilla XSLT. La sintaxis es:

<xsl:value-of select="expresión XPath"/>
  • El atributo select contiene una expresión XPath que indica qué nodo del XML se desea mostrar.
  • El contenido se inserta como texto plano (no se interpretan etiquetas ni nodos hijos).
  • XML entrada
  • XSLT
  • HTML salida

Fragmento del XML (tienda.xml)

<producto id="p01" unidades="25">
  <nombre>Polvos de cuerno de unicornio</nombre>
  <precio>25</precio>
  <imagen>PolvosUnicornio.jpg</imagen>
</producto>

Código en tienda.xsl (plantilla única)

<xsl:template match="/">
  <html>
    <body>
      <h1>Catálogo</h1>
      <xsl:for-each select="tienda/catalogo/producto">
        <div class="producto">
          <p><xsl:value-of select="nombre"/></p>
          <p>Precio: <xsl:value-of select="precio"/> 💰</p>
        </div>
      </xsl:for-each>
    </body>
  </html>
</xsl:template>

Resultado en HTML (tienda.html)

<h1>Catálogo</h1>
<div class="producto">
  <p>Polvos de cuerno de unicornio</p>
  <p>Precio: 25 💰</p>
</div>
  • Si el nodo no existe, no se muestra nada.
  • Solo se muestra el texto del nodo. Si el nodo tiene hijos, estos no se incluyen.
  • Si se desea añadir texto adicional (como símbolos, unidades, etc.), puede colocarse fuera del value-of o combinarse con <xsl:text>.

2. Añadir texto adicional: <xsl:text>​

En XSLT se pueden insertar símbolos, palabras fijas y signos de puntuación mediante <xsl:text>.

<xsl:text>Texto que se quiere insertar</xsl:text>
  • XML entrada
  • XSLT
  • HTML salida
<producto id="p01" unidades="25">
  <nombre>Polvos de cuerno de unicornio</nombre>
  <precio>25</precio>
  <imagen>PolvosUnicornio.jpg</imagen>
</producto>
<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
  <xsl:output method="html" indent="yes"/>
  <xsl:template match="/">
    <html>
      <body>
        <h1>Catálogo</h1>
        <xsl:for-each select="tienda/catalogo/producto">
          <div class="producto">
            <p>
              <xsl:text>Producto: </xsl:text>
              <xsl:value-of select="nombre"/>
              <xsl:text>.</xsl:text>
            </p>
            <p>
              <xsl:text>Precio: </xsl:text>
              <xsl:value-of select="precio"/>
              <xsl:text> 💰</xsl:text>
            </p>
          </div>
        </xsl:for-each>
      </body>
    </html>
  </xsl:template>

</xsl:stylesheet>
<h1>Catálogo</h1>
<div class="producto">
  <p>Producto: Polvos de cuerno de unicornio.</p>
  <p>Precio: 25 💰</p>
</div>

3. Insertar atributos: {} y <xsl:attribute>​

En XSLT, los atributos HTML pueden generarse de dos formas:

  • Usando llaves {} directamente dentro del atributo (más simple y habitual).
  • Usando la instrucción <xsl:attribute> para casos más complejos o condicionales.

Ambas formas permiten insertar valores dinámicos en atributos como src, alt, class, etc.

<elemento atributo="texto/{expresión XPath}/másTexto"/>

Permite insertar el valor de una expresión XPath directamente dentro de un atributo.
<xsl:attribute name="nombreAtributo">
  valor o expresión
</xsl:attribute>

Crea un atributo de forma dinámica dentro de una etiqueta.

Ejemplo: mostrar productos con su imagen​

  • XML entrada
  • XSLT
  • HTML salida
<producto id="p01" unidades="25">
  <nombre>Polvos de cuerno de unicornio</nombre>
  <precio>25</precio>
  <imagen>PolvosUnicornio.jpg</imagen>
</producto>
<xsl:template match="/">
  <html>
    <body>
      <h1>Catálogo</h1>
      <xsl:for-each select="tienda/catalogo/producto">
        <div class="producto">
          <!-- Atributo generado con {} -->
          <img src="img/{imagen}" alt="{nombre}" width="100"/>

          <!-- Atributo generado con <xsl:attribute> -->
          <p>
            <xsl:attribute name="class">
              <xsl:choose>
                <xsl:when test="@unidades &gt; 10">stock-alto</xsl:when>
                <xsl:otherwise>stock-bajo</xsl:otherwise>
              </xsl:choose>
            </xsl:attribute>
            <xsl:value-of select="nombre"/>
          </p>
        </div>
      </xsl:for-each>
    </body>
  </html>
</xsl:template>
<h1>Catálogo</h1>
<div class="producto">
  <img src="img/PolvosUnicornio.jpg" alt="Polvos de cuerno de unicornio" width="100"/>
  <p class="stock-alto">Polvos de cuerno de unicornio</p>
</div>
¿Cuándo usar cada forma?

Si el valor del atributo es directo, se recomienda usar {}.

Si es necesario aplicar condiciones o generar el valor dinámicamente, utiliza <xsl:attribute>.

SituaciónRecomendación
Atributo simple con valor fijo o directo{} dentro del atributo
Atributo condicional o complejo<xsl:attribute>
Necesidad de if / choose<xsl:attribute> + lógica XPath

4. Crear bucles: <xsl:for-each>​

La instrucción <xsl:for-each> permite recorrer un conjunto de nodos seleccionados mediante XPath y aplicar una misma estructura de salida a cada uno de ellos.

Se utiliza cuando se desea generar un bloque de contenido repetido para cada nodo de una lista, como por ejemplo para mostrar varios elementos del XML con el mismo formato.

<xsl:for-each select="expresión XPath">
  <!-- acciones repetidas por cada nodo -->
</xsl:for-each>

Ejemplo: mostrar todos los productos del catálogo​

  • XML entrada
  • XSLT
  • HTML salida
<catalogo>
  <producto id="p01" unidades="25">
    <nombre>Polvos de cuerno de unicornio</nombre>
    <precio>25</precio>
  </producto>
  <producto id="p02" unidades="10">
    <nombre>Raíz de mandrágora</nombre>
    <precio>30</precio>
  </producto>
</catalogo>
<xsl:stylesheet version="1.0"
  xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

  <xsl:output method="html" indent="yes"/>
  <xsl:strip-space elements="*"/>

  <xsl:template match="/">
    <html>
      <body>
        <h1>🧪 Catálogo</h1>
        <xsl:for-each select="tienda/catalogo/producto">
          <div class="producto">
            <p><strong><xsl:value-of select="nombre"/></strong></p>
            <p>Precio: <xsl:value-of select="precio"/> 💰</p>
          </div>
        </xsl:for-each>
      </body>
    </html>
  </xsl:template>

</xsl:stylesheet>
<h1>🧪 Catálogo</h1>
<div class="producto">
  <p><strong>Polvos de cuerno de unicornio</strong></p>
  <p>Precio: 25 💰</p>
</div>
<div class="producto">
  <p><strong>Raíz de mandrágora</strong></p>
  <p>Precio: 30 💰</p>
</div>
¿Cuándo usar <xsl:for-each>?
  • Cuando se quiere aplicar una misma estructura de salida a varios elementos.
  • Cuando se necesita control total sobre el formato dentro del bucle.
  • Cuando se desea acceder al índice de cada elemento con position().

5. Ordenar resultados: <xsl:sort>​

La instrucción <xsl:sort> permite ordenar los elementos que se están recorriendo en un <xsl:for-each> o <xsl:apply-templates>.
El orden se basa en el valor de un nodo o atributo, y se puede personalizar con varios parámetros (order, data-type, etc.).

<xsl:sort select="expresión XPath" order="ascending|descending" data-type="text|number"/>
importante

Siempre debe colocarse dentro de <xsl:for-each> o <xsl:apply-templates>, justo al principio, antes de generar el contenido.

Ejemplo: mostrar los productos del catálogo ordenados por precio (descendente)​

  • XML entrada
  • XSLT
  • HTML salida
<catalogo>
  <producto id="p01" unidades="25">
    <nombre>Polvos de cuerno de unicornio</nombre>
    <precio>25</precio>
  </producto>
  <producto id="p02" unidades="10">
    <nombre>Raíz de mandrágora</nombre>
    <precio>30</precio>
  </producto>
  <producto id="p03" unidades="15">
    <nombre>Escamas de dragón rojo</nombre>
    <precio>50</precio>
  </producto>
</catalogo>
<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

  <xsl:output method="html" indent="yes"/>
  <xsl:strip-space elements="*"/>

  <xsl:template match="/">
    <html>
      <body>
        <h1>🧪 Catálogo (ordenado por precio descendente)</h1>
        <xsl:for-each select="tienda/catalogo/producto">
          <xsl:sort select="precio" data-type="number" order="descending"/>
          <div class="producto">
            <p><strong><xsl:value-of select="nombre"/></strong></p>
            <p>Precio: <xsl:value-of select="precio"/> 💰</p>
          </div>
        </xsl:for-each>
      </body>
    </html>
  </xsl:template>

</xsl:stylesheet>

<div class="producto">
  <p><strong>Escamas de dragón rojo</strong></p>
  <p>Precio: 50 💰</p>
</div>
<div class="producto">
  <p><strong>Raíz de mandrágora</strong></p>
  <p>Precio: 30 💰</p>
</div>
<div class="producto">
  <p><strong>Polvos de cuerno de unicornio</strong></p>
  <p>Precio: 25 💰</p>

Parámetros útiles de <xsl:sort>​

AtributoFunciónEjemplo

6. Condiciones simples: <xsl:if>​

La instrucción <xsl:if> permite mostrar contenido solo cuando se cumple una condición.
Es útil para personalizar el resultado en función de los valores de un nodo o atributo del XML (por ejemplo, si un producto es destacado, si el stock es bajo, etc.).

La condición se indica en el atributo test, utilizando expresiones XPath que devuelven un valor booleano (true o false).

<xsl:if test="condición XPath">
  <!-- contenido si se cumple la condición -->
</xsl:if>

Ejecuta el bloque solo si la condición es verdadera.

Ejemplo: mostrar un mensaje solo si el producto es "destacado"​

  • XML entrada
  • XSLT
  • HTML salida
<producto id="p02" unidades="10" destacado="sí">
  <nombre>Raíz de mandrágora</nombre>
  <precio>30</precio>
</producto>
<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

  <xsl:output method="html" indent="yes"/>
  <xsl:strip-space elements="*"/>

  <xsl:template match="/">
    <html>
      <body>
        <h1>🧪 Catálogo</h1>
        <xsl:for-each select="tienda/catalogo/producto">
          <div class="producto">
            <p><strong><xsl:value-of select="nombre"/></strong></p>
            <p>Precio: <xsl:value-of select="precio"/> 💰</p>

            <xsl:if test="@destacado = 'sí'">
              <p>✨ <strong>¡Producto destacado!</strong></p>
            </xsl:if>
            
          </div>
        </xsl:for-each>
      </body>
    </html>
  </xsl:template>

</xsl:stylesheet>
<h1>🧪 Catálogo</h1>
<div class="producto">
  <p><strong>Raíz de mandrágora</strong></p>
  <p>Precio: 30 💰€</p>
  <p>✨ <strong>¡Producto destacado!</strong></p>
</div>
¿Qué se puede comprobar con <xsl:if>?
  • El valor de un atributo: @destacado = 'sí'
  • Comparaciones numéricas: precio &gt; 50
  • Existencia de un nodo: imagen
  • Combinaciones: @unidades &lt;= 10 and @destacado = 'sí'

7. Condiciones múltiples: <xsl:choose>​

Cuando se necesita evaluar varias condiciones diferentes (como un "if... else if... else"), se utiliza la instrucción <xsl:choose>.
Dentro de ella se escriben uno o varios bloques <xsl:when> y, opcionalmente, un <xsl:otherwise> para el caso por defecto.

Es muy útil para mostrar distintos mensajes según el valor de un nodo o atributo.

<xsl:choose>
  <xsl:when test="condición XPath">
    <!-- contenido si se cumple -->
  </xsl:when>
  <xsl:otherwise>
    <!-- contenido si no se cumple ninguna condición -->
  </xsl:otherwise>
</xsl:choose>

Ejemplo: mostrar un mensaje según el nivel de stock (@unidades)​

  • XML entrada
  • XSLT
  • HTML salida
<producto id="p04" unidades="5">
  <nombre>Patas de culebra albina</nombre>
  <precio>20</precio>
</producto>
<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

  <xsl:output method="html" indent="yes"/>
  <xsl:strip-space elements="*"/>

  <xsl:template match="/">
    <html>
      <body>
        <h1>🧪 Catálogo</h1>
        <xsl:for-each select="tienda/catalogo/producto">
          <div class="producto">
            <p><strong><xsl:value-of select="nombre"/></strong></p>
            <p>Precio: <xsl:value-of select="precio"/> 💰</p>

            <xsl:choose>
              <xsl:when test="@unidades &gt; 10">
                <p class="stock alto">✅ Stock alto</p>
              </xsl:when>
              <xsl:when test="@unidades &gt; 5">
                <p class="stock medio">⚠️ Stock medio</p>
              </xsl:when>
              <xsl:otherwise>
                <p class="stock bajo">🛑 Quedan pocas unidades</p>
              </xsl:otherwise>
            </xsl:choose>

          </div>
        </xsl:for-each>
      </body>
    </html>
  </xsl:template>

</xsl:stylesheet>
<h1>🧪 Catálogo</h1>
<div class="producto">
  <p><strong>Patas de culebra albina</strong></p>
  <p>Precio: 20 💰</p>
  <p class="stock bajo">🛑 Quedan pocas unidades</p>
</div>
¿Cuándo usar <xsl:choose>?
  • Cuando hay más de dos posibles situaciones.
  • Cuando quieres mostrar un contenido distinto según rangos, categorías, etc.
  • Cuando <xsl:if> se quedaría corto o haría el código más difícil de leer.

8. Declarar variables: <xsl:variable>​

En XSLT se pueden declarar variables para guardar valores temporalmente y utilizarlos dentro de la misma plantilla o bloque de código. Son útiles para:

  • Guardar resultados de cálculos.

  • Guardar información extraída del XML.

  • Reutilizar un valor varias veces sin tener que repetir la expresión.

¿Cómo se declara una variable?​

<xsl:variable name="nombreVariable" select="expresiónXPath"/>

name → es el nombre de la variable (sin el símbolo $ al declararla).

select → es la expresión XPath cuyo valor se guardará.

¿Cómo se utiliza una variable?​

Para utilizar una variable , se usa $ delante del nombre:

<xsl:value-of select="$nombreVariable"/>
importante

Una variable en XSLT no se comparte entre plantillas, solo se puede usar dentro de la misma plantilla (o bucle, o condición) donde se ha definido.

Una variable puede contener:

  • Texto fijo.

  • El valor de un nodo.

  • El resultado de un cálculo (por ejemplo, precio × cantidad).

  • La concatenación de varios textos.

variables en xslt

Una vez que se crea una variable con <xsl:variable>, no se puede modificar su valor más adelante. Es decir, se comporta más como una constante que como una variable tradicional en otros lenguajes de programación.

👉 No puedes hacer algo como:

<xsl:variable name="precio" select="25"/>
<!-- ❌ No válido: redeclarar o cambiar valor -->
<xsl:variable name="precio" select="$precio + 10"/>

Ejemplo: calcular el total de un producto (precio × cantidad) en un pedido​

  • XML entrada
  • XSLT
  • HTML salida
<pedido id="ped01" cliente="c02" fecha="2025-04-01">
  <linea producto="p01" cantidad="2"/>
</pedido>
<catalogo>
  <producto id="p01">
    <nombre>Polvos de cuerno de unicornio</nombre>
    <precio>25</precio>
  </producto>
</catalogo>
<xsl:template match="/">
  <html>
    <body>
      <h1>🛒 Pedido</h1>
      <xsl:for-each select="tienda/pedidos/pedido">
        <h2><xsl:value-of select="@id"/></h2>
        <table>
          <tr>
            <th>Producto</th>
            <th>Cantidad</th>
            <th>Total</th>
          </tr>
          <xsl:for-each select="linea">
            <!-- Variables -->
            <xsl:variable name="id" select="@producto"/>
            <xsl:variable name="producto" select="/tienda/catalogo/producto[@id=$id]"/>
            <xsl:variable name="precio" select="number($producto/precio)"/>
            <xsl:variable name="cantidad" select="number(@cantidad)"/>
            <xsl:variable name="total" select="$precio * $cantidad"/>

            <tr>
              <td><xsl:value-of select="$producto/nombre"/></td>
              <td><xsl:value-of select="$cantidad"/></td>
              <td><xsl:value-of select="format-number($total, '#.00')"/> 💰</td>
            </tr>
          </xsl:for-each>
        </table>
      </xsl:for-each>
    </body>
  </html>
</xsl:template>
<h1>🛒 Pedido</h1>
<h2>ped01</h2>
<table>
  <tr>
    <th>Producto</th>
    <th>Cantidad</th>
    <th>Total</th>
  </tr>
  <tr>
    <td>Polvos de cuerno de unicornio</td>
    <td>2</td>
    <td>50.00 💰</td>
  </tr>
</table>
¿Para qué usar <xsl:variable>?
  • Para guardar el resultado de una búsqueda (producto[@id=$id]).
  • Para realizar cálculos (como multiplicar precio × cantidad).
  • Para evitar repetir código si un mismo valor se usa varias veces.

9. Errores comunes en XSLT y cómo evitarlos​

  1. Olvidar vincular el XML con la hoja de estilo

    Asegúrate de incluir esta línea en tu XML:

    <?xml-stylesheet type="text/xsl" href="archivo.xsl"?>

  2. Olvidar declarar el espacio de nombres en la etiqueta raíz del XSL

    Las hojas de estilo XSLT deben comenzar así:

    <xsl:stylesheet version="1.0"
      xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

  3. No comenzar con una plantilla match="/"

    Es imprescindible tener una plantilla que indique al procesador dónde empezar:

    <xsl:template match="/"> ... </xsl:template>

  4. No recorrer los nodos hijos

    Si no usas <xsl:for-each> o <xsl:apply-templates>, solo se procesará el nodo actual.

  5. Olvidar usar comillas en los atributos de las etiquetas

    Todos los atributos deben ir entre comillas:

    <xsl:value-of select="nombre"/>  ✅
    <xsl:value-of select=nombre/>   ❌

  6. Mal uso de rutas XPath

    Es común escribir producto/nombre cuando ya estás dentro de producto, y deberías usar solo nombre.

  7. Errores en condiciones (<xsl:if> o <xsl:choose>)

    Por ejemplo, comparar atributos sin el @:

    @precio &gt; 50  ✅
    precio &gt; 50   ❌

  8. No cerrar etiquetas correctamente

    Especialmente en HTML generado. Un error común es olvidarse de cerrar img, meta, etc.

  9. Usar <xsl:value-of> para mostrar varios nodos

    Solo muestra el primer nodo. Si quieres mostrar varios, usa un bucle:

    <xsl:for-each select="producto">
      <xsl:value-of select="nombre"/>
    </xsl:for-each>

  10. Errores con varaibles

  • No poner $ al usar la variable (select="$precio" ✔️).

  • Intentar cambiar el valor de una variable ya creada (no es posible).

  • Declarar la variable después de intentar usarla.
